
## Определение

**Сегментированное решето Эратосфена** — это модификация классического решета, позволяющая находить все простые числа до заданного большого `n` с использованием значительно меньшего объёма памяти. Вместо хранения булевого массива размера `n`, алгоритм обрабатывает диапазон чисел по частям (сегментам), используя только простые числа до `√n` для отметки составных чисел в каждом сегменте. Это делает алгоритм применимым для `n` порядка 10⁹–10¹², где обычное решето невозможно из-за ограничений памяти.

**Пример:** для `n = 100` сегментированное решето находит все простые числа: 2, 3, 5, 7, …, 97.

---

## Постановка задачи

**Вход:** целое неотрицательное число `n`.  
**Выход:** список всех простых чисел, не превосходящих `n`.

---

## Идея решения

Обычное решето Эратосфена требует массива размера `n`, что неприемлемо для больших `n`. Однако заметим, что для проверки числа на простоту достаточно знать простые числа до `√n`. Сегментированное решето использует этот факт:

1. Найти все простые числа до `√n` с помощью обычного решета.
2. Разбить диапазон `[0, n]` на сегменты (блоки) небольшого размера (обычно `√n` или около того).
3. Для каждого сегмента `[low, high]`:
   - Создать булев массив `mark` размера `high - low + 1`, изначально заполненный `True`.
   - Для каждого простого `p` из найденных на шаге 1:
     - Найти первое кратное `p` в сегменте: `start = max(p*p, ceil(low/p)*p)`.
     - Отметить все числа `start, start+p, start+2p, ...` как составные (установить `False` в `mark`).
   - Все индексы в `mark`, оставшиеся `True`, соответствуют простым числам в данном сегменте.

Таким образом, в каждый момент времени в памяти хранится только один сегмент и массив простых чисел до `√n`, что значительно экономит память.

---

## Алгоритм (пошагово)

1. **Найти простые до `√n`:** используем обычное решето Эратосфена для `limit = int(sqrt(n)) + 1`. Сохраняем найденные простые в список `primes`.
2. **Инициализировать сегменты:** установить `low = limit`, `high = limit * 2`. (Первый сегмент начинается после `limit`, так как числа до `limit` уже обработаны.)
3. **Обрабатывать сегменты, пока `low < n`:**
   - Если `high >= n`, установить `high = n`.
   - Создать массив `mark` размера `high - low + 1`, заполненный `True`.
   - Для каждого `p` в `primes`:
     - Найти первое кратное `p` в сегменте: `start = max(p*p, ((low + p - 1) // p) * p)`.
     - Для `j` от `start` до `high` с шагом `p`:
       - `mark[j - low] = False`.
   - Пройти по `mark`: если `mark[i - low] == True`, то число `i` простое — вывести или сохранить.
   - Обновить `low = high + 1`, `high = low + limit - 1`.
4. **Завершить**, когда `low >= n`.

## Реализация на Python

```python
import math

def segmented_sieve(n):
    if n < 2:
        return []
    
    # Шаг 1: простые до sqrt(n) обычным решетом
    limit = int(math.isqrt(n)) + 1
    is_prime_small = [True] * (limit + 1)
    is_prime_small[0] = is_prime_small[1] = False
    primes = []
    for p in range(2, limit + 1):
        if is_prime_small[p]:
            primes.append(p)
            for multiple in range(p * p, limit + 1, p):
                is_prime_small[multiple] = False
    
    # Шаг 2: обработка сегментов
    result = primes[:]  # простые до sqrt(n) уже известны
    seg_size = limit  # размер сегмента (можно варьировать)
    low = limit
    high = low + seg_size - 1
    
    while low <= n:
        if high > n:
            high = n
        mark = [True] * (high - low + 1)
        for p in primes:
            # первое кратное p в [low, high]
            start = max(p * p, ((low + p - 1) // p) * p)
            for j in range(start, high + 1, p):
                mark[j - low] = False
        for i in range(low, high + 1):
            if mark[i - low]:
                result.append(i)
        low = high + 1
        high = low + seg_size - 1
    
    return result

# Пример
print(segmented_sieve(100))
```

## Сложность

- **Время:** O(n log log n) — теоретически такая же, как у обычного решета, но на практике алгоритм может быть чуть медленнее из-за дополнительных вычислений. Однако он имеет лучшее поведение кэша (локальность обращения к памяти), что ускоряет работу для больших `n`.
- **Память:** O(√n + seg_size), где `seg_size` — размер сегмента (обычно ≈ √n). Это значительно меньше O(n) для больших `n`.

---

## Примечания

- Сегментированное решето особенно полезно, когда `n` велико (например, > 10⁷) и массив размера `n` не помещается в память.
- Размер сегмента можно выбирать исходя из доступной памяти и производительности кэша. Обычно выбирают около `√n` или, например, 10⁵–10⁶ для лучшей производительности.
- Алгоритм требует, чтобы все простые числа до `√n` были найдены заранее, поэтому для очень больших `n` (например, 10¹²) `√n` может всё ещё быть большим, но управляемым.
- Сегментированное решето также может быть использовано для поиска простых чисел в заданном интервале `[L, R]` без необходимости обработки всех чисел от 0 до R.
