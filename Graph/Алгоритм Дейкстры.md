# Алгоритм Дейкстры — Поиск кратчайшего пути

Алгоритм Дейкстры — это классический алгоритм для нахождения **кратчайшего пути** от одной вершины до всех остальных в **взвешенном графе** с **неотрицательными весами ребер**.

Он широко используется в:

- GPS-навигации и маршрутизации
- Планировании сетей
- Игровых алгоритмах поиска пути

---

## 1. Основная идея алгоритма

1. Начинаем с исходной вершины, присваиваем ей расстояние `0`. Всем остальным вершинам присваиваем расстояние `∞`.
2. Выбираем вершину с минимальным расстоянием среди непосещённых.
3. Обновляем расстояния до её соседей:  
```python
dist[neighbor] = min(dist[neighbor], dist[current] + weight(current, neighbor))
```
4. Помечаем вершину как посещённую.
5. Повторяем, пока все вершины не будут посещены или пока минимальное расстояние не станет `∞`.

---

## 2. Реализация на Python (с использованием очереди с приоритетом)

Используем `heapq` для эффективного выбора вершины с минимальным расстоянием.

```python
import heapq

def dijkstra(graph, start):
    # Инициализация расстояний
    distances = {vertex: float('inf') for vertex in graph}
    distances[start] = 0
    
    # Очередь с приоритетом (min-heap)
    priority_queue = [(0, start)]
    
    while priority_queue:
        current_distance, current_vertex = heapq.heappop(priority_queue)
        
        # Если текущее расстояние больше уже найденного, пропускаем
        if current_distance > distances[current_vertex]:
            continue
        
        # Обновляем расстояния соседей
        for neighbor, weight in graph[current_vertex].items():
            distance = current_distance + weight
            if distance < distances[neighbor]:
                distances[neighbor] = distance
                heapq.heappush(priority_queue, (distance, neighbor))
    
    return distances

# Пример графа (взвешенный)
graph = {
    'A': {'B': 4, 'C': 2},
    'B': {'A': 4, 'C': 5, 'D': 10},
    'C': {'A': 2, 'B': 5, 'D': 3},
    'D': {'B': 10, 'C': 3}
}

distances = dijkstra(graph, 'A')
print(distances)
```

**Вывод:**

```
{'A': 0, 'B': 4, 'C': 2, 'D': 5}
```

Это показывает кратчайшие расстояния от вершины `A` до всех остальных вершин.

---

## 3. Особенности алгоритма Дейкстры

- Работает только с **неотрицательными весами ребер**.
    
- Использование очереди с приоритетом (`heapq`) ускоряет поиск минимального расстояния до `O((V+E) log V)`.
    
- Можно модифицировать, чтобы не только находить расстояния, но и строить **пути** до каждой вершины.
    

---

## 4. Сравнение с BFS

|Характеристика|BFS|Дейкстра|
|---|---|---|
|Используемые веса|Только неважные (или одинаковые)|Любые неотрицательные|
|Цель|Посещение всех вершин, поиск кратчайшего пути в невзвешенном графе|Кратчайший путь в взвешенном графе|
|Структура|Очередь|Очередь с приоритетом (min-heap)|
|Сложность|O(V + E)|O((V + E) log V)|
