# Очередь

**Очередь** — это линейная структура данных, которая работает по принципу **FIFO** (_First In, First Out_ — «первым пришёл, первым вышел»). Это означает, что элемент, добавленный первым, будет извлечён первым.

Представьте очередь в магазине: первый человек в очереди обслуживается первым.

- **Добавление элемента (enqueue):** как встать в конец очереди.
- **Удаление элемента (dequeue):** как выйти из начала очереди.

---

## Основные понятия

- **Front (голова):** позиция первого элемента в очереди (отсюда происходит удаление).
- **Rear (хвост):** позиция последнего добавленного элемента (сюда добавляются новые элементы).
- **Size (размер):** текущее количество элементов в очереди.

---

## Типы очередей

### 1. Обычная очередь (Simple Queue)

- Работает строго по принципу FIFO.
- Добавление — в конец, удаление — из начала.

---

### 2. Кольцевая очередь (Circular Queue)

- Использует массив, но при достижении конца массива индексы «замыкаются» в начало.
- Позволяет эффективно использовать память.
- Решает проблему «ложного переполнения» в простой очереди на массиве.

---

### 3. Двусторонняя очередь (Deque — Double Ended Queue)

- Позволяет добавлять и удалять элементы как с начала, так и с конца.
- Может работать как стек и как очередь.

---

## Основные операции очереди

|Операция|Описание|
|---|---|
|`enqueue(x)`|Добавляет элемент `x` в конец очереди.|
|`dequeue()`|Удаляет и возвращает элемент из начала очереди.|
|`peek()`|Возвращает первый элемент без удаления.|
|`isEmpty()`|Возвращает `true`, если очередь пуста.|
|`size()`|Возвращает количество элементов.|
|`isFull()`|Для фиксированной очереди: проверяет переполнение.|

---

## Overflow и Underflow

- **Overflow** — попытка добавить элемент в заполненную очередь фиксированного размера.
- **Underflow** — попытка удалить элемент из пустой очереди.

---

## Применение очереди

Очередь широко используется в программировании:

1. **Планирование задач (Scheduling)**  
    Операционные системы используют очередь для управления процессами.
2. **Обработка запросов**  
    Серверы обрабатывают входящие запросы в порядке их поступления.
3. **Поиск в ширину (BFS)**  
    В алгоритмах на графах используется очередь для обхода уровней.
4. **Буферизация данных**  
    Например, в потоковой передаче данных.
5. **Моделирование реальных процессов**  
    Очереди клиентов, пакетов в сети и т.д.

## Реализация очереди на массиве (Python, фиксированный размер)

```python
class Queue:
    def __init__(self, capacity):
        self.capacity = capacity
        self.arr = [0] * capacity
        self.front = 0
        self.rear = -1
        self.size = 0

    def enqueue(self, x):
        if self.size == self.capacity:
            print("Queue Overflow")
            return
        self.rear += 1
        self.arr[self.rear] = x
        self.size += 1

    def dequeue(self):
        if self.size == 0:
            print("Queue Underflow")
            return -1
        val = self.arr[self.front]
        self.front += 1
        self.size -= 1
        return val

    def peek(self):
        if self.size == 0:
            print("Queue is Empty")
            return -1
        return self.arr[self.front]

    def isEmpty(self):
        return self.size == 0

    def isFull(self):
        return self.size == self.capacity
```

## Кольцевая очередь (Circular Queue, Python)

```python
class CircularQueue:
    def __init__(self, capacity):
        self.capacity = capacity
        self.arr = [0] * capacity
        self.front = 0
        self.rear = -1
        self.size = 0

    def enqueue(self, x):
        if self.size == self.capacity:
            print("Queue Overflow")
            return
        self.rear = (self.rear + 1) % self.capacity
        self.arr[self.rear] = x
        self.size += 1

    def dequeue(self):
        if self.size == 0:
            print("Queue Underflow")
            return -1
        val = self.arr[self.front]
        self.front = (self.front + 1) % self.capacity
        self.size -= 1
        return val

    def peek(self):
        if self.size == 0:
            print("Queue is Empty")
            return -1
        return self.arr[self.front]
```

## Динамическая очередь (на основе списка Python)

```python
class DynamicQueue:
    def __init__(self):
        self.arr = []

    def enqueue(self, x):
        self.arr.append(x)

    def dequeue(self):
        if not self.arr:
            print("Queue Underflow")
            return -1
        return self.arr.pop(0)

    def peek(self):
        if not self.arr:
            print("Queue is Empty")
            return -1
        return self.arr[0]

    def isEmpty(self):
        return len(self.arr) == 0

    def size(self):
        return len(self.arr)
```

## Динамическая очередь на основе связного списка (Python)

```python
# Узел связного списка
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

# Очередь на основе связного списка
class Queue:
    def __init__(self):
        self.front = None
        self.rear = None

    def enqueue(self, data):
        new_node = Node(data)
        if self.rear is None:
            self.front = self.rear = new_node
            return
        self.rear.next = new_node
        self.rear = new_node

    def dequeue(self):
        if self.front is None:
            print("Queue is empty")
            return None
        temp = self.front
        self.front = self.front.next
        if self.front is None:
            self.rear = None
        return temp.data

    def peek(self):
        if self.front is None:
            print("Queue is empty")
            return None
        return self.front.data

    def is_empty(self):
        return self.front is None
```

## Сравнение: фиксированная очередь vs динамическая

|Свойство|Фиксированная очередь|Динамическая очередь|
|---|---|---|
|Ёмкость|Фиксированная|Растёт автоматически|
|Overflow|Возможен|Нет (пока есть память)|
|Эффективность|Быстрее (без перевыделения)|Может быть медленнее при расширении|
|Гибкость|Ограничена размером|Более гибкая|
