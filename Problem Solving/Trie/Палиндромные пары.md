## Определение

**Палиндромные пары** — это задача, в которой дан массив строк. Требуется определить, существует ли пара индексов `(i, j)` (i ≠ j) такая, что конкатенация `words[i] + words[j]` является палиндромом. Это классическая задача на использование Trie для эффективного поиска без перебора всех пар.

---

## Постановка задачи

**Вход:** массив строк `words` (длина n, суммарная длина L).  
**Выход:** `True`, если существует пара (i, j) с i ≠ j, такая что `words[i] + words[j]` является палиндромом, иначе `False`.

**Ограничения:** суммарная длина слов до 10⁵–10⁶.

---

## Идея решения (Trie)

Наивный подход — перебор всех пар O(n² · L) — слишком медленный. Используем **Trie**, в который вставляются **перевёрнутые** слова. Это позволяет быстро проверять, существует ли слово, которое может дополнить данное до палиндрома.

### Ключевые наблюдения

Для двух строк `A` и `B`, конкатенация `A + B` является палиндромом, если:

1. Одна строка является перевёрнутой версией другой. Тогда `A + reverse(A)` — палиндром (если `A` пуста?).
2. Одна строка содержит перевёрнутую версию другой как префикс, а оставшаяся часть сама является палиндромом.

Например, `A = "abc"`, `B = "ba"` → `"abc" + "ba" = "abcba"` — палиндром. Здесь `B` — это перевёрнутый префикс "ab" плюс палиндром "c"? На самом деле, более общий случай: если `B` можно разбить на `reverse(A)` и `palindrome`, то `A + B` будет палиндромом.

Алгоритм с Trie:

1. Вставляем все слова в Trie **в обратном порядке**. При вставке для каждого узла храним:
   - `idx` — индекс слова, которое заканчивается в этом узле (если есть).
   - `indices` — список индексов слов, для которых оставшаяся часть (суффикс от текущей позиции до конца слова) является палиндромом. Это нужно для случая, когда при поиске мы достигаем конца слова, а оставшаяся часть текущего слова (от текущей позиции до конца) должна быть палиндромом.

2. Для каждого слова `word` выполняем поиск в Trie:
   - Проходим по символам `word` слева направо.
   - Если в процессе обхода мы встречаем узел, где `idx != -1` (т.е. некоторое слово заканчивается здесь) и оставшаяся часть `word` от текущей позиции до конца является палиндромом, то пара существует.
   - После прохода всех символов, если в узле есть другие слова (т.е. `indices` содержит индексы, отличные от текущего), то пара существует (это случай, когда одно слово является обратным другому).

3. Также нужно учитывать случай, когда текущее слово является палиндромом, и есть пустая строка, но в задаче обычно слова непустые.

---

## Алгоритм (по шагам)

1. Определить класс `TrieNode`:
   - `children[26]` — массив дочерних узлов.
   - `idx` — индекс слова, заканчивающегося в этом узле (-1, если нет).
   - `indices` — список индексов слов, для которых суффикс от текущего узла до конца является палиндромом.

2. Функция `isPalindrome(word, i, j)` — проверяет, является ли подстрока `word[i..j]` палиндромом.

3. Вставка:
   - Для каждого слова `word` с индексом `idx` проходим по символам с конца к началу.
   - На каждом шаге проверяем, является ли префикс исходного слова (от начала до текущей позиции i) палиндромом. Если да, добавляем `idx` в `indices` текущего узла.
   - Создаём дочерние узлы для текущего символа.
   - После завершения цикла помечаем `node.idx = idx` и добавляем `idx` в `indices` (для пустого суффикса).

4. Поиск:
   - Для каждого слова `word` с индексом `idx`:
     - Начинаем с корня.
     - Проходим по символам `word` слева направо.
     - Если текущий узел содержит `idx != -1` (т.е. какое-то слово заканчивается здесь) и этот `idx` не равен текущему, и оставшаяся часть `word` (от текущей позиции до конца) является палиндромом, возвращаем `True`.
     - Если дочернего узла для текущего символа нет, прерываем поиск.
     - Переходим к дочернему узлу.
   - После прохода всех символов, если у узла есть индексы в `indices`, отличные от текущего, возвращаем `True`.

## Реализация на Python

```python
class TrieNode:
    def __init__(self):
        self.children = [None] * 26
        self.idx = -1               # индекс слова, заканчивающегося здесь
        self.indices = []           # индексы слов, у которых суффикс палиндром

def is_palindrome(s, l, r):
    while l < r:
        if s[l] != s[r]:
            return False
        l += 1
        r -= 1
    return True

def insert(root, word, idx):
    node = root
    # вставляем слово в обратном порядке
    for i in range(len(word) - 1, -1, -1):
        # если префикс word[0..i] является палиндромом,
        # то это слово может образовать пару с тем, что заканчивается здесь
        if is_palindrome(word, 0, i):
            node.indices.append(idx)
        ch = ord(word[i]) - ord('a')
        if not node.children[ch]:
            node.children[ch] = TrieNode()
        node = node.children[ch]
    node.idx = idx
    node.indices.append(idx)

def search(root, word, idx):
    node = root
    for i, ch in enumerate(word):
        # если текущий узел содержит слово и оставшаяся часть word[i:] — палиндром
        if node.idx != -1 and node.idx != idx and is_palindrome(word, i, len(word) - 1):
            return True
        c = ord(ch) - ord('a')
        if not node.children[c]:
            return False
        node = node.children[c]
    # после окончания слова проверяем, есть ли другие слова, заканчивающиеся здесь
    for other_idx in node.indices:
        if other_idx != idx:
            return True
    return False

def palindrome_pair_exists(words):
    root = TrieNode()
    for i, w in enumerate(words):
        insert(root, w, i)
    for i, w in enumerate(words):
        if search(root, w, i):
            return True
    return False

# Пример
words = ["geekf", "geeks", "or", "keeg", "abc", "bc"]
print(palindrome_pair_exists(words))  # True
```

## Сложность

- **Время:** O(n · k²), где n — количество слов, k — средняя длина слова (проверка палиндрома для каждой позиции при вставке). На практике O(сумма длин слов * средняя длина).
- **Память:** O(n · k) для хранения Trie.

## Примечания

- Задача известна как [LeetCode 336. Palindrome Pairs](https://leetcode.com/problems/palindrome-pairs/).
- В этой реализации мы проверяем наличие любой пары (не возвращая сами пары). Для возврата пар нужно модифицировать `search` для сбора индексов.
- Важно правильно обрабатывать пустые строки (если они есть). В коде они будут обработаны как слова длины 0.
- Альтернативный подход: использование хеш-таблицы с перевёрнутыми строками, но Trie эффективнее при большом количестве слов.
