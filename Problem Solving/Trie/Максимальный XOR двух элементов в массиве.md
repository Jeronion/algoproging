## Определение

**Задача о максимальном XOR двух элементов в массиве** заключается в том, чтобы найти такую пару элементов `(a, b)` из заданного массива целых чисел, для которой значение `a XOR b` является максимальным среди всех возможных пар. XOR (исключающее ИЛИ) — это бинарная операция, которая возвращает 1, если биты различны, и 0, если одинаковы. Максимизация XOR эквивалентна поиску пары чисел, у которых биты на каждой позиции различаются как можно больше, начиная со старших битов.

**Пример:**
```
arr = [26, 100, 25, 13, 4, 14]
Максимальный XOR: 126 (например, 100 XOR 26 = 126)
```

---

## Постановка задачи

**Вход:** массив целых неотрицательных чисел `arr` (длина n).  
**Выход:** максимальное значение XOR, достигаемое для любой пары различных элементов (i ≠ j). Если массив содержит менее двух элементов, результат может быть 0 (или не определено).

**Ограничения:** типично n до 10⁵, значения до 10⁹ (32-битные).

---

## Идея решения

Для нахождения максимального XOR двух чисел можно использовать **префиксное дерево (Trie)** для битовых представлений чисел. Основная идея: для каждого числа мы хотим найти другое число, которое имеет как можно больше противоположных битов начиная со старшего (самого значимого) бита. Это эквивалентно тому, что мы строим дерево, где каждый узел представляет бит (0 или 1), и путь от корня к листу соответствует двоичному представлению числа (от старшего бита к младшему).

Алгоритм:
1. Вставляем все числа в Trie (по битам, начиная с самого старшего).
2. Для каждого числа `x` в массиве, проходим по Trie и пытаемся на каждом уровне выбрать противоположный бит (если он существует), чтобы максимизировать XOR. Если противоположный бит отсутствует, идём по существующему.
3. Суммируем выбранные биты (их веса `1 << i`) для получения XOR-значения для данной пары.
4. Обновляем глобальный максимум.

**Почему это работает?**  
Жадный выбор на каждом уровне (старшие биты имеют больший вес) гарантирует максимальный XOR, поскольку старшие биты доминируют над младшими. Если на каком-то уровне существует противоположный бит, мы всегда выбираем его, так как это даёт больший вклад, чем любые комбинации младших битов.

## Реализация на Python

```python
class Node:
    def __init__(self):
        self.zero = None  # ребро для бита 0
        self.one = None   # ребро для бита 1

class Trie:
    def __init__(self):
        self.root = Node()

    def insert(self, num):
        curr = self.root
        for i in range(31, -1, -1):  # от старшего бита (31) до младшего (0)
            bit = (num >> i) & 1
            if bit == 0:
                if not curr.zero:
                    curr.zero = Node()
                curr = curr.zero
            else:
                if not curr.one:
                    curr.one = Node()
                curr = curr.one

    def max_xor_with(self, num):
        curr = self.root
        res = 0
        for i in range(31, -1, -1):
            bit = (num >> i) & 1
            if bit == 0:
                # хотим найти 1, чтобы получить XOR = 1
                if curr.one:
                    curr = curr.one
                    res |= (1 << i)
                else:
                    curr = curr.zero
            else:
                # хотим найти 0
                if curr.zero:
                    curr = curr.zero
                    res |= (1 << i)
                else:
                    curr = curr.one
        return res

def max_xor(arr):
    if len(arr) < 2:
        return 0
    trie = Trie()
    trie.insert(arr[0])
    best = 0
    for num in arr[1:]:
        best = max(best, trie.max_xor_with(num))
        trie.insert(num)
    return best
```

## Сложность

- **Время:** O(n · 32) = O(n) с учётом константного числа бит (обычно 32). Для каждого числа мы выполняем вставку и поиск за O(32).
- **Память:** O(n · 32) для хранения Trie (количество узлов пропорционально общему числу бит вставленных чисел). В худшем случае O(n·32).

---

## Примечания

- Алгоритм работает для неотрицательных чисел. Для отрицательных чисел в Python битовые операции сдвига дают бесконечную последовательность единиц, что нарушит алгоритм, поэтому следует использовать маску или преобразовывать в беззнаковые.
- Количество бит может быть увеличено (например, 31 для чисел ≤ 2³¹-1). Если числа могут быть до 10⁹, достаточно 30 бит (0..30), но 31 безопасно.
- Можно также решить задачу с помощью сортировки и построения максимального XOR с использованием дерева отрезков, но Trie является наиболее элегантным и быстрым.
