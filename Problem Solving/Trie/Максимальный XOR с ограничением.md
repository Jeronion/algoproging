## Определение

**Задача о максимальном XOR с ограничением** формулируется следующим образом: дан массив целых чисел `arr` и список запросов, каждый из которых содержит пару `(xi, mi)`. Для каждого запроса требуется найти максимальное значение `arr[j] XOR xi` среди всех элементов `arr[j]` таких, что `arr[j] ≤ mi`. Если таких элементов нет, ответ должен быть `-1`. Эта задача является классическим примером использования бинарного Trie (префиксного дерева по битам) для эффективного поиска максимального XOR, а также применения сортировки и двух указателей для обработки ограничений.

**Пример:**
```
arr = [0, 1, 2, 3, 4]
queries = [[3, 1], [1, 3], [5, 6]]
Ответы: [3, 3, 7]
```

---

## Постановка задачи

**Вход:**
- `arr` — массив целых чисел (длина n).
- `queries` — список пар `[xi, mi]` (длина q).

**Выход:** массив `ans` длины q, где `ans[i]` — максимальное значение `(arr[j] XOR xi)` при условии `arr[j] ≤ mi`. Если таких элементов нет, то `-1`.

**Ограничения:** типичные для LeetCode 1707: n, q ≤ 10⁵, значения ≤ 10⁹.

---

## Идея решения

Наивное решение — перебирать все элементы массива для каждого запроса — даёт O(n·q) и неэффективно. Оптимизация достигается за счёт использования **Trie** для хранения битового представления чисел и **сортировки** запросов по ограничению `mi`.

### Ключевые наблюдения

1. Для поиска максимального XOR с числом `x` среди множества чисел используется Trie, хранящий биты чисел от старшего (31) к младшему. Жадный алгоритм на Trie позволяет найти число, дающее максимальный XOR за O(32) шагов, выбирая на каждом бите противоположное значение, если оно существует.
2. Если обрабатывать запросы в порядке возрастания `mi`, мы можем постепенно добавлять в Trie только те элементы массива, которые не превосходят текущего `mi`. Это делается с помощью указателя на отсортированный массив.
3. Таким образом, каждый элемент массива вставляется в Trie ровно один раз, а каждый запрос обрабатывается за O(32). Сортировка занимает O(n log n + q log q), что даёт общую сложность O((n+q) log(n+q)).

## Алгоритм (по шагам)

1. **Сортировка массива `arr`** по возрастанию.
2. **Подготовка запросов**: для каждого запроса `(xi, mi)` создаём кортеж `(mi, xi, index)`, где `index` — исходный индекс запроса. Сортируем эти кортежи по `mi`.
3. **Создание пустого Trie** (корень с двумя возможными переходами: левый — бит 0, правый — бит 1).
4. **Инициализация указателя `j = 0`** на отсортированный массив.
5. Для каждого запроса в отсортированном порядке:
   - Пока `j < n` и `arr[j] <= mi`:
     - Вставляем `arr[j]` в Trie (по битам от 31 до 0).
     - Увеличиваем `j`.
   - Если `j == 0` (ни один элемент не был добавлен), ответ для этого запроса равен `-1`.
   - Иначе вычисляем максимальный XOR для `xi` с помощью Trie (жадный обход от старших битов к младшим, выбирая противоположный бит, если он есть).
   - Сохраняем результат в массив `ans` по исходному индексу.
6. Возвращаем `ans`.

### Вставка в Trie

- Узел содержит два указателя: `left` (для бита 0) и `right` (для бита 1).
- Вставка числа: проходим по битам с 31 по 0, создаём нужные узлы.

### Поиск максимального XOR

- Для числа `x` проходим по битам с 31 по 0.
- На каждом бите стараемся выбрать противоположный бит (если текущий бит 0, хотим 1; если 1, хотим 0).
- Если такой узел существует, переходим в него и добавляем соответствующий бит (1 << i) к результату.
- Иначе переходим в узел с тем же битом (результат не увеличивается).

## Реализация на Python

```python
class TrieNode:
    def __init__(self):
        self.left = None   # для бита 0
        self.right = None  # для бита 1

class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, num):
        node = self.root
        for i in range(31, -1, -1):
            bit = (num >> i) & 1
            if bit == 0:
                if node.left is None:
                    node.left = TrieNode()
                node = node.left
            else:
                if node.right is None:
                    node.right = TrieNode()
                node = node.right

    def max_xor(self, x):
        node = self.root
        res = 0
        for i in range(31, -1, -1):
            bit = (x >> i) & 1
            if bit == 0:
                # хотим 1
                if node.right is not None:
                    node = node.right
                    res |= (1 << i)
                else:
                    node = node.left
            else:
                # хотим 0
                if node.left is not None:
                    node = node.left
                    res |= (1 << i)
                else:
                    node = node.right
        return res

def max_xor_queries(arr, queries):
    n = len(arr)
    q = len(queries)
    # сортируем массив
    arr.sort()
    # подготавливаем запросы: (mi, xi, index)
    qry = [(queries[i][1], queries[i][0], i) for i in range(q)]
    qry.sort()
    ans = [0] * q
    trie = Trie()
    j = 0  # указатель в отсортированном массиве

    for mi, xi, idx in qry:
        while j < n and arr[j] <= mi:
            trie.insert(arr[j])
            j += 1
        if j == 0:
            ans[idx] = -1
        else:
            ans[idx] = trie.max_xor(xi)
    return ans

# Пример
arr = [0, 1, 2, 3, 4]
queries = [[3, 1], [1, 3], [5, 6]]
print(max_xor_queries(arr, queries))  # [3, 3, 7]
```

## Сложность

- **Время:** O((n+q) log(n+q) + (n+q)·32). Сортировка массивов O(n log n + q log q), вставка и поиск в Trie O(32) на элемент/запрос.
- **Память:** O(n·32) для хранения Trie в худшем случае (если все числа различны и не имеют общих префиксов). На практике память меньше.

## Примечания

- Задача известна как [LeetCode 1707. Maximum XOR With an Element From Array](https://leetcode.com/problems/maximum-xor-with-an-element-from-array/).
- Для работы с отрицательными числами необходимо учитывать знаковый бит, но в задаче обычно числа неотрицательные (или можно сдвигать диапазон).
- Вместо явного дерева можно использовать массив для хранения ссылок, но класс `TrieNode` с двумя полями делает код более понятным.
- Сортировка запросов по `mi` позволяет избежать повторной вставки элементов в Trie для каждого запроса.
