## Определение

**Задача о подсчёте различных подстрок** заключается в определении количества уникальных (неповторяющихся) подстрок заданной строки. Подстрока — это непрерывная последовательность символов. Две подстроки считаются различными, если их содержимое различается, даже если они начинаются в разных позициях исходной строки. Пустая подстрока не учитывается.

**Пример:**
```
s = "ababa"
Все различные подстроки: "a", "b", "ab", "ba", "aba", "bab", "abab", "baba", "ababa" → 9.
s = "aaa"
Подстроки: "a", "aa", "aaa" → 3.
```

---

## Постановка задачи

**Вход:** строка `s` длины n (1 ≤ n ≤ 3000), состоящая из строчных латинских букв.  
**Выход:** количество различных непустых подстрок (целое число).

---

## Идея решения (Trie)

Ключевая идея: все различные подстроки строки можно представить как все префиксы всех суффиксов строки. Например, для `"ababa"` суффиксы: `"ababa"`, `"baba"`, `"aba"`, `"ba"`, `"a"`. Префиксы каждого суффикса дают все возможные подстроки. Если мы будем вставлять все префиксы всех суффиксов в **префиксное дерево (Trie)**, то каждая новая вставка будет добавлять ровно одну новую подстроку (когда мы создаём новый узел). Таким образом, общее количество созданных узлов в Trie (кроме корня) будет равно числу различных подстрок.

### Почему это работает?

Trie (префиксное дерево) естественным образом удаляет дубликаты: если подстрока уже встречалась ранее, путь к ней уже существует в дереве, и новый узел не создаётся. Каждый раз, когда мы создаём новый узел при вставке, это означает, что мы встретили подстроку, которая не встречалась раньше. Поэтому количество созданных узлов равно количеству различных подстрок.

### Алгоритм

1. Создать корень Trie.
2. Инициализировать счётчик `count = 0`.
3. Для каждого стартового индекса `i` от 0 до n-1 (суффикс `s[i:]`):
   - Установить текущий узел в корень.
   - Для каждого символа `j` от `i` до n-1:
     - Вычислить индекс символа `idx = ord(s[j]) - ord('a')`.
     - Если у текущего узла нет ребёнка для `idx`, то создать нового ребёнка и увеличить `count` на 1.
     - Перейти к этому ребёнку.
4. Вернуть `count`.

## Реализация на Python

```python
class TrieNode:
    def __init__(self):
        self.children = [None] * 26

class Solution:
    def countSubs(self, s: str) -> int:
        root = TrieNode()
        count = 0
        n = len(s)
        
        for i in range(n):
            curr = root
            for j in range(i, n):
                idx = ord(s[j]) - ord('a')
                if curr.children[idx] is None:
                    curr.children[idx] = TrieNode()
                    count += 1
                curr = curr.children[idx]
        
        return count
```

## Сложность

- **Время:** O(n²) — для каждого из n суффиксов мы проходим до конца строки, вставляя подстроки. В худшем случае (для строки с уникальными символами) это O(n²) операций.
- **Память:** O(n²) — в худшем случае количество различных подстрок может быть O(n²), что требует соответствующего количества узлов в Trie.

## Пример работы

Для `s = "ababa"`:

1. Суффикс `"ababa"`: вставляем "a", "ab", "aba", "abab", "ababa" → новые узлы: 5.
2. Суффикс `"baba"`: вставляем "b" (новый), "ba" (новый), "bab" (новый), "baba" (новый) → 4 новых.
3. Суффикс `"aba"`: вставляем "a" (уже есть), "ab" (есть), "aba" (есть) → 0 новых.
4. Суффикс `"ba"`: "b" (есть), "ba" (есть) → 0.
5. Суффикс `"a"`: "a" (есть) → 0.

Итого: 5 + 4 = 9. Ответ 9.

## Примечания

- Ограничение `n ≤ 3000` делает O(n²) время и память приемлемыми.
- Альтернативное решение: использование суффиксного массива или хеширования, но Trie проще для понимания и реализации.
- Задача известна на GeeksforGeeks как "Count of distinct substrings".
- Для более длинных строк (n > 5000) может потребоваться более эффективный подход (например, суффиксный автомат или суффиксное дерево).
