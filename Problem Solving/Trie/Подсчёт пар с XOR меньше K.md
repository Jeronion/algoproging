## Определение

**Задача о подсчёте пар с XOR меньше K** заключается в том, чтобы для заданного массива целых чисел и числа `K` найти количество пар `(i, j)` с `i < j`, для которых `arr[i] XOR arr[j] < K`. Эта задача часто встречается в задачах на битовые операции и эффективно решается с помощью битового Trie (префиксного дерева по битам). Использование Trie позволяет подсчитывать количество пар за линейное время от количества элементов, а не квадратичное.

**Пример:**
```
arr = [1, 2, 3, 4, 5]
K = 2
Пары с XOR < 2: (1,2) -> 3 (нет), (1,3) -> 2 (нет, т.к. <2? 2 не меньше 2), ... нужно посчитать.
```

---

## Постановка задачи

**Вход:**
- `arr` — массив целых чисел (длина n).
- `K` — целое неотрицательное число.

**Выход:** количество пар `(i, j)` (i < j), таких что `arr[i] XOR arr[j] < K`.

**Ограничения:** типичные: n до 10⁵, значения до 10⁹.

---

## Идея решения

Наивное решение — перебор всех пар O(n²) — неэффективно. Используется **битовое Trie**, которое хранит двоичные представления чисел (от старшего бита к младшему). Для каждого числа `x` мы сначала ищем в Trie количество уже вставленных чисел `y`, таких что `x XOR y < K`, а затем вставляем `x` в Trie. Таким образом, мы подсчитываем все пары ровно один раз.

### Как Trie помогает считать XOR < K?

При обходе битов чисел (от старшего к младшему) мы одновременно смотрим на биты числа `x` и числа `K`. На каждом уровне:

- Если бит `K` равен **1**, то:
  - Если мы выберем в Trie путь с тем же битом, что и `x` (т.е. XOR на этом бите даст 0), то всё число `x XOR y` автоматически станет меньше `K` (потому что на этом бите мы уже меньше). Значит, все числа, лежащие в этом поддереве, можно сразу добавить к ответу.
  - Затем мы продолжаем поиск в поддереве с противоположным битом (где XOR на этом бите даст 1), чтобы проверить возможность равенства `K` на старших битах и потенциально меньшего на младших.

- Если бит `K` равен **0**, то:
  - Чтобы итоговый XOR был меньше `K`, на этом бите XOR должен быть 0 (иначе он станет больше). Значит, мы можем идти только в поддерево с таким же битом, как у `x`. Если такого поддерева нет, поиск прерывается.

После обхода всех битов мы получаем количество чисел `y`, для которых `x XOR y < K`. Суммируем это для всех `x` (при этом каждое `y` было вставлено ранее, так что каждая пара учитывается один раз).

## Алгоритм (пошагово)

1. Создать корень Trie. Каждый узел содержит два дочерних указателя (0 и 1) и счётчик `cnt` — количество чисел, проходящих через этот узел.
2. Инициализировать `result = 0`.
3. Для каждого числа `x` из массива (в порядке обхода):
   - Вызвать функцию `countLess(root, x, K)`, которая возвращает количество уже вставленных чисел `y` с `x XOR y < K`.
   - Добавить это значение к `result`.
   - Вставить `x` в Trie (обновляя счётчики на каждом уровне).
4. Вернуть `result`.

### Функция `countLess(root, x, K)`

- Начинаем с корня.
- Для бита `i` от 31 до 0 (или до максимального бита чисел):
  - Если `root` стал `None`, прерываем цикл.
  - `bit_x = (x >> i) & 1`
  - `bit_k = (K >> i) & 1`
  - Если `bit_k == 1`:
    - Если существует дочерний узел с битом `bit_x`, то все числа в этом поддереве дадут XOR с текущим битом 0, что уже меньше `K` (так как на этом бите у `K` единица, а у нас ноль). Добавляем к ответу `cnt` этого поддерева.
    - Затем переходим в поддерево с противоположным битом (`1 - bit_x`), чтобы продолжить поиск чисел, у которых XOR на этом бите равен 1 (равен биту `K`).
  - Если `bit_k == 0`:
    - Чтобы XOR оставался равным 0 на этом бите, нужно идти в поддерево с битом `bit_x`. Если его нет, поиск заканчивается.
- По окончании цикла возвращаем накопленное количество.

## Реализация на Python

```python
class TrieNode:
    def __init__(self):
        self.child = [None, None]   # index 0 for bit 0, 1 for bit 1
        self.cnt = 0                # количество чисел в этом узле

def insert_trie(root, num):
    node = root
    for i in range(31, -1, -1):
        bit = (num >> i) & 1
        if node.child[bit] is None:
            node.child[bit] = TrieNode()
        node = node.child[bit]
        node.cnt += 1

def count_less(root, x, k):
    node = root
    res = 0
    for i in range(31, -1, -1):
        if node is None:
            break
        bit_x = (x >> i) & 1
        bit_k = (k >> i) & 1
        if bit_k == 1:
            # Если есть поддерево с таким же битом как у x,
            # то все числа там дают XOR < K на этом бите
            if node.child[bit_x] is not None:
                res += node.child[bit_x].cnt
            # Идём в противоположное поддерево, чтобы XOR был равен 1
            node = node.child[1 - bit_x]
        else:
            # Идём только в поддерево с таким же битом
            node = node.child[bit_x]
    return res

def count_pairs_xor_less(arr, k):
    root = TrieNode()
    ans = 0
    for num in arr:
        ans += count_less(root, num, k)
        insert_trie(root, num)
    return ans
```

## Сложность

- **Время:** O(n · 32) = O(n), где n — количество элементов, так как каждый элемент вставляется и обрабатывается за 32 шага (фиксированное количество бит).
- **Память:** O(n · 32) в худшем случае для хранения Trie (каждое число может создать до 32 новых узлов).

## Пример работы

```
arr = [1, 2, 3, 4, 5]
K = 2
```
- Вставляем 1: count = 0, вставляем.
- 2: count_less с 2: ищем числа y с 2 XOR y < 2. Проверяем: 2 XOR 1 = 3 (не <2), так что count=0. Вставляем 2.
- 3: 3 XOR 1 = 2 (не <2), 3 XOR 2 = 1 (<2) -> count=1. Вставляем 3.
- 4: 4 XOR 1 = 5, 4 XOR 2 = 6, 4 XOR 3 = 7 -> все >2, count=0.
- 5: 5 XOR 1 = 4, 5 XOR 2 = 7, 5 XOR 3 = 6, 5 XOR 4 = 1 (<2) -> count=1 (пара (4,5)). Итого ans = 0+0+1+0+1 = 2.

## Примечания

- Работает для неотрицательных чисел. Если числа могут быть отрицательными, нужно либо сдвигать диапазон (добавлять смещение), либо рассматривать их как беззнаковые 32-битные.
- Вместо фиксированных 31 бит можно использовать 30 или 63 в зависимости от диапазона. Обычно 31 бит достаточно для чисел до 2³¹-1.
- Trie эффективен, но требует памяти. При n до 10⁵ память приемлема.
