## Определение

Дана матрица размером `m × n`, изначально заполненная числами от 1 до `m·n` в **row‑major** порядке (по строкам). Выполняется последовательность операций трёх типов:

- **type = 1** — поменять местами строки `a` и `b`.
- **type = 2** — поменять местами столбцы `a` и `b`.
- **type = 3** — запросить значение в ячейке `(a, b)` после всех предыдущих операций.

Требуется вернуть массив ответов на все запросы типа 3. Количество операций `q` может достигать 10⁵, а размер матрицы — до 10⁹ ячеек, поэтому **строить саму матрицу невозможно**.

---

## Идея решения

Вместо того чтобы физически перемещать строки и столбцы, мы будем отслеживать, **куда переместилась каждая исходная строка и каждый исходный столбец** под действием перестановок.

Пусть:

- `rows[i]` — это индекс исходной строки, которая **сейчас** находится на позиции `i` (т.е. `rows` — это перестановка, отображающая текущую позицию → исходную строку). Изначально `rows[i] = i`.
- `cols[j]` — аналогично для столбцов: `cols[j]` — исходный столбец, который сейчас находится на позиции `j`.

При обмене строк `a` и `b` меняются местами значения `rows[a]` и `rows[b]`. При обмене столбцов — `cols[a]` и `cols[b]`.

Для запроса значения в позиции `(a, b)` мы:
1. Определяем, какая исходная строка попала в текущую строку `a`: `orig_row = rows[a]`.
2. Определяем, какой исходный столбец попал в текущий столбец `b`: `orig_col = cols[b]`.
3. Значение в этой ячейке равно `orig_row * n + orig_col + 1` (при 0‑индексации).

### Почему это работает?

Изначально значение в ячейке `(i, j)` равно `i·n + j + 1`. Перестановка строк и столбцов линейно изменяет координаты: после серии обменов значение в текущей позиции `(a, b)` равно `(исходная_строка(a))·n + (исходный_столбец(b)) + 1`.

---

## Реализация

```python
class Solution:
    def solveQueries(self, m, n, queries):
        rows = list(range(m))
        cols = list(range(n))
        res = []
        for typ, a, b in queries:
            if typ == 1:
                rows[a], rows[b] = rows[b], rows[a]
            elif typ == 2:
                cols[a], cols[b] = cols[b], cols[a]
            else:
                res.append(rows[a] * n + cols[b] + 1)
        return res
```

---

## Сложность

- **Время:** O(q) — каждая операция выполняется за O(1).
- **Память:** O(m + n) — для хранения перестановок строк и столбцов.

---

## Примечания

- Метод основан на разделении перестановок строк и столбцов.
- Если m и n очень велики (например, 10⁹), необходимо использовать словари для хранения только изменённых индексов. Но в рамках учебного примера код предполагает, что размеры позволяют создать списки.
- Этот подход избегает явного построения матрицы, что было бы невозможно при больших размерах.
