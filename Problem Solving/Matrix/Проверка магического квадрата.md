## Определение

**Магический квадрат** порядка `n` — это квадратная матрица размера `n × n`, заполненная различными целыми числами от 1 до `n²` включительно, такая что суммы чисел в каждой строке, каждом столбце и обеих главных диагоналях равны одному и тому же числу — **магической константе**.

Магическая константа вычисляется по формуле:
$$
S = \frac{n(n^2 + 1)}{2}
$$

---

## Условие

Дан двумерный массив `mat` размера `n × n`. Требуется проверить, является ли он магическим квадратом. Функция `magicSquare` возвращает `True`, если матрица удовлетворяет всем условиям, иначе `False`.

---

## Алгоритм (по шагам)

В коде реализована следующая логика:

1. **Вычисление магической константы** `s = n·(n²+1)//2`.

2. **Проверка строк, столбцов и диагоналей**:
   - Проходим по всем строкам `i`:
     - Суммируем элементы главной диагонали (`mat[i][i]`) и побочной (`mat[i][n-i-1]`).
     - Если сумма текущей строки не равна `s` → `False`.
     - Суммируем элементы столбца `i` (итерируем по `j` и добавляем `mat[j][i]`). Если сумма не равна `s` → `False`.
   - После цикла проверяем, что суммы диагоналей равны `s`.

3. **Проверка диапазона значений**:
   - В процессе проверки столбца также убеждаемся, что каждый элемент `mat[i][j]` находится в пределах от 1 до `n²`. Если нет → `False`.

4. **Проверка уникальности элементов (Negative Marking)**:
   - Используется техника отрицательной маркировки: для каждого элемента `x` вычисляем его ожидаемые координаты в матрице как `( (x-1)//n , (x-1)%n )`. Поскольку числа должны быть от 1 до `n²` и все различны, каждое число соответствует уникальной позиции в нормализованной сетке.
   - Проверяем, что в ячейке `mat[n_i][n_j]` ещё не было отрицательного значения (если оно уже отрицательное, значит число уже встречалось → `False`).
   - Затем делаем это значение отрицательным (умножаем на -1), чтобы пометить как использованное.
   - После прохода всех элементов, если ни одно число не повторялось, возвращаем `True`.

## Реализация на Python

```python
class Solution:
    def magicSquare(self, mat):
        n = len(mat)
        s = (n ** 2) * ((n ** 2) + 1) // 2 // n   # магическая константа

        s_d1, s_d2 = 0, 0

        for i in range(n):
            s_d1 += mat[i][i]
            s_d2 += mat[i][n - i - 1]

            # проверка строки
            if sum(mat[i]) != s:
                return False

            # проверка столбца и диапазона
            s_i = 0
            for j in range(n):
	            # проверка числа на 0 < mat[i][j] < n ** 2 + 1
                if mat[i][j] < 1 or mat[i][j] > n * n:
                    return False
                s_i += mat[j][i]

            if s_i != s:
                return False

        # проверка диагоналей
        if s_d1 != s or s_d2 != s:
            return False

        # проверка уникальности (negative marking)
        for i in range(n):
            for j in range(n):
                x = mat[i][j]
                # если число уже отрицательное, значит его уже обрабатывали
                if x < 0:
                    return False
                # вычисляем "целевую" позицию для числа x
                ni, nj = (x - 1) // n, (x - 1) % n
                if mat[ni][nj] < 0:
                    return False
                mat[ni][nj] = -mat[ni][nj]   # помечаем как использованное

        return True
```

## Сложность

- **Время:** O(n²) — один проход по всем элементам для проверки строк, столбцов и уникальности.
- **Память:** O(1) — если не учитывать модификацию входной матрицы; фактически алгоритм изменяет исходную матрицу (ставит отрицательные значения), что может быть нежелательно. При необходимости можно создать копию.

## Примечания

- **Negative Marking** — эффективный способ проверки уникальности элементов в диапазоне 1..n² без использования дополнительной памяти. Идея в том, что каждому числу соответствует ровно одна ячейка в нормализованной сетке. Если при проходе мы встречаем число, которое должно было бы находиться в уже помеченной отрицательной ячейке, значит число повторяется.
- Важно, что исходная матрица модифицируется. Если нужно сохранить исходные данные, следует создать копию.
- Алгоритм не проверяет, что все числа от 1 до n² присутствуют, но отрицательная маркировка выявит как дубликаты, так и отсутствие чисел (поскольку некоторые ячейки останутся положительными после цикла, но на самом деле это не приведёт к `False`; в коде это не проверяется. Однако если все числа различны и лежат в диапазоне, то их ровно n², и все ячейки станут отрицательными. Отсутствие проверки на то, что все числа от 1 до n² использованы, может дать ложноположительный результат, если есть пропуски, но нет дубликатов. В классической задаче требуется именно полный набор, поэтому после negative marking стоило бы проверить, что все элементы стали отрицательными. В данном коде этого нет, что является недостатком.)
