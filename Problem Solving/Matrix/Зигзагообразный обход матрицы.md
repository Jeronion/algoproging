## Определение

Дан двумерный массив (матрица) размера `n × m`. Требуется вывести все его элементы в **зигзагообразном порядке** — то есть обойти все диагонали, меняя направление: сначала идём по диагонали вверх-вправо, затем по следующей диагонали вниз-влево, и так далее.

Пример для матрицы 3×3:

```
1 2 3
4 5 6
7 8 9
```

Зигзагообразный обход: `1 2 4 7 5 3 6 8 9`.  
(Диагонали: [1], [2,4], [7,5,3], [6,8], [9] — с чередованием направления.)

---

## Идея решения

Общее количество диагоналей в матрице равно `n + m - 1`. Каждая диагональ характеризуется постоянной суммой индексов `s = i + j` (от 0 до n+m-2). Для каждой диагонали нужно определить начальную точку и направление обхода.

- Если `s` чётное (или нечётное — зависит от выбора направления), то диагональ обходится снизу-вверх-вправо (т.е. от нижней границы к верхней).  
- Если `s` нечётное, то обход сверху-вниз-влево.

Однако в приведённом коде используется переменная `d`, которая переключается между 1 и -1, и начальные координаты вычисляются исходя из направления.

### Алгоритм (по шагам)

1. Определить `n = len(mat)`, `m = len(mat[0])`.
2. Инициализировать `d = 1` (направление: 1 — движение вверх-вправо, -1 — вниз-влево).
3. Для каждой диагонали `s` от `0` до `n+m-2`:
   - В зависимости от `d` вычислить начальные `i, j`:
     - Если `d == 1` (движение вверх-вправо), то начинаем с нижней границы диагонали:
       `i = min(s, n-1)`, `j = max(0, s - n + 1)`.
     - Если `d == -1` (движение вниз-влево), то начинаем с верхней границы:
       `i = max(0, s - m + 1)`, `j = min(s, m-1)`.
   - Затем в цикле, пока `0 <= i < n` и `0 <= j < m`, добавляем `mat[i][j]` в результат, и обновляем `i -= d`, `j += d` (двигаемся по диагонали).
   - Меняем `d` на `-d` для следующей диагонали.
4. Вернуть результат.

---

## Реализация на Python

```python
def zig_zag_matrix(mat):
    n = len(mat)
    m = len(mat[0])
    res = []
    d = 1  # 1 - вверх-вправо, -1 - вниз-влево

    for s in range(n + m - 1):
        if d == 1:
            # начало диагонали: идём снизу-вверх
            i = min(s, n - 1)
            j = max(0, s - n + 1)
        else:
            # начало диагонали: идём сверху-вниз
            i = max(0, s - m + 1)
            j = min(s, m - 1)

        while 0 <= i < n and 0 <= j < m:
            res.append(mat[i][j])
            i -= d
            j += d

        d = -d  # меняем направление для следующей диагонали

    return res
```

---

## Сложность

- **Время:** O(n·m) — каждый элемент посещается ровно один раз.
- **Память:** O(1) дополнительно (не считая памяти для результата).

---

## Примечания

- В приведённом коде использовалась переменная `d`, которая чередует знак, что соответствует зигзагообразному обходу.
- Альтернативный подход: вычислять начальные координаты в зависимости от чётности `s`, но предложенный способ с направлением более нагляден.
- Обратите внимание на корректность граничных условий: `i = min(s, n-1)` и т.д. — они гарантируют, что начальная точка лежит на первой или последней строке/столбце в зависимости от направления.
