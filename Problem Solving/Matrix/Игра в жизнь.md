## Определение

**Игра «Жизнь» (Conway's Game of Life)** — клеточный автомат, в котором каждое поколение вычисляется на основе текущего состояния клеток (живые = 1, мёртвые = 0) по следующим правилам (с учётом 8 соседей):

1. Любая живая клетка с **меньше чем двумя** живыми соседями **умирает** (недонаселение).
2. Живая клетка с **двумя или тремя** живыми соседями **выживает**.
3. Живая клетка с **больше чем тремя** живыми соседями **умирает** (перенаселение).
4. Мёртвая клетка ровно с **тремя** живыми соседями **рождается**.

Соседи — это все 8 клеток вокруг (горизонтальные, вертикальные, диагональные). Края считаются мёртвыми.

---

## Решение: кодирование следующего состояния

Используются **промежуточные состояния** (маркеры) в самой матрице:

- `2` — клетка была мёртвой (0) и становится живой (1) в следующем поколении.
- `3` — клетка была живой (1) и становится мёртвой (0) в следующем поколении.

Эти значения позволяют при подсчёте живых соседей считать старые живые клетки (1 и 3), а затем, после полного прохода, преобразовать `2` → `1`, `3` → `0`.

### Алгоритм

1. **Подсчёт живых соседей** для каждой клетки с учётом 8 направлений. Живыми считаются клетки со значениями `1` или `3` (потому что они были живы в предыдущем поколении).
2. **Применение правил** к текущей клетке:
   - Если клетка была живой (`1`) и соседей < 2 или > 3 → помечаем как `3` (умрёт).
   - Если клетка была мёртвой (`0`) и соседей == 3 → помечаем как `2` (родится).
   - Иначе состояние остаётся неизменным (`1` остаётся `1`, `0` остаётся `0`).
3. **Финальный проход**: заменяем все `2` на `1`, все `3` на `0`.

## Реализация на Python

```python
class Solution:
    def findNextGen(self, mat):
        n, m = len(mat), len(mat[0])
        directions = [
            (0,1), (1,0), (0,-1), (-1,0),   # вертикальные/горизонтальные
            (1,1), (-1,-1), (1,-1), (-1,1)   # диагональные
        ]

        for i in range(n):
            for j in range(m):
                live = 0
                for dx, dy in directions:
                    ni, nj = i + dx, j + dy
                    if 0 <= ni < n and 0 <= nj < m:
                        if mat[ni][nj] in (1, 3):   # было живым в прошлом
                            live += 1
                # Применяем правила
                if mat[i][j] == 1 and (live < 2 or live > 3):
                    mat[i][j] = 3      # живая -> умрёт
                elif mat[i][j] == 0 and live == 3:
                    mat[i][j] = 2      # мёртвая -> родится

        # Финальная замена
        for i in range(n):
            for j in range(m):
                if mat[i][j] == 2:
                    mat[i][j] = 1
                elif mat[i][j] == 3:
                    mat[i][j] = 0
```

## Сложность

- **Время:** O(n·m) — каждый элемент обрабатывается дважды (подсчёт соседей + финальная замена). Подсчёт соседей для каждой клетки выполняется за O(1) (8 соседей).
- **Память:** O(1) дополнительно — все изменения происходят на месте, используются только несколько переменных.

## Примечания

- Этот метод позволяет обновлять матрицу **in-place** без дополнительного массива.
- Промежуточные состояния (2, 3) выбраны произвольно, главное, чтобы они не конфликтовали с исходными (0, 1).
- Можно обобщить для любого размера матрицы (не только квадратной).
