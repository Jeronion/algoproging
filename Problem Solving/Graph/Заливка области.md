## Определение

**Заливка области (Flood Fill)** — это алгоритм, который изменяет цвет (значение) всех соседних пикселей (ячеек) одного исходного цвета, начиная с заданной точки, распространяясь в четырёх направлениях (вверх, вниз, влево, вправо). Алгоритм широко используется в графических редакторах (например, «заливка ведром»), в играх и в задачах обработки изображений.

---

## Условие

Задача №112504 с сайта informatics.msk.ru (раздел «Стеки, очереди, деки»).

Дана матрица размером `N × M` (количество строк `N`, столбцов `M`), содержащая целые числа — коды цветов (от 0 до 255). Задана начальная точка `(x, y)` (координаты, где `x` — столбец, `y` — строка) и новый цвет `color`. Требуется выполнить заливку области одного цвета, начиная с указанной точки, распространяясь только по четырём направлениям (вверх, вниз, влево, вправо). Необходимо вывести количество изменённых ячеек и итоговую матрицу после заливки.

**Входные данные (файл input.txt):**
- Первая строка: `N M` (размеры матрицы)
- Вторая строка: `x y` (координаты начальной точки, отсчёт от 0, левый верхний угол)
- Третья строка: `color` (новый цвет)
- Далее `N` строк, каждая содержит `M` целых чисел — исходные цвета.

**Выходные данные (файл output.txt):**
- Первая строка: количество изменённых ячеек.
- Затем `N` строк — итоговая матрица после заливки.

**Пример:**

Вход:
```
5 5
1 0
2
0 1 0 1 1
1 1 1 2 2
0 1 0 2 2
3 3 1 2 2
0 1 1 0 0
```
Выход:
```
5
0 2 0 1 1
2 2 2 2 2
0 2 0 2 2
3 3 1 2 2
0 1 1 0 0
```
Пояснение: начальная точка (1,0) имеет исходный цвет 1 (в матрице строка 0, столбец 1). Заливка распространяется на все соседние ячейки со значением 1, заменяя их на 2. Всего изменено 5 ячеек.

---

## Идея решения

Классическая задача на **обход в ширину (BFS)** или **в глубину (DFS)**. Алгоритм:

1. Прочитать входные данные из файла.
2. Запомнить исходный цвет `old_color` в начальной точке.
3. Если `old_color` уже равен новому цвету, ничего не делаем (количество изменений 0).
4. Иначе создать очередь (deque) и поместить в неё начальную точку.
5. Заменить цвет начальной точки на новый и увеличить счётчик.
6. Пока очередь не пуста:
   - Извлечь точку.
   - Для каждого из четырёх соседей (вверх, вниз, влево, вправо):
     - Если сосед в пределах матрицы и его цвет равен `old_color`:
       - Заменить цвет соседа на новый.
       - Поместить соседа в очередь.
       - Увеличить счётчик.
7. Записать в выходной файл количество изменённых ячеек и получившуюся матрицу.

В приведённом коде используется BFS с явной очередью из модуля `collections.deque`.

## Реализация

```python
from collections import deque

def flood_fill_bfs():
    with open("input.txt", "r") as fin:
        N, M = map(int, fin.readline().split())
        x, y = map(int, fin.readline().split())
        color = int(fin.readline())
        matrix = [list(map(int, fin.readline().split())) for _ in range(N)]

    # Начальная точка (x, y): x — столбец, y — строка
    old_color = matrix[y][x]
    if old_color == color:
        # Ничего не меняется
        with open("output.txt", "w") as fout:
            fout.write("0\n")
            for row in matrix:
                fout.write(" ".join(map(str, row)) + "\n")
        return

    # BFS
    queue = deque()
    queue.append((x, y))
    matrix[y][x] = color
    counter = 1

    directions = [(0, 1), (1, 0), (0, -1), (-1, 0)]  # вниз, вправо, вверх, влево

    while queue:
        cx, cy = queue.popleft()
        for dx, dy in directions:
            nx, ny = cx + dx, cy + dy
            if 0 <= nx < M and 0 <= ny < N and matrix[ny][nx] == old_color:
                matrix[ny][nx] = color
                queue.append((nx, ny))
                counter += 1

    # Запись результата
    with open("output.txt", "w") as fout:
        fout.write(str(counter) + "\n")
        for row in matrix:
            fout.write(" ".join(map(str, row)) + "\n")
```

**Важные детали:**
- Координаты: `x` — номер столбца, `y` — номер строки. В матрице обращение `matrix[y][x]`.
- Используются четыре направления: вниз (0,1), вправо (1,0), вверх (0,-1), влево (-1,0). Можно использовать любой порядок, результат не изменится.
- Если исходный цвет совпадает с новым, заливка не требуется (программа выводит исходную матрицу и 0).

---

## Сложность

- **Время:** O(N·M) — в худшем случае обходятся все ячейки матрицы (например, если вся матрица одного цвета). Каждая ячейка посещается не более одного раза.
- **Память:** O(N·M) для хранения матрицы и очереди (очередь может содержать до O(N·M) элементов в худшем случае).

---

## Примечания

- Вместо BFS можно использовать DFS (рекурсию или стек), но BFS удобнее для итеративного подхода.
- Проверка границ и цвета соседей выполняется за константное время.
- Код явно обрабатывает случай, когда начальный цвет совпадает с новым, чтобы избежать бесконечного цикла (хотя и без этой проверки при равенстве старый и новый цвет одинаковы, алгоритм бы ничего не изменил, но заливка не требуется).
- Пример входных данных и вывода соответствует задаче.
