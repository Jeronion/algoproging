## Определение

Дана квадратная сетка `grid` размера `N × N`, где каждая ячейка содержит положительную стоимость (вес). Требуется найти путь от левого верхнего угла `(0, 0)` до правого нижнего `(N-1, N-1)`, двигаясь только по четырём направлениям (вверх, вниз, влево, вправо), с **минимальной суммарной стоимостью** прохождения ячеек. Стоимость пути — это сумма стоимостей всех посещённых ячеек, включая начальную и конечную.

**Пример:**
```
grid = [[9, 4, 9, 9],
        [6, 7, 6, 4],
        [8, 3, 3, 7],
        [7, 4, 9, 10]]
Вывод: 43
Путь: 9 + 4 + 7 + 3 + 3 + 7 + 10 = 43
```

---

## Постановка задачи

**Вход:**
- `grid` — матрица размера `N × N` с положительными целыми стоимостями (1 ≤ cost ≤ 500).

**Выход:** минимальная суммарная стоимость пути из `(0,0)` в `(N-1, N-1)`.

**Ограничения:** 1 ≤ N ≤ 500.

---

## Идея решения (алгоритм Дейкстры)

Задача сводится к поиску кратчайшего пути во взвешенном графе, где вершины — это ячейки сетки, а рёбра соединяют соседние ячейки с весом, равным стоимости **целевой** ячейки (или стоимость перехода можно считать стоимостью следующей клетки). В данной реализации вес ребра из текущей клетки `(i,j)` в соседнюю `(ni,nj)` равен `grid[ni][nj]`. Начальная клетка имеет вес `grid[0][0]`. Таким образом, суммарная стоимость пути — это сумма стоимостей всех посещённых ячеек (включая начальную).

Поскольку все веса неотрицательны (положительны), мы можем применить **алгоритм Дейкстры** с приоритетной очередью. Он гарантирует нахождение минимальной стоимости за O(N² log N) (при размере сетки N×N).

### Почему не BFS?

BFS даёт кратчайший путь по количеству рёбер, но здесь рёбра имеют разные веса, поэтому нужен алгоритм для взвешенных графов.

---

## Алгоритм (пошагово)

1. Создаём матрицу `dist` размера `N × N`, заполняем `inf`. `dist[0][0] = grid[0][0]`.
2. Создаём мин-кучу `pq` и помещаем `(dist[0][0], 0, 0)`.
3. Пока куча не пуста:
   - Извлекаем `(d, i, j)` — минимальное расстояние до `(i,j)`.
   - Если `d > dist[i][j]`, пропускаем (устаревшая запись).
   - Для каждого из четырёх соседей `(ni, nj)`:
     - Если `ni, nj` в пределах сетки и `dist[i][j] + grid[ni][nj] < dist[ni][nj]`:
       - Обновляем `dist[ni][nj]` и добавляем в кучу `(dist[ni][nj], ni, nj)`.
4. По завершении возвращаем `dist[N-1][N-1]`.

---

## Реализация

```python
import heapq

class Solution:
    def minimumCostPath(self, grid):
        N = len(grid)
        # направления движения (вниз, вверх, вправо, влево)
        dirs = [(0, 1), (1, 0), (0, -1), (-1, 0)]
        dist = [[float('inf')] * N for _ in range(N)]
        dist[0][0] = grid[0][0]
        pq = [(dist[0][0], 0, 0)]

        while pq:
            d, i, j = heapq.heappop(pq)
            # если текущее расстояние больше уже найденного, пропускаем
            if d > dist[i][j]:
                continue
            for di, dj in dirs:
                ni, nj = i + di, j + dj
                if 0 <= ni < N and 0 <= nj < N:
                    new_cost = dist[i][j] + grid[ni][nj]
                    if new_cost < dist[ni][nj]:
                        dist[ni][nj] = new_cost
                        heapq.heappush(pq, (new_cost, ni, nj))
        return dist[N-1][N-1]
```

---

## Сложность

- **Время:** O(N² log N), где N² — количество ячеек. Каждая ячейка может быть добавлена в кучу несколько раз, но в худшем случае O(N²) операций, каждая за O(log N²) = O(log N).
- **Память:** O(N²) для хранения матрицы расстояний.

---

## Примечания

- Алгоритм Дейкстры работает, так как все веса положительные (стоимости ячеек ≥ 1).
- Если бы веса могли быть нулевыми или отрицательными, потребовался бы другой подход (например, Беллмана-Форда или 0-1 BFS для нулевых весов).
- В коде используется `float('inf')` для инициализации бесконечности.
- Движение возможно во всех четырёх направлениях, поэтому граф не является деревом.
