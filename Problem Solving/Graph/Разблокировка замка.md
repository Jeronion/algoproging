## Определение

У нас есть замок с четырьмя колёсами, каждое показывает цифру от 0 до 9. Начальное состояние – "0000". За один ход можно повернуть **одно** колесо на один шаг вперёд или назад (с циклическим переходом между 0 и 9). Некоторые комбинации заблокированы: если замок попадает в такую комбинацию, он застревает и двигаться дальше нельзя. Дана целевая комбинация. Требуется найти **минимальное количество ходов**, чтобы добраться из "0000" в цель, не попадая в заблокированные состояния. Если это невозможно, вернуть `-1`.

**Пример:**
```
barriers = ["0201","0101","0102","1212","2002"], target = "0202"
Вывод: 6
```

---

## Постановка задачи

**Вход:**
- `barriers` – список строк (заблокированные комбинации).
- `target` – целевая строка.

**Выход:** минимальное число ходов, или `-1` при невозможности.

**Ограничения:** строки всегда длины 4, цифры от 0 до 9, количество барьеров невелико (в условии не указано строго, но обычно до 10^4).

---

## Идея решения (граф состояний)

Задача сводится к поиску кратчайшего пути в графе, где вершины – все возможные комбинации (от 0000 до 9999), а рёбра соединяют комбинации, отличающиеся ровно одной цифрой на ±1 (с учётом цикличности). Некоторые вершины помечены как заблокированные – их нельзя посещать. Нужно найти минимальное число рёбер от старта до цели. Поскольку все рёбра имеют вес 1, применяем **поиск в ширину (BFS)**. Для ускорения можно использовать **двунаправленный BFS**, который запускает поиск одновременно от старта и от цели и останавливается при встрече.

---

## Подходы к решению

### 1. Односторонний BFS (с множествами)

Классический BFS из стартовой вершины. Используем множество `visited` для посещённых и множество `blocked` для заблокированных. На каждом шаге генерируем 8 соседей (4 колеса × 2 направления). Первое достижение цели даёт минимальное число ходов.

**Плюсы:** простота реализации.  
**Минусы:** при большом количестве состояний (до 10 000) BFS всё равно обойдёт все достижимые вершины, что приемлемо.

### 2. Односторонний BFS с индексацией (оптимизация)

Вместо строк можно закодировать каждую комбинацию как целое число от 0 до 9999 (например, `"1234"` → `1234`). Тогда `visited` и `blocked` хранятся в виде массивов `bool` длины 10000, что быстрее и экономичнее, чем множества строк. Генерация соседей выполняется непосредственно над строкой или над числом.

### 3. Двунаправленный BFS (ожидаемый подход)

Запускаем BFS одновременно от старта и от цели. На каждом шаге расширяем ту сторону, у которой меньше текущая граница (фронт). Если при генерации соседа встречается состояние, уже посещённое с другой стороны, то пути встретились, и суммарная длина = `steps_left + steps_right + 1`. Это часто сокращает число просмотренных состояний (в среднем до ~2·√(количество состояний)).

## Реализация (двунаправленный BFS с индексацией)

```python
from collections import deque

def get_neighbors(s):
    """Генерирует 8 соседей для строки s длиной 4."""
    res = []
    for i in range(4):
        # вперёд
        a = list(s)
        a[i] = '0' if a[i] == '9' else str(int(a[i]) + 1)
        res.append(''.join(a))
        # назад
        b = list(s)
        b[i] = '9' if b[i] == '0' else str(int(b[i]) - 1)
        res.append(''.join(b))
    return res

def unlock_lock(barriers, target):
    start = "0000"
    blocked = set(barriers)
    if start in blocked or target in blocked:
        return -1
    if start == target:
        return 0

    # Фронты и visited-множества для двух сторон
    front_start = {start}
    front_target = {target}
    visited_start = {start}
    visited_target = {target}
    steps = 0

    while front_start and front_target:
        # Расширяем меньший фронт
        if len(front_start) > len(front_target):
            front_start, front_target = front_target, front_start
            visited_start, visited_target = visited_target, visited_start

        next_front = set()
        for state in front_start:
            for nxt in get_neighbors(state):
                if nxt in blocked:
                    continue
                if nxt in visited_target:
                    return steps + 1
                if nxt not in visited_start:
                    visited_start.add(nxt)
                    next_front.add(nxt)
        front_start = next_front
        steps += 1

    return -1
```

**Альтернативная реализация (обычный BFS с индексацией):**

```python
def encode(s):
    return int(s)

def unlock_lock_bfs(barriers, target):
    start = "0000"
    blocked = [False] * 10000
    for b in barriers:
        blocked[encode(b)] = True
    if blocked[encode(start)]:
        return -1
    if start == target:
        return 0

    q = deque([start])
    visited = [False] * 10000
    visited[encode(start)] = True
    steps = 0

    while q:
        for _ in range(len(q)):
            cur = q.popleft()
            for nxt in get_neighbors(cur):
                id_nxt = encode(nxt)
                if blocked[id_nxt] or visited[id_nxt]:
                    continue
                if nxt == target:
                    return steps + 1
                visited[id_nxt] = True
                q.append(nxt)
        steps += 1
    return -1
```

---

## Сложность

- **Обычный BFS:** O(10^4 · 8) = O(80000) в худшем случае (поскольку всего 10 000 состояний), но может быть меньше из-за барьеров. Память O(10^4).
- **Двунаправленный BFS:** в среднем значительно быстрее, но асимптотически также O(10^4). Однако на практике сокращает число посещённых состояний почти вдвое.

---

## Примечания

- Граф состояний имеет размер 10000, что делает даже простой BFS очень быстрым.
- Барьеры могут включать стартовую или целевую вершину – тогда ответ сразу -1.
- Функция `get_neighbors` генерирует все возможные переходы за O(8·4) = O(32) на каждое состояние, что константно.
- В данной задаче из-за малого размера графа подойдёт любой из подходов, но двунаправленный BFS демонстрирует элегантный приём для поиска кратчайших путей.
