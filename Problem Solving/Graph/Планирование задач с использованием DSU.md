## Определение

**Задача о планировании задач** (Job Sequencing Problem) заключается в выборе подмножества заданий с максимальной суммарной прибылью, где каждое задание требует 1 единицу времени для выполнения и должно быть завершено к своему дедлайну. В каждый момент времени может выполняться только одно задание. Требуется найти максимальную прибыль и количество выполненных заданий. Классический жадный алгоритм сортирует задания по убыванию прибыли и назначает каждому заданию самый поздний доступный слот до дедлайна. Оптимизация с использованием DSU позволяет быстро находить свободный слот.

---

## Постановка задачи

Даны три массива: `id[]`, `deadline[]`, `profit[]`. Для каждого задания `i` известны его идентификатор, дедлайн и прибыль. Каждое задание занимает 1 единицу времени. Нужно выбрать подмножество заданий, которое можно выполнить в срок (каждое до своего дедлайна) так, чтобы суммарная прибыль была максимальной. Вернуть количество выбранных заданий и максимальную прибыль.

**Пример:**
```
id = [1, 2, 3, 4]
deadline = [4, 1, 1, 1]
profit = [20, 10, 40, 30]
Вывод: 2 60
```

## Идея решения (DSU)

### Жадный подход

1. Отсортировать задания по убыванию прибыли.
2. Для каждого задания найти самый поздний свободный временной слот `t` такой, что `1 ≤ t ≤ deadline`. Если такой слот есть, назначить задание на этот слот и добавить прибыль.

Наивная реализация поиска слота требует перебора, что даёт O(n·d). Использование **системы непересекающихся множеств (DSU)** позволяет находить свободный слот почти за O(1) (с обратной функцией Аккермана).

### Как работает DSU здесь?

- Создаём `d+1` множеств, где `d = max(deadline)`. Каждое множество представляет временной слот `t` (от 1 до d) и содержит информацию о ближайшем свободном слоте.
- Изначально каждый слот `t` является отдельным множеством, где `parent[t] = t`.
- При поиске свободного слота для задания с дедлайном `d` вызываем `find(d)`, который возвращает самый поздний доступный слот ≤ d. Если возвращённый слот `> 0`, мы занимаем его (выполняем задание) и объединяем множество этого слота с множеством `slot-1` (устанавливаем родителя `slot` как `slot-1`). Таким образом, при следующем поиске для того же дедлайна `find` вернёт уже меньший свободный слот.

### Почему это работает?

- `find(x)` возвращает корень множества, представляющего самый большой доступный слот ≤ x.
- После занятия слота `t` мы делаем `parent[t] = find(t-1)`, что связывает слот `t` с предыдущим свободным. Это гарантирует, что будущий запрос для любого дедлайна ≥ t вернёт следующий свободный слот (t-1, затем t-2, ...).
- Таким образом, каждый слот используется не более одного раза, и мы всегда назначаем задание на максимально возможный слот, что не мешает другим заданиям с более ранними дедлайнами.

---

## Алгоритм

1. Найти максимальный дедлайн `max_d` среди всех заданий.
2. Инициализировать DSU с `max_d + 1` элементами (индексы 0..max_d). `parent[i] = i`.
3. Отсортировать задания по убыванию прибыли.
4. Для каждого задания `(profit, deadline)`:
   - `slot = find(deadline)` — самый поздний свободный слот ≤ deadline.
   - Если `slot > 0`:
     - Выполнить задание (увеличить счётчик и добавить прибыль).
     - Выполнить `union(slot-1, slot)`, т.е. установить `parent[slot] = find(slot-1)` (или просто `parent[slot] = slot-1` с последующей оптимизацией).
5. Вернуть количество и суммарную прибыль.

## Реализация на Python

```python
class DisjointSet:
    def __init__(self, n):
        self.parent = list(range(n + 1))

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def merge(self, u, v):
        # u - слот, который становится родителем для v
        self.parent[v] = u

def job_sequencing(deadline, profit):
    n = len(deadline)
    jobs = list(zip(profit, deadline))
    jobs.sort(reverse=True)  # по убыванию прибыли
    max_d = max(deadline)
    dsu = DisjointSet(max_d)
    count = 0
    total_profit = 0
    for p, d in jobs:
        slot = dsu.find(d)
        if slot > 0:
            count += 1
            total_profit += p
            dsu.merge(dsu.find(slot - 1), slot)
    return count, total_profit

# Пример
deadline = [2, 1, 2, 1, 1]
profit = [100, 19, 27, 25, 15]
print(job_sequencing(deadline, profit))  # (2, 127)
```

---

## Сложность

- **Время:** O(n log n) на сортировку + O(n · α(max_d)) на операции DSU, где α — обратная функция Аккермана (почти константа). В целом O(n log n).
- **Память:** O(max_d) для хранения массива `parent`.

---

## Примечания

- Задача известна также как "Job Sequencing with Deadlines".
- DSU здесь используется для эффективного нахождения свободного слота, что улучшает наивный алгоритм с O(n·d) до почти O(n log n).
- Если дедлайны могут быть большими (до 10⁵ и более), память O(max_d) приемлема.
- Важно, что `find(deadline)` может вернуть 0, если все слоты до дедлайна заняты.
