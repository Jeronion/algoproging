## Определение

Дан квадратная шахматная доска размером `n × n` (координаты от 1 до n). Заданы начальная и целевая позиции коня. Требуется найти **минимальное количество ходов**, за которое конь может добраться из начальной позиции в целевую. Конь ходит буквой "Г" — 2 клетки по одной оси и 1 по другой.

**Пример:**
```
n = 3, начальная (3,3), целевая (1,2) → 1 ход
n = 6, (4,5) → (1,1) → 3 хода
```

---

## Идея решения (BFS)

Задача о поиске кратчайшего пути в невзвешенном графе, где вершины — клетки доски, рёбра — возможные ходы коня. Используется **поиск в ширину (BFS)**, так как BFS находит минимальное количество шагов.

### Алгоритм

1. Если начальная позиция совпадает с целевой, вернуть 0.
2. Создать очередь, в которую помещаем кортеж `(x, y, steps)`.
3. Создать множество `visited` для посещённых клеток.
4. Пока очередь не пуста:
   - Извлечь клетку `(x, y, steps)`.
   - Для каждого из 8 возможных ходов коня:
     - Вычислить новые координаты `(nx, ny)`.
     - Если `(nx, ny)` выходит за пределы доски (1..n) или уже посещена — пропустить.
     - Если `(nx, ny)` совпадает с целевой позицией — вернуть `steps + 1`.
     - Иначе пометить как посещённую и добавить в очередь с `steps + 1`.
5. Если BFS завершился без нахождения (не должно случиться), вернуть -1 (или 0).

## Реализация

```python
from collections import deque

def min_knight_moves(n, start, target):
    if start == target:
        return 0
    moves = [(1,2), (1,-2), (-1,2), (-1,-2),
             (2,1), (2,-1), (-2,1), (-2,-1)]
    queue = deque([(start[0], start[1], 0)])
    visited = set()
    visited.add((start[0], start[1]))
    while queue:
        x, y, dist = queue.popleft()
        for dx, dy in moves:
            nx, ny = x + dx, y + dy
            if 1 <= nx <= n and 1 <= ny <= n and (nx, ny) not in visited:
                if (nx, ny) == (target[0], target[1]):
                    return dist + 1
                visited.add((nx, ny))
                queue.append((nx, ny, dist + 1))
    return -1
```

## Сложность

- **Время:** O(n²) в худшем случае — BFS посещает все клетки доски. Для n ≤ 1000 это 10⁶ клеток, что допустимо.
- **Память:** O(n²) для хранения посещённых клеток (множество) и очереди.

---

## Примечания

- BFS гарантирует минимальное количество шагов.
- Важно проверять границы доски и избегать повторных посещений.
- Если доска большая, можно использовать массив `visited` размера (n+1)×(n+1) для ускорения вместо множества.
- Координаты заданы в 1‑индексации, что учтено в проверках.
