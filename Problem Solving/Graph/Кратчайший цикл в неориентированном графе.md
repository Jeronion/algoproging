## Определение

**Цикл** в неориентированном графе — это путь, начинающийся и заканчивающийся в одной вершине, без повторения рёбер или вершин (кроме стартовой/конечной). Длина цикла измеряется количеством рёбер. Требуется найти **кратчайший цикл** (с минимальным числом рёбер) в заданном графе. Если граф ациклический, вернуть `-1`.

**Пример:**
```
V = 7, edges = [[0,5], [0,6], [5,1], [6,1], [6,2], [2,3], [3,4], [1,4]]
Кратчайший цикл: 0 → 5 → 1 → 6 → 0 (длина 4)
```

---

## Постановка задачи

**Вход:**
- `V` — количество вершин.
- `edges` — список рёбер (неориентированных) вида `[u, v]`.

**Выход:** длина кратчайшего цикла, или `-1`, если циклов нет.

**Ограничения:** 1 ≤ V ≤ 10³, 0 ≤ E ≤ 10³.

---

## Идея решения (BFS из каждой вершины)

Алгоритм основан на том, что любой цикл можно обнаружить, запуская **поиск в ширину (BFS)** из каждой вершины и отслеживая обратные рёбра. При BFS мы строим дерево обхода. Если при обработке вершины `u` мы встречаем соседа `v`, который уже посещён и **не является родителем** `u` (и `u` не является родителем `v`), то мы нашли цикл. Его длина вычисляется как `dist[u] + dist[v] + 1` (сумма расстояний от корня до `u` и до `v` плюс ребро между ними). Берём минимум по всем таким случаям и по всем стартовым вершинам.

Этот подход работает для неориентированных графов без кратных рёбер. Для каждой стартовой вершины BFS даёт кратчайшие расстояния от неё; рёбра, соединяющие вершины с одного уровня (или вершины, не являющиеся предком-потомком), указывают на цикл.

### Почему это работает?

Любой цикл в графе может быть представлен как путь от некоторой вершины `s` до двух вершин `u` и `v`, которые соединены ребром, не принадлежащим дереву BFS. Тогда длина цикла = `dist[s][u] + dist[s][v] + 1`. BFS из `s` находит все такие пары. Так как мы запускаем BFS из каждой вершины, мы гарантированно найдём минимальный цикл.

---

## Алгоритм (пошагово)

1. Построить список смежности `graph`.
2. Инициализировать `ans = INF`.
3. Для каждой вершины `s` от 0 до V-1:
   - Запустить BFS из `s`, используя очередь.
   - Поддерживать массивы `dist[]` (расстояние от `s`) и `parent[]` (предок в дереве BFS).
   - Изначально `dist[s] = 0`, `parent[s] = -1`.
   - Пока очередь не пуста:
     - Извлечь `u`.
     - Для каждого соседа `v`:
       - Если `dist[v] == INF`, то `dist[v] = dist[u] + 1`, `parent[v] = u`, добавить в очередь.
       - Иначе если `parent[u] != v` и `parent[v] != u` (т.е. ребро не является обратным к родительскому), то нашли цикл: обновить `ans = min(ans, dist[u] + dist[v] + 1)`.
4. Если `ans == INF`, вернуть `-1`, иначе `ans`.

---

## Реализация

```python
from collections import deque, defaultdict

class Solution:
    def shortCycle(self, V, edges):
        graph = defaultdict(list)
        for u, v in edges:
            graph[u].append(v)
            graph[v].append(u)

        INF = 10**9
        ans = INF

        for start in range(V):
            dist = [INF] * V
            parent = [-1] * V
            q = deque([start])
            dist[start] = 0

            while q:
                u = q.popleft()
                for v in graph[u]:
                    if dist[v] == INF:
                        dist[v] = dist[u] + 1
                        parent[v] = u
                        q.append(v)
                    elif parent[u] != v and parent[v] != u:
                        # найден цикл
                        ans = min(ans, dist[u] + dist[v] + 1)

        return -1 if ans == INF else ans
```

---

## Сложность

- **Время:** O(V·(V+E)) — BFS из каждой вершины. Для V, E ≤ 10³ это приемлемо (≈ 10⁶ операций).
- **Память:** O(V+E) для хранения графа и массивов.

---

## Примечания

- Алгоритм подходит для небольших графов (V ≤ 1000). Для больших графов существуют более сложные методы (например, поиск кратчайшего цикла через BFS с удалением рёбер).
- В неориентированном графе простой цикл длины 3 (треугольник) обнаруживается, когда `u` и `v` — соседи, а расстояние до них от старта равно 1 и 1 соответственно, тогда `1+1+1=3`.
- Если граф содержит кратные рёбра, алгоритм может их некорректно обрабатывать; в условии задачи, скорее всего, кратных рёбер нет.
