## Определение

Даны два слова `s` и `e` (стартовое и целевое), а также список `words` уникальных слов одинаковой длины. Требуется найти **длину кратчайшей последовательности преобразований** от `s` к `e`, где каждое преобразование меняет ровно одну букву, и каждое промежуточное слово (включая `e`) должно присутствовать в `words`. Если такой последовательности нет, вернуть `0`.

**Пример:**
```
words = ["des", "der", "dfr", "dgt", "dfs"], s = "der", e = "dfs"
Ответ: 3 (der → dfr → dfs)
```

---

## Постановка задачи

**Вход:**
- `words` — список строк (все одной длины).
- `s`, `e` — две строки (длина такая же).
- Все слова состоят из строчных латинских букв.

**Выход:** минимальное количество преобразований (длина последовательности) от `s` к `e`. Если путь отсутствует, вернуть `0`.

**Ограничения:** длина слов ≤ 10, количество слов ≤ 5000.

---

## Идея решения (BFS)

Задачу можно свести к поиску кратчайшего пути в невзвешенном графе, где вершины — слова, а рёбра соединяют слова, отличающиеся ровно одной буквой. Стартовая вершина — `s` (даже если её нет в `words`), целевая — `e` (присутствует в `words`). Поскольку все рёбра имеют вес 1, для поиска кратчайшего пути используется **поиск в ширину (BFS)**.

Для эффективной генерации соседей мы не перебираем все слова в `words` (это дало бы O(n·m) на вершину), а перебираем все позиции в слове (длина до 10) и все 26 букв, формируя новое слово. Если новое слово присутствует в `words` и ещё не посещено, оно добавляется в очередь. Посещённые слова удаляются из множества `words`, чтобы не обрабатывать их повторно.

Этот подход гарантирует, что первое достижение `e` будет кратчайшим путём.

### Особенности реализации

- В коде не проверяется, содержится ли `s` в списке `words`; это не обязательно, так как мы начинаем с `s` и ищем только переходы в множество `words`.
- Множество `words` используется как неvisited; после посещения слово удаляется, что экономит память и ускоряет проверки.
- Если `s == e`, ответ должен быть 1? В задаче обычно длина последовательности, включая `s`, т.е. если `s` уже равно `e`, ответ 1. В коде это не обработано, но по условию `s` и `e` различны? В условии сказано "two distinct words", так что это не требуется.

## Алгоритм

1. Поместить `s` в очередь и начать счётчик шагов `res = 0`.
2. Пока очередь не пуста:
   - Увеличить `res` на 1 (текущий уровень BFS).
   - Для всех слов на текущем уровне:
     - Извлечь слово `word` из очереди.
     - Для каждой позиции `j` в слове:
       - Для каждой буквы от `'a'` до `'z'`:
         - Создать новое слово `new_word` заменой буквы в позиции `j`.
         - Если `new_word` отсутствует в множестве `words`, пропустить.
         - Если `new_word == e`, вернуть `res + 1` (добавляем целевое слово как следующий шаг).
         - Удалить `new_word` из множества (пометить как посещённое) и добавить в очередь.
3. Если цикл завершился без возврата — путь не найден, вернуть `0`.

## Реализация

```python
from collections import deque

class Solution:
    def wordLadder(self, words, s, e):
        m = len(words[0])
        st = set(words)
        q = deque([s])
        res = 0

        while q:
            res += 1
            for _ in range(len(q)):
                word = q.popleft()
                for j in range(m):
                    for i in range(26):
                        c = chr(ord('a') + i)
                        new_word = word[:j] + c + word[j+1:]
                        if new_word not in st:
                            continue
                        if new_word == e:
                            return res + 1
                        st.remove(new_word)
                        q.append(new_word)
        return 0
```

## Сложность

- **Время:** O(длина_слова · 26 · количество_посещённых_слов). В худшем случае O(L·26·N), где L — длина слова (≤10), N — количество слов (≤5000). Это очень эффективно.
- **Память:** O(N) для множества и очереди.

---

## Примечания

- Использование множества `st` и удаление посещённых слов позволяет не заводить отдельный `visited` set.
- В коде не учитывается случай `s == e`; если такое возможно, следует сразу вернуть 1 (или 0, в зависимости от определения). В условии задачи говорится о различных словах.
- Все слова состоят из строчных букв, поэтому перебор 26 букв корректен.
