## Определение

Дан неориентированный взвешенный граф, в котором между некоторыми парами вершин существуют два параллельных ребра: **прямое** с весом `w1` и **изогнутое** с весом `w2`. Требуется найти кратчайший путь от вершины `a` к вершине `b`, который может содержать **не более одного** изогнутого ребра. Если такого пути нет, вернуть `-1`.

**Пример:**
```
V = 4, a = 1, b = 3
edges = [[0,1,1,4], [0,2,2,4], [0,3,3,1], [1,3,6,5]]
Вывод: 2
Объяснение: путь 1→0 (прямое 1), затем 0→3 (изогнутое 1) → итого 2.
```

---

## Постановка задачи

**Вход:**
- `V` — количество вершин.
- `edges` — список рёбер вида `[u, v, w1, w2]` (два ребра между u и v).
- `src`, `end` — начальная и конечная вершины.

**Выход:** минимальная стоимость пути с учётом ограничения (не более одного изогнутого ребра), или `-1` если путь невозможен.

**Ограничения:** V, E ≤ 10⁶; веса неотрицательные.

---

## Идея решения

Задача решается **модифицированным алгоритмом Дейкстры** с расширенным состоянием. Мы отслеживаем не только вершину, но и использовали ли мы уже изогнутое ребро. Состояние: `(вершина, флаг_использования_кривого_ребра)`. Это даёт два набора расстояний:

- `dist[v][0]` — минимальное расстояние до `v` без использования кривого ребра.
- `dist[v][1]` — минимальное расстояние до `v` с использованием ровно одного кривого ребра.

При релаксации ребра `(u, v)` с весами `w1` (прямое) и `w2` (кривое):

- **Прямое ребро**: можно использовать всегда, состояние не меняется.  
  `dist[v][used] = min(dist[v][used], dist[u][used] + w1)`.
- **Кривое ребро**: можно использовать только если `used == False` (ещё не использовали кривое). После этого флаг становится `True`.  
  `dist[v][1] = min(dist[v][1], dist[u][0] + w2)`.

Таким образом, мы строим граф состояний размера `2V` и запускаем Дейкстру из `(src, 0)`. Ответом будет `min(dist[end][0], dist[end][1])`.

---

## Алгоритм

1. Построить список смежности с кортежами `(w1, w2, v)` для каждого ребра (добавляем в обе стороны).
2. Инициализировать `dist = [[inf, inf] for _ in range(V)]`, `dist[src] = [0, 0]`.
3. Поместить в min-heap `(0, src, False)`.
4. Пока куча не пуста:
   - Извлечь `(d, u, used)`.
   - Если `d != dist[u][used]`, пропустить.
   - Для каждого соседа `(w1, w2, v)`:
     - Если `d + w2 < dist[v][1]` и `used == False`:
       - Обновить `dist[v][1] = d + w2`, добавить `(dist[v][1], v, True)`.
     - Если `d + w1 < dist[v][used]`:
       - Обновить `dist[v][used] = d + w1`, добавить `(dist[v][used], v, used)`.
5. Ответ = `min(dist[end][0], dist[end][1])`, если не `inf`, иначе `-1`.

## Реализация на Python

```python
from collections import defaultdict
import heapq

class Solution:
    def shortestPath(self, V, src, end, edges):
        graph = defaultdict(list)
        for u, v, w1, w2 in edges:
            graph[u].append((w1, w2, v))
            graph[v].append((w1, w2, u))
        
        dist = [[float('inf'), float('inf')] for _ in range(V)]
        dist[src] = [0, 0]
        pq = [(0, src, 0)]   # (расстояние, вершина, флаг_использования)
        
        while pq:
            d, u, used = heapq.heappop(pq)
            if d != dist[u][used]:
                continue
            for w1, w2, v in graph[u]:
                # Прямое ребро (всегда доступно)
                if d + w1 < dist[v][used]:
                    dist[v][used] = d + w1
                    heapq.heappush(pq, (dist[v][used], v, used))
                # Кривое ребро (только если ещё не использовали)
                if used == 0 and d + w2 < dist[v][1]:
                    dist[v][1] = d + w2
                    heapq.heappush(pq, (dist[v][1], v, 1))
        
        ans = min(dist[end])
        return -1 if ans == float('inf') else ans
```

## Сложность

- **Время:** O((V+E) log V) — стандартная сложность Дейкстры на графе состояний (2V вершин, 2E рёбер).
- **Память:** O(V + E) для хранения графа и массивов расстояний.

---

## Примечания

- Алгоритм модифицирует Дейкстру для работы с **ограничением на использование особого типа рёбер**.
- Так как веса неотрицательные, Дейкстра применима.
- Если граф содержит несколько параллельных рёбер между одними и теми же вершинами, они будут обработаны корректно (список смежности содержит все варианты).
- В коде флаг `used` — 0 или 1; при использовании кривого ребра он становится 1.
