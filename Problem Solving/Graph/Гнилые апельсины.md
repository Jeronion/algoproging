## Определение

Дана сетка `mat` размером `n × m`, каждая ячейка которой может иметь значение:
- `0` — пустая клетка,
- `1` — свежий апельсин,
- `2` — гнилой апельсин.

Гнилой апельсин заражает все соседние (сверху, снизу, слева, справа) свежие апельсины за 1 единицу времени. Необходимо найти **минимальное время**, за которое все свежие апельсины станут гнилыми. Если это невозможно (какой-то свежий апельсин изолирован от всех гнилых), вернуть `-1`.

**Пример:**
```
Вход: mat = [[2,1,0,2,1],
             [1,0,1,2,1],
             [1,0,0,2,1]]
Выход: 2
```

---

## Постановка задачи

- Начальное состояние: несколько гнилых апельсинов (источников).
- Процесс распространения: каждый источник заражает соседние свежие апельсины за 1 шаг, новые гнилые становятся источниками на следующем шаге.
- Требуется найти время, когда все свежие апельсины будут заражены, или определить, что некоторые останутся свежими.

---

## Идея решения (Multi-Source BFS)

Задача сводится к **поиску кратчайшего расстояния** от множества источников (гнилых апельсинов) до всех свежих. Так как все рёбра имеют вес 1, для этого идеально подходит **обход в ширину (BFS)**, запущенный одновременно из всех начальных гнилых клеток. Время заражения свежего апельсина — это его расстояние до ближайшего источника в терминах шагов BFS.

### Алгоритм

1. **Инициализация очереди:** Найти все гнилые апельсины (`mat[i][j] == 2`) и поместить их в очередь вместе с временем `0`.
2. **BFS:** Пока очередь не пуста:
   - Извлечь клетку и её время.
   - Для каждого из 4 соседей:
     - Если соседняя клетка содержит свежий апельсин (`1`), то заражаем его (меняем на `2`), добавляем в очередь с временем `time + 1`, обновляем максимальное время.
3. **Проверка:** После завершения BFS проверить, остались ли в сетке свежие апельсины (`1`). Если да — вернуть `-1`, иначе вернуть максимальное зафиксированное время.

### Почему это работает?

BFS от нескольких источников гарантирует, что мы посещаем каждую клетку за минимальное количество шагов от любого источника. Поскольку все операции заражения занимают одинаковое время, кратчайший путь в графе соответствует минимальному времени заражения.

## Реализация на Python

```python
from collections import deque

class Solution:
    def orangesRot(self, mat):
        n, m = len(mat), len(mat[0])
        directions = [(0,1), (1,0), (0,-1), (-1,0)]
        q = deque()
        fresh = 0
        # Инициализация очереди и подсчёт свежих
        for i in range(n):
            for j in range(m):
                if mat[i][j] == 2:
                    q.append((i, j, 0))
                elif mat[i][j] == 1:
                    fresh += 1
        
        time = 0
        while q:
            x, y, t = q.popleft()
            time = max(time, t)
            for dx, dy in directions:
                nx, ny = x + dx, y + dy
                if 0 <= nx < n and 0 <= ny < m and mat[nx][ny] == 1:
                    mat[nx][ny] = 2
                    fresh -= 1
                    q.append((nx, ny, t + 1))
        
        return -1 if fresh > 0 else time
```

---

## Сложность

- **Время:** O(n·m) — каждая клетка посещается не более одного раза.
- **Память:** O(n·m) в худшем случае для очереди.

---

## Примечания

- В коде используется подсчёт `fresh` для быстрой проверки, остались ли свежие апельсины, что избавляет от повторного прохода по всей матрице в конце. Однако в некоторых реализациях это делается отдельным циклом.
- Важно не забывать, что пустые клетки (`0`) не влияют на процесс и не могут быть заражены.
- Если изначально нет свежих апельсинов (`fresh == 0`), ответ `0`, что обрабатывается автоматически.
