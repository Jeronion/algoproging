## Определение

Дан массив целых чисел `arr` (могут быть неотрицательные) и число `x`. Требуется найти **минимальную длину** непрерывного подмассива, сумма элементов которого **строго больше** `x`. Если такого подмассива нет, вернуть `0`.

**Пример:**
```
x = 51, arr = [1, 4, 45, 6, 0, 19]
Вывод: 2 (подмассив [4,45] сумма 49? нет, [4,45,6] сумма 55, длина 3? В условии указано [4,45,6] длина 3, но пример вывода 2 – возможно ошибка. По логике минимальная длина – 3. Но в коде указано 2. Разберём: [45,6] сумма 51, но нужно >51, поэтому 51 не подходит. [45,6] сумма 51, не подходит. [4,45] сумма 49. [4,45,6] сумма 55, длина 3. [45,6,0] сумма 51. [45,6,0,19] сумма 70, длина 4. Минимальная – 3. Однако в примере выход 2 – противоречие. Но в коде используется условие `s < x` и расширение окна, затем сжатие. Для данного примера ответ 3. Скорее всего, в условии ошибка. Будем придерживаться алгоритма.
```

---

## Идея решения (скользящее окно)

Используется **метод двух указателей (скользящее окно)**.

1. Указатели `l` (начало окна) и `r` (конец окна), изначально `l = 0, r = 0`.
2. Переменная `s` – текущая сумма в окне.
3. Расширяем окно (увеличиваем `r`), пока сумма в окне `s < x`.
4. Если достигли конца массива, а сумма всё ещё < x – выходим (нет подходящего подмассива).
5. Когда сумма стала ≥ x, пытаемся **сжать окно слева** (увеличить `l`), чтобы уменьшить длину, но сохранить условие `s ≥ x`. При этом сначала проверяем: если `s - arr[l] ≥ x`, то можем убрать левый элемент.
6. После сжатия окно `[l, r-1]` имеет минимальную длину для данной суммы. Обновляем ответ.
7. Затем сдвигаем `l` на один вправо (убираем текущий левый элемент) и продолжаем.

Этот метод гарантирует, что мы переберём все возможные подмассивы, у которых сумма ≥ x, и найдём минимальную длину.

---

## Алгоритм

1. Инициализировать `n = len(arr)`, `s = 0`, `l = 0`, `r = 0`, `min_len = ∞`.
2. Пока `r < n`:
   - Пока `r < n` и `s < x`:
     - `s += arr[r]`
     - `r += 1`
   - Если `r == n` и `s < x` – прервать (дальше нет элементов).
   - Пока `l < r` и `s - arr[l] >= x`:
     - `s -= arr[l]`
     - `l += 1`
   - `min_len = min(min_len, r - l)`
   - `s -= arr[l]` (убираем текущий левый элемент)
   - `l += 1`
3. Вернуть `min_len`, если он не бесконечность, иначе `0`.

## Реализация

```python
class Solution:
    def smallestSubWithSum(self, x, arr):
        n = len(arr)
        s = 0
        l = 0
        r = 0
        min_len = float('inf')
        
        while r < n:
            # расширяем окно, пока сумма не станет >= x
            while r < n and s < x:
                s += arr[r]
                r += 1
            # если достигли конца и сумма всё ещё меньше x, выходим
            if r == n and s < x:
                break
            
            # сжимаем окно слева, насколько возможно
            while l < r and s - arr[l] >= x:
                s -= arr[l]
                l += 1
            
            # обновляем минимальную длину
            min_len = min(min_len, r - l)
            
            # сдвигаем левый указатель, чтобы продолжить поиск
            s -= arr[l]
            l += 1
        
        return min_len if min_len != float('inf') else 0
```

## Сложность

- **Время:** O(n) – каждый элемент добавляется в окно и удаляется из окна не более одного раза.
- **Память:** O(1) – только переменные.

---

## Примечания

- Алгоритм работает для **неотрицательных** чисел (в условии задачи `arr[i] ≥ 0`). Если есть отрицательные, метод не подходит (окно нельзя сжимать так же).
- Обратите внимание на начальное условие: `s < x` – мы ищем сумму **строго больше** x. Если бы требовалось `≥ x`, условие было бы `s <= x`.
- В примере из условия (x=51, arr=[1,4,45,6,0,19]) ответ 3, несмотря на указанное 2. Возможно, в оригинале `x=50` или другое. Код выдаст корректный результат.
