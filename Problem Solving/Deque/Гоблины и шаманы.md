## Условие задачи

В очереди к шаману действуют два типа гоблинов:

- **Обычный** — встаёт в конец очереди.
- **Привилегированный** — встаёт **ровно в середину** очереди. Если длина очереди нечётна, привилегированный встаёт сразу **за центральным** элементом.

Требуется обработать серию запросов:

- `+ i` — обычный гоблин с номером `i` встаёт в конец.
- `* i` — привилегированный гоблин `i` встаёт в середину.
- `-` — первый гоблин уходит к шаману; вывести его номер.

Все номера гоблинов различны (1 ≤ i ≤ N). Количество запросов N ≤ 10⁵.

**Пример:**

```
Вход:
7
+ 1
+ 2
-      # вывод 1
+ 3
* 4
-      # вывод 2
-      # вывод 3
```

Выход:
```
1
2
3
```

---

## Идея решения

Прямая реализация на списке (вставка в середину) даст O(n) на операцию, что при N=10⁵ слишком медленно. Нужно добиться O(1) (амортизированно) для всех операций.

Используем **два дека**:
- `left` — первая половина очереди (ближе к выходу).
- `right` — вторая половина (ближе к хвосту).

Поддерживаем **инвариант**:
- `len(left) >= len(right)` (количество элементов в левой части либо равно, либо на 1 больше правой).
- Всегда `left` содержит первую часть, `right` — вторую, причём все элементы `left` «раньше» элементов `right`.

При такой организации:
- **Удаление из начала** — `left.popleft()`.
- **Обычный гоблин** (в конец) — `right.append(i)`.
- **Привилегированный** (в середину) — добавляем либо в конец `left`, либо в начало `right`, в зависимости от текущего соотношения размеров, чтобы сохранить инвариант.

После каждой операции балансируем деки, чтобы `len(left)` отличалось от `len(right)` не более чем на 1 (и левая часть была не меньше).

## Алгоритм

### Инициализация
```python
from collections import deque
left = deque()   # левая половина (первые элементы)
right = deque()  # правая половина (последние элементы)
```

### Обработка запроса

1. **`-`** (удаление из начала):
   - Если `left` пуст, переложить из `right` в `left` (но по инварианту такого не должно быть).
   - Вывести `left.popleft()`.

2. **`+ i`** (обычный гоблин в конец):
   - `right.append(i)`.
   - После добавления, если `len(left) < len(right)`, перекладываем один элемент из `right` в `left`.

3. **`* i`** (привилегированный в середину):
   - Если `len(left) > len(right)`, вставляем в **начало** `right` (`right.appendleft(i)`).
   - Иначе (равны) вставляем в **конец** `left` (`left.append(i)`).
   - После добавления балансируем: если `len(left) > len(right) + 1`, перекладываем один элемент из `left` в начало `right` (`right.appendleft(left.pop())`).

### Балансировка после операций

Чтобы инвариант всегда выполнялся, после добавления в `right` (обычный) или вставки в середину (привилегированный) необходимо скорректировать длины:

- Если `len(left) < len(right)`, перемещаем один элемент из `right` в `left` (`left.append(right.popleft())`).
- Если `len(left) > len(right) + 1`, перемещаем один элемент из `left` в начало `right` (`right.appendleft(left.pop())`).

В коде, приведённом в условии, эти коррекции вынесены после основного действия и выполняются для всех типов запросов.

## Реализация на Python

```python
from collections import deque

def process_goblin_queue():
    N = int(input())
    left = deque()
    right = deque()

    for _ in range(N):
        parts = input().split()
        if parts[0] == '-':               # удалить из начала
            print(left.popleft())
        elif parts[0] == '+':             # обычный в конец
            right.append(parts[1])
        else:                             # '*' – привилегированный в середину
            # если левая часть длиннее – в начало правой, иначе в конец левой
            if len(left) > len(right):
                right.appendleft(parts[1])
            else:
                left.append(parts[1])

        # Балансировка
        if len(left) < len(right):
            left.append(right.popleft())
        elif len(left) > len(right) + 1:
            right.appendleft(left.pop())
```

### Пояснение к коду

- **`left`** и **`right`** — деки, поддерживающие инвариант.
- Для `+` добавляем в `right`, затем корректируем.
- Для `*` в зависимости от длин решаем, куда добавить. После добавления также корректируем.
- Корректировка после каждого запроса гарантирует, что левая часть всегда содержит либо столько же, либо на один элемент больше правой.
- При удалении просто выводим первый элемент левой части (гарантированно существует).

## Сложность

- **Время:** Каждая операция (вставка, удаление, перекладывание) выполняется за O(1) благодаря `deque`. Балансировка перекладывает не более одного элемента за операцию, поэтому амортизированное время — O(1) на запрос.
- **Память:** O(N) для хранения элементов.
