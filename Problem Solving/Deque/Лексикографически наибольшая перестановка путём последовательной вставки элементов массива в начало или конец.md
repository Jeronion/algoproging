## Определение

Дан массив `arr` размера `n`. Требуется построить перестановку, последовательно вставляя элементы исходного массива (в том же порядке) либо в начало, либо в конец нового массива (изначально пустого), чтобы получить **лексикографически наибольший** возможный результат.

**Пример:**
```
arr = [3, 1, 2, 4] → 4 3 1 2
arr = [1, 2, 3, 4, 5] → 5 4 3 2 1
```

---

## Идея решения (жадный алгоритм с деком)

Используем двустороннюю очередь (`deque`). Первый элемент кладём в пустую очередь. Для каждого следующего элемента:
- Если текущий элемент **≥** первого элемента в очереди (фронта), то вставляем его в **начало** (чтобы большие числа оказались слева).
- Иначе (элемент меньше фронта) вставляем в **конец**.

Этот жадный подход гарантирует, что на каждом шаге мы максимизируем лексикографический порядок результирующей последовательности.

### Алгоритм

1. Создать пустой `deque`.
2. Добавить в него `arr[0]`.
3. Для каждого `i` от 1 до n-1:
   - Если `arr[i] >= dq[0]`, добавить `arr[i]` в начало (`appendleft`).
   - Иначе добавить в конец (`append`).
4. Преобразовать `deque` в список и вернуть.

---

## Реализация на Python

```python
from collections import deque

def largest_permutation(arr):
    dq = deque()
    dq.append(arr[0])
    for i in range(1, len(arr)):
        if arr[i] >= dq[0]:
            dq.appendleft(arr[i])
        else:
            dq.append(arr[i])
    return list(dq)
```

---

## Сложность

- **Время:** O(n) – один проход по массиву.
- **Память:** O(n) – для хранения дека.

---

## Примечания

- Алгоритм использует жадный выбор: всегда ставим максимальный из доступных элементов в начало, чтобы получить наибольшую последовательность.
- Дек позволяет эффективно добавлять элементы с обоих концов за O(1).
- Результат – лексикографически наибольшая перестановка среди всех возможных вариантов вставок.
