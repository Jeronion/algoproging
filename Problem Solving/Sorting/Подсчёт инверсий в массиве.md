## Определение

**Инверсия** в массиве — это пара индексов `(i, j)` таких, что `i < j` и `arr[i] > arr[j]`. Иными словами, это пара элементов, расположенных в «неправильном» порядке.

**Инверсионный подсчёт** — это количество таких пар. Для отсортированного по возрастанию массива инверсий нет (0), а для массива, отсортированного по убыванию, количество инверсий максимально (равно `n*(n-1)/2`).

---

## Идея решения (через сортировку слиянием)

Наивный подход с двумя вложенными циклами требует O(n²). Оптимальный метод использует **сортировку слиянием** и работает за O(n log n).

Основная идея: в процессе слияния двух отсортированных половин (левой и правой), если текущий элемент правой половины меньше текущего элемента левой половины, то **все оставшиеся элементы в левой половине** также больше этого элемента правой половины (так как левая половина отсортирована). Следовательно, можно сразу добавить количество этих элементов к общему счётчику инверсий.

Таким образом, алгоритм:
1. Рекурсивно разбиваем массив на две половины.
2. Считаем инверсии в левой половине, в правой половине и при слиянии.
3. При слиянии, когда берём элемент из правой половины, увеличиваем счётчик на количество оставшихся элементов в левой половине.

---

## Алгоритм (по шагам)

1. **Разделение:** если `l < r`, найти середину `mid = (l + r) // 2`.
2. **Рекурсивный вызов:** подсчитать инверсии в левой половине (`countInv(arr, l, mid)`) и в правой (`countInv(arr, mid+1, r)`).
3. **Слияние и подсчёт кросс-инверсий:** в функции `countAndMerge`:
   - Создать временные копии левой и правой половин.
   - Инициализировать указатели `i`, `j` и счётчик `res = 0`.
   - Пока обе половины не обработаны:
     - Если `left[i] <= right[j]`: копируем `left[i]` в массив, сдвигаем `i`.
     - Иначе ( `left[i] > right[j]` ): копируем `right[j]` в массив, сдвигаем `j` и добавляем к `res` количество оставшихся элементов в левой половине (`(n1 - i)`), так как каждый из них образует инверсию с `right[j]`.
   - Добавить оставшиеся элементы из любой половины.
   - Вернуть `res`.
4. Суммировать результаты левой, правой и кросс-инверсий.

## Реализация на Python

```python
def count_and_merge(arr, left, mid, right):
    # Создаём временные массивы для левой и правой половин
    L = arr[left:mid+1]
    R = arr[mid+1:right+1]
    i = j = 0
    k = left
    inversions = 0

    while i < len(L) and j < len(R):
        if L[i] <= R[j]:
            arr[k] = L[i]
            i += 1
        else:
            arr[k] = R[j]
            j += 1
            # Все оставшиеся в L элементы больше текущего R[j]
            inversions += len(L) - i
        k += 1

    # Копируем остатки
    while i < len(L):
        arr[k] = L[i]
        i += 1
        k += 1
    while j < len(R):
        arr[k] = R[j]
        j += 1
        k += 1
    return inversions

def count_inversions(arr, left, right):
    inversions = 0
    if left < right:
        mid = (left + right) // 2
        inversions += count_inversions(arr, left, mid)
        inversions += count_inversions(arr, mid + 1, right)
        inversions += count_and_merge(arr, left, mid, right)
    return inversions

def inversion_count(arr):
    # Чтобы не изменять исходный массив, работаем с копией
    temp = arr[:]
    return count_inversions(temp, 0, len(temp) - 1)

# Пример
arr = [4, 3, 2, 1]
print(inversion_count(arr))  # 6
```

## Сложность

- **Время:** O(n log n) — рекуррентное соотношение T(n) = 2T(n/2) + O(n) даёт O(n log n).
- **Память:** O(n) — временные массивы при слиянии, а также глубина рекурсии O(log n).

---

## Примечания

- Алгоритм **модифицирует** переданный массив (сортирует его). Если нужно сохранить исходный массив, следует передавать копию.
- Данный метод считается оптимальным для подсчёта инверсий в массиве (сравним с сортировкой слиянием).
- Отличие от наивного подхода: здесь одновременно происходит сортировка и подсчёт, что даёт выигрыш в скорости на больших данных.
- При наличии равных элементов условие `L[i] <= R[j]` гарантирует, что равные элементы не считаются инверсией (требуется строгое `>`).
