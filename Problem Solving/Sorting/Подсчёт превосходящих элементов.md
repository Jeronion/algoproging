## Определение

Дан массив **различных** целых чисел `arr`. Для каждого элемента `arr[i]` **превосходящим** называется такой элемент `arr[j]`, где `j > i` и `arr[j] > arr[i]`. Требуется для каждого элемента вывести количество его превосходителей (элементов справа, которые больше него).

**Пример:**
```
Вход: arr = [2, 7, 5, 3, 8, 1]
Выход: [4, 1, 1, 1, 0, 0]
Объяснение: для 2 справа больше: 7,5,3,8 → 4; для 7 → 8; для 5 → 8; для 3 → 8; для 8 и 1 — 0.
```

---

## Идея решения (через сортировку слиянием)

Наивный подход — двойной цикл за O(n²). Оптимальный метод использует **сортировку слиянием** и работает за O(n log n).

Ключевое наблюдение: при слиянии двух отсортированных половин (левой и правой), если текущий элемент левой половины `left[i]` **меньше** текущего элемента правой половины `right[j]`, то все оставшиеся элементы в правой половине (от `j` до конца) тоже больше `left[i]` (так как правая половина отсортирована по возрастанию). Следовательно, можно добавить количество этих элементов к счётчику превосходителей для `left[i]`.

Таким образом, в процессе обычного слияния мы накапливаем для каждого элемента количество элементов справа (из правой половины), которые больше его. Поскольку рекурсивный спуск разбивает массив на всё более мелкие части, в итоге для каждого элемента будут учтены все превосходители из правой части на всех уровнях слияния.

### Алгоритм

1. Создаём словарь (или массив, если значения не слишком велики) для хранения счётчиков превосходителей для каждого значения элемента.
2. Запускаем сортировку слиянием на копии исходного массива (чтобы не потерять исходный порядок). В процессе слияния:
   - Берём левую половину `L` и правую `R` (обе отсортированы).
   - Указатели `i` и `j` проходят по `L` и `R`.
   - Если `L[i] < R[j]`, то все элементы `R[j:]` больше `L[i]`, поэтому увеличиваем счётчик для `L[i]` на `len(R) - j`, затем помещаем `L[i]` в результирующий массив и сдвигаем `i`.
   - Иначе ( `L[i] > R[j]` ) просто помещаем `R[j]` в результат и сдвигаем `j`.
3. После завершения сортировки слиянием в словаре будут накоплены корректные количества превосходителей для каждого элемента.
4. Формируем итоговый массив, проходя по исходному порядку элементов и извлекая значения из словаря.

## Реализация

```python
def merge(arr, left, mid, right, counter):
    # левая половина: arr[left:mid+1], правая: arr[mid+1:right+1]
    L = arr[left:mid+1]
    R = arr[mid+1:right+1]
    i = j = 0
    k = left
    while i < len(L) and j < len(R):
        if L[i] < R[j]:
            # все оставшиеся в R больше L[i]
            counter[L[i]] += len(R) - j
            arr[k] = L[i]
            i += 1
        else:
            arr[k] = R[j]
            j += 1
        k += 1
    # остатки
    while i < len(L):
        arr[k] = L[i]
        i += 1
        k += 1
    while j < len(R):
        arr[k] = R[j]
        j += 1
        k += 1

def merge_sort(arr, left, right, counter):
    if left < right:
        mid = (left + right) // 2
        merge_sort(arr, left, mid, counter)
        merge_sort(arr, mid + 1, right, counter)
        merge(arr, left, mid, right, counter)

def find_surpasser(arr):
    if not arr:
        return []
    counter = {x: 0 for x in arr}
    dup = arr[:]          # копия для сортировки
    merge_sort(dup, 0, len(arr)-1, counter)
    return [counter[x] for x in arr]

# Пример
arr = [2, 7, 5, 3, 8, 1]
print(find_surpasser(arr))  # [4, 1, 1, 1, 0, 0]
```

## Сложность

- **Время:** O(n log n) — стандартная сложность сортировки слиянием.
- **Память:** O(n) — для временных массивов при слиянии и словаря.

---

## Примечания

- Алгоритм работает только для **различных** элементов. Если элементы могут повторяться, условие `L[i] < R[j]` следует заменить на `L[i] <= R[j]`? Нет, для равных элементов превосходителем считается строго больший, поэтому в случае равенства нельзя добавлять счётчик. При наличии дубликатов потребуется модификация.
- Метод аналогичен подсчёту инверсий, но здесь мы считаем не пары `(i,j)` с `i<j` и `arr[i] > arr[j]`, а пары с `arr[i] < arr[j]`. По сути, это «количество больших справа».
- Если значения элементов велики, словарь `counter` можно заменить на `defaultdict(int)`.
