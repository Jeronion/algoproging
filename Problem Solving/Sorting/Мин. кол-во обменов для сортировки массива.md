## Определение

Дан массив **различных** целых чисел. Требуется найти **минимальное количество обменов** (любых двух элементов), необходимое для сортировки массива по возрастанию.

**Пример:**
```
Вход: [2, 8, 5, 4] → Выход: 1 (меняем 8 и 4)
Вход: [10, 19, 6, 3, 5] → Выход: 2 (10↔3, 19↔5)
```

---

## Идея решения (графовый подход)

1. **Сопоставление позиций:** отсортируем массив и запомним, где каждый элемент должен оказаться. Для этого создаём список пар `(значение, исходный индекс)`, сортируем его по значению. Тогда `i`-й элемент в отсортированном списке должен стоять на позиции `i` (в 0-индексации), а его текущий индекс — `arrpos[i][0]`.

2. **Построение графа:** рассматриваем перестановку, где вершины — это индексы, а ребро идёт от `i` к `arrpos[i][0]` (текущая позиция элемента, который должен быть на месте `i`). Получаем несколько циклов.

3. **Обмены внутри цикла:** для цикла длины `cycle_size` требуется `cycle_size - 1` обменов, чтобы расставить все элементы на свои места.

4. **Суммирование:** общее количество обменов = сумма `(cycle_size - 1)` по всем циклам.

---

## Алгоритм

1. Создать список `arrpos` из пар `(элемент, исходный индекс)`.
2. Отсортировать `arrpos` по значению элемента.
3. Инициализировать массив `visited` для отслеживания обработанных индексов.
4. Для каждого индекса `i`:
   - Если `visited[i]` или элемент уже на своём месте (`arrpos[i][0] == i`), пропустить.
   - Иначе начать обход цикла: `j = i`, `cycle_size = 0`.
     - Пока `not visited[j]`: отметить `visited[j] = True`, перейти к `j = arrpos[j][0]`, увеличить `cycle_size`.
   - Добавить к ответу `cycle_size - 1`.
5. Вернуть сумму.

---

## Реализация на Python

```python
class Solution:
    def minSwaps(self, arr):
        n = len(arr)
        # Список (значение, индекс)
        arrpos = list(enumerate(arr))
        # Сортируем по значению
        arrpos.sort(key=lambda x: x[1])
        
        visited = [False] * n
        swaps = 0
        
        for i in range(n):
            if visited[i] or arrpos[i][0] == i:
                continue
            
            cycle_size = 0
            j = i
            while not visited[j]:
                visited[j] = True
                j = arrpos[j][0]
                cycle_size += 1
            if cycle_size > 0:
                swaps += cycle_size - 1
        
        return swaps
```

---

## Сложность

- **Время:** O(n log n) — из-за сортировки.
- **Память:** O(n) — для хранения пар и массива посещений.

---

## Примечания

- Алгоритм работает только для **различных** элементов.
- Каждый цикл длины `k` требует ровно `k-1` обменов.
- Этот метод оптимален, так как каждый обмен исправляет положение хотя бы одного элемента и не может исправить больше одного элемента за раз в цикле.
- Задача эквивалентна нахождению количества обменов в перестановке, которая задаётся отображением текущих позиций в правильные.
