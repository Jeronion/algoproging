## Определение

Дан поток целых чисел (массив `arr`). После каждого добавленного элемента нужно определить **k-й наибольший** элемент среди всех обработанных. Если в потоке меньше `k` элементов, ответ — `-1`. Требуется вернуть список результатов для всех позиций.

**Пример:**
```
arr = [1, 2, 3, 4, 5, 6], k = 4
Вывод: [-1, -1, -1, 1, 2, 3]
```

---

## Постановка задачи

**Вход:**
- `arr` — массив целых чисел (размер n ≤ 10⁵).
- `k` — порядковый номер (1 ≤ k ≤ n).

**Выход:** массив `res` длины `n`, где `res[i]` — k-й наибольший элемент после обработки первых `i+1` элементов (или -1, если элементов меньше k).

---

## Идея решения (min-heap размера k)

Используем **минимальную кучу (min-heap)** размера `k`, которая будет хранить `k` **наибольших** элементов из уже обработанного потока. Тогда корень этой кучи (`heap[0]`) будет минимальным среди них, то есть k-м наибольшим элементом.

**Алгоритм для каждого нового элемента `x`:**

1. Если размер кучи меньше `k`, просто добавляем `x` в кучу.
2. Иначе если `x` больше корня кучи (`heap[0]`), то заменяем корень на `x` (выполняем `heappushpop` или `heappop` + `heappush`).
3. После обновления кучи, если её размер равен `k`, то k-й наибольший — `heap[0]`, иначе `-1`.

Этот подход гарантирует, что в куче всегда находятся k наибольших элементов, а корень — это k-й по величине.

---

## Алгоритм (пошагово)

1. Инициализировать пустую min-heap `min_heap`.
2. Создать пустой список `res`.
3. Для каждого `x` в `arr`:
   - Если `len(min_heap) < k`:
     - `heappush(min_heap, x)`
   - Иначе если `x > min_heap[0]`:
     - `heapq.heappushpop(min_heap, x)` (заменяет минимальный на `x`)
   - После вставки:
     - Если `len(min_heap) == k`, то `res.append(min_heap[0])`, иначе `res.append(-1)`.
4. Вернуть `res`.

---

## Реализация на Python

```python
import heapq

class Solution:
    def kthLargest(self, arr, k):
        min_heap = []
        res = []
        for x in arr:
            if len(min_heap) < k:
                heapq.heappush(min_heap, x)
            else:
                if x > min_heap[0]:
                    heapq.heappushpop(min_heap, x)
            if len(min_heap) == k:
                res.append(min_heap[0])
            else:
                res.append(-1)
        return res
```

## Альтернативная (данная) реализация

В предоставленном коде используется иной порядок: сначала создаётся куча из первых `k` элементов, затем для каждого следующего элемента сначала выводится текущий k-й наибольший (до вставки нового), после чего куча обновляется. В конце добавляется последний результат. Этот подход также корректен и даёт те же результаты.

```python
import heapq

class Solution:
    def kthLargest(self, arr, k):
        res = [-1] * (k - 1)
        heap = arr[:k]
        heapq.heapify(heap)
        for i in range(k, len(arr)):
            res.append(heap[0])
            if arr[i] >= heap[0]:
                heapq.heappush(heap, arr[i])
                heapq.heappop(heap)
        res.append(heap[0])
        return res
```

## Сложность

- **Время:** O(n log k) — каждая операция с кучей (вставка, удаление) требует O(log k). Всего n операций.
- **Память:** O(k) для хранения кучи, плюс O(n) для результата (обязательно).

---

## Примечания

- Если `k = 1`, то это просто поиск максимума в потоке, и куча размера 1 даёт его за O(1) на элемент после первой вставки.
- Если все элементы одинаковы, алгоритм всё равно работает корректно.
- В Python `heapq` реализует min-heap, поэтому для хранения k наибольших используем обычный min-heap.
- При использовании `heapq.heappushpop` сначала добавляется новый элемент, затем удаляется минимальный, что эффективнее, чем `heappop` + `heappush`.
