## Определение

Дано бинарное дерево поиска (BST), которое одновременно является **полным бинарным деревом**. Требуется преобразовать его в **минимальную кучу (Min Heap)** таким образом, чтобы для каждого узла выполнялось условие: все значения в левом поддереве были **меньше** всех значений в правом поддереве. Структура дерева (форма) не изменяется, меняются только значения узлов.

**Пример:**
```
Исходное BST (полное):
        4
       / \
      2   6
     / \ / \
    1  3 5  7

После преобразования (Min Heap):
        1
       / \
      2   3
     / \ / \
    4  5 6  7
```
Preorder-обход даёт: 1, 2, 4, 5, 3, 6, 7 (но это не обязательно preorder; в коде preorder-заполнение даёт порядок 1,2,3,4,5,6,7, что соответствует preorder на полном дереве, но это не совсем preorder? В примере вывода preorder: 1 2 3 4 5 6 7. Это preorder именно для этого дерева? Да, если дерево полное и мы заполняем preorder отсортированными значениями, то получается такая последовательность.

---

## Постановка задачи

Дан корень BST, который также является полным бинарным деревом. Преобразовать его в Min Heap, не меняя структуру (только значения узлов). При этом должно сохраняться свойство: для каждого узла все значения в его левом поддереве должны быть меньше всех значений в правом поддереве.

---

## Идея решения

1. Выполнить **inorder-обход** BST и сохранить значения в массив. Inorder обход BST даёт отсортированную последовательность.
2. Выполнить **preorder-обход** дерева и присваивать узлам значения из массива в том порядке, в котором они встречаются при preorder-обходе.
   - Preorder-обход на полном бинарном дереве гарантирует, что родитель всегда получает меньшее значение, чем его дети (поскольку мы присваиваем значения в порядке возрастания). Это обеспечивает свойство min-heap.
   - Кроме того, из-за того, что preorder сначала посещает корень, затем левое поддерево, затем правое, а значения в массиве отсортированы, левое поддерево получает меньшие значения, чем правое, что удовлетворяет дополнительному условию.

Таким образом, преобразование выполняется за O(n) времени с использованием O(n) дополнительной памяти для хранения inorder-массива.

---

## Алгоритм

1. Создать пустой список `inorder_arr`.
2. Выполнить inorder-обход корня и заполнить `inorder_arr` значениями узлов (они будут отсортированы).
3. Выполнить preorder-обход дерева, при этом для каждого посещаемого узла:
   - Присвоить `node.data = inorder_arr[index]`, где `index` — текущая позиция в массиве.
   - Увеличить `index`.
4. После завершения preorder-обхода дерево становится Min Heap с требуемыми свойствами.

## Реализация на Python

```python
class Node:
    def __init__(self, data):
        self.data = data
        self.left = None
        self.right = None

def inorder_traversal(root, inorder_arr):
    if root is None:
        return
    inorder_traversal(root.left, inorder_arr)
    inorder_arr.append(root.data)
    inorder_traversal(root.right, inorder_arr)

def preorder_fill(root, inorder_arr, index):
    if root is None:
        return index
    root.data = inorder_arr[index]
    index += 1
    index = preorder_fill(root.left, inorder_arr, index)
    index = preorder_fill(root.right, inorder_arr, index)
    return index

def bst_to_min_heap(root):
    inorder_arr = []
    inorder_traversal(root, inorder_arr)
    preorder_fill(root, inorder_arr, 0)

def preorder_print(root):
    if root is None:
        return
    print(root.data, end=" ")
    preorder_print(root.left)
    preorder_print(root.right)

# Пример использования
if __name__ == "__main__":
    # Построение BST (полное дерево)
    root = Node(4)
    root.left = Node(2)
    root.right = Node(6)
    root.left.left = Node(1)
    root.left.right = Node(3)
    root.right.left = Node(5)
    root.right.right = Node(7)

    bst_to_min_heap(root)
    preorder_print(root)  # Вывод: 1 2 3 4 5 6 7
```

## Сложность

- **Время:** O(n), где n — количество узлов. Inorder-обход занимает O(n), preorder-заполнение также O(n).
- **Память:** O(n) для хранения inorder-массива плюс O(h) для стека рекурсии (h — высота дерева).

---

## Примечания

- Исходное дерево должно быть **полным** (complete binary tree), чтобы preorder-обход давал корректное распределение для min-heap. Однако алгоритм работает для любого бинарного дерева, но только в полном дереве preorder гарантирует, что родитель всегда меньше детей и что левое поддерево получает меньшие значения, чем правое.
- Если дерево не полное, условие о левом и правом поддеревьях может нарушиться, но свойство min-heap (родитель меньше детей) всё равно будет выполняться при любом обходе, если мы присваиваем значения в порядке возрастания в preorder? Нет, для min-heap нужно, чтобы родитель был меньше детей, но при preorder на произвольном дереве это не гарантируется. Поэтому условие "complete binary tree" важно.
- Метод не требует изменения структуры дерева, только перезапись данных.
