## Определение

Дано `n` отсортированных по возрастанию списков (строк матрицы). Требуется найти **наименьший диапазон** `[l, r]` (включительно), такой что в этом диапазоне содержится **хотя бы один элемент** из каждого из `n` списков. Если существует несколько диапазонов одинаковой длины, выбрать тот, у которого меньше `l`.

**Пример:**
```
mat = [[4, 7, 9, 12, 15],
       [0, 8, 10, 14, 20],
       [6, 12, 16, 30, 50]]
Вывод: [6, 8]
Объяснение: 7 (из первого), 8 (из второго), 6 (из третьего) покрываются диапазоном [6, 8].
```

---

## Постановка задачи

**Вход:** матрица `mat` размера `n × k`, каждая строка отсортирована по возрастанию.
**Выход:** массив из двух чисел — [l, r] (минимальный диапазон).

**Ограничения:** n, k ≤ 500, значения ≤ 10⁵.

---

## Идея решения (min-heap + скользящее окно)

Используем **min-heap** для хранения текущих кандидатов из каждого списка. В куче храним кортежи `(значение, индекс_списка, индекс_элемента)`. Также поддерживаем переменную `mx` — максимальное значение среди элементов, находящихся в куче (правый конец текущего диапазона).

Алгоритм:

1. Помещаем в кучу **первый** элемент из каждого списка. Вычисляем `mx` как максимум из них.
2. Инициализируем ответ `res = [ -inf, inf ]` (очень широкий диапазон).
3. Пока куча не пуста:
   - Извлекаем минимальный элемент `(val, i, j)` из кучи — это левый конец текущего диапазона.
   - Текущий правый конец — `mx`.
   - Если текущий диапазон `[val, mx]` короче, чем `res`, обновляем `res` (если длины равны, берём меньший `val`).
   - Если в списке `i` есть следующий элемент (индекс `j+1`), то добавляем его в кучу и обновляем `mx = max(mx, mat[i][j+1])`.
   - Если следующего элемента нет, то мы не можем продолжать, так как этот список закончился, и мы больше не сможем покрыть все списки. Завершаем цикл.
4. Возвращаем `res`.

Этот алгоритм является вариацией **k‑way merge** с отслеживанием максимума в окне.

## Реализация

```python
import heapq

class Solution:
    def findSmallestRange(self, mat):
        k = len(mat)
        heap = []
        mx = float('-inf')
        for i in range(k):
            val = mat[i][0]
            mx = max(mx, val)
            heapq.heappush(heap, (val, i, 0))
        
        res = [float('-inf'), float('inf')]
        while True:
            val, row, col = heapq.heappop(heap)
            # обновляем ответ, если текущий диапазон меньше
            if res[1] - res[0] > mx - val:
                res = [val, mx]
            # пытаемся продвинуться в этой строке
            if col + 1 < len(mat[row]):
                next_val = mat[row][col + 1]
                mx = max(mx, next_val)
                heapq.heappush(heap, (next_val, row, col + 1))
            else:
                break
        return res
```

---

## Сложность

- **Время:** O(n·k·log n), где n — количество списков, k — длина каждого списка. Каждый элемент может быть добавлен в кучу не более одного раза.
- **Память:** O(n) для хранения кучи.

---

## Примечания

- Алгоритм гарантирует, что будет найден диапазон минимальной длины, так как мы перебираем все возможные «окна», содержащие по одному элементу из каждого списка, и выбираем наименьшее.
- Если несколько диапазонов имеют одинаковую длину, выбор с меньшим `l` обеспечивается условием `res[1] - res[0] > mx - val` (строгое сравнение) — при равной длине мы не обновляем ответ, оставляя текущий с меньшим `l`.
- Метод также известен как «наименьший диапазон, покрывающий все списки».
