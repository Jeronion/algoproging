## Определение

Дан массив целых положительных чисел `arr` и целое число `k`. Требуется найти `k` **наибольших** элементов массива и вернуть их в порядке **убывания**. Если `k` равно размеру массива, возвращаем весь массив, отсортированный по убыванию.

**Пример:**
```
arr = [12, 5, 787, 1, 23], k = 2 → [787, 23]
```

---

## Постановка задачи

**Вход:**
- `arr` — массив целых чисел (размер до 10⁶).
- `k` — количество элементов, которые нужно вернуть (1 ≤ k ≤ arr.size()).

**Выход:** массив из `k` наибольших элементов, отсортированных по убыванию.

---

## Идея решения (min-heap размера k)

Вместо сортировки всего массива (O(n log n)) можно использовать **минимальную кучу (min-heap)** размера `k`. В куче будем хранить `k` наибольших элементов, встреченных на данный момент.

Алгоритм:

1. Создаём min-heap из первых `k` элементов массива (heapify за O(k)).
2. Для каждого следующего элемента `arr[i]` (начиная с индекса `k`):
   - Если `arr[i]` больше корня кучи (минимального элемента в куче), то заменяем корень на `arr[i]`: удаляем минимальный и добавляем новый элемент (push + pop).
3. В результате в куче останутся `k` наибольших элементов массива.
4. Извлекаем все элементы из кучи (они будут в порядке возрастания), затем переворачиваем список, чтобы получить убывающий порядок.

Этот подход работает за O(n log k) времени и использует O(k) дополнительной памяти.

---

## Алгоритм (пошагово)

1. Если `k == len(arr)`, можно просто отсортировать массив по убыванию и вернуть его (но это не обязательно, алгоритм и так корректно обработает).
2. Создать кучу `min_heap` из первых `k` элементов: `heapq.heapify(min_heap)`.
3. Для `i` от `k` до `n-1`:
   - Если `arr[i] > min_heap[0]` (корень кучи — минимальный из k элементов):
     - Выполнить `heapq.heappushpop(min_heap, arr[i])` (или push затем pop).
4. После цикла извлечь все элементы из кучи в список `res` (pop в цикле).
5. Развернуть `res` (так как из кучи они идут в порядке возрастания) и вернуть.

## Реализация на Python

```python
import heapq

class Solution:
    def kLargest(self, arr, k):
        # создаём min-heap из первых k элементов
        min_heap = arr[:k]
        heapq.heapify(min_heap)
        
        # обрабатываем остальные элементы
        for i in range(k, len(arr)):
            if arr[i] > min_heap[0]:
                heapq.heappushpop(min_heap, arr[i])
        
        # извлекаем все элементы из кучи
        res = []
        while min_heap:
            res.append(heapq.heappop(min_heap))
        # возвращаем в убывающем порядке
        return res[::-1]
```

## Сложность

- **Время:** O(n log k), где n — размер массива. Построение кучи O(k), каждый из `n-k` элементов может быть вставлен и удалён за O(log k).
- **Память:** O(k) для хранения кучи.

---

## Примечания

- Если `k` близко к `n`, сортировка всего массива (O(n log n)) может быть сопоставима по времени, но для больших `n` и малых `k` этот метод эффективнее.
- В Python модуль `heapq` реализует min-heap.
- Альтернативное решение: сортировка по убыванию и взятие первых `k` элементов — O(n log n) времени и O(1) дополнительной памяти (если сортировать на месте). Для больших `n` и малых `k` лучше использовать кучу.
- Если массив содержит дубликаты, они будут корректно обработаны (в ответ попадут значения, а не индексы).
