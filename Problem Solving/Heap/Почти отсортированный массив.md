## Определение

Дан массив `arr[]`, в котором каждый элемент находится на расстоянии не более `k` позиций от своей правильной позиции в отсортированном массиве. Требуется восстановить полный порядок, переставив элементы **на месте**, не используя встроенные функции сортировки.

**Пример:**
```
arr = [2, 3, 1, 4], k = 2
Правильный порядок: [1, 2, 3, 4]
Элементы сдвинуты:
1 (индекс 2) → 0
2 (0) → 1
3 (1) → 2
4 (3) → 3
```

---

## Постановка задачи

Дан массив `arr` размера `n` и целое `k` (0 ≤ k < n). Каждый элемент `arr[i]` в итоговом отсортированном массиве должен оказаться на позиции `j`, такой что `|i - j| ≤ k`. Нужно отсортировать массив, переставляя элементы **на месте**, без использования `sort()` или аналогичных методов. Ожидается, что алгоритм будет эффективным для больших `n` и малых `k`.

---

## Идея решения (мини-куча размера k+1)

Если элемент может сдвинуться не более чем на `k` позиций влево, то при обработке массива слева направо для текущей позиции `i` правильный элемент должен находиться среди первых `k+1` элементов, начиная с `i`. Поэтому можно использовать **минимальную кучу** (min-heap) размера `k+1`:

1. Помещаем в кучу первые `k+1` элементов.
2. Для каждой позиции `i` от `0` до `n - k - 2` (т.е. пока остаются элементы для подстановки):
   - Извлекаем минимальный элемент из кучи и помещаем его в `arr[i]`.
   - Добавляем в кучу следующий элемент из массива (с индексом `i + k + 1`), если он существует.
3. Когда все элементы из конца массива добавлены, извлекаем оставшиеся элементы из кучи и помещаем их в конец массива.

Таким образом, на каждом шаге мы выбираем наименьший доступный элемент, что гарантирует правильный порядок.

### Почему размер кучи равен k+1?

Для позиции `i` возможные кандидаты — элементы с индексами `i .. i+k` (так как элемент не может сдвинуться влево более чем на `k`). Поэтому в куче должны храниться элементы, начиная с текущего индекса `i` и до `i+k`. После того как мы поместили элемент в `i`, мы добавляем следующий элемент `i+k+1`, чтобы окно сместилось.

---

## Алгоритм (пошагово)

1. Если `k == 0` (массив уже отсортирован), можно сразу вернуть массив.
2. Создать min-heap из первых `min(n, k+1)` элементов.
3. Для `i` от `0` до `n - k - 1`:
   - Извлечь минимальный элемент из кучи и записать его в `arr[i]`.
   - Если `i + k + 1 < n`, добавить в кучу `arr[i + k + 1]`.
4. После того как все элементы из хвоста добавлены, в куче останутся оставшиеся элементы (их будет не более `k`). Извлечь их по порядку и записать в конец массива.
5. Вернуть отсортированный массив.

## Реализация

```python
import heapq

class Solution:
    def nearlySorted(self, arr, k):
        n = len(arr)
        # Если k == 0, массив уже отсортирован
        if k == 0:
            return arr
        
        # Min-heap из первых k+1 элементов
        min_heap = arr[:k+1]
        heapq.heapify(min_heap)
        
        # Основной цикл
        for i in range(n - k - 1):
            arr[i] = heapq.heappop(min_heap)
            heapq.heappush(min_heap, arr[i + k + 1])
        
        # Оставшиеся элементы
        idx = n - k - 1
        while min_heap:
            arr[idx] = heapq.heappop(min_heap)
            idx += 1
        
        return arr
```

---

## Сложность

- **Время:** O(n log k). На каждом шаге выполняется `pop` и `push` над кучей размера `k+1`, каждая операция O(log k). Всего O(n) таких операций.
- **Память:** O(k) для хранения кучи.

---

## Примечания

- Алгоритм особенно эффективен, когда `k` мало по сравнению с `n`. Если `k` близко к `n`, сложность стремится к O(n log n), и проще использовать обычную сортировку.
- Метод работает **на месте** (меняет исходный массив) и не требует дополнительной памяти, кроме кучи.
- Встроенные функции сортировки не используются, что соответствует условию.
- Если `k = 0`, куча не нужна, массив уже отсортирован.
