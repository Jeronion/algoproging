## Определение

**Максимальная куча (max-heap)** — это полное бинарное дерево, в котором для каждого узла выполняется свойство: значение родителя **больше или равно** значениям его детей. При хранении кучи в массиве (в порядке уровней, начиная с индекса 0) элементы располагаются так:

- Корень находится на индексе `0`.
- Для узла с индексом `i`:
  - Левый ребёнок: `2*i + 1`
  - Правый ребёнок: `2*i + 2` (если существует)

Задача: по данному массиву определить, является ли он корректной максимальной кучей.

---

## Постановка задачи

Дан массив целых чисел `arr`. Требуется проверить, удовлетворяет ли он свойству максимальной кучи: для каждого индекса `i`, где `2*i+1 < n` (левый ребёнок существует), должно выполняться `arr[i] >= arr[2*i+1]`, и аналогично для правого ребёнка, если он есть. Если все условия выполнены, вернуть `True`, иначе `False`.

**Пример:**
```
arr = [90, 15, 10, 7, 12, 2, 7, 3]
Проверка:
- 90 >= 15 и 90 >= 10 (ok)
- 15 >= 7 и 15 >= 12 (ok)
- 10 >= 2 и 10 >= 7 (ok)
- 7 >= 3 (ok)
=> True
```

---

## Идея решения (итеративный обход)

В максимальной куче достаточно проверить все **внутренние узлы** (узлы, у которых есть хотя бы один ребёнок). В массиве внутренние узлы находятся на индексах от `0` до `(n-2)//2` (включительно). Для каждого такого узла проверяем его детей:

- Если левый ребёнок существует и его значение больше родителя → не куча.
- Если правый ребёнок существует и его значение больше родителя → не куча.

Если все проверки пройдены успешно, массив является максимальной кучей.

Этот подход не требует дополнительной памяти и выполняется за O(n) времени.

---

## Алгоритм

1. Найти длину массива `n`.
2. Пройти по индексам `i` от `0` до `(n-2)//2` (включительно):
   - Проверить левого ребёнка: если `arr[2*i + 1] > arr[i]`, вернуть `False`.
   - Проверить правого ребёнка: если `2*i + 2 < n` и `arr[2*i + 2] > arr[i]`, вернуть `False`.
3. Если цикл завершился без нарушений, вернуть `True`.

---

## Реализация на Python

```python
def isMaxHeap(arr):
    n = len(arr)
    # Проверяем только внутренние узлы
    for i in range((n - 2) // 2 + 1):
        # Левый ребёнок существует всегда для внутреннего узла (кроме последнего уровня)
        if arr[2 * i + 1] > arr[i]:
            return False
        # Правый ребёнок может отсутствовать
        if 2 * i + 2 < n and arr[2 * i + 2] > arr[i]:
            return False
    return True
```

---

## Сложность

- **Время:** O(n), где n — размер массива. Каждый внутренний узел проверяется один раз.
- **Память:** O(1) — используются только переменные и цикл.

---

## Примечания

- Индексы внутренних узлов вычисляются по формуле `(n-2)//2`, так как последний узел, у которого есть хотя бы один ребёнок, находится на этой позиции.
- В массиве размером `n` листьями являются элементы с индексами от `(n//2)` до `n-1`. Их проверять не требуется.
- Алгоритм предполагает, что массив представляет собой полное бинарное дерево (заполнение уровней слева направо), что всегда верно для кучи, хранимой в массиве.
- Если массив пуст или содержит один элемент, он считается корректной кучей (условия не нарушаются).
