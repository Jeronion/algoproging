## Определение

Даны `n` канатов с заданными длинами (массив `arr`). Требуется соединить все канаты в один, причём стоимость соединения двух канатов равна сумме их длин. Нужно найти **минимальную общую стоимость**, чтобы получить один канат.

**Пример:**
```
arr = [4, 3, 2, 6]
Оптимальный порядок:
- соединить 2 и 3 → стоимость 5, массив [4, 5, 6]
- соединить 4 и 5 → стоимость 9, массив [9, 6]
- соединить 9 и 6 → стоимость 15
Итого: 5 + 9 + 15 = 29.
```

---

## Постановка задачи

Дан массив целых чисел `arr`, представляющих длины канатов. Нужно вернуть минимальную общую стоимость соединения всех канатов в один. Если канатов меньше двух, стоимость равна `0`.

---

## Идея решения (жадный алгоритм с min-heap)

Чтобы минимизировать общую стоимость, нужно на каждом шаге соединять **два самых коротких** каната. Это классическая жадная стратегия: чем раньше мы соединим короткие канаты, тем меньше они будут участвовать в последующих сложениях.

Используем **минимальную кучу (min-heap)**, чтобы всегда быстро извлекать два наименьших элемента. После соединения сумма длин добавляется обратно в кучу. Процесс повторяется, пока в куче не останется один элемент.

### Алгоритм

1. Преобразовать массив в min-heap (heapify).
2. Инициализировать `total_cost = 0`.
3. Пока в куче больше одного элемента:
   - Извлечь два наименьших элемента: `a = heappop()`, `b = heappop()`.
   - Стоимость соединения: `cost = a + b`.
   - Добавить `cost` к `total_cost`.
   - Вставить `cost` обратно в кучу (новый канат).
4. Вернуть `total_cost`.

---

## Реализация

```python
import heapq

class Solution:
    def minCost(self, arr):
        heapq.heapify(arr)
        total = 0
        while len(arr) > 1:
            first = heapq.heappop(arr)
            second = heapq.heappop(arr)
            cost = first + second
            total += cost
            heapq.heappush(arr, cost)
        return total
```

---

## Сложность

- **Время:** O(n log n) — heapify за O(n), каждый из `n-1` шагов извлекает и вставляет по два элемента, каждая операция O(log n). Итого O(n log n).
- **Память:** O(n) для хранения кучи (используется исходный массив, преобразованный на месте).

---

## Примечания

- Жадный подход работает, потому что задача обладает свойством оптимальной подструктуры: на каждом шаге выбор двух наименьших канатов не увеличивает общую стоимость.
- Если в массиве один канат, ответ `0`.
- В Python `heapq` реализует min-heap, что идеально подходит для этой задачи.
