## Определение

Дана последовательность чисел, поступающих по одному (поток данных). После каждого нового числа требуется вычислить **медиану** текущего набора чисел.

- Если количество чисел **нечётное**, медиана — это средний элемент в отсортированном порядке.
- Если количество чисел **чётное**, медиана — это среднее арифметическое двух средних элементов.

**Пример:**
```
Поток: 5, 15, 1, 3, 2, 8
Медианы: 5.0, 10.0, 5.0, 4.0, 3.0, 4.0
```

---

## Постановка задачи

Дан массив `arr` (поток). Требуется вернуть список `res` размера `n`, где `res[i]` — медиана после обработки первых `i+1` элементов. Ожидаемая сложность O(n log n) времени и O(n) памяти.

---

## Идея решения (две кучи)

Используем две кучи:

- **Левая куча (max-heap)** — хранит **меньшую половину** элементов. Доступ к максимуму этой половины осуществляется за O(1).
- **Правая куча (min-heap)** — хранит **большую половину** элементов. Доступ к минимуму этой половины — O(1).

Поддерживаем инвариант:
- `len(left) == len(right)` или `len(left) == len(right) + 1`.
- Все элементы в `left` ≤ все элементы в `right`.

**Алгоритм добавления нового элемента:**

1. Добавляем новый элемент в `left` (max-heap). Для этого в Python используем `heapq.heappush(left, -x)` (так как `heapq` реализует min-heap, мы храним отрицательные значения).
2. Если после вставки `left` и `right` не пусты и максимальный элемент `left` больше минимального элемента `right`, то нарушается порядок. Перекладываем максимальный из `left` в `right` и минимальный из `right` в `left` (или проще: выталкиваем из `left` в `right`, затем из `right` в `left`, чтобы восстановить порядок). В коде используется вариант: если `-left[0] > right[0]`, то меняем местами вершины.
3. Если размер `left` превышает размер `right` более чем на 1, перемещаем вершину из `left` в `right`.

**Вычисление медианы:**

- Если текущее количество элементов (i+1) **нечётное**, медиана — это максимум `left` (т.е. `-left[0]`).
- Если **чётное**, медиана — среднее арифметическое максимума `left` и минимума `right`: `(right[0] - left[0]) / 2` (так как в `left` хранятся отрицательные значения, `-left[0]` даёт положительное).

---

## Алгоритм (пошагово)

1. Инициализировать `left = []` (max-heap) и `right = []` (min-heap), `res = []`.
2. Для каждого `x` в `arr`:
   - Добавить `x` в `left`: `heappush(left, -x)`.
   - Если `left` и `right` не пусты и `-left[0] > right[0]`:
     - `max_left = -heappop(left)`
     - `min_right = heappop(right)`
     - `heappush(left, -min_right)`
     - `heappush(right, max_left)`
   - Если `len(left) > len(right) + 1`:
     - `heappush(right, -heappop(left))`
   - Вычислить медиану:
     - Если `(i+1) % 2 == 1`: `median = -left[0]`
     - Иначе: `median = (right[0] - left[0]) / 2.0`
   - Добавить `median` в `res`.
3. Вернуть `res`.

## Реализация на Python

```python
import heapq

class Solution:
    def getMedian(self, arr):
        left = []   # max-heap (храним отрицательные значения)
        right = []  # min-heap
        res = []
        for i, x in enumerate(arr):
            # добавляем в left
            heapq.heappush(left, -x)
            # если нарушен порядок между кучами
            if left and right and -left[0] > right[0]:
                max_left = -heapq.heappop(left)
                min_right = heapq.heappop(right)
                heapq.heappush(left, -min_right)
                heapq.heappush(right, max_left)
            # балансировка размеров
            if len(left) > len(right) + 1:
                heapq.heappush(right, -heapq.heappop(left))
            # вычисление медианы
            if (i + 1) % 2 == 1:
                res.append(-left[0])
            else:
                res.append((right[0] - left[0]) / 2.0)
        return res
```

## Сложность

- **Время:** O(n log n) — каждая вставка и удаление из кучи требуют O(log n). Всего n операций.
- **Память:** O(n) — для хранения элементов в кучах.

---

## Примечания

- В Python `heapq` — min-heap, поэтому для max-heap мы храним отрицательные значения.
- Важно поддерживать баланс размеров куч, чтобы медиана всегда была доступна за O(1).
- Алгоритм также известен как «медиана в потоке» и часто используется в задачах реального времени.
