## Условие

Задача известна как [LeetCode 373. Find K Pairs with Smallest Sums](https://leetcode.com/problems/find-k-pairs-with-smallest-sums/).

Даны два отсортированных по возрастанию целочисленных массива `nums1` и `nums2`, а также целое число `k`. Требуется вернуть `k` наименьших пар `(u, v)`, где `u ∈ nums1`, `v ∈ nums2`, упорядоченных по возрастанию суммы `u + v`. В случае равенства сумм порядок пар не важен.

**Пример:**
```
nums1 = [1,7,11], nums2 = [2,4,6], k = 3
Вывод: [[1,2],[1,4],[1,6]]
Объяснение: наименьшие суммы: 1+2=3, 1+4=5, 1+6=7, следующие 7+2=9 и т.д.
```

---

## Идея решения

Так как оба массива отсортированы, минимальная сумма всегда будет у пары `(0,0)`. Далее возможные кандидаты — пары `(0,1)` и `(1,0)`. Из них выбирается наименьшая, затем добавляются её соседи и т.д. Этот процесс напоминает алгоритм Дейкстры на виртуальном графе, где вершины — индексы `(i, j)`, рёбра ведут к `(i, j+1)` и `(i+1, j)`, а вес ребра — сумма элементов.

Для эффективного извлечения минимальной суммы используется **min-куча**, в которую помещаются пары индексов вместе с их суммой. Также необходимо отслеживать уже посещённые пары индексов, чтобы не добавлять их повторно.

### Алгоритм

1. Создать min-кучу `heap` и множество `visited`.
2. Поместить в кучу стартовую пару `(0, 0)` с суммой `nums1[0] + nums2[0]`. Добавить `(0,0)` в `visited`.
3. Пока куча не пуста и длина результата меньше `k`:
   - Извлечь из кучи элемент `(сумма, i, j)`.
   - Добавить в результат пару `[nums1[i], nums2[j]]`.
   - Если `j+1` меньше длины `nums2` и `(i, j+1)` ещё не посещена:
     - Добавить `(i, j+1)` в `visited`.
     - Поместить в кучу `(nums1[i] + nums2[j+1], i, j+1)`.
   - Если `i+1` меньше длины `nums1` и `(i+1, j)` ещё не посещена:
     - Добавить `(i+1, j)` в `visited`.
     - Поместить в кучу `(nums1[i+1] + nums2[j], i+1, j)`.
4. Вернуть результат.

## Реализация

```python
import heapq

def kSmallestPairs(nums1: List[int], nums2: List[int], k: int) -> List[List[int]]:
    if not nums1 or not nums2:
        return []

    heap = []
    visited = set()
    result = []

    # Стартовая пара
    heapq.heappush(heap, (nums1[0] + nums2[0], 0, 0))
    visited.add((0, 0))

    while heap and len(result) < k:
        _, i, j = heapq.heappop(heap)
        result.append([nums1[i], nums2[j]])

        # Добавляем (i, j+1)
        if j + 1 < len(nums2) and (i, j + 1) not in visited:
            visited.add((i, j + 1))
            heapq.heappush(heap, (nums1[i] + nums2[j + 1], i, j + 1))

        # Добавляем (i+1, j)
        if i + 1 < len(nums1) and (i + 1, j) not in visited:
            visited.add((i + 1, j))
            heapq.heappush(heap, (nums1[i + 1] + nums2[j], i + 1, j))

    return result
```

## Сложность

- **Время:** O(k log k) — каждая операция с кучей (push/pop) занимает O(log size), а размер кучи не превышает O(k). Количество итераций — не более k.
- **Память:** O(k) — для хранения кучи и множества `visited`.

---

## Примечания

- Множество `visited` необходимо для избежания дублирования пар. Например, пара `(1,1)` может быть добавлена как из `(1,0)`, так и из `(0,1)`.
- Если `k` очень велико (например, больше чем возможное количество пар), алгоритм вернёт все возможные пары.
- Вместо `visited` можно использовать двумерный массив булевых значений, если размеры массивов невелики.
- Альтернативное решение: перебирать все возможные суммы для одного массива и использовать бинарный поиск для другого, но это менее эффективно.
