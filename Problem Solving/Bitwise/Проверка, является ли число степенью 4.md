## Определение

Дано целое число `n`. Требуется определить, является ли оно **степенью числа 4**, то есть существует ли целое неотрицательное `k` такое, что `n = 4^k`.

**Примеры:**
- `16 = 4^2` → `true`
- `20` → `false`
- `1 = 4^0` → `true`

---

## 1. Метод с битовыми операциями (O(log n))

### Идея

Любая степень 4 является также степенью 2. У степени 2 в двоичной записи ровно один бит, равный 1. У степени 4 этот единственный бит находится на **чётной позиции** (если считать позиции с 0 справа). Например:
- `1` (4^0) = `1` (бит на позиции 0)
- `4` (4^1) = `100` (бит на позиции 2)
- `16` (4^2) = `10000` (бит на позиции 4)

Позиция бита — это показатель степени двойки. Если этот показатель чётный, то число является степенью 4.

### Алгоритм

1. Проверить, что `n > 0` и `n & (n - 1) == 0` (т.е. `n` — степень двойки).
2. Найти позицию единственного единичного бита (сдвигами вправо, пока `n > 1`, считая количество сдвигов).
3. Если количество сдвигов чётное → `true`, иначе `false`.

### Реализация

```python
def isPowerOfFour_bit(n):
    if n <= 0 or (n & (n - 1)) != 0:
        return False
    count = 0
    while n > 1:
        n >>= 1
        count += 1
    return count % 2 == 0
```

### Сложность

- **Время:** O(log n) — количество битов в числе.
- **Память:** O(1).

## 2. Метод битовой маски (O(1))

### Идея

Как и выше, число должно быть степенью 2. Кроме того, его единственный единичный бит должен находиться на чётной позиции. Это можно проверить с помощью маски `0xAAAAAAAA` (в двоичной записи: `10101010101010101010101010101010`), у которой все биты на **нечётных** позициях установлены в 1. Если `n & 0xAAAAAAAA == 0`, значит, в числе нет единиц на нечётных позициях, следовательно, его бит (если он один) находится на чётной позиции.

---

### Алгоритм

1. Проверить `n > 0` и `(n & (n - 1)) == 0` (степень 2).
2. Проверить `(n & 0xAAAAAAAA) == 0` (бит на чётной позиции).
3. Если оба условия истинны → `true`, иначе `false`.

---

### Реализация

```python
def isPowerOfFour_mask(n):
    return n > 0 and (n & (n - 1)) == 0 and (n & 0xAAAAAAAA) == 0
```

### Сложность

- **Время:** O(1) — несколько битовых операций.
- **Память:** O(1).

---

## Примечания

- Отрицательные числа и ноль не являются степенями 4 (по определению степени натурального числа).
- В методе с маской `0xAAAAAAAA` подразумевается 32-битное представление. Для больших чисел в Python это также работает, так как маска расширяется неявно.
- Альтернативная маска: `0x55555555` (биты на чётных позициях), тогда условие было бы `(n & 0x55555555) != 0`, но обычно используют проверку на отсутствие битов в нечётных позициях.
