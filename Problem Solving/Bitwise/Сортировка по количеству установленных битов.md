## Определение

Дан массив целых чисел `arr`. Требуется отсортировать массив **по убыванию количества единичных битов** в двоичном представлении каждого числа. При этом для чисел с одинаковым количеством битов необходимо сохранить их **относительный порядок** (стабильная сортировка).

**Пример:**
```
Вход: [5, 2, 3, 9, 4, 6, 7, 15, 32]
Выход: [15, 7, 5, 3, 9, 6, 2, 4, 32]
Объяснение:
15 (1111) – 4 бита
7  (0111) – 3 бита
5  (0101) – 2 бита
3  (0011) – 2 бита
9  (1001) – 2 бита
6  (0110) – 2 бита
2  (0010) – 1 бит
4  (0100) – 1 бит
32 (10000) – 1 бит
```

---

## Идея решения

Используется **блочная сортировка (bucket sort)**, так как количество возможных значений битов ограничено (для чисел до 10⁶ достаточно 20 битов, но берём 32).

1. Создаём массив из 32 пустых списков (бакетов), где индекс соответствует количеству установленных битов.
2. Для каждого элемента массива вычисляем количество битов (например, через `countBits`) и добавляем элемент в соответствующий бакет.
3. Проходим по бакетам в порядке **убывания** количества битов (от 31 до 0) и последовательно выписываем все элементы из каждого бакета в исходный массив.

Так как мы не перемешиваем элементы внутри одного бакета, их относительный порядок сохраняется, что обеспечивает стабильность.

## Реализация на Python

```python
class Solution:
    def countBits(self, n):
        b = 0
        while n > 0:
            b += n & 1
            n >>= 1
        return b
    
    def sortBySetBitCount(self, arr, n):
        # создаём 32 пустых списка (максимум битов для int)
        buckets = [[] for _ in range(32)]
        
        # распределяем элементы по бакетам
        for num in arr:
            bits = self.countBits(num)
            buckets[bits].append(num)
        
        # собираем результат (от большего количества битов к меньшему)
        idx = 0
        for i in range(31, -1, -1):
            for num in buckets[i]:
                arr[idx] = num
                idx += 1
```

---

## Сложность

- **Время:** O(n) — один проход для подсчёта битов и распределения, один проход для сбора. Подсчёт битов для каждого элемента выполняется за O(log max_val), но в данном случае это константа (32 бита).
- **Память:** O(n) — для хранения бакетов.

---

## Примечания

- В коде использован ручной подсчёт битов (сдвиги и побитовое И). Можно заменить на `bin(num).count('1')` или `int.bit_count()` (Python 3.8+).
- Бакетов 32 достаточно, так как `arr[i] ≤ 10⁶` (2²⁰ ≈ 1 млн). Для больших чисел можно взять 64.
- Сортировка стабильна благодаря последовательному добавлению в бакеты.
- Задача известна на GeeksforGeeks: "Sort by Set Bit Count".
