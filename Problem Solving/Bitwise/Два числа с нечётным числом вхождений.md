## Определение

Дан массив целых положительных чисел, в котором **все числа встречаются чётное количество раз**, кроме **ровно двух чисел**, которые встречаются нечётное количество раз. Требуется найти эти два числа и вернуть их в порядке убывания.

**Пример:**
```
Вход: [4, 2, 4, 5, 2, 3, 3, 1]
Выход: [5, 1]
Объяснение: 5 встречается 1 раз, 1 встречается 1 раз, остальные – чётное число раз.
```

---

## Идея решения (побитовый XOR)

1. **XOR всех элементов** даёт `x ^ y`, где `x` и `y` – искомые числа (все остальные сокращаются, так как `a ^ a = 0`).
2. Находим **правый установленный бит** в `xor = x ^ y` (например, `xor & -xor`). Этот бит равен 1 там, где `x` и `y` различаются.
3. Разделяем все числа на две группы:
   - у которых этот бит установлен,
   - у которых этот бит не установлен.
   Так как `x` и `y` различаются в этом бите, они попадут в разные группы.
4. Вычисляем XOR внутри каждой группы – в каждой группе останется одно из искомых чисел (все остальные числа в группе встречаются чётное число раз и сокращаются).
5. Возвращаем два полученных числа в порядке убывания.

---

## Алгоритм

1. Инициализировать `xor_all = 0`.
2. Для каждого `num` в `arr`: `xor_all ^= num`.
3. Найти правый установленный бит: `rightmost = xor_all & -xor_all`.
4. Инициализировать `ans = [0, 0]`.
5. Для каждого `num` в `arr`:
   - Если `num & rightmost == 0`, то `ans[0] ^= num`
   - Иначе `ans[1] ^= num`.
6. Отсортировать `ans` по убыванию и вернуть.

---

## Реализация на Python

```python
class Solution:
    def twoOddNum(self, arr):
        xor_all = 0
        for num in arr:
            xor_all ^= num
        
        # Правый установленный бит
        rightmost = xor_all & -xor_all
        
        ans = [0, 0]
        for num in arr:
            if num & rightmost == 0:
                ans[0] ^= num
            else:
                ans[1] ^= num
        
        return sorted(ans, reverse=True)
```

---

## Сложность

- **Время:** O(n) – один проход для XOR всех элементов, второй проход для разделения.
- **Память:** O(1) – только несколько переменных.

---

## Пример работы

`arr = [4, 2, 4, 5, 2, 3, 3, 1]`

1. `xor_all = 4^2^4^5^2^3^3^1 = (4^4)^(2^2)^(3^3)^5^1 = 0^0^0^5^1 = 4` (двоичное `100`).
2. `rightmost = 4 & -4 = 4` (бит на позиции 2).
3. Разделение:
   - Группа 0 (бит не установлен): `2,2,4,4,3,3,?` – на самом деле: числа без бита 2: 2 (010), 2, 4 (100) – у 4 бит 2 установлен? 4 = 100, бит 2 (значение 4) установлен, значит 4 попадает в группу 1. Пересчитаем:
     - `4` (100) – бит 2 установлен → группа 1.
     - `2` (010) – бит 2 не установлен → группа 0.
     - `5` (101) – бит 2 установлен → группа 1.
     - `1` (001) – не установлен → группа 0.
     - `3` (011) – бит 2? 3 = 011, бит 2 (4) не установлен → группа 0.
   - Группа 0: XOR = 2^2^3^3^1 = 0^0^1 = 1.
   - Группа 1: XOR = 4^4^5 = 0^5 = 5.
4. Результат: `[5, 1]`.

---

## Примечания

- Метод работает только при условии, что все остальные числа встречаются чётное количество раз.
- `xor & -xor` выделяет самый правый установленный бит (в двоичном представлении).
- Порядок возврата – убывающий, поэтому после вычислений применяется сортировка (или можно сразу сохранять в правильном порядке).
