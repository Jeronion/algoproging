## Условие задачи

Дано выражение, которое может содержать скобки четырёх видов: `()`, `[]`, `{}`, `<>`. Необходимо проверить, является ли скобочная последовательность правильной (правильная вложенность и порядок). Любые другие символы следует игнорировать.

**Входные данные:** одна строка.

**Выходные данные:** `YES`, если выражение правильное, иначе `NO`.

**Примеры:**

| Вход     | Выход |
|----------|-------|
| `<()[]>` | YES   |
| `([{}]`  | NO    |

---

## Идея решения

Классическая задача на **стек** (). При обходе строки:
- Открывающие скобки `([{<` помещаем в стек.
- Закрывающие скобки `)]}>` проверяем: если стек не пуст и верхняя скобка соответствует текущей закрывающей, то убираем верхнюю (удаляем пару). Иначе выражение неверно.
- Все остальные символы игнорируем.
- После обработки всей строки стек должен быть пустым — тогда выражение правильное.

---

## Алгоритм

1. Создать пустой стек.
2. Создать словарь соответствия закрывающей скобки открывающей:
   ```
   pairs = {')': '(', ']': '[', '}': '{', '>': '<'}
   ```
3. Для каждого символа `c` во входной строке:
   - Если `c` не является скобкой (т.е. `c not in "()[]{}<>"`), пропустить.
   - Если `c` — открывающая скобка (`c in "([{<"`), поместить её в стек.
   - Если `c` — закрывающая скобка:
     - Если стек пуст → выражение неверно (закрывающая скобка без открывающей).
     - Иначе взять верхнюю скобку из стека `top = stack.pop()` и сравнить с `pairs[c]`. Если не равны → выражение неверно.
4. После обработки всех символов:
   - Если стек пуст → выражение верно (`YES`).
   - Иначе → остались не закрытые открывающие скобки → `NO`.

## Реализация на Python

```python
from collections import deque

def brackets_check():
    seq = input()
    stack = deque()
    brackets_dict = {">": "<", ")": "(", "}": "{", "]": "["}
    for i in seq:
        if i not in "(){}[]<>":
            continue
        if i in "{[<(":
            stack.append(i)
        elif stack:
            if brackets_dict[i] != stack.pop():
                return False
        else:
            return False
    else:
        return not stack
```

**Пояснение:** здесь используется `deque` как стек (операции `append` и `pop` с конца). Словарь `brackets_dict` отображает закрывающую скобку на открывающую.

---

## Сложность

- **Время:** O(n), где n — длина строки. Каждый символ обрабатывается один раз.
- **Память:** O(n) в худшем случае (если все символы — открывающие скобки).
