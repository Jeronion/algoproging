## Определение

Дан массив целых чисел `arr`. Для каждого размера окна `k` (от 1 до n) рассмотрим все непрерывные подмассивы длины `k`. Найдём минимальный элемент в каждом таком подмассиве, затем возьмём **максимум** среди этих минимальных значений. Требуется вернуть список таких максимумов для `k = 1, 2, ..., n`.

**Пример:**
```
arr = [10, 20, 30] → [30, 20, 10]
Объяснение:
k=1: минимумы [10,20,30] → max = 30
k=2: минимумы [10,20], [20,30] → max = 20
k=3: минимум [10,20,30] → max = 10
```

---

## Идея решения (стек + границы)

Прямой перебор всех окон требует O(n²) или O(n³). Оптимальное решение O(n) основано на следующем наблюдении: **каждый элемент `arr[i]` является минимумом во всех окнах, которые не содержат элементов, меньших `arr[i]`, и ограничены ближайшими меньшими элементами слева и справа**.

Для каждого элемента найдём:
- `prev_smaller` – индекс ближайшего слева элемента, **строго меньшего** текущего.
- `next_smaller` – индекс ближайшего справа элемента, **строго меньшего** текущего.

Тогда максимальное окно, в котором `arr[i]` является минимумом, имеет размер `wsize = next_smaller - prev_smaller - 1`. Для этого размера окна (а также для всех меньших окон, содержащих `i`) значение `arr[i]` является кандидатом на максимум минимума.

### Алгоритм

1. Вычислить для каждого элемента `i`:
   - `prev[i]` – индекс предыдущего меньшего (или -1).
   - `next[i]` – индекс следующего меньшего (или n).
2. Создать массив `windowMax` длины `n+1`, инициализированный нулями.
3. Для каждого `i`:
   - `wsize = next[i] - prev[i] - 1`
   - `windowMax[wsize] = max(windowMax[wsize], arr[i])`
4. Пройти по `windowMax` от `n` до 1, заполняя `res[k-1] = max(windowMax[k], res[k])` (так как для большего окна максимум минимума не может быть меньше, чем для меньшего окна, содержащего те же элементы? На самом деле, если для размера `k` есть некоторое значение `x`, то для всех меньших размеров значение может быть не меньше `x` (потому что минимум в меньшем окне может быть больше или равен). Поэтому после накопления прямых значений, делаем обратный проход: `res[i] = max(res[i], res[i+1])`.

## Реализация

```python
def max_of_min_for_all_windows(arr):
    n = len(arr)
    # previous smaller element (strictly smaller)
    prev = [-1] * n
    stack = []
    for i in range(n):
        while stack and arr[stack[-1]] >= arr[i]:
            stack.pop()
        prev[i] = stack[-1] if stack else -1
        stack.append(i)

    # next smaller element (strictly smaller)
    nxt = [n] * n
    stack.clear()
    for i in range(n):
        while stack and arr[stack[-1]] >= arr[i]:
            idx = stack.pop()
            nxt[idx] = i
        stack.append(i)

    window_max = [0] * (n + 1)
    for i in range(n):
        length = nxt[i] - prev[i] - 1
        window_max[length] = max(window_max[length], arr[i])

    res = [0] * n
    cur = 0
    for k in range(n, 0, -1):
        cur = max(cur, window_max[k])
        res[k-1] = cur
    return res

# Пример
arr = [10, 20, 30, 50, 10, 70, 30]
print(max_of_min_for_all_windows(arr))
# [70, 30, 20, 10, 10, 10, 10]
```

---

## Сложность

- **Время:** O(n) — вычисление prev и next за линейное время, заполнение `window_max` за O(n), финальный проход O(n).
- **Память:** O(n) — для массивов `prev`, `next`, `window_max`.

## Примечания

- Важно, что при нахождении `prev` и `next` используется условие `>=` (строгое `>=`), чтобы для равных элементов один из них считался меньшим и не увеличивал окно неверно. Это стандартный подход, чтобы каждый элемент был учтён в своей максимальной области минимальности.
- После вычисления `window_max` для каждого размера окна, мы делаем обратный проход, так как для меньшего окна максимум минимума не может быть меньше, чем для большего (поскольку в большем окне минимум может быть меньше, а максимум минимумов – это максимальное значение среди минимумов по всем окнам данного размера; для меньшего окна можно взять ту же самую позицию, что и для большего, и получить минимум не меньше, так как окно уже). Однако строгое доказательство: пусть `f(k)` = max(min для окон размера k). Тогда `f(k) >= f(k+1)`? Да, это свойство монотонного убывания. Поэтому мы можем просто проходить от больших k к меньшим и поддерживать максимум.
