## Определение

Дан массив целых чисел `arr`. Для каждого элемента `arr[i]` нужно найти **первый элемент слева**, который **строго меньше** текущего. Если такого элемента нет, возвращается `-1`.

**Пример:**
```
arr = [1, 6, 2] → [-1, 1, 1]
arr = [1, 5, 0, 3, 4, 5] → [-1, 1, -1, 0, 3, 4]
```

---

## Идея решения (монотонный стек)

Используем стек для хранения элементов, которые могут быть кандидатами на роль предыдущего меньшего для последующих элементов. Стек поддерживается **возрастающим** (от меньшего к большему). При проходе слева направо:

- Для текущего элемента `arr[i]` удаляем из стека все элементы, которые **не меньше** текущего (т.е. `>= arr[i]`), поскольку они не могут быть ответом ни для текущего, ни для будущих элементов (так как текущий элемент находится ближе и меньше).
- Если после удаления стек не пуст, то вершина стека – это ближайший слева меньший элемент. Иначе ответ `-1`.
- Затем помещаем текущий элемент в стек.

### Алгоритм

1. Инициализировать пустой стек `st` и массив `res` размера `n`.
2. Для `i = 0` до `n-1`:
   - Пока стек не пуст и `st[-1] >= arr[i]`, выполнить `st.pop()`.
   - Если стек пуст, `res[i] = -1`, иначе `res[i] = st[-1]`.
   - Добавить `arr[i]` в стек.
3. Вернуть `res`.

---

## Реализация

```python
class Solution:
    def prevSmaller(self, arr):
        n = len(arr)
        res = [0] * n
        stack = []
        for i in range(n):
            while stack and stack[-1] >= arr[i]:
                stack.pop()
            res[i] = stack[-1] if stack else -1
            stack.append(arr[i])
        return res
```

---

## Сложность

- **Время:** O(n) — каждый элемент помещается в стек и удаляется из него не более одного раза.
- **Память:** O(n) — в худшем случае стек может содержать все элементы (например, для строго возрастающей последовательности).

---

## Примечания

- Алгоритм аналогичен поиску **следующего большего элемента**, но с обходом в обратную сторону и условием на меньше.
- Если требуется **предыдущий меньший или равный**, условие меняется на `stack[-1] > arr[i]`.
- Стек в данном решении хранит сами значения, а не индексы (это допустимо, так как для ответа нужно только значение, а не позиция). Однако если нужны индексы, можно хранить индексы и сравнивать значения через `arr[stack[-1]]`.
