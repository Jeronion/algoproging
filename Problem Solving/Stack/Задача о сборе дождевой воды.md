## Определение

Дан массив `arr` неотрицательных целых чисел, где каждое число представляет высоту столбца гистограммы (ширина столбца = 1). После дождя между столбцами может задерживаться вода. Требуется вычислить **общее количество единиц воды**, которое может быть задержано.

**Пример:**
```
arr = [3, 0, 1, 0, 4, 0, 2] → 10
arr = [1, 2, 3, 4] → 0
```

---

## Идея решения

Вода может быть задержана над столбцом `i`, если слева и справа от него есть более высокие столбцы. Количество воды над `i` равно `min(максимальная высота слева, максимальная высота справа) - arr[i]` (если разность положительна). Суммируя по всем `i`, получаем ответ.

### Подход 1: Префиксные и суффиксные максимумы (O(n) времени, O(n) памяти)

Вычисляем два вспомогательных массива:
- `left[i]` – максимальная высота среди элементов `arr[0..i]`.
- `right[i]` – максимальная высота среди элементов `arr[i..n-1]`.
Тогда вода над `i` равна `min(left[i], right[i]) - arr[i]` (если >0).

```python
def trap_rain_water_prefix(arr):
    n = len(arr)
    if n < 3:
        return 0
    left = [0] * n
    right = [0] * n
    left[0] = arr[0]
    for i in range(1, n):
        left[i] = max(left[i-1], arr[i])
    right[n-1] = arr[n-1]
    for i in range(n-2, -1, -1):
        right[i] = max(right[i+1], arr[i])
    water = 0
    for i in range(1, n-1):
        water += max(0, min(left[i], right[i]) - arr[i])
    return water
```

### Подход 2: Два указателя (O(n) времени, O(1) памяти)

Используем два указателя `left` и `right`, которые движутся с двух концов. Поддерживаем максимальные высоты слева (`l_max`) и справа (`r_max`). В каждый момент, если `l_max < r_max`, то вода над `left` определяется левым максимумом, иначе – правым.

```python
def trap_rain_water_two_pointers(arr):
    if len(arr) < 3:
        return 0
    left, right = 0, len(arr) - 1
    l_max = arr[left]
    r_max = arr[right]
    water = 0
    while left < right:
        if l_max <= r_max:
            left += 1
            l_max = max(l_max, arr[left])
            water += l_max - arr[left]
        else:
            right -= 1
            r_max = max(r_max, arr[right])
            water += r_max - arr[right]
    return water
```

### Подход 3: Стек (O(n) времени, O(n) памяти)

Используем стек для хранения индексов. Когда текущая высота больше высоты на вершине стека, вычисляем воду, накапливающуюся между предыдущей и текущей высотами.

```python
def trap_rain_water_stack(arr):
    stack = []
    water = 0
    for i in range(len(arr)):
        while stack and arr[i] > arr[stack[-1]]:
            top = stack.pop()
            if not stack:
                break
            distance = i - stack[-1] - 1
            bounded_height = min(arr[i], arr[stack[-1]]) - arr[top]
            water += distance * bounded_height
        stack.append(i)
    return water
```

## Сложность

| Подход                     | Время   | Память |
|----------------------------|---------|--------|
| Префикс/суффикс            | O(n)    | O(n)   |
| Два указателя              | O(n)    | O(1)   |
| Стек                       | O(n)    | O(n)   |

---

## Примечания

- Подход с двумя указателями является наиболее эффективным по памяти и часто ожидается на собеседованиях.
- Алгоритм работает для любых неотрицательных высот.
- Важно, что вода задерживается только между столбцами, а не на них самих.
