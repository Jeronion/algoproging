## Определение

Даны два целых числа `n` (размер массива) и `k` (количество стеков). Требуется реализовать `k` стеков, используя **один массив** размера `n`. Все стеки должны динамически разделять общее пространство, поддерживая операции:

- `push(x, i)` – добавить элемент `x` в `i`-й стек (0‑индексация).
- `pop(i)` – удалить и вернуть верхний элемент из `i`-го стека, или выдать сообщение об ошибке, если стек пуст.

---

## 1. Наивный подход: фиксированные сегменты (O(1) времени, O(n+k) памяти)

### Идея

Разделить массив на `k` равных (или почти равных) сегментов. Каждый стек получает свою непрерывную область. Храним массив `top[]` размера `k`, где `top[i]` – индекс вершины `i`-го стека (изначально на один меньше начала сегмента).

**Плюсы:** простота.  
**Минусы:** неэффективное использование памяти – один стек может переполниться, даже если в других сегментах есть свободное место.

### Реализация

```python
class kStacksFixed:
    def __init__(self, n, k):
        self.seg_size = n // k
        self.arr = [0] * n
        self.top = [i * self.seg_size - 1 for i in range(k)]

    def push(self, x, i):
        end = (i + 1) * self.seg_size - 1
        if self.top[i] == end:
            print(f"Stack {i} overflow")
            return
        self.top[i] += 1
        self.arr[self.top[i]] = x

    def pop(self, i):
        start = i * self.seg_size
        if self.top[i] < start:
            print(f"Stack {i} underflow")
            return -1
        val = self.arr[self.top[i]]
        self.top[i] -= 1
        return val
```

## 2. Оптимизированный подход: динамическое использование свободных ячеек (O(1) времени, O(n+k) памяти)

### Идея

Все стеки делят массив **динамически**. Используем три вспомогательные структуры:

- `arr[n]` – основной массив для хранения элементов.
- `top[k]` – для каждого стека хранит индекс вершины (или `-1`, если стек пуст).
- `next[n]` – выполняет две роли:
  - Для занятой ячейки: указывает на предыдущий элемент в том же стеке (связывает элементы стека в обратном порядке).
  - Для свободной ячейки: указывает на следующий свободный индекс (образует **список свободных ячеек**).
- `freeTop` – индекс начала списка свободных ячеек.

Инициализация:  
`next[i] = i+1` для `i = 0..n-2`, `next[n-1] = -1`.  
`freeTop = 0` (первая свободная ячейка).  
Все `top[i] = -1`.

### Операция push(x, i)

1. Если `freeTop == -1` – массив заполнен, переполнение.
2. Берём первый свободный индекс `idx = freeTop`.
3. Обновляем `freeTop = next[idx]` (отцепляем ячейку от свободного списка).
4. Кладём `x` в `arr[idx]`.
5. Устанавливаем `next[idx] = top[i]` (связываем с предыдущей вершиной стека).
6. Обновляем `top[i] = idx`.

### Операция pop(i)

1. Если `top[i] == -1` – стек пуст.
2. Берём индекс вершины `idx = top[i]`.
3. Запоминаем значение `val = arr[idx]`.
4. Обновляем `top[i] = next[idx]` (перемещаем вершину на предыдущий элемент).
5. Возвращаем ячейку в список свободных:  
   `next[idx] = freeTop`, `freeTop = idx`.
6. Возвращаем `val`.

### Реализация

```python
class kStacks:
    def __init__(self, n, k):
        self.arr = [0] * n
        self.top = [-1] * k
        self.next = list(range(1, n)) + [-1]
        self.freeTop = 0

    def push(self, x, i):
        if self.freeTop == -1:
            print("Stack Overflow")
            return
        idx = self.freeTop
        self.freeTop = self.next[idx]
        self.arr[idx] = x
        self.next[idx] = self.top[i]
        self.top[i] = idx

    def pop(self, i):
        if self.top[i] == -1:
            print("Stack Underflow")
            return -1
        idx = self.top[i]
        self.top[i] = self.next[idx]
        self.next[idx] = self.freeTop
        self.freeTop = idx
        return self.arr[idx]
```

---

## Пример использования

```python
st = kStacks(9, 3)
st.push(10, 1)
st.push(20, 1)
st.push(30, 1)
st.push(100, 0)
st.push(200, 0)
print(st.pop(1))  # 30
print(st.pop(1))  # 20
print(st.pop(0))  # 200
st.push(2, 2)
```

**Вывод:**
```
30
20
200
```

---

## Сложность

- **Время** каждой операции (`push`, `pop`): O(1).
- **Память:** O(n + k) для массивов `arr`, `top`, `next`.

---

## Примечания

- Оптимизированный подход полностью исключает внутреннюю фрагментацию, так как любая свободная ячейка может быть использована любым стеком.
- Требуется дополнительная память для массива `next` (размер n) и `top` (размер k), что незначительно по сравнению с размером хранимых элементов.
- Этот метод также известен как **«K стеков в массиве с использованием списка свободных ячеек»**.
