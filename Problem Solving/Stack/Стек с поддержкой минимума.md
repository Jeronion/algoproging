## Определение

Необходимо реализовать стек, поддерживающий операции `push(x)`, `pop()`, `peek()` и `getMin()` за **O(1)** времени. `getMin()` должен возвращать наименьший элемент в стеке (или -1, если стек пуст). Допустимо использование дополнительной памяти, но операции должны быть константными.

**Пример:**
```
push(2), push(3), peek() → 3, pop(), getMin() → 2, push(1), getMin() → 1
```

---

## 1. С использованием вспомогательного стека (O(1) времени, O(n) памяти)

### Идея

Храним два стека:
- `st` – основные элементы.
- `minSt` – на каждом шаге содержит текущий минимум (дублируя его для каждого элемента).

При `push(x)`: добавляем `x` в `st`; в `minSt` добавляем `x`, если `x <= minSt[-1]`, иначе дублируем `minSt[-1]`.  
При `pop()`: удаляем из обоих стеков.  
`getMin()` возвращает `minSt[-1]`.

```python
class MinStack:
    def __init__(self):
        self.st = []
        self.minSt = []

    def push(self, x):
        self.st.append(x)
        if not self.minSt or x <= self.minSt[-1]:
            self.minSt.append(x)
        else:
            self.minSt.append(self.minSt[-1])

    def pop(self):
        if self.st:
            self.st.pop()
            self.minSt.pop()

    def peek(self):
        return -1 if not self.st else self.st[-1]

    def getMin(self):
        return -1 if not self.minSt else self.minSt[-1]
```

## 2. С использованием пары в стеке (O(1) времени, O(n) памяти)

### Идея

Храним в стеке пары `(значение, минимум до этого уровня)`.  
При `push(x)`: новый минимум = `min(x, предыдущий минимум)`.  
`pop()`, `peek()`, `getMin()` работают тривиально.

```python
class MinStackPair:
    def __init__(self):
        self.st = []

    def push(self, x):
        new_min = x if not self.st else min(x, self.st[-1][1])
        self.st.append((x, new_min))

    def pop(self):
        if self.st:
            self.st.pop()

    def peek(self):
        return -1 if not self.st else self.st[-1][0]

    def getMin(self):
        return -1 if not self.st else self.st[-1][1]
```

## 3. Без дополнительной памяти (O(1) времени, O(1) памяти)

### Идея

Используем одну переменную `minEle` для хранения текущего минимума. При `push` значения меньшего, чем `minEle`, вместо него сохраняем **кодированное** значение `2*x - minEle`, а `minEle` обновляем на `x`. При `pop` декодируем предыдущий минимум.

**Формулы:**
- Если `x < minEle`: в стек кладём `2*x - minEle`, затем `minEle = x`.
- При извлечении, если `top < minEle`, то предыдущий минимум = `2*minEle - top`.

```python
class MinStackOpt:
    def __init__(self):
        self.st = []
        self.minEle = -1

    def push(self, x):
        if not self.st:
            self.minEle = x
            self.st.append(x)
        elif x < self.minEle:
            self.st.append(2 * x - self.minEle)
            self.minEle = x
        else:
            self.st.append(x)

    def pop(self):
        if not self.st:
            return
        top = self.st.pop()
        if top < self.minEle:
            self.minEle = 2 * self.minEle - top

    def peek(self):
        if not self.st:
            return -1
        top = self.st[-1]
        return self.minEle if top < self.minEle else top

    def getMin(self):
        return -1 if not self.st else self.minEle
```

## Сравнение подходов

| Подход | Время | Память | Сложность кода |
|--------|-------|--------|----------------|
| Вспомогательный стек | O(1) | O(n) | Простой |
| Пара в стеке | O(1) | O(n) | Очень простой |
| Без доп. памяти | O(1) | O(1) | Средняя |

---

## Примечания

- Подход без дополнительной памяти использует **кодирование** и корректно работает для любых целых чисел.
- При использовании вспомогательного стека или пар память линейна, но реализация проще.
- Все методы обеспечивают O(1) для всех операций.
