## Определение

Дан массив `arr`, где `arr[i]` — цена акции в день `i`. Для каждого дня `i` требуется вычислить **размах** (span) — количество **подряд идущих дней** до `i` (включая сам день `i`), в которых цена была **меньше или равна** цене в день `i`. Другими словами, нужно найти ближайший предыдущий день, когда цена была **выше** текущей, и размах = расстояние между этими днями.

**Пример:**
```
arr = [100, 80, 90, 120]
Размахи: [1, 1, 2, 4]
Объяснение:
- день 0 (100): нет предыдущих → 1
- день 1 (80): предыдущий 100 > 80 → 1
- день 2 (90): предыдущий 80 ≤ 90, но 100 > 90 → дни 1 и 2 → 2
- день 3 (120): все предыдущие ≤ 120 → 4
```

---

## Идея решения (монотонный стек)

Классический подход использует стек для поиска **предыдущего большего элемента**. Однако в приведённом коде используется обход **справа налево**: для каждого элемента мы определяем, для каких будущих элементов он является «стенкой», ограничивающей размах. В результате в стеке хранятся индексы элементов, для которых ещё не найден больший элемент справа.

### Алгоритм (обход справа налево)

1. Инициализируем массив `res` нулями.
2. Инициализируем пустой стек.
3. Проходим по массиву `i` от `n-1` до `0`:
   - Пока стек не пуст и `arr[st[-1]] < arr[i]` (текущий элемент больше, чем элемент на вершине стека), это означает, что для элемента `st[-1]` текущий элемент является ближайшим большим слева? Нет, внимательно: мы идём справа налево, значит, `st[-1]` — это индекс некоторого элемента, который находится правее. Если `arr[i] > arr[st[-1]]`, то для элемента справа (в стеке) текущий элемент `i` является **первым большим слева** (т.е. ближайшим слева, который больше). Тогда размах для `st[-1]` будет `st[-1] - i`. Записываем это в `res[st[-1]]` и удаляем этот индекс из стека.
   - Добавляем текущий индекс `i` в стек.
4. После цикла в стеке останутся индексы, для которых нет большего слева (т.е. все предыдущие цены меньше или равны). Для них размах = `индекс + 1` (дней от начала до этого дня включительно).
5. Возвращаем `res`.

---

## Реализация

```python
class Solution:
    def calculateSpan(self, arr):
        n = len(arr)
        res = [0] * n
        stack = []
        # Обход справа налево
        for i in range(n - 1, -1, -1):
            while stack and arr[stack[-1]] < arr[i]:
                idx = stack.pop()
                res[idx] = idx - i
            stack.append(i)
        # Для оставшихся в стеке (нет большего слева)
        while stack:
            idx = stack.pop()
            res[idx] = idx + 1
        return res
```

---

## Сложность

- **Время:** O(n) — каждый индекс попадает в стек и выходит из него не более одного раза.
- **Память:** O(n) — для стека и массива результатов.

## Примечания

- Альтернативный классический подход — обход слева направо с хранением в стеке индексов, для которых ещё не найден больший элемент, и вычислением размаха как `i - stack[-1]` (где `stack[-1]` — предыдущий больший элемент).
- Данный код решает задачу, используя поиск **ближайшего большего слева** (через обход справа налево), что даёт тот же результат.
- Если все цены одинаковы, то размах для каждого дня равен `i+1`.
