## Определение

**Задача о покупке и продаже акций с периодом охлаждения** — это одна из классических задач динамического программирования, в которой трейдер может совершать неограниченное количество транзакций, но после продажи акций он обязан **пропустить один день** (cooldown), прежде чем снова сможет покупать. Цель — максимизировать общую прибыль.

**Пример:**
```
prices = [1, 2, 3, 0, 2]
Максимальная прибыль = 3
Последовательность: buy at 1, sell at 3, cooldown, buy at 0, sell at 2 → (3-1)+(2-0)=4? Нет, правильный ответ: buy 1, sell 2, cooldown, buy 0, sell 2 → 1+2=3. В некоторых вариантах ответ 3.
```

---

## Постановка задачи

**Вход:** массив `prices` (длина n), где `prices[i]` — цена акции в день i.
**Выход:** максимальная прибыль, которую можно получить, при условии, что:
- Можно покупать и продавать неограниченное количество раз.
- Нельзя владеть более чем одной акцией одновременно.
- После продажи акции следует **один день охлаждения** (нельзя покупать на следующий день).
- Можно продать и купить в тот же день? Нет, продажа и покупка в один день не разрешены, так как после продажи идёт cooldown.

---

## Идея решения

Задача решается с помощью **динамического программирования с конечными состояниями (state machine)**. В каждый день мы можем находиться в одном из двух состояний:
- `buy` — мы можем купить акцию (или пропустить день).
- `sell` — мы можем продать акцию (или пропустить день).

Определим:
- `dp[i][1]` — максимальная прибыль на день i, когда мы **можем покупать** (т.е. мы не владеем акцией и не на cooldown).
- `dp[i][0]` — максимальная прибыль на день i, когда мы **можем продавать** (т.е. мы владеем акцией и на следующий день будет cooldown).

Рекуррентные соотношения (с конца к началу):
- Если мы можем покупать (`buy`): у нас есть два варианта:
  1. Купить акцию сегодня: `-prices[i] + dp[i+1][0]` (потратили деньги, но теперь мы владеем акцией, значит на следующий день мы можем продавать).
  2. Пропустить день: `dp[i+1][1]` (остаёмся в состоянии покупки).
  Берём максимум: `dp[i][1] = max(-prices[i] + dp[i+1][0], dp[i+1][1])`.
- Если мы можем продавать (`sell`): варианты:
  1. Продать акцию сегодня: `prices[i] + dp[i+2][1]` (получаем прибыль, но теперь у нас cooldown, поэтому следующий день мы не можем покупать, только через день — `i+2`).
  2. Пропустить день: `dp[i+1][0]` (остаёмся в состоянии продажи).
  Берём максимум: `dp[i][0] = max(prices[i] + dp[i+2][1], dp[i+1][0])`.

Базовые случаи: за пределами массива прибыль равна 0.

Ответ: `dp[0][1]` — максимальная прибыль с первого дня, когда можно покупать.

---

## Подход 1: Табличный Bottom-Up DP (O(n) времени, O(n) памяти)

Используем двумерный массив `dp` размера `(n+2) × 2`, где `dp[i][1]` и `dp[i][0]` определены выше. Заполняем с конца к началу.

### Реализация

```python
def maxProfit(prices):
    n = len(prices)
    dp = [[0] * 2 for _ in range(n + 2)]
    for i in range(n - 1, -1, -1):
        # buy state
        dp[i][1] = max(-prices[i] + dp[i + 1][0], dp[i + 1][1])
        # sell state
        dp[i][0] = max(prices[i] + dp[i + 2][1], dp[i + 1][0])
    return dp[0][1]
```

### Сложность

- **Время:** O(n)
- **Память:** O(n)

## Подход 2: Оптимизация по памяти (O(n) времени, O(1) памяти) — ожидаемый подход

Заметим, что для вычисления `dp[i][1]` нужны значения из `i+1` дня, а для `dp[i][0]` нужны значения из `i+2` дня. Таким образом, нам не нужно хранить всю таблицу, достаточно хранить только:
- `ahead1` — значения для дня `i+1` (массив из двух элементов: [sell_state, buy_state]).
- `ahead2` — значения для дня `i+2`.

На каждой итерации мы вычисляем `curr` (текущий день) и сдвигаем указатели: `ahead2 = ahead1`, `ahead1 = curr`.

### Алгоритм

1. Инициализировать `ahead1 = [0, 0]`, `ahead2 = [0, 0]`.
2. Для `i` от `n-1` до `0`:
   - Вычислить `curr[1] = max(-prices[i] + ahead1[0], ahead1[1])`.
   - Вычислить `curr[0] = max(prices[i] + ahead2[1], ahead1[0])`.
   - Обновить `ahead2 = ahead1[:]`, `ahead1 = curr[:]`.
3. Вернуть `ahead1[1]`.

### Реализация

```python
def maxProfit(prices):
    n = len(prices)
    ahead1 = [0, 0]  # dp[i+1]
    ahead2 = [0, 0]  # dp[i+2]
    curr = [0, 0]    # dp[i]
    for i in range(n - 1, -1, -1):
        curr[1] = max(-prices[i] + ahead1[0], ahead1[1])
        curr[0] = max(prices[i] + ahead2[1], ahead1[0])
        ahead2 = ahead1[:]
        ahead1 = curr[:]
    return ahead1[1]
```

### Сложность

- **Время:** O(n)
- **Память:** O(1)

## Примечания

- Состояние `buy` означает, что мы не владеем акцией и можем покупать (т.е. либо ничего не делаем, либо покупаем).
- Состояние `sell` означает, что мы владеем акцией и можем продавать (или держать).
- После продажи мы переходим в `buy` только через один день, поэтому при продаже используется `dp[i+2][1]`.
- Если `n` мало, массив `dp` или `ahead` обрабатывают выход за границы через нули (благодаря инициализации нулями).
