## Определение

Даны две строки `txt` и `pat`. Требуется подсчитать количество различных способов, которыми `pat` может быть образована как **подпоследовательность** `txt` (не обязательно непрерывная, но порядок символов должен сохраняться). Ответ гарантированно помещается в 32‑битное целое.

**Пример:**
```
txt = "abba", pat = "aba"
Подпоследовательности: 
- индексы 0,1,2 → "aba"
- индексы 0,1,3 → "aba"
Итого 2 способа.
```

---

## Постановка задачи

**Вход:** строки `txt` и `pat` (длины до 10³, строчные буквы).
**Выход:** количество подпоследовательностей `txt`, равных `pat`.

---

## Идея решения (динамическое программирование)

Пусть `dp[i][j]` — количество способов образовать префикс `pat[0..i]` (длиной i+1) как подпоследовательность префикса `txt[0..j]` (длиной j+1).

Для каждого символа `txt[j]` у нас есть два варианта:
- Не использовать его в подпоследовательности: тогда количество способов равно `dp[i][j-1]`.
- Использовать его, если он равен `pat[i]`: тогда добавляем `dp[i-1][j-1]` (количество способов образовать предыдущий префикс паттерна до этого символа).

Таким образом, рекуррентное соотношение:

$$
dp[i][j] = dp[i][j-1] + \begin{cases}
dp[i-1][j-1], & \text{если } txt[j] = pat[i] \\
0, & \text{иначе}
\end{cases}
$$

Базовые случаи:
- Для `i=0` (первый символ `pat[0]`): `dp[0][j]` — количество вхождений `pat[0]` в `txt[0..j]`.
- Для `j=0` (только один символ `txt[0]`): `dp[i][0] = 1`, если `txt[0] == pat[i]` и `i=0`, иначе 0.

В коде используется двумерный массив `(m x n)`, где `m = len(pat)`, `n = len(txt)`. Сначала заполняется первая строка для `pat[0]`, затем остальные строки для остальных символов `pat`.

---

## Алгоритм (пошагово)

1. Инициализировать `dp` размером `m × n` нулями.
2. Заполнить первую строку `dp[0][j]`:
   - `dp[0][0] = 1`, если `txt[0] == pat[0]`, иначе 0.
   - Для `j` от 1 до `n-1`: `dp[0][j] = dp[0][j-1] + (1 if txt[j] == pat[0] else 0)`.
3. Для `i` от 1 до `m-1`:
   - Для `j` от 1 до `n-1`:
     - `dp[i][j] = dp[i][j-1]` (не использовать `txt[j]`).
     - Если `txt[j] == pat[i]`, то добавить `dp[i-1][j-1]` (использовать `txt[j]`).
4. Вернуть `dp[m-1][n-1]`.

---

## Реализация

```python
class Solution:
    def subseqCount(self, txt, pat):
        m, n = len(pat), len(txt)
        dp = [[0] * n for _ in range(m)]
        
        # Первая строка: подпоследовательности из pat[0]
        dp[0][0] = 1 if txt[0] == pat[0] else 0
        for j in range(1, n):
            dp[0][j] = dp[0][j-1]
            if txt[j] == pat[0]:
                dp[0][j] += 1
        
        # Остальные строки
        for i in range(1, m):
            for j in range(1, n):
                dp[i][j] = dp[i][j-1]   # не использовать txt[j]
                if txt[j] == pat[i]:
                    dp[i][j] += dp[i-1][j-1]
        
        return dp[-1][-1]
```

## Сложность

- **Время:** O(m·n), где m = len(pat), n = len(txt). (Поскольку m ≤ n, а в условии `1 ≤ txt.size() ≤ pat.size()`? В условии указано `1 ≤ txt.size() ≤ pat.size() ≤ 103`, значит, m может быть больше n? Нет, `txt.size() ≤ pat.size()`, т.е. n ≥ m. В любом случае O(m·n).)
- **Память:** O(m·n) для таблицы. Можно оптимизировать до O(n), так как для вычисления текущей строки нужна только предыдущая.

---

## Примечания

- Порядок символов важен, поэтому это подпоследовательности, а не подстроки.
- При равенстве символов мы учитываем как возможность использовать текущий символ, так и пропустить его.
- Гарантируется, что результат помещается в 32‑битное целое, поэтому переполнения не будет.
- Если `pat` длиннее `txt`, ответ всегда 0, но в условии `txt.size() ≤ pat.size()`, т.е. это не может быть? На самом деле, если `pat` длиннее `txt`, то `dp` просто даст 0.
- Альтернативное решение: можно сжать пространство до одного измерения, обновляя массив справа налево.
