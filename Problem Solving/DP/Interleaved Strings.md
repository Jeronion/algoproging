## Определение

Даны три строки: `s1`, `s2` и `s3`. Требуется определить, является ли `s3` результатом **перемежения** (interleaving) строк `s1` и `s2`. Перемежение означает, что мы можем получить `s3`, сохраняя относительный порядок символов из `s1` и `s2` (каждая строка остаётся подпоследовательностью `s3`), при этом все символы `s1` и `s2` используются ровно один раз. Другими словами, `s3` можно разбить на две подпоследовательности, которые соответствуют `s1` и `s2`.

**Пример:**
```
s1 = "AAB", s2 = "AAC", s3 = "AAAABC" → true
s1 = "AB", s2 = "C", s3 = "ACB" → true
s1 = "YX", s2 = "X", s3 = "XXY" → false
```

---

## Постановка задачи

**Входные данные:**
- `s1` — строка 1 (длина n).
- `s2` — строка 2 (длина m).
- `s3` — строка 3 (длина l).

**Выход:** `true`, если `s3` является перемежением `s1` и `s2`, иначе `false`.

**Ограничения:** 1 ≤ n, m ≤ 300, 1 ≤ l ≤ 600, все строки состоят из заглавных латинских букв (в условии не указано, но обычно так).

---

## Идея решения (динамическое программирование)

Пусть `dp[i][j]` — может ли префикс `s3` длины `i+j` быть сформирован перемежением префикса `s1` длины `i` и префикса `s2` длины `j`. Тогда:

- Если оба префикса пусты (`i=0, j=0`), то `dp[0][0] = true`.
- Если `i > 0`, то мы можем взять последний символ `s1` (индекс `i-1`) и проверить, что он равен `s3[i+j-1]` и `dp[i-1][j]` истинно.
- Если `j > 0`, то аналогично для `s2`.

Таким образом, рекуррентное соотношение:
```
dp[i][j] = (s3[i+j-1] == s1[i-1] и dp[i-1][j]) или (s3[i+j-1] == s2[j-1] и dp[i][j-1])
```

Базовые случаи:
- Заполняем первую строку `dp[0][j]`: проверяем, что первые `j` символов `s3` равны префиксу `s2`.
- Первый столбец `dp[i][0]`: проверяем, что первые `i` символов `s3` равны префиксу `s1`.

После заполнения таблицы ответ — `dp[n][m]`.

## Реализация

```python
class Solution:
    def isInterleave(self, s1: str, s2: str, s3: str) -> bool:
        n, m = len(s1), len(s2)
        if n + m != len(s3):
            return False

        # dp[i][j] = можно ли получить s3[:i+j] из s1[:i] и s2[:j]
        dp = [[False] * (m + 1) for _ in range(n + 1)]
        dp[0][0] = True

        # Заполняем первую строку (i = 0)
        for j in range(1, m + 1):
            dp[0][j] = dp[0][j - 1] and s3[j - 1] == s2[j - 1]

        # Заполняем первый столбец (j = 0)
        for i in range(1, n + 1):
            dp[i][0] = dp[i - 1][0] and s3[i - 1] == s1[i - 1]

        # Основной цикл
        for i in range(1, n + 1):
            for j in range(1, m + 1):
                # пробуем взять символ из s1
                if s3[i + j - 1] == s1[i - 1]:
                    dp[i][j] = dp[i][j] or dp[i - 1][j]
                # пробуем взять символ из s2
                if s3[i + j - 1] == s2[j - 1]:
                    dp[i][j] = dp[i][j] or dp[i][j - 1]

        return dp[n][m]
```

---

## Сложность

- **Время:** O(n·m) — заполнение таблицы размером (n+1)×(m+1).
- **Память:** O(n·m) — хранение двумерного массива.

Можно оптимизировать до O(m), используя только две строки, но для ограничений 300×300 это не критично.

---

## Примечания

- Проверка длины `s3` обязательна: если `len(s3) != n + m`, сразу возвращаем `False`.
- В коде используется `dp[i][j] or ...`, поскольку может быть два способа прийти в данную клетку.
- Альтернативно можно использовать одномерный массив для экономии памяти.
