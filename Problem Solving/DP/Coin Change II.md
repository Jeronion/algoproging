## Определение

**Задача о размене монет (Coin Change)** в варианте с минимальным количеством монет является классической задачей динамического программирования. Даны номиналы монет (массив `coins[]`) и целевая сумма `sum`. Требуется найти **минимальное количество монет**, необходимое для получения данной суммы, используя неограниченное количество монет каждого номинала. Если это невозможно, вернуть `-1`.

В отличие от задачи о количестве способов размена, здесь мы минимизируем число монет.

**Примеры:**
- `coins = [25, 10, 5], sum = 30` → 2 (25 + 5)
- `coins = [9, 6, 5, 1], sum = 19` → 3 (9 + 9 + 1)
- `coins = [4, 6, 2], sum = 5` → -1 (невозможно)

---

## Постановка задачи

**Входные данные:**
- `coins[]` — массив целых чисел (номиналы монет, все положительные).
- `sum` — целевая сумма (целое неотрицательное число).

**Выход:** минимальное количество монет, или `-1`, если сумма не может быть составлена.

---

## Ключевые свойства

### 1. Оптимальная подструктура

Для каждого номинала монеты мы можем либо взять её (уменьшив сумму на `coins[i]` и добавив 1 к счётчику), либо не брать (перейти к следующему номиналу). Таким образом, решение для суммы `sum` и набора монет с индекса `i` выражается через решения подзадач с меньшей суммой или с меньшим набором монет.

### 2. Перекрывающиеся подзадачи

При рекурсивном переборе одни и те же состояния `(i, sum)` возникают многократно. Это позволяет применить динамическое программирование.

---

## Основные подходы к решению

### 1. Наивная рекурсия (экспоненциальная)

Без кэширования сложность экспоненциальна, так как перебираются все комбинации монет. Неэффективна для больших `sum`.

### 2. Рекурсия с мемоизацией (Top‑Down DP)

Используется двумерный массив `memo[i][sum]`, где `i` — индекс текущего номинала (от 0 до n-1), `sum` — оставшаяся сумма. Базовые случаи:
- `sum == 0` → 0 монет.
- `i == n` или `sum < 0` → бесконечность (невозможно).

Рекуррентное соотношение:
```
if sum - coins[i] >= 0:
    take = 1 + minCoins(i, sum - coins[i])
else:
    take = ∞
noTake = minCoins(i+1, sum)
memo[i][sum] = min(take, noTake)
```

**Сложность:** O(n·sum) по времени, O(n·sum) по памяти.

### 3. Табличный метод (Bottom‑Up DP)

Заполняется таблица `dp[i][j]` — минимальное количество монет для первых `i` номиналов (0..i-1) и суммы `j`. Итерация по i и j.

Формула:
```
if j >= coins[i-1]:
    dp[i][j] = min(dp[i-1][j], 1 + dp[i][j - coins[i-1]])
else:
    dp[i][j] = dp[i-1][j]
```

Здесь важно, что при взятии монеты мы остаёмся на том же `i` (неограниченное использование). Ответ: `dp[n][sum]`.

**Сложность:** O(n·sum) времени, O(n·sum) памяти.

### 4. Пространственно-оптимизированный подход (одномерный массив)

Так как для вычисления `dp[i][j]` нужны значения из текущей строки (для того же i) и из предыдущей, можно использовать один массив `dp` длиной `sum+1` и обновлять его слева направо (в отличие от 0/1 рюкзака, где обновление справа налево).

**Алгоритм:**
```
dp = [inf] * (sum + 1)
dp[0] = 0
for each coin in coins:
    for j from coin to sum:
        dp[j] = min(dp[j], 1 + dp[j - coin])
return dp[sum] if dp[sum] != inf else -1
```

Это работает, потому что мы можем использовать монету несколько раз (бесконечный запас).

**Сложность:** O(n·sum) времени, O(sum) памяти.

## Реализации на Python

### 1. Рекурсия с мемоизацией

```python
def minCoinsRecur(i, sum, coins, memo):
    if sum == 0:
        return 0
    if i == len(coins) or sum < 0:
        return float('inf')
    if memo[i][sum] != -1:
        return memo[i][sum]

    take = float('inf')
    if coins[i] > 0 and sum - coins[i] >= 0:
        take = minCoinsRecur(i, sum - coins[i], coins, memo)
        if take != float('inf'):
            take += 1

    noTake = minCoinsRecur(i + 1, sum, coins, memo)
    memo[i][sum] = min(take, noTake)
    return memo[i][sum]

def minCoins(coins, sum):
    memo = [[-1] * (sum + 1) for _ in range(len(coins))]
    ans = minCoinsRecur(0, sum, coins, memo)
    return -1 if ans == float('inf') else ans
```

### 2. Табличный Bottom‑Up (двумерный)

```python
def minCoins(coins, sum):
    n = len(coins)
    dp = [[float('inf')] * (sum + 1) for _ in range(n + 1)]
    for i in range(n + 1):
        dp[i][0] = 0

    for i in range(1, n + 1):
        for j in range(1, sum + 1):
            if j >= coins[i-1]:
                dp[i][j] = min(dp[i-1][j], 1 + dp[i][j - coins[i-1]])
            else:
                dp[i][j] = dp[i-1][j]
    return dp[n][sum] if dp[n][sum] != float('inf') else -1
```

### 3. Оптимизированный по памяти (одномерный)

```python
def minCoins(coins, sum):
    dp = [float('inf')] * (sum + 1)
    dp[0] = 0
    for coin in coins:
        for j in range(coin, sum + 1):
            if dp[j - coin] != float('inf'):
                dp[j] = min(dp[j], 1 + dp[j - coin])
    return dp[sum] if dp[sum] != float('inf') else -1
```

## Сравнение подходов

| Подход               | Время      | Память    | Примечание                      |
|----------------------|------------|-----------|---------------------------------|
| Наивная рекурсия     | Экспонента | O(sum)    | Неэффективна                    |
| Мемоизация           | O(n·sum)   | O(n·sum)  | Проста в реализации             |
| Табуляция (2D)       | O(n·sum)   | O(n·sum)  | Классический DP                 |
| Оптимизированный (1D)| O(n·sum)   | O(sum)    | **Рекомендуется** для практики  |

## Связь с другими задачами

- **Coin Change (количество способов)** — аналогичная задача, но считается количество комбинаций, а не минимум монет.
- **Задача о рюкзаке (неограниченный)** — похожая структура, но там максимизируется ценность при ограничении на вес.
- **0/1 рюкзак** — отличается тем, что каждый предмет можно взять только один раз (обновление массива справа налево).
- **Минимальное количество монет с ограниченным запасом** — модификация, где каждая монета доступна в ограниченном количестве.

## Применение

- Выдача сдачи в кассовых аппаратах.
- Оптимизация размена валют.
- Задачи планирования с минимальными затратами ресурсов.
- Комбинаторная оптимизация в играх и головоломках.

## Заключение

Задача о минимальном количестве монет является отличным примером применения динамического программирования для задач с неограниченным использованием элементов. Она демонстрирует важность выбора правильного порядка обновления массива (слева направо для неограниченного доступа) и показывает разницу между 0/1 и неограниченным рюкзаком. Понимание этой задачи помогает освоить базовые паттерны DP и подготовиться к более сложным комбинаторным задачам.
