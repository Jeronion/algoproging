## Определение

**Наибольшая цепочка строк (Longest String Chain)** — это задача, в которой дан массив слов `words`. Требуется найти максимальную длину цепочки, где каждое следующее слово может быть получено из предыдущего удалением **ровно одного** символа (порядок оставшихся символов сохраняется). Иными словами, слово `A` является предшественником слова `B`, если `A` можно получить из `B` удалением одного символа. Цепочка может начинаться с любого слова, и каждое следующее слово должно быть на один символ длиннее предыдущего. Порядок слов в цепочке соответствует увеличению длины.

**Пример:**
```
words = ["a", "b", "ba", "bca", "bda", "bdca"]
Наибольшая цепочка: "a" → "ba" → "bda" → "bdca" (длина 4)
```

---

## Постановка задачи

**Вход:** массив `words` (строки, состоящие из строчных букв).  
**Выход:** длина самой длинной цепочки, где каждое следующее слово получается удалением одного символа из предыдущего.

**Ограничения:** n — количество слов, m — максимальная длина слова. Обычно n ≤ 10³–10⁴, m ≤ 16–20.

---

## Идея решения (DP + хеширование)

Эта задача решается с помощью **динамического программирования** с использованием хеш-таблицы. Основная идея:

1. Отсортировать слова по длине (по возрастанию). Это гарантирует, что при обработке слова все его возможные предшественники (более короткие слова) уже будут обработаны.
2. Использовать словарь `dp`, где ключ — слово, значение — длина наибольшей цепочки, заканчивающейся на этом слове.
3. Для каждого слова `w`:
   - Инициализировать `dp[w] = 1` (цепочка из одного слова).
   - Для каждой позиции `i` в слове `w` (от 0 до len(w)-1) создать слово-предшественник `pred = w[:i] + w[i+1:]` (удаление одного символа).
   - Если `pred` существует в `dp`, то мы можем продлить цепочку: `dp[w] = max(dp[w], dp[pred] + 1)`.
4. Отслеживать глобальный максимум `res` и вернуть его.

**Почему это работает?**  
Сортировка по длине гарантирует, что все возможные предшественники (короче на 1 символ) уже обработаны и их `dp` значения известны. Таким образом, мы строим цепочки снизу вверх.

---

## Алгоритм

1. Отсортировать `words` по длине (ключ `len`).
2. Создать словарь `dp = {}`.
3. Инициализировать `res = 0`.
4. Для каждого `w` в отсортированном списке:
   - `dp[w] = 1`
   - Для `i` от 0 до `len(w)-1`:
     - `pred = w[:i] + w[i+1:]`
     - Если `pred` в `dp`:
       - `dp[w] = max(dp[w], dp[pred] + 1)`
   - `res = max(res, dp[w])`
5. Вернуть `res`.

---

## Реализация на Python

```python
def longestStringChain(words):
    # Сортируем по длине
    words.sort(key=len)
    dp = {}
    res = 1

    for w in words:
        dp[w] = 1
        # Генерируем все возможные предшественники
        for i in range(len(w)):
            pred = w[:i] + w[i+1:]
            if pred in dp:
                dp[w] = max(dp[w], dp[pred] + 1)
        res = max(res, dp[w])

    return res

# Пример
words = ["a", "b", "ba", "bca", "bda", "bdca"]
print(longestStringChain(words))  # 4
```

## Сложность

- **Время:** O(n · log n + n · m²), где n — количество слов, m — максимальная длина слова. Сортировка занимает O(n log n), а для каждого слова мы генерируем m предшественников, каждый из которых создаёт строку длины m (операция O(m)). В сумме O(n · m²).
- **Память:** O(n · m) для хранения слов в словаре.

## Примечания

- Задача также известна как "Longest String Chain" на LeetCode (№ 1048).
- Важно, что удаление символа происходит **ровно один раз** и порядок символов сохраняется.
- Если слова могут содержать одинаковые длины, сортировка по длине не определяет порядок внутри группы, но это не влияет на результат, так как предшественник всегда короче.
- Для ускорения можно использовать префиксные деревья (trie), но хеш-таблица с удалением символов достаточно эффективна при малых m.
