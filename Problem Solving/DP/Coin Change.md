## Определение

**Coin Change (Count Ways)** — это классическая задача динамического программирования, в которой даны массив монет различных номиналов `coins[]` и целевая сумма `sum`. Требуется подсчитать количество способов, которыми можно представить `sum` с использованием неограниченного количества монет каждого номинала. Разные порядки одних и тех же монет считаются одним способом (комбинации, а не перестановки).

**Пример:**
```
coins = [1, 2, 3], sum = 4
Способы: [1,1,1,1], [1,1,2], [2,2], [1,3] → 4 способа.
```

---

## Постановка задачи

**Вход:**
- `coins` — массив целых чисел (номиналы монет, положительные).
- `sum` — целевая сумма (целое неотрицательное).

**Выход:** количество способов (целое число), или `0`, если ни одного способа нет.

**Ограничения:** 1 ≤ sum ≤ 10³, 1 ≤ coins[i] ≤ 10⁴, 1 ≤ coins.size() ≤ 10³. Ответ гарантированно помещается в 32-битное целое.

---

## Идея решения (динамическое программирование снизу вверх)

Используем **одномерный массив `dp`**, где `dp[i]` — количество способов получить сумму `i`. Инициализируем `dp[0] = 1` (пустой набор монет). Затем для каждой монеты `c` обновляем массив `dp` слева направо:

```
for c in coins:
    for i in range(c, sum+1):
        dp[i] += dp[i-c]
```

Это соответствует использованию каждой монеты неограниченное количество раз, причём порядок монет не важен (комбинации). Внешний цикл по монетам гарантирует, что каждая комбинация учитывается ровно один раз.

### Почему это работает?

- `dp[i]` накапливает количество способов с использованием уже обработанных монет.
- При добавлении новой монеты `c`, мы можем добавить её к любой сумме `i-c`, которая уже была сформирована. Это даёт все способы, где `c` используется хотя бы один раз.
- Так как мы идём по возрастанию `i`, мы позволяем использовать монету `c` несколько раз (неограниченно).

---

## Алгоритм (пошагово)

1. Создать массив `dp` размера `sum+1`, заполнить нулями.
2. Установить `dp[0] = 1` (один способ получить сумму 0 — не брать монет).
3. Для каждой монеты `c` в `coins`:
   - Для `i` от `c` до `sum` (включительно):
     - `dp[i] += dp[i-c]`.
4. Вернуть `dp[sum]`.

---

## Реализация на Python

```python
class Solution:
    def count(self, coins, sum):
        dp = [0] * (sum + 1)
        dp[0] = 1
        for c in coins:
            for i in range(c, sum + 1):
                dp[i] += dp[i - c]
        return dp[sum]
```

---

## Сложность

- **Время:** O(n·sum), где n — количество типов монет, sum — целевая сумма.
- **Память:** O(sum) для массива `dp`.

---

## Примечания

- Если `sum == 0`, ответ всегда `1` (пустой набор).
- Если ни одна монета не может использоваться для получения `sum`, ответ `0`.
- Задача является частным случаем задачи о рюкзаке с неограниченным количеством предметов.
- Порядок монет в `coins` не влияет на результат (внешний цикл всё равно пройдёт по всем).
- Если бы требовались перестановки (порядок важен), мы бы использовали другой подход: `dp[i] = sum(dp[i-c] for c in coins)` с внешним циклом по `i`.
