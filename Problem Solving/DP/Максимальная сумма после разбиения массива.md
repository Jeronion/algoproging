## Определение

**Задача о максимальной сумме после разбиения массива** (Maximum Sum After Partitioning) — это задача, в которой дан массив целых чисел `arr` и целое число `k`. Требуется разбить массив на непрерывные подмассивы (сегменты), каждый длиной не более `k`. В каждом сегменте все элементы заменяются на максимальный элемент этого сегмента. Затем все элементы суммируются. Нужно максимизировать итоговую сумму, выбрав оптимальное разбиение.

Эта задача является классическим примером использования динамического программирования для оптимизации разбиения массива с ограничением на длину сегмента.

---

## Постановка задачи

**Входные данные:**
- `arr` — массив целых чисел (длина n).
- `k` — максимальная длина подмассива.

**Выход:** максимально возможная сумма после применения операции.

**Ограничения:** типичные: 1 ≤ n ≤ 500, 1 ≤ k ≤ n, 0 ≤ arr[i] ≤ 10⁴ (в LeetCode 1043).

---

## Идея решения (динамическое программирование)

Пусть `dp[i]` — максимальная сумма, которую можно получить для первых `i` элементов массива (т.е. для префикса `arr[0..i-1]`). Тогда мы можем рассмотреть последний сегмент, заканчивающийся в позиции `i-1`, который имеет длину `len` (1 ≤ len ≤ k). В этом сегменте все элементы будут заменены на максимальный элемент внутри него. Этот максимум мы можем вычислить, расширяя сегмент влево.

Для каждого возможного `len` от 1 до `k` (и `i - len >= 0`):
- Найти `max_val` в сегменте `arr[i-len .. i-1]` (при расширении мы можем поддерживать текущий максимум).
- Стоимость этого сегмента: `max_val * len`.
- Общая сумма для такого разбиения: `dp[i - len] + max_val * len`.
- `dp[i]` — максимум из всех таких значений.

Базовый случай: `dp[0] = 0` (для пустого массива).

Ответ: `dp[n]`.

### Почему это работает?

Мы рассматриваем все возможные позиции начала последнего сегмента. Так как сегмент не может быть длиннее `k`, мы перебираем только `k` вариантов. При этом для каждого варианта мы знаем оптимальную сумму для оставшейся части (`dp[i - len]`). Это даёт оптимальную подструктуру и перекрывающиеся подзадачи, что позволяет использовать DP.

---

## Алгоритм (пошагово)

1. Создать массив `dp` размера `n+1`, заполненный нулями.
2. Для `i` от 1 до `n`:
   - Инициализировать `cur_max = 0`.
   - Для `len` от 1 до `min(k, i)` (текущая длина сегмента, заканчивающегося в `i-1`):
     - Обновить `cur_max = max(cur_max, arr[i - len])` (так как мы расширяем сегмент влево, новый элемент может быть больше).
     - Вычислить `dp[i] = max(dp[i], dp[i - len] + cur_max * len)`.
3. Вернуть `dp[n]`.

---

## Реализация на Python

```python
def max_sum_after_partitioning(arr, k):
    n = len(arr)
    dp = [0] * (n + 1)  # dp[i] — макс. сумма для первых i элементов

    for i in range(1, n + 1):
        cur_max = 0
        for length in range(1, min(k, i) + 1):
            cur_max = max(cur_max, arr[i - length])
            dp[i] = max(dp[i], dp[i - length] + cur_max * length)

    return dp[n]
```

---

## Сложность

- **Время:** O(n·k) — для каждого индекса мы перебираем не более k длин.
- **Память:** O(n) — массив `dp`.

## Примечания

- Задача известна на LeetCode как [1043. Partition Array for Maximum Sum](https://leetcode.com/problems/partition-array-for-maximum-sum/).
- Алгоритм работает для любых целых чисел (не только положительных), но в условии обычно числа неотрицательные.
- Если `k` велико (например, равно n), то мы просто находим максимум во всём массиве и умножаем на n (однако при `k = n` мы можем разбить на один сегмент, но это даст не обязательно максимум, потому что можно разбить на несколько сегментов, чтобы увеличить сумму? На самом деле, если `k = n`, то мы можем взять весь массив как один сегмент, но также можно разбить на несколько, и ответ будет максимумом по всем разбиениям, что делает алгоритм всё ещё корректным).
- DP массив можно сократить до O(k) с использованием скользящего окна, но для n до 500 это не обязательно.
