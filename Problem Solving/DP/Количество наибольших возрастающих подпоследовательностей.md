## Определение

**Наибольшая возрастающая подпоследовательность (LIS)** — это подпоследовательность максимальной длины, в которой элементы строго возрастают. Задача может требовать не только найти длину LIS, но и подсчитать количество таких подпоследовательностей. Важно, что LIS могут различаться индексами элементов, даже если значения одинаковы. Например, для массива `[10, 10, 10, 10]` все четыре элемента по отдельности образуют LIS длины 1, поэтому количество LIS равно 4.

**Пример:**
```
arr = [1, 3, 5, 4, 7]
LIS длины 4: [1,3,5,7] и [1,3,4,7] → количество = 2.
```

---

## Постановка задачи

**Вход:** массив `arr` длины n (целые числа).  
**Выход:** количество различных наибольших возрастающих подпоследовательностей (LIS). Различными считаются подпоследовательности, отличающиеся хотя бы одним индексом.

---

## Идея решения (Tabulation DP)

Используем два одномерных массива:
- `lis[i]` — длина LIS, заканчивающейся на индексе `i`.
- `cnt[i]` — количество LIS длины `lis[i]`, заканчивающихся на индексе `i`.

Инициализация: `lis[i] = 1`, `cnt[i] = 1` для всех `i` (каждый элемент сам по себе является LIS длины 1).

Для каждой пары индексов `(i, j)`, где `j < i` и `arr[j] < arr[i]` (строгое возрастание), проверяем:
- Если `lis[j] + 1 > lis[i]`, то мы нашли новую, более длинную последовательность. Обновляем `lis[i] = lis[j] + 1` и `cnt[i] = cnt[j]`.
- Если `lis[j] + 1 == lis[i]`, то мы нашли ещё одну последовательность такой же длины. Добавляем `cnt[j]` к `cnt[i]`.

После заполнения массивов `lis` и `cnt`:
- Находим `max_len = max(lis)`.
- Суммируем все `cnt[i]`, для которых `lis[i] == max_len`. Это и будет общее количество LIS.

Этот подход является прямым расширением классического DP для LIS (O(n²)) и позволяет одновременно подсчитывать количество последовательностей.

---

## Алгоритм (пошагово)

1. Инициализировать массивы `lis` и `cnt` длиной `n` единицами.
2. Для `i` от 1 до `n-1`:
   - Для `j` от 0 до `i-1`:
     - Если `arr[j] < arr[i]`:
       - Если `lis[j] + 1 > lis[i]`:
         - `lis[i] = lis[j] + 1`
         - `cnt[i] = cnt[j]`
       - Иначе если `lis[j] + 1 == lis[i]`:
         - `cnt[i] += cnt[j]`
3. Найти `max_len = max(lis)`.
4. Вернуть `sum(cnt[i] for i in range(n) if lis[i] == max_len)`.

---

## Реализация на Python

```python
def number_of_LIS(arr):
    n = len(arr)
    lis = [1] * n
    cnt = [1] * n

    for i in range(1, n):
        for j in range(i):
            if arr[j] < arr[i]:
                if lis[j] + 1 > lis[i]:
                    lis[i] = lis[j] + 1
                    cnt[i] = cnt[j]
                elif lis[j] + 1 == lis[i]:
                    cnt[i] += cnt[j]

    max_len = max(lis)
    return sum(cnt[i] for i in range(n) if lis[i] == max_len)

# Пример
arr = [1, 3, 5, 4, 7]
print(number_of_LIS(arr))  # 2
```

---

## Сложность

- **Время:** O(n²) — двойной цикл по индексам.
- **Память:** O(n) — два массива длины n.

---

## Пример работы

Для `arr = [1, 3, 5, 4, 7]`:

| i | arr[i] | lis[i] | cnt[i] |
|---|--------|--------|--------|
| 0 | 1      | 1      | 1      |
| 1 | 3      | 2 (1→3) | 1      |
| 2 | 5      | 3 (1→3→5) | 1      |
| 3 | 4      | 3 (1→3→4) | 1      |
| 4 | 7      | 4 (1→3→5→7 или 1→3→4→7) | 2      |

`max_len = 4`, `cnt[4] = 2` → ответ 2.

---

## Примечания

- Если массив пуст, ответ 0.
- Если все элементы равны, то `lis[i] = 1` для всех, `cnt[i] = 1`. `max_len = 1`, сумма `cnt` = n, что корректно.
- Алгоритм подсчитывает подпоследовательности по индексам, а не по значениям. Если значения повторяются, разные индексы считаются разными подпоследовательностями.
- Для больших n (n > 10⁴) O(n²) может быть слишком медленным. Существует решение за O(n log n) с использованием дерева Фенвика или сегментного дерева, которое также позволяет подсчитывать количество LIS. Однако O(n²) проще для понимания и часто достаточно для средних ограничений.
