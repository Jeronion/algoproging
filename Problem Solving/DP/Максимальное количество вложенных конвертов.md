## Определение

**Задача о вложении конвертов** — это классическая задача на поиск наибольшей цепочки вложенных прямоугольников. Даны `N` конвертов, каждый задан парой `(W, H)` (ширина и высота). Конверт `A` можно вложить в конверт `B`, если `W_A < W_B` и `H_A < H_B`. Требуется найти максимальное количество конвертов, которые можно вложить друг в друга (каждый следующий строго больше предыдущего). Поворачивать конверты нельзя.

**Пример:**
```
envelopes = [(4,3), (5,3), (5,6), (1,2)]
Оптимальная цепочка: (1,2) → (4,3) → (5,6) → длина 3.
```

---

## Постановка задачи

**Вход:** массив пар `(w, h)` — размеры конвертов.  
**Выход:** максимальное количество конвертов, которые можно вложить друг в друга (строгое возрастание по обоим измерениям).

---

## Идея решения

Эта задача сводится к поиску **наибольшей возрастающей подпоследовательности (LIS)** среди высот после специальной сортировки.

### Ключевое наблюдение

Если отсортировать конверты по ширине **по возрастанию**, а при равной ширине — по высоте **по убыванию**, то задача сведения к LIS на массиве высот. Почему?

- Сортировка по ширине обеспечивает, что при рассмотрении конвертов в порядке сортировки ширина у нас уже не убывает.
- Если ширины равны, мы сортируем высоты по убыванию. Это предотвращает случай, когда два конверта с одинаковой шириной могут быть "вложены" друг в друга (поскольку ширина должна быть строго меньше, а равная ширина не подходит). Убывание высот гарантирует, что в последовательности высот мы не сможем взять два конверта с одинаковой шириной, так как для LIS нужна строго возрастающая высота, а убывание делает равные высоты идущими в неправильном порядке.

После такой сортировки задача превращается в поиск **длины наибольшей строго возрастающей подпоследовательности** в массиве высот. Это классическая задача, решаемая за O(n log n) с использованием бинарного поиска.

---

## Алгоритм (эффективный подход)

1. **Сортировка:**
   - Отсортировать конверты по ширине **по возрастанию**.
   - Если ширины равны, отсортировать по высоте **по убыванию**.
   - Например, `(5,3)` и `(5,6)` при сортировке станут `(5,6)` перед `(5,3)`.

2. **Поиск LIS среди высот:**
   - Создать массив `dp`, который будет хранить минимальные возможные "хвосты" для возрастающих подпоследовательностей.
   - Для каждой высоты `h` из отсортированного списка (в порядке сортировки):
     - Найти позицию `pos` первого элемента в `dp`, который **≥** `h` (нижняя граница, `bisect_left`).
     - Если `pos == len(dp)`, добавить `h` в конец `dp`.
     - Иначе заменить `dp[pos]` на `h`.
   - Длина `dp` в конце — это максимальное количество вложенных конвертов.

**Почему это работает?**  
Сортировка гарантирует, что ширина не убывает, а при равной ширине высота убывает, чтобы не допустить ложных включений. Затем LIS на высотах даёт максимальную цепочку, где каждая следующая высота строго больше предыдущей, что вместе с условием на ширину (которая также строго возрастает благодаря сортировке) даёт корректную цепочку вложений.

## Реализация на Python

```python
import bisect

def max_envelopes(envelopes):
    # Сортируем: ширина по возрастанию, высота по убыванию при равной ширине
    envelopes.sort(key=lambda x: (x[0], -x[1]))
    
    # Извлекаем только высоты
    heights = [h for _, h in envelopes]
    
    # Поиск LIS на массиве высот (строго возрастающий)
    dp = []
    for h in heights:
        pos = bisect.bisect_left(dp, h)  # lower_bound для строгого возрастания
        if pos == len(dp):
            dp.append(h)
        else:
            dp[pos] = h
    
    return len(dp)

# Пример
envelopes = [(4,3), (5,3), (5,6), (1,2)]
print(max_envelopes(envelopes))  # 3
```

---

## Сложность

- **Время:** O(N log N) — сортировка O(N log N) + проход по всем элементам с бинарным поиском O(N log N).
- **Память:** O(N) для хранения высот и массива `dp`.

---

## Примечания

- Важно, что условие вложения требует **строгого** неравенства по обеим размерностям. Сортировка с убыванием высот при равной ширине предотвращает случаи, когда два конверта с одинаковой шириной могут быть ошибочно включены в цепочку.
- Если конверты можно поворачивать, задача усложняется (нужно рассматривать оба варианта ориентации), но в классической постановке поворот запрещён.
- Данный подход является стандартным решением задачи "Russian Doll Envelopes" (LeetCode 354).
