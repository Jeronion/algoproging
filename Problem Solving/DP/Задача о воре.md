## Определение

**Stickler Thief** — это классическая задача динамического программирования. Дан массив `arr[]`, где `arr[i]` представляет сумму денег в i-м доме, расположенном в линию. Вор не может грабить два соседних дома (так как сработает сигнализация). Требуется найти максимальную сумму, которую он может украсть, не нарушая это ограничение.

**Пример:**
```
arr = [6, 7, 1, 3, 8, 2, 4]
Оптимальный выбор: 1-й, 3-й, 5-й и 7-й дома → 6 + 1 + 8 + 4 = 19.
```

---

## Постановка задачи

**Вход:** массив целых чисел `arr` размера `n` (1 ≤ n ≤ 10⁵, 1 ≤ arr[i] ≤ 10⁴).
**Выход:** максимальная сумма, которую можно получить, выбирая непоследовательные элементы.

---

## Идея решения (динамическое программирование)

Пусть `dp[i]` — максимальная сумма, которую можно получить, рассматривая первые `i+1` домов (0-индексация). Для каждого дома `i` есть два варианта:

1. Не грабить дом `i` — тогда ответ равен `dp[i-1]`.
2. Грабить дом `i` — тогда нужно добавить `arr[i]` к оптимальному решению для домов до `i-2` (чтобы не взять соседний), т.е. `dp[i-2] + arr[i]`.

Таким образом, рекуррентное соотношение:
```
dp[i] = max(dp[i-1], dp[i-2] + arr[i])
```

Базовые случаи:
- `dp[0] = arr[0]`
- `dp[1] = max(arr[0], arr[1])`

Ответ: `dp[n-1]`.

Для экономии памяти можно хранить только два последних значения, но в приведённом коде используется модификация на месте (in-place), где массив `arr` переиспользуется для хранения `dp`.

### Особенность реализации (in-place)

Код изменяет исходный массив, сохраняя в `arr[i]` значение `dp[i]` для текущего `i`. Для `i=0` и `i=1` обрабатываются отдельно. Для `i=2` используется `arr[2] += arr[0]` (что эквивалентно `arr[2] + dp[0]`, так как `dp[1]` может быть больше, но для `i=2` правильнее было бы `max(arr[2] + dp[0], dp[1])`. Однако код использует `arr[2] += arr[0]`, а затем в конце возвращает `max(arr[-1], arr[-2])`. Это работает потому, что для `i=2` `dp[2] = max(arr[2] + dp[0], dp[1])`. В коде после `arr[2] += arr[0]` получается `arr[2] = arr[2] + dp[0]`, но не учитывается вариант `dp[1]`. Однако в конце берётся максимум между последними двумя элементами, что компенсирует это. Давайте проверим: для `i=2` после операции `arr[2] += arr[0]`, `arr[2]` становится `arr[2] + arr[0]`. Но `dp[1]` хранится в `arr[1]`. В конце ответ `max(arr[-1], arr[-2])` — это максимум из последнего и предпоследнего, что эквивалентно `max(dp[n-1], dp[n-2])`. Таким образом, для `i=2` мы не обновляем `dp[2]` до максимума, но в конце максимум из двух последних даст правильный результат, потому что `dp[2]` не нужно хранить для дальнейших вычислений? Дальше для `i=3` используется `max(arr[1], arr[0])`? На самом деле в коде для `i >= 3` используется `arr[i] += max(arr[i-2], arr[i-3])`. Это соответствует рекуррентности `dp[i] = arr[i] + max(dp[i-2], dp[i-3])`. Это другая формулировка: если мы грабим дом `i`, то предыдущий взятый дом может быть либо `i-2`, либо `i-3` (потому что `i-1` нельзя). Это эквивалентно `max(dp[i-2], dp[i-3]) + arr[i]`. Тогда для `i=2` эта формула не подходит, поэтому обрабатывается отдельно. После обработки всех элементов ответ — максимум из двух последних `dp` значений, так как оптимальное решение может заканчиваться либо на последнем, либо на предпоследнем доме. В коде это `max(arr[-1], arr[-2])`. Таким образом, код корректен и использует O(1) дополнительной памяти, изменяя исходный массив.

---

## Алгоритм

1. Если `n == 1`: вернуть `arr[0]`.
2. Если `n == 2`: вернуть `max(arr[0], arr[1])`.
3. `arr[2] += arr[0]` (dp[2] с учётом грабежа дома 2 и дома 0; но на самом деле это не полное dp[2], но оно будет использовано для вычислений).
4. Для `i` от `3` до `n-1`:
   - `arr[i] += max(arr[i-2], arr[i-3])`.
5. Вернуть `max(arr[-1], arr[-2])`.

---

## Реализация на Python

```python
class Solution:
    def findMaxSum(self, arr):
        n = len(arr)
        if n == 1:
            return arr[0]
        if n == 2:
            return max(arr[0], arr[1])
        
        arr[2] += arr[0]
        for i in range(3, n):
            arr[i] += max(arr[i-2], arr[i-3])
        return max(arr[-1], arr[-2])
```

## Сложность

- **Время:** O(n) — один проход по массиву.
- **Память:** O(1) — используется только модифицированный входной массив (без дополнительной памяти).

## Примечания

- Задача также известна как «House Robber» (LeetCode 198), но там обычно используется DP с двумя переменными. В данном решении используется in-place модификация и нестандартная рекуррентность, но она корректна.
- Если нельзя изменять входной массив, можно использовать две переменные `prev2` и `prev1`.
- Альтернативное стандартное решение:
  ```python
  prev2 = 0
  prev1 = arr[0]
  for i in range(1, n):
      curr = max(prev1, prev2 + arr[i])
      prev2, prev1 = prev1, curr
  return prev1
  ```
