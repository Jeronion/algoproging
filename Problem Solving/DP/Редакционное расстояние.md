## Определение

**Редакционное расстояние (расстояние Левенштейна)** — это минимальное количество операций (вставка, удаление, замена), необходимых для преобразования одной строки в другую. Все операции имеют одинаковую стоимость (обычно 1). Это классическая задача динамического программирования, широко применяемая в обработке текста, биоинформатике, проверке орфографии и сравнении строк.

**Пример:**
```
s1 = "geek", s2 = "gesek" → 1 (вставка 's')
s1 = "GEEXSFRGEEKKS", s2 = "GEEKSFORGEEKS" → 3
```

---

## Постановка задачи

Даны две строки `s1` и `s2` длиной `m` и `n` соответственно. Разрешены три операции над `s1`:
- **Вставка** любого символа
- **Удаление** любого символа
- **Замена** любого символа на другой

Все операции имеют стоимость 1. Требуется найти минимальное количество операций для преобразования `s1` в `s2`.

---

## Идея решения (динамическое программирование)

Пусть `dp[i][j]` — минимальное количество операций для преобразования префикса `s1[0..i-1]` в `s2[0..j-1]`.

Рассмотрим последние символы `s1[i-1]` и `s2[j-1]`:

- Если они **совпадают**, то ничего делать не нужно: `dp[i][j] = dp[i-1][j-1]`.
- Если **не совпадают**, то можно применить одну из трёх операций:
  1. **Вставить** символ `s2[j-1]` в `s1`: `dp[i][j-1] + 1`.
  2. **Удалить** символ `s1[i-1]` из `s1`: `dp[i-1][j] + 1`.
  3. **Заменить** `s1[i-1]` на `s2[j-1]`: `dp[i-1][j-1] + 1`.

Выбираем минимум из трёх.

**Базовые случаи:**
- `dp[i][0] = i` — для преобразования `s1[0..i-1]` в пустую строку нужно `i` удалений.
- `dp[0][j] = j` — для преобразования пустой строки в `s2[0..j-1]` нужно `j` вставок.

Ответ: `dp[m][n]`.

---

## Рекуррентное соотношение

```
if s1[i-1] == s2[j-1]:
    dp[i][j] = dp[i-1][j-1]
else:
    dp[i][j] = 1 + min(dp[i][j-1],     # вставка
                       dp[i-1][j],     # удаление
                       dp[i-1][j-1])   # замена
```

## Реализации

### 1. Классическое DP (таблица) — O(m·n) времени, O(m·n) памяти

```python
def edit_distance(s1, s2):
    m, n = len(s1), len(s2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    
    for i in range(m + 1):
        dp[i][0] = i
    for j in range(n + 1):
        dp[0][j] = j
    
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i-1] == s2[j-1]:
                dp[i][j] = dp[i-1][j-1]
            else:
                dp[i][j] = 1 + min(dp[i][j-1], dp[i-1][j], dp[i-1][j-1])
    
    return dp[m][n]
```

### 2. Оптимизация по памяти (два массива) — O(m·n) времени, O(n) памяти

```python
def edit_distance_two_rows(s1, s2):
    m, n = len(s1), len(s2)
    prev = [j for j in range(n + 1)]
    curr = [0] * (n + 1)
    
    for i in range(1, m + 1):
        curr[0] = i
        for j in range(1, n + 1):
            if s1[i-1] == s2[j-1]:
                curr[j] = prev[j-1]
            else:
                curr[j] = 1 + min(curr[j-1], prev[j], prev[j-1])
        prev = curr[:]
    
    return prev[n]
```

### 3. Дальнейшая оптимизация (один массив) — O(m·n) времени, O(n) памяти

```python
def edit_distance_one_array(s1, s2):
    m, n = len(s1), len(s2)
    dp = [j for j in range(n + 1)]
    
    for i in range(1, m + 1):
        prev = dp[0]       # dp[i-1][0]
        dp[0] = i          # dp[i][0]
        for j in range(1, n + 1):
            temp = dp[j]   # dp[i-1][j] (старое значение)
            if s1[i-1] == s2[j-1]:
                dp[j] = prev
            else:
                dp[j] = 1 + min(dp[j-1], dp[j], prev)
            prev = temp
    
    return dp[n]
```

**Пояснение:** `prev` хранит диагональное значение `dp[i-1][j-1]`. `dp[j]` до обновления — `dp[i-1][j]`, после обновления становится `dp[i][j]`. `dp[j-1]` уже обновлён для текущей строки и равен `dp[i][j-1]`.

## Сложность

| Подход | Время | Память |
|--------|-------|--------|
| Таблица | O(m·n) | O(m·n) |
| Два массива | O(m·n) | O(n) |
| Один массив | O(m·n) | O(n) |

## Связь с LCS

Если не использовать операцию замены, то минимальное количество операций (вставок и удалений) для преобразования `s1` в `s2` равно:

```
m + n - 2 * LCS(s1, s2)
```

где LCS — длина наибольшей общей подпоследовательности. Это объясняется тем, что символы, входящие в LCS, можно оставить нетронутыми, а остальные удалить из `s1` и вставить из `s2`.

---

## Примечания

- Расстояние Левенштейна является метрикой (удовлетворяет неравенству треугольника).
- Существуют модификации с разными стоимостями операций.
- Для очень длинных строк (например, > 10⁵) O(m·n) может быть слишком медленным, но на практике обычно используются оптимизации (например, алгоритм Укконена для ограниченного расстояния).
