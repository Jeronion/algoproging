## Определение

**Задача о сумме подмножества** (Subset Sum) — это классическая задача динамического программирования. Дан массив неотрицательных целых чисел `arr` и целевая сумма `sum`. Требуется определить, существует ли подмножество элементов (не обязательно непрерывное) из `arr`, сумма которых в точности равна `sum`.

**Пример:**
```
arr = [3, 34, 4, 12, 5, 2], sum = 9
Ответ: True (подмножество {4, 5}).
```

---

## Постановка задачи

**Вход:** массив `arr` (длина n, все элементы неотрицательны) и целое `sum`.  
**Выход:** `True`, если существует подмножество с суммой `sum`, иначе `False`.

---

## Идея решения

Задача сводится к выбору некоторых элементов так, чтобы их сумма равнялась `target`. Это классический частный случай задачи о рюкзаке 0/1 (каждый элемент либо берётся, либо нет). Решается динамическим программированием: `dp[i][j]` — можно ли получить сумму `j` из первых `i` элементов.

Рекуррентное соотношение:
- Если `j < arr[i-1]` (текущий элемент больше целевой суммы), то `dp[i][j] = dp[i-1][j]` (не берём).
- Иначе `dp[i][j] = dp[i-1][j] OR dp[i-1][j - arr[i-1]]` (не берём или берём).

Базовый случай: `dp[i][0] = True` (пустое подмножество даёт сумму 0).

---

## 1. Табличный подход (Bottom‑Up DP) – O(n·sum) времени, O(n·sum) памяти

Этот подход строит полную таблицу размером `(n+1) × (sum+1)`, где каждая ячейка хранит булево значение достижимости. Он прост для понимания и служит основой для оптимизации.

### Алгоритм

1. Создать матрицу `dp` размера `(n+1) × (sum+1)`, заполненную `False`.
2. Для всех `i` от 0 до n установить `dp[i][0] = True`.
3. Для каждого элемента `i` (1..n) и для каждой суммы `j` (1..sum):
   - Если `j < arr[i-1]`, то `dp[i][j] = dp[i-1][j]` (элемент не помещается).
   - Иначе `dp[i][j] = dp[i-1][j] or dp[i-1][j - arr[i-1]]`.
4. Ответ — `dp[n][sum]`.

### Реализация

```python
def isSubsetSum(arr, sum):
    n = len(arr)
    dp = [[False] * (sum + 1) for _ in range(n + 1)]
    for i in range(n + 1):
        dp[i][0] = True

    for i in range(1, n + 1):
        for j in range(1, sum + 1):
            if j < arr[i-1]:
                dp[i][j] = dp[i-1][j]
            else:
                dp[i][j] = dp[i-1][j] or dp[i-1][j - arr[i-1]]
    return dp[n][sum]
```

## 2. Пространственно-оптимизированный подход – O(n·sum) времени, O(sum) памяти

Заметим, что для вычисления текущей строки `dp[i]` нужна только предыдущая строка `dp[i-1]`. Поэтому можно хранить только две строки (или даже одну, обновляя справа налево). Это снижает память с O(n·sum) до O(sum).

### 2.1. Два одномерных массива (prev и curr)

Храним `prev` — предыдущую строку, и `curr` — текущую. После заполнения `curr` копируем его в `prev`.

### Реализация с двумя массивами

```python
def isSubsetSum(arr, sum):
    n = len(arr)
    prev = [False] * (sum + 1)
    curr = [False] * (sum + 1)
    prev[0] = True

    for i in range(1, n + 1):
        for j in range(sum + 1):
            if j < arr[i-1]:
                curr[j] = prev[j]
            else:
                curr[j] = prev[j] or prev[j - arr[i-1]]
        prev = curr[:]   # копируем текущую строку
    return prev[sum]
```

### 2.2. Один одномерный массив (обновление справа налево)

Можно обойтись одним массивом `dp`, обновляя его в обратном порядке (`j` от `sum` до `arr[i-1]`), чтобы не использовать один элемент несколько раз.

```python
def isSubsetSum(arr, sum):
    dp = [False] * (sum + 1)
    dp[0] = True
    for x in arr:
        for j in range(sum, x - 1, -1):
            if dp[j - x]:
                dp[j] = True
    return dp[sum]
```

Этот вариант наиболее компактен и эффективен по памяти.

---

## Сложность

| Подход                | Время      | Память    |
|-----------------------|------------|-----------|
| Полная таблица        | O(n·sum)   | O(n·sum)  |
| Два массива (prev/curr) | O(n·sum) | O(sum)    |
| Один массив (обратный)  | O(n·sum) | O(sum)    |

---

## Примечания

- Если `sum` очень велика (например, > 10⁶), даже O(sum) памяти может быть проблематично, но в типичных задачах это приемлемо.
- Алгоритм работает только для неотрицательных чисел (в условии задачи обычно так и есть).
- Задача является **NP-полной**, но псевдополиномиальное решение (зависящее от `sum`) эффективно при умеренных значениях `sum`.
