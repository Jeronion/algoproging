## Определение

**Задача об оптимальной стратегии для игры** — это классическая задача динамического программирования, в которой два игрока по очереди выбирают монеты с концов ряда (первую или последнюю). Игроки стремятся максимизировать сумму своих монет. Требуется найти максимальную сумму, которую может гарантированно получить первый игрок, если оба играют оптимально (т.е. противник также выбирает наилучшие ходы для себя).

**Пример:**
```
coins = [5, 3, 7, 10]
Оптимальный выбор для первого игрока: сначала взять 10 (последний), затем 5 (первый) → сумма = 15.
```

---

## Постановка задачи

**Входные данные:** массив `arr` чётной длины, где `arr[i]` — стоимость i-й монеты.  
**Выход:** максимальная сумма, которую может собрать первый игрок, если оба играют оптимально.

**Ограничения:** `n ≤ 10³` (для DP O(n²)), но есть и пространственно-оптимизированные решения.

---

## Почему жадный алгоритм не работает?

Выбор наибольшей монеты на каждом ходу не гарантирует оптимальности. Например:
```
[8, 15, 3, 7]
- Жадный: первый берёт 8, второй 15, первый 7, второй 3 → сумма первого = 15.
- Оптимальный: первый берёт 7, второй 8, первый 15, второй 3 → сумма первого = 22.
```
Поэтому необходим перебор вариантов с учётом будущих ходов противника.

---

## Идея решения (интервальное DP)

Пусть `dp[i][j]` — максимальная сумма, которую может получить **текущий игрок** (тот, чей ход), рассматривая подмассив `arr[i..j]`. Если `i > j` (пустой подмассив), то сумма равна 0.

Если игрок берёт `arr[i]`, то противник будет играть оптимально на оставшемся подмассиве `arr[i+1..j]`. В этом случае игрок получит `arr[i] + (сумма, которую противник оставит ему)`. Но поскольку противник хочет минимизировать нашу прибыль, он выберет такой ход, чтобы мы получили минимум из двух возможных вариантов.

Таким образом, рекуррентное соотношение:

```
dp[i][j] = max(
    arr[i] + min(dp[i+2][j], dp[i+1][j-1]),
    arr[j] + min(dp[i+1][j-1], dp[i][j-2])
)
```

Здесь:
- В первом случае мы берём `arr[i]`. Противник затем может взять `arr[i+1]` или `arr[j]`. Если противник берёт `arr[i+1]`, мы остаёмся с подмассивом `[i+2, j]`; если берёт `arr[j]`, остаётся `[i+1, j-1]`. Противник выберет вариант, минимизирующий нашу будущую прибыль, поэтому берём минимум из двух.
- Во втором случае аналогично для `arr[j]`.

Базовый случай: `dp[i][i] = arr[i]` (одна монета).

Ответ: `dp[0][n-1]`.

**Альтернативная интерпретация (разность очков):**  
Можно определить `dp[j]` как максимальную разность (текущий игрок - противник) для подмассива. Тогда формула упрощается, и мы можем использовать одномерный массив.

## Подходы к решению

### 1. Наивная рекурсия (экспоненциальная)

Рекурсивный перебор всех вариантов без мемоизации даёт O(2ⁿ). Подходит только для очень маленьких n.

### 2. Рекурсия с мемоизацией (Top‑Down DP)

Используется двумерный массив `dp[n][n]` для хранения уже вычисленных значений. Это снижает сложность до O(n²).

```python
def maxAmount(arr):
    n = len(arr)
    dp = [[-1]*n for _ in range(n)]
    def solve(i, j):
        if i > j:
            return 0
        if dp[i][j] != -1:
            return dp[i][j]
        # берём arr[i]
        take_first = arr[i] + min(solve(i+2, j), solve(i+1, j-1))
        # берём arr[j]
        take_last = arr[j] + min(solve(i+1, j-1), solve(i, j-2))
        dp[i][j] = max(take_first, take_last)
        return dp[i][j]
    return solve(0, n-1)
```

### 3. Табличный DP (Bottom‑Up) — O(n²) времени и O(n²) памяти

Заполняем таблицу по возрастанию длины подмассива (gap).

```python
def maximumAmount(arr):
    n = len(arr)
    dp = [[0]*n for _ in range(n)]
    for gap in range(n):
        for i in range(n - gap):
            j = i + gap
            if gap == 0:
                dp[i][j] = arr[i]
            else:
                x = dp[i+2][j] if i+2 <= j else 0
                y = dp[i+1][j-1] if i+1 <= j-1 else 0
                z = dp[i][j-2] if i <= j-2 else 0
                dp[i][j] = max(arr[i] + min(x, y),
                               arr[j] + min(y, z))
    return dp[0][n-1]
```

### 4. Пространственно-оптимизированный подход (ожидаемый) — O(n²) времени и O(n) памяти

Можно переформулировать задачу как поиск максимальной разности очков между игроками. Для подмассива `arr[i..j]` определим `dp[j]` как максимальную разность (текущий игрок − противник), которую можно получить для этого подмассива.

Если текущий игрок берёт `arr[i]`, то его преимущество будет `arr[i] - dp_opponent`, где `dp_opponent` — это разность для оставшейся части, но с противоположным знаком. Итеративно можно пересчитывать значения, используя только один массив.

Формула для обновления (при обходе i от n-1 до 0):
```
dp[j] = max(arr[i] - dp[j], arr[j] - dp[j-1])
```
где `dp[j]` до обновления соответствует подмассиву `[i+1..j]`, а `dp[j-1]` — `[i..j-1]`.

В конце `dp[n-1]` даёт разность (первый игрок − второй). Общая сумма всех монет известна, поэтому:
```
первый_игрок = (total_sum + dp[n-1]) // 2
```

```python
def maximumAmount(arr):
    n = len(arr)
    total = sum(arr)
    dp = [0] * n
    for i in range(n-1, -1, -1):
        for j in range(i, n):
            if i == j:
                dp[j] = arr[i]
            else:
                dp[j] = max(arr[i] - dp[j], arr[j] - dp[j-1])
    return (total + dp[n-1]) // 2
```

## Сложность

| Подход | Время | Память |
|--------|-------|--------|
| Рекурсивный | O(2ⁿ) | O(n) |
| Мемоизация | O(n²) | O(n²) |
| Табличный | O(n²) | O(n²) |
| **Оптимизированный (1D)** | **O(n²)** | **O(n)** |

## Примечания

- Задача известна на GeeksforGeeks как "Optimal Strategy for a Game" и на LeetCode как "Predict the Winner" (№ 486, но там другой подсчёт — разница очков, а не сумма первого игрока).
- В пространственно-оптимизированном подходе важно правильно понимать, что `dp[j]` до обновления хранит значение для `[i+1..j]`, а `dp[j-1]` — для `[i..j-1]`, что позволяет использовать один массив.
- Если массив имеет чётную длину, оба игрока получают равное количество ходов, но в задаче это не принципиально.
