## Определение

Дана задача о лягушке, которая хочет добраться с первой ступеньки на последнюю, имея возможность прыгать либо на следующую ступеньку, либо через одну. Каждый прыжок имеет стоимость, равную абсолютной разнице высот между ступеньками. Требуется найти минимальную суммарную стоимость, чтобы достичь последней ступеньки.

**Пример:**
```
height = [20, 30, 40, 20]
Оптимальный путь: 0 → 1 → 3 (20→30→20)
Стоимость: |30-20| + |20-30| = 10 + 10 = 20
```

---

## Постановка задачи

**Вход:** массив `height` длины `n`, где `height[i]` — высота i-й ступеньки (0-индексация). Лягушка стартует с ступеньки 0 и должна достичь ступеньки n-1. Разрешены прыжки длиной 1 или 2. Стоимость прыжка с i на j равна `|height[i] - height[j]|`.

**Выход:** минимальная суммарная стоимость.

**Ограничения:** 1 ≤ n ≤ 10⁵, 0 ≤ height[i] ≤ 10⁴.

---

## Идея решения

Это классическая задача динамического программирования. Пусть `dp[i]` — минимальная стоимость достижения ступеньки `i`. Тогда:

- Для `i = 0`: `dp[0] = 0` (старт).
- Для `i = 1`: `dp[1] = |height[1] - height[0]|` (единственный способ — прыжок с 0).
- Для `i ≥ 2`: можно прийти либо с `i-1` (прыжок длины 1), либо с `i-2` (прыжок длины 2):
  ```
  dp[i] = min(dp[i-1] + |height[i] - height[i-1]|,
              dp[i-2] + |height[i] - height[i-2]|)
  ```

Ответ — `dp[n-1]`.

Для эффективного использования памяти заметим, что для вычисления `dp[i]` нужны только два предыдущих значения, поэтому можно хранить только две последние переменные вместо всего массива.

---

## Алгоритм

1. Если `n == 1`, вернуть `0`.
2. Инициализировать `prev2 = 0` (dp[0]), `prev1 = abs(height[1] - height[0])` (dp[1]).
3. Для `i` от 2 до `n-1`:
   - `curr = min(prev1 + abs(height[i] - height[i-1]), prev2 + abs(height[i] - height[i-2]))`.
   - Обновить: `prev2 = prev1`, `prev1 = curr`.
4. Вернуть `prev1`.

---

## Реализация на Python

```python
class Solution:
    def minCost(self, height):
        n = len(height)
        if n == 1:
            return 0
        prev2 = 0  # dp[0]
        prev1 = abs(height[1] - height[0])  # dp[1]
        for i in range(2, n):
            curr = min(prev1 + abs(height[i] - height[i-1]),
                       prev2 + abs(height[i] - height[i-2]))
            prev2, prev1 = prev1, curr
        return prev1
```

---

## Сложность

- **Время:** O(n) — один проход по массиву.
- **Память:** O(1) — используются только несколько переменных.

---

## Примечания

- Если `n = 1`, лягушка уже на последней ступеньке, стоимость 0.
- Задача может быть обобщена на прыжки любой длины (до k), тогда потребуется массив из k последних значений или использование очереди.
- Входные данные могут быть большими (до 10⁵), поэтому линейный алгоритм оптимален.
