## Определение

**Задача об уникальных путях в сетке с препятствиями** — это классическая задача динамического программирования, в которой требуется найти количество способов добраться из левого верхнего угла `(0, 0)` в правый нижний `(n-1, m-1)` прямоугольной сетки, двигаясь только вправо и вниз, при этом некоторые клетки заблокированы (непроходимы). Задача является обобщением более простой версии (без препятствий) и часто используется для демонстрации применения DP для подсчёта комбинаторных путей.

**Пример:**
```
grid = [[0,0,0],
        [0,1,0],
        [0,0,0]]
Ответ: 2 (единственные пути обходят препятствие).
```

---

## Постановка задачи

**Входные данные:**
- `grid` — двумерный массив размера `n × m`, где `0` обозначает проходимую ячейку, `1` — заблокированную.

**Выход:** количество различных путей от `(0,0)` до `(n-1, m-1)`, двигаясь только вправо и вниз, не заходя в заблокированные клетки. Если путь невозможен, вернуть `0`.

**Ограничения:** `1 ≤ n·m ≤ 10^6`, ответ помещается в 32-битное целое.

---

## Идея решения

Ключевая идея: количество способов добраться до клетки `(i, j)` равно сумме способов попасть в клетку сверху `(i-1, j)` и слева `(i, j-1)`, при условии, что текущая клетка не заблокирована. Это следует из того, что движение возможно только вправо и вниз.

Пусть `dp[i][j]` — количество путей до `(i, j)`. Тогда:
- Если `grid[i][j] == 1` (заблокирована), то `dp[i][j] = 0`.
- Иначе `dp[i][j] = dp[i-1][j] + dp[i][j-1]` (с учётом границ).

Базовый случай: `dp[0][0] = 1`, если стартовая клетка не заблокирована, иначе `0`.

Обычно для этого используют двумерный массив `dp` размером `n × m`. Однако в приведённом коде используется **модификация исходной матрицы на месте**, что экономит память: вместо создания новой матрицы, значения путей записываются в `grid` с отрицательным знаком (чтобы отличать от препятствий `1`). Это позволяет обойтись без дополнительной памяти.

---

## Реализация (с модификацией матрицы)

```python
class Solution:
    def uniquePaths(self, grid):
        n = len(grid)
        m = len(grid[0])
        if grid[0][0] == 1:
            return 0
        grid[0][0] = -1   # стартовая клетка: 1 способ (отрицательное значение)

        for i in range(n):
            for j in range(m):
                if grid[i][j] == 1:   # заблокированная клетка
                    continue
                if i > 0 and grid[i-1][j] != 1:
                    grid[i][j] += grid[i-1][j]
                if j > 0 and grid[i][j-1] != 1:
                    grid[i][j] += grid[i][j-1]

        return -grid[n-1][m-1] if grid[n-1][m-1] < 0 else 0
```

### Пояснение кода

1. Если начальная клетка заблокирована, сразу возвращаем `0`.
2. Устанавливаем `grid[0][0] = -1`, что означает «1 путь» (отрицательное число используется для отличия от `1`, обозначающего препятствие). Далее при сложении мы будем накапливать отрицательные значения.
3. Проходим по всем ячейкам в порядке слева направо, сверху вниз (это гарантирует, что значения сверху и слева уже вычислены).
4. Для каждой проходимой ячейки (не `1`):
   - Если есть верхний сосед и он не заблокирован (`grid[i-1][j] != 1`), прибавляем его значение (которое отрицательное).
   - Если есть левый сосед и он не заблокирован, прибавляем его значение.
5. В конце, если значение в правом нижнем углу отрицательно (т.е. есть путь), возвращаем его абсолютное значение, иначе `0`.

### Почему отрицательные числа?

Это трюк, позволяющий использовать исходный массив как для хранения препятствий (1), так и для хранения количества путей (отрицательные числа). При сложении отрицательных чисел мы получаем отрицательную сумму, которая по модулю равна количеству путей. В конце берём модуль. Это экономит память и упрощает код.

## Альтернативные подходы

### 1. Классическое DP с отдельным массивом

Создаётся двумерный массив `dp` размером `n × m`. Заполняется по тому же принципу.

```python
def uniquePaths(grid):
    n, m = len(grid), len(grid[0])
    if grid[0][0] == 1:
        return 0
    dp = [[0]*m for _ in range(n)]
    dp[0][0] = 1
    for i in range(n):
        for j in range(m):
            if grid[i][j] == 1:
                continue
            if i > 0 and grid[i-1][j] != 1:
                dp[i][j] += dp[i-1][j]
            if j > 0 and grid[i][j-1] != 1:
                dp[i][j] += dp[i][j-1]
    return dp[n-1][m-1]
```

### 2. Оптимизация по памяти (одномерный массив)

Поскольку для вычисления текущей строки нужны только значения из предыдущей строки и текущей строки слева, можно использовать одномерный массив `dp` длиной `m`. Обновление происходит слева направо.

```python
def uniquePaths(grid):
    n, m = len(grid), len(grid[0])
    if grid[0][0] == 1:
        return 0
    dp = [0]*m
    dp[0] = 1
    for i in range(n):
        for j in range(m):
            if grid[i][j] == 1:
                dp[j] = 0
            else:
                if i == 0 and j == 0:
                    continue
                if j > 0:
                    dp[j] += dp[j-1]
                # Для j=0 значение остаётся из предыдущей строки (если не заблокирована)
    return dp[m-1]
```

Этот подход использует O(m) памяти, что предпочтительнее для больших матриц.

---

## Сложность

- **Время:** O(n·m) — проход по всем клеткам.
- **Память:** O(1) для модификации на месте, O(n·m) для классического DP, O(m) для оптимизированного одномерного.

---

## Примечания

- Стартовая клетка не должна быть заблокирована, иначе ответ 0.
- Путь всегда идёт только вправо и вниз, поэтому это направленный ациклический граф (DAG), что позволяет использовать DP без циклов.
- Если целевая клетка заблокирована, ответ автоматически 0 (так как при достижении `(n-1, m-1)` она будет `1`, и мы пропустим её).
- В коде с модификацией матрицы важно не путать `1` (препятствие) с отрицательными числами (количеством путей). При проверке соседа на препятствие используется строгое `!= 1`, чтобы не отсечь отрицательные значения.

## Связь с другими задачами

- **Unique Paths (без препятствий)** — частный случай, когда все клетки проходимы. Тогда ответ можно вычислить по формуле комбинаторики: `C(n+m-2, n-1)`.
- **Minimum Path Sum** — аналогичная задача, но вместо подсчёта путей ищется минимальная сумма весов на пути.
- **Задача о роботе в сетке** — часто формулируется как "робот может двигаться вниз и вправо, посчитать количество маршрутов".

## Заключение

Задача об уникальных путях в сетке с препятствиями — отличный пример применения динамического программирования для подсчёта комбинаторных объектов. Она демонстрирует, как можно эффективно использовать DP на матрицах, а также показывает, как можно экономить память, модифицируя исходные данные (с осторожностью). Понимание этой задачи помогает освоить базовые приёмы работы с двумерными DP и подготавливает к более сложным задачам на сетках.
