## Определение

**Дера́нжмент** (или **субфакториал**) — это перестановка элементов множества, в которой ни один элемент не остаётся на своём исходном месте. Иными словами, это такая перестановка `π` множества `{1, 2, …, n}`, что `π(i) ≠ i` для всех `i`. Количество таких перестановок обозначается `!n` (читается как «субфакториал n») или `D(n)`.

**Пример:**
- Для `n = 1`: перестановка `[1]` не является дера́нжментом, так как `1` остаётся на месте → `D(1) = 0`.
- Для `n = 2`: перестановка `[2, 1]` — единственный дера́нжмент → `D(2) = 1`.
- Для `n = 3`: дера́нжменты: `[2, 3, 1]`, `[3, 1, 2]` → `D(3) = 2`.
- Для `n = 4`: всего 9 дера́нжментов.

---

## Постановка задачи

Дано натуральное число `n` (1 ≤ n ≤ 10⁶). Требуется вычислить количество дера́нжментов для `n` элементов, т.е. `D(n)`.

**Вход:** целое число `n`.
**Выход:** `D(n)` — количество перестановок без неподвижных точек.

---

## Идея решения (рекуррентное соотношение)

Количество дера́нжментов подчиняется простому рекуррентному соотношению:

$$
D(n) = (n-1) \cdot \bigl(D(n-1) + D(n-2)\bigr)
$$

с начальными условиями:
- `D(1) = 0`
- `D(2) = 1`

**Обоснование:** рассмотрим элемент `n`. В дера́нжменте он должен стоять на позиции `k` (где `k ≠ n`). Есть `(n-1)` способов выбрать `k`. Теперь рассмотрим два случая:

1. Элемент `k` оказывается на позиции `n`. Тогда остальные `n-2` элементов должны образовывать дера́нжмент → `D(n-2)` способов.
2. Элемент `k` **не** оказывается на позиции `n`. Тогда мы можем «переименовать» задачу для `n-1` элементов, где элемент `k` теперь считается «запрещённым» для позиции `n`, что эквивалентно дера́нжменту размера `n-1` → `D(n-1)` способов.

Таким образом, для каждого из `(n-1)` возможных `k` получаем `D(n-1) + D(n-2)` вариантов, что даёт формулу.

---

## Алгоритм (оптимизированный по памяти)

Поскольку для вычисления `D(n)` нужны только два предыдущих значения, можно обойтись без массива и использовать две переменные.

1. Если `n == 1`, вернуть `0`.
2. Если `n == 2`, вернуть `1`.
3. Инициализировать `prev2 = 0` (D(1)), `prev1 = 1` (D(2)).
4. Для `i` от `3` до `n`:
   - `curr = (i - 1) * (prev1 + prev2)`
   - `prev2 = prev1`
   - `prev1 = curr`
5. Вернуть `prev1`.

---

## Реализация на Python

```python
def derangement_count(n):
    if n == 1:
        return 0
    if n == 2:
        return 1

    prev2 = 0   # D(1)
    prev1 = 1   # D(2)

    for i in range(3, n + 1):
        curr = (i - 1) * (prev1 + prev2)
        prev2 = prev1
        prev1 = curr

    return prev1
```

---

## Сложность

- **Время:** O(n) — один цикл до `n`.
- **Память:** O(1) — используются только две переменные.

---

## Примеры

```python
print(derangement_count(1))  # 0
print(derangement_count(2))  # 1
print(derangement_count(3))  # 2
print(derangement_count(4))  # 9
print(derangement_count(5))  # 44
```

## Примечания

- Дера́нжменты широко используются в комбинаторике и теории вероятностей, например, в задаче о встречах (письма, случайным образом разложенные по конвертам, не попадают в свои).
- Существует также формула через включения-исключения:
  $$
  D(n) = n! \sum_{k=0}^{n} \frac{(-1)^k}{k!}
  $$
  но рекурсивный подход эффективнее для больших `n`, так как работает за O(n) и не требует вычисления факториалов с плавающей точкой.
