## Определение

Дана длина отрезка `n`. Разрешено разрезать его на части длиной `x`, `y` или `z` (каждую часть можно использовать любое количество раз). Требуется найти **максимальное количество отрезков**, на которое можно разрезать исходный отрезок, используя только указанные длины. Если это невозможно, вернуть `0`.

**Пример:**
```
n = 4, x = 2, y = 1, z = 1 → можно разрезать на 4 отрезка по 1 → ответ 4.
n = 5, x = 5, y = 3, z = 2 → можно разрезать на 3 и 2 → ответ 2.
n = 7, x = 8, y = 9, z = 10 → невозможно → ответ 0.
```

---

## Постановка задачи

**Вход:** целые числа `n`, `x`, `y`, `z` (1 ≤ n, x, y, z ≤ 10⁴).
**Выход:** максимальное число отрезков (целое).

---

## Идея решения (динамическое программирование)

Задача сводится к поиску максимального числа предметов (отрезков) с суммарным весом (длиной) ровно `n`. Это классическая задача о неограниченном рюкзаке, где вес каждого предмета = его длине, а ценность = 1.

Будем использовать DP снизу вверх:

- `dp[i]` — максимальное количество отрезков, которыми можно заполнить длину `i`.
- Инициализируем `dp[0] = 0`; все остальные `dp[i] = -1` (означает недостижимость).
- Для каждого `i` от 1 до `n`:
  - Если `i >= x` и `dp[i-x] != -1`, то `dp[i] = max(dp[i], dp[i-x] + 1)`.
  - Аналогично для `y` и `z`.
- Если `dp[n] == -1`, вернуть `0`, иначе `dp[n]`.

**Почему это работает?**  
Длина `i` может быть получена из длины `i-x` добавлением одного отрезка длины `x`. Если `i-x` достижимо, то и `i` достижимо, причём число отрезков увеличивается на 1. Берём максимум из трёх возможных вариантов (используя `x`, `y` или `z`).

---

## Реализация на Python

```python
class Solution:
    def maximizeCuts(self, n, x, y, z):
        dp = [-1] * (n + 1)
        dp[0] = 0
        for i in range(1, n + 1):
            if i >= x and dp[i - x] != -1:
                dp[i] = max(dp[i], dp[i - x] + 1)
            if i >= y and dp[i - y] != -1:
                dp[i] = max(dp[i], dp[i - y] + 1)
            if i >= z and dp[i - z] != -1:
                dp[i] = max(dp[i], dp[i - z] + 1)
        return 0 if dp[n] == -1 else dp[n]
```

---

## Сложность

- **Время:** O(n) — один проход по всем значениям длины.
- **Память:** O(n) для массива `dp`.

---

## Примечания

- Если `n` меньше всех длин, ответ будет `0`.
- Порядок `x, y, z` не важен, все они проверяются.
- Можно оптимизировать по памяти, используя только несколько последних значений, но поскольку `x, y, z` могут быть разными, и длина `i` зависит от `i-x, i-y, i-z`, которые не обязательно идут подряд, проще использовать массив.
