## Определение

**Задача о разрезании стержня** (Rod Cutting Problem) — это классическая задача динамического программирования. Дан стержень длиной `n` дюймов и массив `price[]`, где `price[i]` (1-индексация) обозначает стоимость куска длиной `i` дюймов. Требуется разрезать стержень на части (целой длины) и продать их так, чтобы получить **максимальную суммарную выручку**. Разрезы можно делать произвольным образом, каждый кусок продаётся по цене, соответствующей его длине. Разрешается не резать стержень вовсе (продать целиком).

**Пример:**
```
price = [1, 5, 8, 9, 10, 17, 17, 20] (n = 8)
Оптимальный вариант: разрезать на длины 2 и 6 → 5 + 17 = 22.
```

---

## Постановка задачи

**Входные данные:**
- `price` — массив длины `n` (1 ≤ n ≤ 10³), где `price[i]` (1-индексация) — цена куска длиной `i`.
- `n` — длина стержня (равна `len(price)`).

**Выход:** максимальная выручка (целое число).

**Ограничения:** 1 ≤ price[i] ≤ 10⁶.

---

## Идея решения (динамическое программирование)

Обозначим `dp[i]` — максимальную выручку, которую можно получить от стержня длины `i`. Для длины `i` мы можем отрезать первый кусок длиной `j` (1 ≤ j ≤ i) и продать его по цене `price[j-1]` (поскольку `price` индексируется с 0, но соответствует длине 1). Оставшаяся часть длины `i-j` будет разрезана оптимально, давая выручку `dp[i-j]`. Таким образом, получаем рекуррентное соотношение:

```
dp[i] = max_{1 ≤ j ≤ i} ( price[j-1] + dp[i-j] )
```

Базовый случай: `dp[0] = 0` (стержень нулевой длины не даёт выручки). Мы вычисляем `dp[i]` для всех `i` от 1 до `n` в порядке возрастания, так как `dp[i]` зависит от `dp[i-j]` с меньшей длиной.

Ответ: `dp[n]`.

---

## Реализация

В предоставленном коде используется одномерный массив `dp` размера `n+1` (индексы от 0 до n). Внешний цикл `for i in range(1, n+1)` заполняет `dp[i]`. Внутренний цикл `for j in range(1, i+1)` перебирает длину первого куска `j`. Внутри вычисляется `price[j-1] + dp[i-j]`, и обновляется максимум.

```python
class Solution:
    def cutRod(self, price):
        n = len(price)
        dp = [0] * (n + 1)
        for i in range(1, n + 1):
            for j in range(1, i + 1):
                dp[i] = max(dp[i], price[j - 1] + dp[i - j])
        return dp[n]
```

### Пояснение кода

- `dp[0] = 0` (стержень длины 0 не стоит ничего).
- Для каждого `i` (длина стержня) мы пробуем все возможные длины первого куска `j`.
- `price[j-1]` — цена куска длины `j` (т.к. массив 0-индексированный).
- `dp[i-j]` — оптимальная выручка от оставшейся части.
- Берём максимум по всем `j`.
- В конце возвращаем `dp[n]`.

---

## Сложность

- **Время:** O(n²) — два вложенных цикла по i и j.
- **Память:** O(n) — используется один массив `dp`.

---

## Пример работы

Для `price = [1, 5, 8, 9, 10, 17, 17, 20]`, n = 8:

Вычисления `dp`:
- dp[1] = max(price[0] + dp[0]) = 1
- dp[2] = max(price[0]+dp[1]=2, price[1]+dp[0]=5) = 5
- dp[3] = max(1+5=6, 5+1=6, 8+0=8) = 8
- dp[4] = max(1+8=9, 5+5=10, 8+1=9, 9+0=9) = 10
- ...
- dp[8] = 22 (как в примере).

---

## Примечания

- Массив `price` интерпретируется как 1-индексированный, поэтому `price[0]` соответствует длине 1.
- Задача является вариацией **неограниченного рюкзака**, где предметы — это куски различной длины, каждый может быть использован несколько раз (в разных разрезах). Однако здесь есть зависимость: сумма длин кусков должна равняться n.
- Альтернативный подход: можно решать через рекурсию с мемоизацией, но итеративный метод проще и эффективнее.

---

## Связь с другими задачами

- **Неограниченный рюкзак** — аналогичная структура, но там максимизируется ценность при ограничении на вес, и предметы имеют вес и ценность. Здесь вес куска равен его длине, а ценность — `price[длина]`.
- **Coin Change (минимум монет)** — похожая задача, но там минимизируется количество монет при неограниченном использовании.
- **Задача о раскрое** — обобщение для двумерных материалов.

## Заключение

Задача о разрезании стержня — это классическое упражнение на динамическое программирование, демонстрирующее технику построения решения снизу вверх для задач с оптимальной подструктурой и перекрывающимися подзадачами. Она помогает понять разницу между 0/1 рюкзаком и неограниченным рюкзаком, а также важность правильного порядка обновления массива.
