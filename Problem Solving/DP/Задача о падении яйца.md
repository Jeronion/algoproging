## Определение

**Задача о падении яйца** — это классическая задача динамического программирования, в которой дано `n` яиц и `k` этажей. Необходимо определить минимальное количество попыток (бросков), гарантирующее нахождение **критического этажа** — самого высокого этажа, с которого яйцо можно бросить, и оно не разобьётся.

Условия:
- Яйцо разбивается при падении с этажа `f` и выше; не разбивается при падении с этажа ниже `f`.
- Если яйцо разбивается, оно больше не может быть использовано.
- Если яйцо не разбивается, его можно использовать снова.
- Нужно найти минимальное количество бросков, достаточное для гарантированного определения критического этажа в худшем случае.

**Пример:**
- 1 яйцо, 10 этажей → нужно 10 бросков (приходится проверять последовательно снизу вверх).
- 2 яйца, 10 этажей → достаточно 4 бросков (оптимальная стратегия).

---

## Постановка задачи

**Входные данные:**
- `n` — количество яиц (целое положительное).
- `k` — количество этажей (целое положительное).

**Выход:** минимальное количество бросков, гарантирующее нахождение критического этажа.

**Ограничения:** типичные: 1 ≤ n ≤ 50, 1 ≤ k ≤ 10⁴ (в некоторых версиях до 10⁹).

---

## Идея решения (оптимизированный DP)

В классическом решении используется DP `dp[i][j]` — минимальное количество бросков для `i` яиц и `j` этажей. Однако существует более эффективный подход, основанный на изменении логики: вместо того чтобы искать минимальные броски для заданного числа этажей, мы вычисляем максимальное количество этажей, которое можно проверить за `m` бросков, имея `n` яиц.

Обозначим `dp[m][i]` — максимальное количество этажей, которое можно покрыть (гарантированно проверить) за `m` бросков, имея `i` яиц. Рекуррентное соотношение:

```
dp[m][i] = 1 + dp[m-1][i-1] + dp[m-1][i]
```

Пояснение:
- Мы бросаем яйцо с некоторого этажа.
- Если яйцо разбивается, у нас остаётся `m-1` бросков и `i-1` яиц, и мы можем проверить `dp[m-1][i-1]` этажей ниже.
- Если яйцо не разбивается, у нас остаётся `m-1` бросков и `i` яиц, и мы можем проверить `dp[m-1][i]` этажей выше.
- Плюс текущий этаж, который мы проверили (1).

Таким образом, `dp[m][i]` — это максимальное количество этажей, которое можно покрыть.

Нам нужно найти минимальное `m`, такое что `dp[m][n] >= k`. Это и будет ответ.

---

## Оптимизация по памяти

Для вычисления `dp[m][i]` требуется только предыдущая строка `dp[m-1]`. Поэтому можно использовать одномерный массив `dp` размера `n+1`, обновляя его справа налево:

```
for m in range(1, max_moves+1):
    for i in range(n, 0, -1):
        dp[i] = 1 + dp[i] + dp[i-1]
    if dp[n] >= k:
        return m
```

Начальное условие: `dp[0] = 0` (с 0 яиц нельзя проверить ни одного этажа). При одном броске и одном яйце можно проверить 1 этаж, и т.д.

Этот подход даёт O(n·m) времени и O(n) памяти, где `m` — ответ (количество бросков). Поскольку `m` растёт как `O(log k)` для больших `k`, алгоритм очень эффективен.

## Реализация на Python

### 1. Оптимизированный (одномерный массив) — ожидаемый

```python
def egg_drop(n, k):
    dp = [0] * (n + 1)   # dp[i] — максимальное количество этажей для i яиц
    moves = 0
    while dp[n] < k:
        moves += 1
        for i in range(n, 0, -1):
            dp[i] = 1 + dp[i] + dp[i - 1]
    return moves
```

**Пояснение:**  
- `dp[0]` всегда 0.
- На каждой итерации `moves` мы обновляем массив `dp` для всех `i` от `n` до 1, используя значения из предыдущего шага (так как обновление справа налево предотвращает использование уже обновлённых значений).
- Когда `dp[n]` становится ≥ `k`, текущее `moves` и есть ответ.

### 2. Классический DP (таблица) — для справки

```python
def egg_drop_classic(n, k):
    # dp[i][j] — минимальные броски для i яиц и j этажей
    dp = [[0] * (k + 1) for _ in range(n + 1)]
    for i in range(1, n + 1):
        dp[i][1] = 1   # 1 этаж → 1 бросок
        dp[i][0] = 0   # 0 этажей → 0 бросков
    for j in range(1, k + 1):
        dp[1][j] = j   # 1 яйцо → j бросков (последовательно)

    for i in range(2, n + 1):
        for j in range(2, k + 1):
            dp[i][j] = float('inf')
            for x in range(1, j + 1):
                # бросаем с этажа x
                # если разбилось: dp[i-1][x-1]; если нет: dp[i][j-x]
                dp[i][j] = min(dp[i][j], 1 + max(dp[i-1][x-1], dp[i][j-x]))
    return dp[n][k]
```

## Сложность

| Подход | Время | Память |
|--------|-------|--------|
| Классический DP | O(n·k²) | O(n·k) |
| Оптимизированный DP (двумерный) | O(m·n) | O(m·n) |
| **Оптимизированный (одномерный)** | **O(m·n)** | **O(n)** |

Где `m` — минимальное количество бросков, которое обычно не превышает `ceil(log2(k)) + n` и сильно меньше `k` для больших `k`.

---

## Пример

`n = 2, k = 10`

- `dp = [0, 0, 0]` (индексы 0,1,2)
- moves=1: 
  - i=2: dp[2] = 1 + dp[2] + dp[1] = 1+0+0=1
  - i=1: dp[1] = 1 + dp[1] + dp[0] = 1+0+0=1
  - dp[2] = 1 < 10
- moves=2:
  - i=2: dp[2] = 1 + 1 + 1 = 3
  - i=1: dp[1] = 1 + 1 + 0 = 2
  - dp[2] = 3 < 10
- moves=3:
  - i=2: dp[2] = 1 + 3 + 2 = 6
  - i=1: dp[1] = 1 + 2 + 0 = 3
  - dp[2] = 6 < 10
- moves=4:
  - i=2: dp[2] = 1 + 6 + 3 = 10 ≥ 10 → возвращаем 4.

---

## Примечания

- Задача известна как **Egg Dropping Problem** (LeetCode 887, GeeksforGeeks).
- Оптимизированное решение часто используется в интервью, так как оно очень эффективно по памяти.
- При `n` = 1 ответ всегда `k`.
- При `k` = 0 или 1 ответы тривиальны (0 или 1).
- В оптимизированном подходе мы не вычисляем непосредственно `dp` для всех этажей, а ищем минимальное количество бросков, что позволяет работать с очень большими `k` (до 10⁹).
