## Определение

**Наибольшая общая подстрока** двух строк — это самая длинная непрерывная последовательность символов, которая встречается в обеих строках. В отличие от подпоследовательности, подстрока должна идти подряд без пропусков.

**Пример:**
```
s1 = "GeeksforGeeks"
s2 = "GeeksQuiz"
Наибольшая общая подстрока: "Geeks" (длина 5).
```

---

## Постановка задачи

**Вход:** две строки `s1` и `s2` (длины m и n).  
**Выход:** длина самой длинной непрерывной подстроки, присутствующей в обеих строках. Если общей подстроки нет, ответ 0.

---

## Идея решения

В отличие от Наибольшая общая подпоследовательность (LCS), здесь требуется непрерывность. Динамическое программирование для этой задачи использует таблицу `dp[i][j]`, где `dp[i][j]` — длина наибольшей общей подстроки, заканчивающейся на позициях `i-1` в `s1` и `j-1` в `s2`.

**Рекуррентное соотношение:**
```
если s1[i-1] == s2[j-1]:
    dp[i][j] = dp[i-1][j-1] + 1
иначе:
    dp[i][j] = 0
```

Ответ — максимальное значение во всей таблице.

---

## Подходы к решению

### 1. Наивная рекурсия (экспоненциальная)

Рассматриваются все пары индексов `(i, j)` и рекурсивно вычисляется длина общего суффикса. Сложность O(m·n·min(m,n)), что очень медленно.

### 2. Динамическое программирование (таблица) – O(m·n) времени, O(m·n) памяти

Построение таблицы `dp` размером `(m+1) × (n+1)`. Заполнение по строкам. Ведётся глобальный максимум.

### 3. Оптимизация по памяти (два одномерных массива) – O(m·n) времени, O(n) памяти

Поскольку каждое состояние зависит только от диагонального значения из предыдущей строки, можно хранить только две строки: `prev` и `curr`. Это снижает память до O(n).

## Реализации

### Классическое DP (таблица)

```python
def longest_common_substring(s1, s2):
    m, n = len(s1), len(s2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    best = 0
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i-1] == s2[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
                best = max(best, dp[i][j])
            else:
                dp[i][j] = 0
    return best
```

### Оптимизированное по памяти (два массива)

```python
def longest_common_substring_optimized(s1, s2):
    m, n = len(s1), len(s2)
    prev = [0] * (n + 1)
    best = 0
    for i in range(1, m + 1):
        curr = [0] * (n + 1)
        for j in range(1, n + 1):
            if s1[i-1] == s2[j-1]:
                curr[j] = prev[j-1] + 1
                best = max(best, curr[j])
            else:
                curr[j] = 0
        prev = curr
    return best
```

## Сложность

| Подход                | Время      | Память    |
|-----------------------|------------|-----------|
| Наивная рекурсия      | O(m·n·k)   | O(k)      |
| DP (таблица)          | O(m·n)     | O(m·n)    |
| Оптимизированное DP   | O(m·n)     | O(n)      |

---

## Примечания

- Задача отличается от **Longest Common Subsequence** (LCS), где подстрока может быть прерывистой. Здесь используется непрерывность, поэтому при несовпадении сбрасываем `dp[i][j] = 0`.
- Можно также решить с помощью суффиксных массивов или хешей, но DP достаточно эффективен для строк длиной до нескольких тысяч.
- Если требуется сама подстрока, а не длина, можно запоминать её начало при обновлении максимума.

## Заключение

Задача о наибольшей общей подстроке является классическим упражнением на динамическое программирование и часто встречается в собеседованиях и олимпиадах. Понимание разницы между подстрокой и подпоследовательностью, а также умение оптимизировать память — важные навыки для решения подобных задач.
