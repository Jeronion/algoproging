## Определение

**Наибольшее подмножество с делимостью** — это задача, в которой дан массив различных положительных целых чисел. Требуется найти подмножество максимального размера, такое что для любой пары элементов `(a, b)` в этом подмножестве либо `a` делится на `b`, либо `b` делится на `a`. Иными словами, все элементы в подмножестве должны быть попарно сравнимы по отношению делимости.

**Пример:**
```
arr = [1, 16, 7, 8, 4]
Наибольшее подмножество: [1, 4, 8, 16] (каждое число делится на предыдущее).
```

---

## Постановка задачи

**Вход:** массив `arr` различных положительных целых чисел (длина n).  
**Выход:** подмножество максимального размера, удовлетворяющее условию попарной делимости. Если таких подмножеств несколько, можно вернуть любое.

**Ограничения:** типичные: n ≤ 10³–10⁴, значения до 10⁹.

---

## Идея решения

Задача решается динамическим программированием после сортировки массива. Ключевое наблюдение: если массив отсортирован по возрастанию, то для любого элемента `arr[i]` все его делители находятся слева (меньшие числа). Тогда мы можем построить наибольшее подмножество, где каждый следующий элемент делится на предыдущий. Это похоже на задачу о наибольшей возрастающей подпоследовательности (LIS), но с условием делимости вместо строгого возрастания.

Однако в предоставленном коде массив сортируется **по убыванию**. Это тоже допустимо: тогда мы проверяем, делится ли больший элемент на меньший (что эквивалентно). Выбор направления сортировки не влияет на результат, но упрощает логику: для каждого `i` мы ищем `j < i`, где `arr[j] % arr[i] == 0` (поскольку `arr[j] >= arr[i]` после сортировки по убыванию). Так мы строим подмножество, где каждый следующий элемент (меньший) делится на предыдущий (больший). В конце восстанавливаем подмножество с помощью массива `parent`.

### Алгоритм (по шагам)

1. Отсортировать массив по убыванию (чтобы большие числа были в начале).
2. Инициализировать массив `dp` длины `n`, где `dp[i] = 1` (каждый элемент сам по себе образует подмножество размера 1).
3. Инициализировать массив `parent` длиной `n` со значением `-1` (для восстановления ответа).
4. Для каждого `i` от 1 до `n-1`:
   - Для каждого `j` от 0 до `i-1`:
     - Если `arr[j] % arr[i] == 0` (т.е. `arr[j]` делится на `arr[i]`) и `dp[i] < dp[j] + 1`, то обновить `dp[i] = dp[j] + 1` и `parent[i] = j`.
5. Найти индекс `last_index` с максимальным `dp` и запомнить `max_size`.
6. Восстановить подмножество, проходя по `parent` от `last_index` до `-1` и добавляя элементы в результат.
7. Вернуть результат (можно развернуть, чтобы получить порядок возрастания, но в коде возвращается в порядке убывания).

## Реализация на Python

```python
def largestDivisibleSubset(arr):
    # Сортируем по убыванию
    arr.sort(reverse=True)
    n = len(arr)
    dp = [1] * n
    parent = [-1] * n

    max_size = 1
    last_index = 0

    for i in range(1, n):
        for j in range(i):
            if arr[j] % arr[i] == 0 and dp[i] < dp[j] + 1:
                dp[i] = dp[j] + 1
                parent[i] = j
        if dp[i] > max_size:
            max_size = dp[i]
            last_index = i

    # Восстановление подмножества
    res = []
    while last_index != -1:
        res.append(arr[last_index])
        last_index = parent[last_index]

    return res

# Пример
arr = [1, 16, 7, 8, 4]
print(largestDivisibleSubset(arr))  # [16, 8, 4, 1] или [8, 4, 1]? На самом деле [16, 8, 4, 1]
```

**Примечание:** В коде возвращается подмножество в порядке убывания (как в примере вывода). Если требуется возрастающий порядок, можно развернуть результат.

---

## Сложность

- **Время:** O(n²) — двойной цикл по индексам.
- **Память:** O(n) — массивы `dp` и `parent`.

---

## Почему это работает?

После сортировки по убыванию, для каждого `i` мы рассматриваем все предыдущие элементы `j`, которые больше или равны `arr[i]`. Если `arr[j]` делится на `arr[i]`, то мы можем добавить `arr[i]` в конец подмножества, заканчивающегося на `arr[j]`, увеличивая его размер. Это аналогично построению цепочки, где каждый следующий элемент делится на предыдущий. Таким образом, `dp[i]` хранит максимальную длину цепочки, заканчивающейся на `arr[i]`. Максимальное значение `dp` даёт размер наибольшего подмножества, а `parent` позволяет восстановить саму цепочку.

---

## Примечания

- Если в массиве есть дубликаты, условие делимости становится тривиальным (число делится само на себя), но в задаче обычно элементы различны.
- Можно сортировать и по возрастанию, тогда условие будет `arr[i] % arr[j] == 0` и обновление `dp[i]` с `dp[j] + 1`; восстановление будет аналогичным.
- Для больших n (например, > 10⁴) O(n²) может быть медленным. Существуют более сложные подходы с использованием теории чисел и хеширования, но для средних ограничений этот метод достаточно эффективен.
- Подмножество не обязано быть уникальным; алгоритм находит одно из максимальных.
