## Определение

**Wildcard Pattern Matching** (сопоставление с шаблоном, содержащим wildcard-символы) — это задача проверки, соответствует ли заданная строка `txt` шаблону `pat`, который может содержать два специальных символа:
- `?` — соответствует ровно одному любому символу.
- `*` — соответствует любой последовательности символов (включая пустую).

Требуется, чтобы шаблон полностью покрывал всю строку `txt`.

**Примеры:**
- `txt = "abcde", pat = "a?c*"` → `true` (`?` → `b`, `*` → `de`)
- `txt = "baaabab", pat = "a*ab"` → `false` (первый символ не совпадает)
- `txt = "abc", pat = "*"` → `true` (`*` покрывает всю строку)

---

## Постановка задачи

**Вход:**
- `txt` — строка текста (длина n).
- `pat` — шаблон с wildcard-символами (длина m).

**Выход:** `true`, если `pat` полностью соответствует `txt`, иначе `false`.

**Ограничения:** типичные: 1 ≤ n, m ≤ 10³–10⁴ (зависит от реализации).

---

## Идея решения

Задача решается динамическим программированием, но может быть эффективно решена и жадным способом за счёт особого поведения `*`. Мы рассмотрим два подхода:

1. **Пространственно-оптимизированное DP** (O(m) памяти) — стандартное решение, использующее только две строки таблицы.
2. **Жадный обход с двумя указателями** (O(1) памяти) — оптимальный для многих случаев, так как не требует дополнительной памяти.

---

## Подход 1: Пространственно-оптимизированное DP

### Идея

Обозначим `dp[i][j]` — можно ли сопоставить первые `i` символов `txt` с первыми `j` символами `pat`. Рекуррентное соотношение:

- Если `pat[j-1] == txt[i-1]` или `pat[j-1] == '?'`:
  `dp[i][j] = dp[i-1][j-1]`
- Если `pat[j-1] == '*'`:
  `dp[i][j] = dp[i-1][j] or dp[i][j-1]`
- Иначе `false`.

Базовые случаи:
- `dp[0][0] = true`
- Для пустого `txt` (`i=0`): `dp[0][j]` истинно только если все символы `pat[0..j-1]` — `*`.
- Для пустого `pat` (`j=0`): `dp[i][0] = false` при `i>0`.

Поскольку переход использует только предыдущую строку и текущую строку, мы можем хранить два одномерных массива: `prev` (предыдущая строка) и `curr` (текущая).

### Алгоритм

1. Инициализировать `prev` длиной `m+1` нулями.
2. Для `i` от 0 до `n`:
   - Создать `curr` нулевой массив.
   - Установить `curr[0] = (i == 0)` (пустой шаблон соответствует только пустому тексту).
   - Для `j` от 1 до `m`:
     - Если `i == 0`:
       `curr[j] = curr[j-1] and pat[j-1] == '*'`
     - Иначе:
       Если `pat[j-1] == '?'` или `txt[i-1] == pat[j-1]`:
         `curr[j] = prev[j-1]`
       Иначе если `pat[j-1] == '*'`:
         `curr[j] = prev[j] or curr[j-1]`
       Иначе `curr[j] = False`
   - После заполнения `curr` присвоить `prev = curr`.
3. Вернуть `prev[m]`.

### Реализация

```python
def wildcard_match_dp(txt, pat):
    n, m = len(txt), len(pat)
    prev = [False] * (m + 1)

    for i in range(n + 1):
        curr = [False] * (m + 1)
        curr[0] = (i == 0)  # пустой шаблон совпадает только с пустым текстом

        for j in range(1, m + 1):
            if i == 0:
                # если текст пуст, то pat должен состоять только из '*'
                curr[j] = curr[j-1] and pat[j-1] == '*'
            else:
                if pat[j-1] == '?' or txt[i-1] == pat[j-1]:
                    curr[j] = prev[j-1]
                elif pat[j-1] == '*':
                    curr[j] = prev[j] or curr[j-1]
                # иначе остаётся False

        prev = curr

    return prev[m]
```

### Сложность

- **Время:** O(n·m)
- **Память:** O(m)

## Подход 2: Жадный обход (Simple Traversal)

### Идея

Используем два указателя `i` (по тексту) и `j` (по шаблону). При встрече `*` запоминаем позицию в шаблоне и в тексте, чтобы можно было вернуться и попробовать другое разбиение. Алгоритм работает следующим образом:

- Если символы совпадают или `pat[j] == '?'`, двигаем оба указателя вперёд.
- Если `pat[j] == '*'`, запоминаем текущие `i` и `j` как точку возврата, затем увеличиваем `j` (пробуем сопоставить `*` с пустой строкой).
- Если не совпало и есть сохранённая звёздочка, возвращаемся к ней: увеличиваем `i` на 1 (звёздочка "съедает" ещё один символ текста) и сбрасываем `j` на позицию после звёздочки.
- Если ни одно условие не выполнено, возвращаем `False`.

В конце, если остались символы в шаблоне, они должны быть только `*` (их можно пропустить).

### Алгоритм

1. Инициализировать `i = 0`, `j = 0`, `star_idx = -1`, `match_idx = 0`.
2. Пока `i < n`:
   - Если `j < m` и `pat[j] == '?'` или `pat[j] == txt[i]`:
     - `i += 1`, `j += 1`
   - Иначе если `j < m` и `pat[j] == '*'`:
     - `star_idx = j`
     - `match_idx = i`
     - `j += 1`
   - Иначе если `star_idx != -1`:
     - `j = star_idx + 1`
     - `match_idx += 1`
     - `i = match_idx`
   - Иначе:
     - вернуть `False`
3. Пропустить все `*` в конце шаблона: пока `j < m` и `pat[j] == '*'`: `j += 1`.
4. Вернуть `j == m`.

### Реализация

```python
def wildcard_match_greedy(txt, pat):
    n, m = len(txt), len(pat)
    i = j = 0
    star_idx = -1
    match_idx = 0

    while i < n:
        if j < m and (pat[j] == '?' or pat[j] == txt[i]):
            i += 1
            j += 1
        elif j < m and pat[j] == '*':
            star_idx = j
            match_idx = i
            j += 1
        elif star_idx != -1:
            j = star_idx + 1
            match_idx += 1
            i = match_idx
        else:
            return False

    # остаток шаблона должен состоять только из '*'
    while j < m and pat[j] == '*':
        j += 1

    return j == m
```

### Сложность

- **Время:** O(n·m) в худшем случае (из-за backtracking), но на практике часто линейный. В худшем случае (много звёздочек) может быть O(n·m).
- **Память:** O(1)

---

## Сравнение подходов

| Подход | Время | Память | Примечание |
|--------|-------|--------|------------|
| DP (O(m) памяти) | O(n·m) | O(m) | Надёжный, простой, гарантирует корректность. |
| Жадный обход | O(n·m) (худший), O(n+m) (средний) | O(1) | Быстрый на практике, но может быть медленным на специальных случаях. |

---

## Примечания

- Жадный алгоритм особенно эффективен, если `*` встречается часто, так как он не требует заполнения таблицы.
- DP-подход предпочтительнее, если важна гарантированная производительность и предсказуемость.
- Символ `?` в обоих подходах обрабатывается одинаково.
- Если строка или шаблон пусты, алгоритмы корректно обрабатывают граничные случаи (проверка длины и обработка `*`).
