## Определение

**Битонная последовательность** — это последовательность, которая сначала строго возрастает, а затем строго убывает. Иными словами, существует пиковый элемент, слева от которого элементы возрастают, а справа — убывают. Математически последовательность `a1, a2, ..., ak` является битонной, если существует индекс `i` (1 < i < k) такой, что:

```
a1 < a2 < ... < ai > ai+1 > ... > ak
```

**Наибольшая битонная подпоследовательность (LBS)** — это подпоследовательность (не обязательно непрерывная) заданного массива, которая является битонной и имеет максимальную длину. Элементы подпоследовательности сохраняют исходный порядок.

**Пример:**
```
arr = [12, 11, 40, 5, 3, 1]
Наибольшая битонная подпоследовательность: [12, 40, 5, 3, 1] (длина 5)
```

---

## Постановка задачи

**Вход:** массив `arr` длины n (n ≥ 1).  
**Выход:** длина наибольшей битонной подпоследовательности. Если битонная подпоследовательность не может быть построена (т.е. нет строгого возрастания и убывания одновременно), вернуть `0`.

---

## Идея решения (динамическое программирование)

Для каждого индекса `i` мы можем рассмотреть его как потенциальный "пик" битонной последовательности. Тогда длина битонной последовательности с пиком в `i` равна:

```
LBS(i) = LIS_ending_at(i) + LDS_starting_at(i) - 1
```

где:
- `LIS_ending_at(i)` — длина наибольшей возрастающей подпоследовательности, заканчивающейся в `i`.
- `LDS_starting_at(i)` — длина наибольшей убывающей подпоследовательности, начинающейся в `i`.

Чтобы последовательность была битонной, обе части должны иметь длину не менее 2? На самом деле, достаточно, чтобы `LIS > 1` и `LDS > 1`, чтобы пик не был крайним (иначе последовательность была бы просто возрастающей или убывающей). В некоторых вариантах допускаются вырожденные случаи (только возрастание или только убывание), но в классической постановке требуется наличие обеих частей.

### Алгоритм

1. Вычислить `leftLIS[i]` — длину LIS, заканчивающейся в `i`. Используем стандартное DP за O(n²):
   ```
   leftLIS[i] = 1 + max(leftLIS[j]) для всех j < i, где arr[j] < arr[i]
   ```

2. Вычислить `rightLDS[i]` — длину LDS, начинающейся в `i`. Аналогично, но проходим справа налево:
   ```
   rightLDS[i] = 1 + max(rightLDS[j]) для всех j > i, где arr[j] < arr[i] (убывание)
   ```

3. Для каждого `i` вычислить `candidate = leftLIS[i] + rightLDS[i] - 1`. Берём максимум среди всех `i`, где `leftLIS[i] > 1` и `rightLDS[i] > 1`.

4. Если максимум остался 0, возвращаем 0 (нет битонной подпоследовательности).

## Реализация на Python

```python
def longest_bitonic_sequence(arr):
    n = len(arr)
    if n < 3:
        return 0

    # LIS слева
    left_lis = [1] * n
    for i in range(1, n):
        for j in range(i):
            if arr[j] < arr[i]:
                left_lis[i] = max(left_lis[i], left_lis[j] + 1)

    # LDS справа
    right_lds = [1] * n
    for i in range(n-2, -1, -1):
        for j in range(n-1, i, -1):
            if arr[j] < arr[i]:   # для убывания: arr[i] > arr[j]
                right_lds[i] = max(right_lds[i], right_lds[j] + 1)

    max_len = 0
    for i in range(n):
        if left_lis[i] > 1 and right_lds[i] > 1:
            max_len = max(max_len, left_lis[i] + right_lds[i] - 1)

    return max_len

# Пример
arr = [12, 11, 40, 5, 3, 1]
print(longest_bitonic_sequence(arr))  # 5
```

---

## Сложность

- **Время:** O(n²) — двойной цикл для LIS и LDS.
- **Память:** O(n) — два массива.

---

## Примечания

- Если массив содержит менее 3 элементов, битонная последовательность невозможна (в классическом определении), поэтому возвращаем 0.
- Можно оптимизировать вычисление LIS и LDS до O(n log n) с использованием бинарного поиска, но это усложнит код и не изменит общую структуру.
- Если допустимы вырожденные случаи (только возрастание или только убывание), то условие `> 1` можно убрать, и тогда ответом будет максимум из всех `leftLIS[i] + rightLDS[i] - 1`, включая случаи, где одна из частей равна 1.
