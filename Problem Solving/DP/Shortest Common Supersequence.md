## Определение

**Кратчайшая общая суперпоследовательность (Shortest Common Supersequence, SCS)** двух строк — это кратчайшая строка, которая содержит обе исходные строки как **подпоследовательности** (не обязательно непрерывно). Задача заключается в нахождении длины такой строки. SCS тесно связана с задачей о наибольшей общей подпоследовательности (LCS), так как общие символы можно включить только один раз.

**Пример:**
```
s1 = "geek", s2 = "eke"
SCS = "geeke" (длина 5)
```

---

## Постановка задачи

**Вход:** две строки `s1` и `s2` (длины m и n).  
**Выход:** длина кратчайшей строки, содержащей обе строки как подпоследовательности.

---

## Идея решения (общая)

Обозначим `scs(i, j)` — длина кратчайшей общей суперпоследовательности для префиксов `s1[0..i-1]` и `s2[0..j-1]`. Рекуррентное соотношение:

- Если `i == 0` → ответ `j` (все символы из `s2`).
- Если `j == 0` → ответ `i` (все символы из `s1`).
- Если `s1[i-1] == s2[j-1]` → `1 + scs(i-1, j-1)`.
- Иначе → `1 + min(scs(i-1, j), scs(i, j-1))`.

Это соотношение можно реализовать через DP. Однако далее мы рассмотрим два эффективных подхода с оптимизацией памяти.

---

## Ожидаемый подход 1: Пространственно-оптимизированное DP (O(n) памяти)

### Идея

В стандартном табличном DP `dp[i][j]` зависит только от:
- `dp[i-1][j-1]` (диагональ),
- `dp[i-1][j]` (сверху),
- `dp[i][j-1]` (слева).

Поэтому для вычисления текущей строки нужны только значения из предыдущей строки и текущей строки слева. Можно хранить два одномерных массива: `prev` (предыдущая строка) и `curr` (текущая строка). После заполнения `curr` становится `prev` для следующей итерации.

### Алгоритм

1. Инициализировать `prev` размером `n+1`, где `prev[j] = j` (базовый случай: если `s1` пуста, то SCS — это вся `s2`).
2. Для `i` от 1 до `m`:
   - `curr[0] = i` (базовый случай: если `s2` пуста, то SCS — вся `s1`).
   - Для `j` от 1 до `n`:
     - Если `s1[i-1] == s2[j-1]`, то `curr[j] = 1 + prev[j-1]`.
     - Иначе `curr[j] = 1 + min(prev[j], curr[j-1])`.
   - После заполнения `curr` присвоить `prev = curr` (создать копию).
3. Вернуть `prev[n]`.

### Реализация

```python
def shortest_common_supersequence_len(s1, s2):
    m, n = len(s1), len(s2)
    prev = [0] * (n + 1)
    for j in range(n + 1):
        prev[j] = j  # если s1 пуста

    for i in range(1, m + 1):
        curr = [0] * (n + 1)
        curr[0] = i  # если s2 пуста
        for j in range(1, n + 1):
            if s1[i-1] == s2[j-1]:
                curr[j] = 1 + prev[j-1]
            else:
                curr[j] = 1 + min(prev[j], curr[j-1])
        prev = curr[:]  # обновляем предыдущую строку

    return prev[n]
```

### Сложность

- **Время:** O(m·n)
- **Память:** O(n)

## Ожидаемый подход 2: Через длину LCS (O(n) памяти)

### Идея

Существует простая связь между длиной SCS и длиной наибольшей общей подпоследовательности (LCS):

```
Длина(SCS) = len(s1) + len(s2) - длина(LCS)
```

Это следует из того, что общие символы (найденные в LCS) можно включить в суперпоследовательность только один раз, а все остальные символы добавляются по одному разу. Поэтому достаточно вычислить длину LCS, а затем применить формулу.

Длину LCS можно вычислить с помощью оптимизированного DP с одним массивом (или двумя строками). Такой подход также требует O(m·n) времени и O(n) памяти.

### Алгоритм

1. Вычислить длину LCS для `s1` и `s2` с использованием двух одномерных массивов:
   - `prev` и `curr` размера `n+1`.
   - Если символы совпадают: `curr[j] = 1 + prev[j-1]`.
   - Иначе: `curr[j] = max(prev[j], curr[j-1])`.
2. После заполнения всех строк `lcs_len = prev[n]`.
3. Ответ: `m + n - lcs_len`.

### Реализация

```python
def shortest_common_supersequence_len_via_lcs(s1, s2):
    m, n = len(s1), len(s2)
    prev = [0] * (n + 1)
    curr = [0] * (n + 1)

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i-1] == s2[j-1]:
                curr[j] = 1 + prev[j-1]
            else:
                curr[j] = max(prev[j], curr[j-1])
        prev, curr = curr[:], [0] * (n + 1)  # сброс curr для следующей строки

    lcs_len = prev[n]
    return m + n - lcs_len
```

**Примечание:** в этом коде `curr` сбрасывается, но можно использовать два массива и менять их местами, как в предыдущем подходе.

---

## Сравнение подходов

| Подход | Время | Память | Основа |
|--------|-------|--------|--------|
| Табличный DP | O(m·n) | O(m·n) | Прямой DP |
| Space Optimized (prev/curr) | O(m·n) | O(n) | Прямой DP с оптимизацией памяти |
| LCS + формула | O(m·n) | O(n) | Использование связи с LCS |

Оба ожидаемых подхода имеют одинаковую сложность, но второй (через LCS) часто проще для понимания, если уже известна задача о LCS.

---

## Примечания

- Если требуется не только длина, но и сама строка SCS, можно модифицировать DP для восстановления (обычно через обратный проход по таблице, но тогда память O(m·n) неизбежна или нужно использовать более сложные техники).
- Связь с LCS делает эту задачу хорошим примером для демонстрации того, как одна задача может быть сведена к другой.
- Входные строки могут содержать любые символы (не только строчные буквы).
- Алгоритмы работают для пустых строк (ответ — длина другой строки).
