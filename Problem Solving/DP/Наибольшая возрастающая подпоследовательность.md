## Определение

**Наибольшая возрастающая подпоследовательность (LIS)** — это подпоследовательность заданной последовательности, в которой элементы строго возрастают, и которая имеет максимально возможную длину. Подпоследовательность не обязана быть непрерывной, но порядок элементов сохраняется. Задача нахождения LIS является классической задачей динамического программирования и имеет множество применений в биологии, информатике и оптимизации.

**Пример:**
```
arr = [3, 10, 2, 1, 20]
LIS: [3, 10, 20] или [2, 10, 20] или [1, 10, 20] — длина 3.
```

---

## Постановка задачи

**Вход:** массив `arr` длины n (целые числа, могут быть отрицательными).  
**Выход:** длина наибольшей строго возрастающей подпоследовательности.

---

## Подход 1: Динамическое программирование (Bottom-Up) — O(n²) времени, O(n) памяти

### Идея

Пусть `dp[i]` — длина LIS, заканчивающейся на элементе с индексом `i`. Изначально `dp[i] = 1` (подпоследовательность из одного элемента). Для каждого `i` мы перебираем все предыдущие индексы `j < i`. Если `arr[j] < arr[i]`, то мы можем продлить подпоследовательность, заканчивающуюся на `j`, добавив `arr[i]`, и обновить `dp[i] = max(dp[i], dp[j] + 1)`. После заполнения `dp` ответом будет максимальное значение в массиве `dp`.

Этот подход прост в реализации и понимании, но имеет квадратичную сложность, что неприемлемо для больших массивов (n > 10⁴).

### Реализация

```python
def lengthOfLIS_dp(arr):
    n = len(arr)
    dp = [1] * n
    for i in range(1, n):
        for j in range(i):
            if arr[j] < arr[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    return max(dp)
```

### Сложность

- **Время:** O(n²)
- **Память:** O(n)

## Подход 2: Бинарный поиск (Patience Sorting) — O(n log n) времени, O(n) памяти

### Идея

Этот алгоритм, также известный как **Patience Sorting**, моделирует процесс, аналогичный раскладыванию карт. Мы поддерживаем вспомогательный массив `tails`, где `tails[i]` — минимально возможное значение последнего элемента возрастающей подпоследовательности длины `i+1`. Массив `tails` всегда отсортирован по возрастанию.

Для каждого элемента `x` из входного массива:

- Если `x` больше последнего элемента `tails`, добавляем `x` в конец `tails` (увеличивая длину LIS).
- Иначе находим позицию `pos` в `tails` первого элемента, который **≥** `x` (нижняя граница, `lower_bound`), и заменяем `tails[pos]` на `x`.

Почему это работает? Замена на меньшее значение сохраняет длину подпоследовательности, но улучшает потенциал для будущих элементов, позволяя построить более длинную возрастающую последовательность.

**Пример:**
```
arr = [3, 10, 2, 1, 20]
tails = []
3 -> tails = [3]
10 -> 10 > 3 -> tails = [3, 10]
2 -> lower_bound([3,10], 2) = 0 -> tails = [2, 10]
1 -> lower_bound([2,10], 1) = 0 -> tails = [1, 10]
20 -> 20 > 10 -> tails = [1, 10, 20]
Ответ: len(tails) = 3
```

### Реализация

```python
import bisect

def lengthOfLIS_binary(arr):
    tails = []
    for x in arr:
        pos = bisect.bisect_left(tails, x)  # lower_bound
        if pos == len(tails):
            tails.append(x)
        else:
            tails[pos] = x
    return len(tails)
```

### Сложность

- **Время:** O(n log n) — бинарный поиск для каждого элемента.
- **Память:** O(n) для массива `tails`.

---

## Сравнение подходов

| Критерий | DP O(n²) | Бинарный поиск O(n log n) |
|----------|----------|---------------------------|
| Время | O(n²) | O(n log n) |
| Память | O(n) | O(n) |
| Простота реализации | Очень прост | Средняя (нужен бинарный поиск) |
| Восстановление последовательности | Возможно (с доп. массивом) | Требуется дополнительная обработка |
| Применимость | Для небольших n (≤ 10⁴) | Для больших n (до 10⁵–10⁶) |

---

## Восстановление самой подпоследовательности

Если требуется не только длина, но и сама последовательность, можно использовать DP с запоминанием предков (`parent`). Для бинарного поиска восстановление сложнее, но возможно с использованием дополнительных массивов. Обычно на практике достаточно длины.

---

## Примечания

- LIS требует строгого возрастания. Для нестрогого возрастания (неубывания) нужно использовать `bisect_right` вместо `bisect_left`.
- Алгоритм с бинарным поиском не даёт саму подпоследовательность, только её длину. Для восстановления можно хранить позиции и использовать обратный проход.
- Существуют также варианты для поиска LIS в двумерных массивах (например, для прямоугольников) с использованием сортировки и дерева отрезков.
