## Определение

Дано `n` ступенек. Человек может подниматься на 1 или 2 ступеньки за раз. Требуется подсчитать количество различных способов достичь вершины (порядок шагов важен). Это классическая задача, решение которой выражается через числа Фибоначчи.

**Пример:**
```
n = 1 → 1 способ
n = 2 → 2 способа: [1,1], [2]
n = 3 → 3 способа: [1,1,1], [1,2], [2,1]
n = 4 → 5 способов: [1,1,1,1], [1,1,2], [1,2,1], [2,1,1], [2,2]
```

---

## Постановка задачи

**Вход:** целое число `n` (1 ≤ n ≤ 40 в задаче, но подход работает для любых n).
**Выход:** количество способов добраться до n-й ступеньки.

---

## Идея решения

Пусть `f(n)` — количество способов достичь ступеньки `n`. Чтобы попасть на ступеньку `n`, можно либо прийти с `n-1` (сделав шаг длины 1), либо с `n-2` (сделав шаг длины 2). Таким образом, получаем рекуррентное соотношение:

```
f(n) = f(n-1) + f(n-2)
```

с начальными условиями:
- `f(1) = 1`
- `f(2) = 2` (способы: [1,1] и [2])

Это соотношение совпадает с числами Фибоначчи, но со сдвигом: `f(n) = Fib(n+1)`, где `Fib(1)=1, Fib(2)=1`.

Обычное решение — использовать динамическое программирование O(n). Однако для очень больших `n` (например, до 10^9) требуется более быстрый метод. **Возведение матрицы в степень** позволяет вычислить `f(n)` за O(log n).

### Матричное представление

Систему можно записать в матричном виде:

```
[ f(n)   ]   = [1 1] * [f(n-1)]
[ f(n-1) ]     [1 0]   [f(n-2)]
```

Тогда:

```
[ f(n)   ]   = M^(n-2) * [f(2)]
[ f(n-1) ]             [f(1)]
```

где `M = [[1, 1], [1, 0]]`.

Для `n=1` или `n=2` ответы известны; для `n >= 3` вычисляем `M^(n-2)` и умножаем на вектор начальных условий. Альтернативно, можно вычислить `M^(n-1)` и взять сумму первой строки (как в коде).

---

## Реализация на Python (с матричным возведением в степень)

```python
class Solution:
    def multiply(self, A, B):
        m, n, p = len(A), len(A[0]), len(B[0])
        res = [[0] * p for _ in range(m)]
        for i in range(m):
            for k in range(n):
                if A[i][k]:
                    for j in range(p):
                        res[i][j] += A[i][k] * B[k][j]
        for i in range(m):
            for j in range(p):
                A[i][j] = res[i][j]

    def power(self, mat, exp):
        # единичная матрица 2x2
        result = [[1, 0], [0, 1]]
        base = [row[:] for row in mat]
        while exp > 0:
            if exp & 1:
                self.multiply(result, base)
            self.multiply(base, base)
            exp >>= 1
        return result

    def countWays(self, n: int) -> int:
        if n == 0 or n == 1:
            return 1
        M = [[1, 1], [1, 0]]
        powM = self.power(M, n - 1)
        # результат: f(n) = powM[0][0] + powM[0][1] (потому что M^(n-1) * [1,0] дает первую строку)
        return powM[0][0] + powM[0][1]
```

## Сложность

- **Время:** O(log n) — бинарное возведение в степень матрицы 2×2.
- **Память:** O(1) — используются только несколько матриц 2×2.

---

## Примечания

- Для малых `n` (≤ 40) проще использовать DP, но матричный метод демонстрирует мощь для больших `n`.
- Числа Фибоначчи растут экспоненциально, поэтому для `n` около 10^9 результат может быть огромным (выходить за пределы 64-битных целых), но в Python целые не ограничены.
- Задача также может быть решена через формулу Бине, но матричный метод более универсален для других рекуррентных соотношений.
