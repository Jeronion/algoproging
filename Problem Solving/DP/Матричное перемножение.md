## Определение

**Задача о порядке перемножения матриц (Matrix Chain Multiplication)** заключается в нахождении минимального количества скалярных умножений, необходимых для вычисления произведения цепочки матриц: `A1 * A2 * ... * An`.

Матрицы можно перемножать в любом порядке (расставляя скобки), так как умножение матриц ассоциативно. Результат от порядка не зависит, но количество операций может сильно различаться.

**Пример:**
Даны матрицы:
- `A1`: 10 × 20
- `A2`: 20 × 5
- `A3`: 5 × 15

Оптимальный порядок: `(A1 * A2) * A3` → 10·20·5 + 10·5·15 = 1000 + 750 = 1750 умножений.  
Порядок `A1 * (A2 * A3)` дал бы 20·5·15 + 10·20·15 = 1500 + 3000 = 4500.

---

## Постановка задачи

**Вход:** массив `p` размера `n+1`, где `p[i]` — количество строк в матрице `A_i` (для `i` от 0 до n-1), а `p[i+1]` — количество столбцов матрицы `A_i`. Матрицы считаются совместимыми: `A_i` имеет размер `p[i] × p[i+1]`, поэтому её столбцы равны строкам следующей матрицы `p[i+1] = p[i+1]`.

**Выход:** минимальное количество скалярных умножений.

**Почему не нужна проверка совместимости?**  
Входные данные гарантируют, что все матрицы можно перемножить. Массив `p` строится именно так: для каждой матрицы `i` размеры `p[i]` и `p[i+1]`. Следовательно, размеры согласованы автоматически. При умножении двух частей, полученных после разбиения в точке `k`, левая часть имеет размер `p[i] × p[k]`, правая — `p[k] × p[j]`. Они совместимы, потому что у левой столбцы равны `p[k]`, у правой строки равны `p[k]`. Формула `p[i] * p[k] * p[j]` учитывает количество умножений при перемножении этих двух матриц.

---

## Идея решения (динамическое программирование)

Обозначим `dp[i][j]` — минимальное количество умножений для перемножения матриц с `i`-й по `j`-ю (включительно). Базовый случай: `dp[i][i] = 0` (одна матрица, умножать нечего).

Рекуррентное соотношение: для каждого `k` от `i` до `j-1` мы можем разбить цепочку на две части: `[i..k]` и `[k+1..j]`. Тогда общая стоимость = стоимость левой части + стоимость правой части + стоимость умножения двух получившихся матриц.

Стоимость умножения левой части (размер `p[i] × p[k]`) и правой (`p[k] × p[j]`) равна `p[i] * p[k] * p[j]`.

Таким образом:
```
dp[i][j] = min_{k = i..j-1} ( dp[i][k] + dp[k+1][j] + p[i] * p[k] * p[j] )
```

Ответ: `dp[0][n-1]`.

## Алгоритм (Bottom-Up Tabulation)

Мы заполняем таблицу по возрастанию длины отрезка `len = j - i + 1`. Начинаем с `len = 2` (две матрицы), затем `len = 3` и т.д. до `len = n`. Это гарантирует, что все подзадачи меньшей длины уже вычислены.

### Реализация на Python

```python
def matrix_chain_order(p):
    n = len(p) - 1          # количество матриц
    dp = [[0] * n for _ in range(n)]

    # length — количество матриц в цепочке
    for length in range(2, n + 1):
        for i in range(n - length + 1):
            j = i + length - 1
            dp[i][j] = float('inf')
            for k in range(i, j):
                cost = dp[i][k] + dp[k+1][j] + p[i] * p[k+1] * p[j+1]
                # Обратите внимание: в коде из запроса используется p[i]*p[k]*p[j] с другой индексацией.
                # Здесь индексы: p[i] — строки Ai, p[k+1] — столбцы Ak (и строки A_{k+1}), p[j+1] — столбцы Aj.
                # В запросе использовали массив p размером n (не n+1) и формулы чуть отличались, но суть та же.
                dp[i][j] = min(dp[i][j], cost)
    return dp[0][n-1]
```

**Пояснение индексов (для массива p длиной n+1):**
- `p[i]` — строки матрицы `i` (0-индексация)
- `p[i+1]` — столбцы матрицы `i` (и строки матрицы `i+1`)
- При разбиении в точке `k` (где `i ≤ k < j`):
  - Левая часть: матрицы `i..k` → результат размером `p[i] × p[k+1]`
  - Правая часть: матрицы `k+1..j` → результат размером `p[k+1] × p[j+1]`
  - Стоимость умножения: `p[i] * p[k+1] * p[j+1]`

В коде из запроса использовался массив `arr` длины n (без +1) и формула `arr[i] * arr[k] * arr[j]`. Это эквивалентно, если считать, что `arr[i]` — это `p[i]`, а `arr[j]` — `p[j+1]`. В любом случае логика та же.

## Пример

`p = [2, 1, 3, 4]` → матрицы:
- A1: 2×1
- A2: 1×3
- A3: 3×4

Заполним `dp`:

- `dp[0][1] = 2*1*3 = 6`
- `dp[1][2] = 1*3*4 = 12`
- Для `dp[0][2]`: k=0 → `dp[0][0] + dp[1][2] + 2*1*4 = 0 + 12 + 8 = 20`; k=1 → `dp[0][1] + dp[2][2] + 2*3*4 = 6 + 0 + 24 = 30` → минимум = 20.

Ответ: 20.

---

## Сложность

- **Время:** O(n³) — три вложенных цикла.
- **Память:** O(n²) — таблица `dp`.

---

## Примечания

- Задача часто используется для иллюстрации **интервального DP**.
- Если матрицы несовместимы, задача не имеет смысла; на практике входные данные всегда корректны.
- Существует оптимизация до O(n²) с использованием свойства четырёхугольного неравенства (оптимизация Кнута), но это выходит за рамки базового решения.
