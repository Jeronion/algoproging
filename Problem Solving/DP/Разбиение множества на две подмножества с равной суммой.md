## Определение

**Задача о разбиении** (Partition Problem) — это задача, в которой дан массив целых чисел. Требуется определить, можно ли разбить его на два подмножества с **равной суммой** элементов. Каждый элемент должен принадлежать ровно одному подмножеству. По сути, это частный случай задачи о сумме подмножества (Subset Sum) с целью `target = sum(arr) / 2`.

**Пример:**
```
arr = [1, 5, 11, 5] → true (можно разбить на {1, 5, 5} и {11})
arr = [1, 5, 3] → false (сумма 9 нечётна или не достижима)
```

---

## Постановка задачи

**Вход:** массив целых чисел `arr` (длина n, значения могут быть положительными).  
**Выход:** `true`, если массив можно разбить на две части с одинаковой суммой, иначе `false`.

---

## Идея решения

Обозначим общую сумму `total = sum(arr)`. Если `total` нечётна, ответ сразу `false`, так как сумму нельзя разделить на две равные целые части. Иначе задача сводится к поиску подмножества с суммой `target = total // 2`. Если такое подмножество существует, то оставшиеся элементы автоматически дадут вторую половину.

Это классическая задача о сумме подмножества (Subset Sum) с ограничением на точное равенство `target`. Решается методами динамического программирования.

---

## Подходы к решению

### 1. Наивная рекурсия (экспоненциальная)

Перебираем все подмножества (O(2ⁿ)), проверяя, есть ли сумма `target`. Неэффективно для больших n.

### 2. Рекурсия с мемоизацией (Top‑Down DP)

Используем `memo[i][sum]`, где `i` — количество рассмотренных элементов, `sum` — текущая накопленная сумма. Запоминаем результаты для ускорения. Сложность O(n·target) по времени и O(n·target) по памяти.

### 3. Табличное DP (Bottom‑Up)

Строим таблицу `dp[i][j]` — можно ли получить сумму `j` из первых `i` элементов. Базовый случай: `dp[i][0] = true`. Переход: `dp[i][j] = dp[i-1][j] or (j >= arr[i-1] and dp[i-1][j-arr[i-1]])`. Сложность O(n·target), память O(n·target).

### 4. Пространственно-оптимизированное DP (ожидаемый подход)

Заметим, что для вычисления текущей строки нам нужна только предыдущая строка. Используем два одномерных массива `prev` и `curr` (или даже один массив с обновлением справа налево). Это снижает память до O(target), сохраняя время O(n·target).

---

## Реализация (оптимизированная по памяти)

В приведённом решении используется два массива `prev` и `curr` для хранения достижимости сумм. На каждом шаге обновляем `curr` на основе `prev`, затем копируем.

```python
class Solution:
    def equalPartition(self, arr):
        total = sum(arr)
        # Если сумма нечётна, разбиение невозможно
        if total % 2 != 0:
            return False
        
        target = total // 2
        n = len(arr)
        
        # prev[j] — можно ли получить сумму j из предыдущих элементов
        prev = [False] * (target + 1)
        curr = [False] * (target + 1)
        prev[0] = True  # пустое подмножество даёт сумму 0
        
        for i in range(1, n + 1):
            for j in range(target + 1):
                if j < arr[i-1]:
                    curr[j] = prev[j]          # не берём элемент
                else:
                    curr[j] = prev[j] or prev[j - arr[i-1]]  # не берём или берём
            prev = curr[:]   # копируем текущую строку для следующей итерации
        
        return prev[target]
```

**Альтернативная оптимизация с одним массивом (обновление справа налево):**

```python
def equalPartition(arr):
    total = sum(arr)
    if total % 2:
        return False
    target = total // 2
    dp = [False] * (target + 1)
    dp[0] = True
    for x in arr:
        for j in range(target, x - 1, -1):
            if dp[j - x]:
                dp[j] = True
    return dp[target]
```

Этот вариант использует только один массив и обновляет его в обратном порядке, чтобы не использовать один элемент несколько раз.

---

## Сложность

- **Время:** O(n · target), где target = sum(arr)/2. В худшем случае target может быть до 10⁵–10⁶ (в зависимости от ограничений).
- **Память:** O(target) для оптимизированной версии (один или два массива).

---

## Примечания

- Задача является NP-полной в общем случае, но для небольших значений суммы эффективно решается DP.
- Если сумма нечётна, сразу возвращаем `False`.
- Алгоритм работает только для положительных чисел (в условии обычно все числа положительные).
- Код использует два массива для наглядности, но можно использовать один массив с обратным обходом, что ещё экономит память.
