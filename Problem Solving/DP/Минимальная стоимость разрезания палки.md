## Определение

**Задача о разрезании палки** — это классическая задача интервального динамического программирования. Дана палка длиной `n` и список позиций `cuts`, где её нужно разрезать. Разрезы можно выполнять в любом порядке. Стоимость разрезания отрезка палки равна его длине. Требуется найти минимальную суммарную стоимость, чтобы выполнить все разрезы.

Эта задача является аналогом задачи **Matrix Chain Multiplication** (минимальное количество умножений) и часто используется для иллюстрации интервального DP. Она имеет множество приложений в оптимизации распиловки материалов, планировании производства и обработке дерева.

**Пример:**
```
n = 7, cuts = [1, 3, 4]
Оптимальный порядок: разрезать сначала по 3 (стоимость = 7), затем по 1 (стоимость = 3), затем по 4 (стоимость = 4) → общая стоимость = 14.
```

---

## Постановка задачи

**Входные данные:**
- `n` — длина палки (целое положительное число).
- `cuts` — массив целых чисел, обозначающих позиции разрезов (все позиции строго между 0 и n, могут быть не отсортированы).

**Выход:** минимальная суммарная стоимость выполнения всех разрезов.

**Ограничения:** типичные: `1 ≤ n ≤ 10⁶`, количество разрезов `k ≤ 100` (в некоторых задачах до 10⁵, но DP O(k³) работает только для небольших k). В GeeksforGeeks обычно `k ≤ 100`.

---

## Идея решения (интервальное DP)

Пусть мы добавим фиктивные разрезы на концах палки: 0 и n. Тогда мы можем рассматривать задачу как выбор порядка разрезания отрезков между этими точками.

Обозначим `dp[i][j]` — минимальная стоимость разрезания отрезка палки от позиции `cuts[i]` до `cuts[j]` (включительно), где `cuts` — массив всех точек разреза (включая 0 и n). Базовый случай: если `j == i+1` (между двумя соседними точками нет разрезов), стоимость равна 0.

Для отрезка `[i, j]` (длина > 1) мы должны сделать хотя бы один разрез внутри. Пусть первым разрезом будет в позиции `k` (где `i < k < j`). Тогда стоимость этого разреза равна длине отрезка: `cuts[j] - cuts[i]`. После этого мы должны разрезать левую часть `[i, k]` и правую `[k, j]` оптимальным образом. Таким образом, рекуррентное соотношение:

```
dp[i][j] = min_{k = i+1 .. j-1} (
    dp[i][k] + dp[k][j] + (cuts[j] - cuts[i])
)
```

Мы вычисляем `dp` для всех отрезков, увеличивая их длину. Ответ: `dp[0][m-1]`, где `m` — количество точек в `cuts` (включая 0 и n).

---

## Алгоритм (Bottom-Up Tabulation)

1. Добавить 0 и n в массив `cuts` и отсортировать его.
2. Обозначить `m = len(cuts)`.
3. Создать таблицу `dp` размером `m × m`, инициализированную нулями.
4. Для `length` от 2 до `m-1` (длина отрезка в терминах количества точек, т.е. количество отрезков между точками):
   - Для `left` от 0 до `m - length - 1`:
     - `right = left + length`.
     - Инициализировать `min_cost = ∞`.
     - Для `k` от `left + 1` до `right - 1`:
       - `cost = dp[left][k] + dp[k][right] + (cuts[right] - cuts[left])`.
       - `min_cost = min(min_cost, cost)`.
     - `dp[left][right] = min_cost`.
5. Вернуть `dp[0][m-1]`.

---

## Реализация на Python

```python
def min_cut_cost(n, cuts):
    # Добавляем границы
    cuts.append(0)
    cuts.append(n)
    cuts.sort()
    m = len(cuts)

    # dp[i][j] — мин. стоимость разрезания отрезка от cuts[i] до cuts[j]
    dp = [[0] * m for _ in range(m)]

    # length — количество отрезков между точками (не длина палки)
    for length in range(2, m):
        for left in range(m - length):
            right = left + length
            min_cost = float('inf')
            # Перебираем первый разрез
            for k in range(left + 1, right):
                cost = dp[left][k] + dp[k][right] + (cuts[right] - cuts[left])
                if cost < min_cost:
                    min_cost = cost
            dp[left][right] = min_cost

    return dp[0][m - 1]
```

---

## Сложность

- **Время:** O(k³), где `k` — количество точек разреза (включая границы). Для k ≤ 100 это приемлемо.
- **Память:** O(k²) для хранения таблицы `dp`.

---

## Примечания

- Задача известна как **Minimum Cost to Cut a Stick** (LeetCode 1547) и на GeeksforGeeks (Minimum Cost to Cut a Board into Squares — но там другая).
- Если количество разрезов велико (> 200), O(k³) не подходит, но обычно в таких задачах k ≤ 100.
- Альтернативный подход: можно использовать мемоизацию (рекурсия + кэш), но табуляция проще и быстрее.
- Порядок разрезов не влияет на конечный результат, но влияет на стоимость; задача ищет оптимальный порядок.
- Аналогия с Matrix Chain Multiplication: там стоимость — умножение матриц, здесь — длина отрезка.
