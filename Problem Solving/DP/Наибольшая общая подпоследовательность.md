## Определение

**Наибольшая общая подпоследовательность (LCS)** двух строк — это самая длинная последовательность символов, которая может быть получена из обеих строк путём удаления некоторых (возможно, ни одного) символов без изменения порядка оставшихся. Например, для строк `"AGGTAB"` и `"GXTXAYB"` LCS равна `"GTAB"` длиной 4 (или `"GXAB"`? на самом деле `"GTAB"`). Это классическая задача динамического программирования.

---

## Постановка задачи

Даны две строки `S1` и `S2` длины `m` и `n` соответственно. Требуется найти длину их наибольшей общей подпоследовательности.

**Пример:**
```
S1 = "AGGTAB"
S2 = "GXTXAYB"
LCS = "GTAB" (длина 4)
```

---

## Идея решения (динамическое программирование)

Пусть `dp[i][j]` — длина LCS для префиксов `S1[0..i-1]` и `S2[0..j-1]`. Рекуррентное соотношение:

- Если `S1[i-1] == S2[j-1]`, то `dp[i][j] = 1 + dp[i-1][j-1]`.
- Иначе `dp[i][j] = max(dp[i-1][j], dp[i][j-1])`.

Базовые случаи: `dp[0][j] = 0`, `dp[i][0] = 0`.

Ответ — `dp[m][n]`.

Это соотношение основано на проверке последних символов: если они равны, мы включаем их в LCS и решаем подзадачу для оставшихся префиксов. Если не равны, то LCS либо не использует последний символ первой строки, либо последний символ второй.

## Реализации

### 1. Классическое DP (таблица) — O(m·n) времени, O(m·n) памяти

```python
def lcs_table(S1, S2):
    m, n = len(S1), len(S2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if S1[i-1] == S2[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    return dp[m][n]
```

### 2. Оптимизация по памяти (две строки) — O(m·n) времени, O(min(m,n)) памяти

Поскольку для вычисления текущей строки нужна только предыдущая, можно хранить только две строки.

```python
def lcs_two_rows(S1, S2):
    m, n = len(S1), len(S2)
    # Чтобы минимизировать память, можно сделать m <= n, но здесь просто две строки
    prev = [0] * (n + 1)
    curr = [0] * (n + 1)
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if S1[i-1] == S2[j-1]:
                curr[j] = prev[j-1] + 1
            else:
                curr[j] = max(prev[j], curr[j-1])
        prev, curr = curr, [0] * (n + 1)  # обновляем предыдущую строку и сбрасываем curr
    return prev[n]
```

Можно использовать `prev = curr[:]` вместо создания нового списка, но это увеличит время. Вместо этого используем два списка и меняем их местами.

### 3. Дальнейшая оптимизация (один массив) — O(m·n) времени, O(min(m,n)) памяти

Используем один массив `dp` размером `(n+1)`, где `dp[j]` хранит значение для текущей строки. Для вычисления нужен `prev` (диагональное значение).

```python
def lcs_one_array(S1, S2):
    m, n = len(S1), len(S2)
    # Чтобы уменьшить память, можно убедиться, что m >= n, но алгоритм работает и так.
    dp = [0] * (n + 1)
    for i in range(1, m + 1):
        prev = dp[0]  # dp[0] всегда 0
        for j in range(1, n + 1):
            temp = dp[j]  # сохраняем старое dp[j] (оно будет dp[i-1][j] после обновления)
            if S1[i-1] == S2[j-1]:
                dp[j] = prev + 1
            else:
                dp[j] = max(dp[j-1], dp[j])
            prev = temp
    return dp[n]
```

**Пояснение:** `dp[j]` до обновления содержит значение `dp[i-1][j]` (с предыдущей строки). `dp[j-1]` уже обновлён на текущей строке, поэтому `dp[j-1]` — это `dp[i][j-1]`. `prev` хранит диагональное значение `dp[i-1][j-1]` (из предыдущей итерации, до обновления). При совпадении используем `prev + 1`, иначе `max(dp[j-1], dp[j])`.

## Сложность

| Подход | Время | Память |
|--------|-------|--------|
| Таблица | O(m·n) | O(m·n) |
| Две строки | O(m·n) | O(min(m,n)) |
| Один массив | O(m·n) | O(min(m,n)) |

На практике для минимизации памяти обычно используют подход с одним массивом, так как он наиболее эффективен.

---

## Примечания

- LCS находит **длину**, а не саму последовательность. Для восстановления последовательности потребуется дополнительная логика (обратный проход по таблице или хранение родительских ссылок).
- Если строки очень длинные (например, > 10⁵), O(m·n) может быть слишком медленным. Существуют более быстрые алгоритмы (например, алгоритм Хантера-Ширра), но они сложнее.
- Задача LCS тесно связана с **редакционным расстоянием** (Levenshtein distance) и **наибольшей общей подстрокой**.
- Использование одного массива требует аккуратного сохранения `prev`, так как значение перезаписывается.
