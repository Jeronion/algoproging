## Определение

**Задача о самом длинном возрастающем пути в матрице** заключается в поиске максимальной длины пути, проходящего по ячейкам матрицы, где значения строго возрастают при переходе от одной ячейки к соседней (по четырём направлениям: вверх, вниз, влево, вправо). Каждая ячейка может быть использована не более одного раза в пути (что автоматически выполняется при строгом возрастании, так как циклы невозможны). Требуется найти длину самого длинного такого пути в данной матрице.

**Пример:**
```
matrix = [[1, 2, 3],
          [4, 5, 6],
          [7, 8, 9]]
Самый длинный возрастающий путь: 1→2→3→6→9 (длина 5).
```

---

## Постановка задачи

**Вход:** матрица `matrix` размера `n × m` (1 ≤ n, m ≤ 500), содержащая целые числа (положительные или отрицательные, но в большинстве примеров положительные).  
**Выход:** длина самого длинного строго возрастающего пути (количество ячеек в пути).

Движение разрешено только в четырёх направлениях (вверх, вниз, влево, вправо). Путь должен быть строго возрастающим (каждое следующее значение больше предыдущего). Переходы по диагонали запрещены.

---

## Идея решения (DFS с мемоизацией)

Наивный рекурсивный перебор из каждой ячейки приводит к экспоненциальной сложности, так как пути пересекаются. Однако задача обладает свойствами **оптимальной подструктуры** и **перекрывающихся подзадач**, что позволяет применить динамическое программирование.

Для каждой ячейки `(i, j)` мы можем вычислить длину самого длинного возрастающего пути, начинающегося в этой ячейке. Обозначим её `dp[i][j]`. Тогда:

```
dp[i][j] = 1 + max(dp[neighbor]) для всех соседей, у которых значение > matrix[i][j].
```

Если таких соседей нет, `dp[i][j] = 1`.

Поскольку путь строго возрастающий, граф направлен от меньших значений к большим, что гарантирует отсутствие циклов. Мы можем вычислить `dp` с помощью DFS с мемоизацией (рекурсивно, сохраняя результаты в отдельной матрице). При посещении ячейки, если её значение уже вычислено, возвращаем его; иначе запускаем рекурсивный обход всех четырёх соседей, которые больше текущей, и выбираем максимум.

---

## Алгоритм (пошагово)

1. Создать матрицу `memo` размера `n × m`, заполненную `-1` (означает, что значение ещё не вычислено).
2. Для каждой ячейки `(i, j)` вызвать рекурсивную функцию `dfs(i, j)`:
   - Если `memo[i][j] != -1`, вернуть сохранённое значение.
   - Инициализировать `best = 1`.
   - Для каждого из четырёх соседей `(ni, nj)`:
     - Если сосед в пределах матрицы и `matrix[ni][nj] > matrix[i][j]`:
       - `best = max(best, 1 + dfs(ni, nj))`
   - Сохранить `memo[i][j] = best` и вернуть `best`.
3. Ответ — максимум среди всех `dfs(i, j)`.

---

## Реализация

```python
class Solution:
    def longestIncreasingPath(self, matrix):
        if not matrix or not matrix[0]:
            return 0
        n, m = len(matrix), len(matrix[0])
        memo = [[-1] * m for _ in range(n)]
        directions = [(0, 1), (1, 0), (0, -1), (-1, 0)]
        
        def dfs(i, j):
            if memo[i][j] != -1:
                return memo[i][j]
            best = 1
            for dx, dy in directions:
                ni, nj = i + dx, j + dy
                if 0 <= ni < n and 0 <= nj < m and matrix[ni][nj] > matrix[i][j]:
                    best = max(best, 1 + dfs(ni, nj))
            memo[i][j] = best
            return best
        
        ans = 0
        for i in range(n):
            for j in range(m):
                ans = max(ans, dfs(i, j))
        return ans
```

---

## Сложность

- **Время:** O(n·m) — каждая ячейка обрабатывается ровно один раз, а для каждой из них проверяется 4 соседа.
- **Память:** O(n·m) для матрицы мемоизации плюс глубина рекурсии (в худшем случае O(n·m) для цепочки, но обычно меньше).

## Альтернативный подход (Kahn's algorithm, топологическая сортировка)

Можно представить матрицу как ориентированный граф, где ребро направлено от меньшего значения к большему. Поскольку значения строго возрастают, граф ациклический. Тогда задача сводится к поиску самого длинного пути в DAG, что можно решить через топологическую сортировку (алгоритм Кана).

1. Вычислить входящие степени для каждой ячейки (количество соседей с меньшим значением).
2. Поместить все ячейки с in-degree = 0 в очередь.
3. Проводить BFS уровень за уровнем: на каждом уровне увеличивать длину пути на 1. При обработке ячейки уменьшать in-degree её больших соседей; если in-degree становится 0, добавлять соседа в очередь.
4. Количество уровней — это длина самого длинного пути.

Этот подход также работает за O(n·m) времени и O(n·m) памяти, но требует дополнительной матрицы in-degree и очереди.

```python
from collections import deque

def longestIncreasingPathKahn(matrix):
    n, m = len(matrix), len(matrix[0])
    directions = [(0,1), (1,0), (0,-1), (-1,0)]
    indeg = [[0]*m for _ in range(n)]
    for i in range(n):
        for j in range(m):
            for dx, dy in directions:
                ni, nj = i+dx, j+dy
                if 0 <= ni < n and 0 <= nj < m and matrix[ni][nj] < matrix[i][j]:
                    indeg[i][j] += 1
    q = deque()
    for i in range(n):
        for j in range(m):
            if indeg[i][j] == 0:
                q.append((i, j))
    ans = 0
    while q:
        ans += 1
        for _ in range(len(q)):
            i, j = q.popleft()
            for dx, dy in directions:
                ni, nj = i+dx, j+dy
                if 0 <= ni < n and 0 <= nj < m and matrix[ni][nj] > matrix[i][j]:
                    indeg[ni][nj] -= 1
                    if indeg[ni][nj] == 0:
                        q.append((ni, nj))
    return ans
```

## Примечания

- Оба подхода дают одинаковый результат; выбор зависит от предпочтений. DFS с мемоизацией более интуитивен для рекурсивного мышления, а алгоритм Кана является итеративным и хорошо иллюстрирует применение топологической сортировки.
- Если матрица содержит равные значения, они не считаются возрастающими, поэтому переходы между равными числами запрещены.
- В задачах с большими матрицами (например, 1000×1000) рекурсивный DFS может переполнить стек, поэтому алгоритм Кана (BFS) является предпочтительным с точки зрения безопасности.
- Движение по диагонали не разрешено, что упрощает проверку соседей.

## Связь с другими задачами

- **Longest Increasing Subsequence** (LIS) — одномерный аналог, но там последовательность не обязательно соседняя.
- **Maximum Path Sum in Matrix** — похожая задача, но там максимизируется сумма, а не длина.
- **Word Search** — поиск слова в матрице с использованием DFS.
- **Graph Longest Path in DAG** — общая задача, которую можно решить через топологическую сортировку.

## Заключение

Задача о самом длинном возрастающем пути в матрице является классическим примером применения динамического программирования на графах, заданных неявно (соседство в сетке). Благодаря свойству ацикличности (строгое возрастание), рекурсивное решение с мемоизацией работает эффективно. Альтернативный подход через топологическую сортировку демонстрирует мощь теории графов для решения задач на матрицах. Оба метода имеют линейную сложность и подходят для размеров до нескольких сотен или тысяч элементов.
