## Определение

**Задача о покупке и продаже акций с комиссией за транзакцию** — это вариация классической задачи на торговлю акциями, где трейдер может совершать неограниченное количество транзакций, но каждая продажа облагается фиксированной комиссией `fee`. Цель — максимизировать общую прибыль. Эта задача является важным шагом между неограниченными транзакциями (без комиссии) и более сложными ограничениями.

**Пример:**
```
prices = [1, 3, 2, 8, 4, 9], fee = 2
Максимальная прибыль: 8
Объяснение: buy at 1, sell at 8 (прибыль 7), затем buy at 4, sell at 9 (прибыль 5) → 7+5=12? Нет, с учётом комиссии: 8-1-2=5, 9-4-2=3, итого 8.
```

---

## Постановка задачи

**Вход:**
- `prices` — массив цен акций (длина n).
- `fee` — комиссия за одну транзакцию (при продаже).

**Выход:** максимальная прибыль, которую можно получить, совершая неограниченное количество транзакций, при условии, что:
- Нельзя владеть более чем одной акцией одновременно.
- При каждой продаже уплачивается комиссия `fee`.

**Ограничения:** 1 ≤ n ≤ 10⁵, 0 ≤ prices[i] ≤ 10⁴, 0 ≤ fee ≤ 10⁴.

---

## Идея решения

Используется **динамическое программирование с конечными состояниями**. В каждый день мы можем находиться в одном из двух состояний:
- `buy` (или `noStock`) — мы не владеем акцией и можем купить.
- `sell` (или `inHand`) — мы владеем акцией и можем продать.

Рекуррентные соотношения (с конца к началу):
- Если мы не владеем акцией (`buy`):
  ```
  dp[i][buy] = max(dp[i+1][buy], dp[i+1][sell] - prices[i])
  ```
  (пропустить день или купить сегодня)
- Если мы владеем акцией (`sell`):
  ```
  dp[i][sell] = max(dp[i+1][sell], prices[i] - fee + dp[i+1][buy])
  ```
  (пропустить день или продать сегодня с вычетом комиссии)

Базовый случай: за пределами массива прибыль равна 0.

Ответ: `dp[0][buy]` — максимальная прибыль с первого дня, когда мы не владеем акцией.

---

## Подход 1: Табличный Bottom-Up DP (O(n) времени, O(n) памяти)

### Идея

Строим таблицу `dp` размера `(n+1) × 2`, где `dp[i][0]` соответствует состоянию `buy` (нет акции), а `dp[i][1]` — состоянию `sell` (есть акция). Заполняем с конца к началу.

### Реализация

```python
def maxProfit(prices, fee):
    n = len(prices)
    dp = [[0] * 2 for _ in range(n + 1)]
    
    for i in range(n - 1, -1, -1):
        # состояние "нет акции" (можем купить)
        dp[i][0] = max(dp[i + 1][0], dp[i + 1][1] - prices[i])
        # состояние "есть акция" (можем продать)
        dp[i][1] = max(dp[i + 1][1], prices[i] - fee + dp[i + 1][0])
    
    return dp[0][0]
```

### Сложность

- **Время:** O(n)
- **Память:** O(n)

## Подход 2: Пространственно-оптимизированный (O(n) времени, O(1) памяти) — ожидаемый подход

### Идея

Заметим, что для вычисления `dp[i]` нужны только значения из `dp[i+1]`. Поэтому можно обойтись двумя переменными:
- `noStock` — максимальная прибыль, когда мы не владеем акцией (аналог `dp[i][0]`).
- `inHand` — максимальная прибыль, когда мы владеем акцией (аналог `dp[i][1]`).

Идём с конца к началу и обновляем эти переменные:

```
newNoStock = max(noStock, inHand - prices[i])
newInHand = max(inHand, prices[i] - fee + noStock)
```

После обновления присваиваем новые значения.

### Алгоритм

1. Инициализировать `noStock = 0`, `inHand = 0` (на последний день прибыль 0).
2. Для `i` от `n-1` до `0`:
   - `newNoStock = max(noStock, inHand - prices[i])`
   - `newInHand = max(inHand, prices[i] - fee + noStock)`
   - `noStock, inHand = newNoStock, newInHand`
3. Вернуть `noStock`.

### Реализация

```python
def maxProfit(prices, fee):
    noStock = 0
    inHand = 0
    
    for i in range(len(prices) - 1, -1, -1):
        newNoStock = max(noStock, inHand - prices[i])
        newInHand = max(inHand, prices[i] - fee + noStock)
        noStock, inHand = newNoStock, newInHand
    
    return noStock
```

### Сложность

- **Время:** O(n)
- **Память:** O(1)

---

## Сравнение подходов

| Подход | Время | Память | Примечание |
|--------|-------|--------|------------|
| Табличный DP | O(n) | O(n) | Простой для понимания, но тратит память |
| Оптимизированный | O(n) | O(1) | **Рекомендуется** для больших массивов |

---

## Примечания

- Комиссия вычитается при продаже, поэтому в состоянии `sell` используется `prices[i] - fee`.
- Оба подхода корректно обрабатывают случай, когда выгоднее не совершать транзакции (если комиссия слишком высока).
- Порядок обновления переменных в оптимизированном подходе важен: мы используем старые значения `noStock` и `inHand` для вычисления новых.
- Задача является обобщением "Best Time to Buy and Sell Stock II" (без комиссии), где `fee = 0`.
