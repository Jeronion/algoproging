## Определение

**Палиндромное разбиение строки** — это разбиение строки на подстроки таким образом, что каждая подстрока является палиндромом. Задача заключается в нахождении минимального количества разрезов, необходимых для такого разбиения.

Например, для строки `"geek"` минимальное количество разрезов равно 2: `"g" | "ee" | "k"`. (Хотя можно и `"g" | "e" | "e" | "k"` — 3 разреза). Оптимальное разбиение: `"g"` (палиндром), `"ee"` (палиндром), `"k"` (палиндром) → 2 разреза.

---

## Постановка задачи

**Вход:** строка `s` длины `n` (1 ≤ n ≤ 2000, в некоторых задачах до 10⁵, но обычно ≤ 2000).  
**Выход:** минимальное количество разрезов, чтобы разбить `s` на палиндромные подстроки.

---

## Идея решения (динамическое программирование)

Задача решается двумя этапами:

1. **Предварительное вычисление палиндромов** — создание таблицы `isPalin[i][j]`, где `isPalin[i][j]` равно `True`, если подстрока `s[i..j]` является палиндромом. Это делается за O(n²) с использованием динамического программирования.
2. **Вычисление минимальных разрезов** — создание массива `dp[i]`, где `dp[i]` — минимальное количество разрезов для префикса `s[0..i]`. Если `s[0..i]` — палиндром, то `dp[i] = 0`. Иначе перебираем все возможные позиции `j` (1 ≤ j ≤ i), такие что подстрока `s[j..i]` — палиндром, и тогда `dp[i] = min(dp[i], 1 + dp[j-1])`.

Таким образом, мы постепенно вычисляем минимальные разрывы для каждого префикса, используя уже вычисленные значения для меньших префиксов и информацию о палиндромах.

---

## Алгоритм (пошагово)

1. **Построение таблицы палиндромов `isPalin`**:
   - Инициализировать `isPalin[i][i] = True` для всех `i` (подстроки длины 1).
   - Для длины `length` от 2 до `n`:
     - Для `i` от 0 до `n - length`:
       - `j = i + length - 1`.
       - Если `s[i] == s[j]` и (`length == 2` или `isPalin[i+1][j-1]`), то `isPalin[i][j] = True`.

2. **Вычисление минимальных разрезов**:
   - Создать массив `dp` размера `n`, заполнить большим числом (например, `n`).
   - `dp[0] = 0` (односимвольная строка всегда палиндром).
   - Для `i` от 1 до `n-1`:
     - Если `isPalin[0][i]` истинно, то `dp[i] = 0`.
     - Иначе:
       - Для `j` от `i` до 1:
         - Если `isPalin[j][i]` истинно, то `dp[i] = min(dp[i], 1 + dp[j-1])`.
   - Ответ: `dp[n-1]`.

## Реализация на Python

```python
def palindromePartitioning(s):
    n = len(s)
    # 1. Таблица палиндромов
    isPalin = [[False] * n for _ in range(n)]
    for i in range(n):
        isPalin[i][i] = True
    for length in range(2, n + 1):
        for i in range(n - length + 1):
            j = i + length - 1
            if s[i] == s[j] and (length == 2 or isPalin[i+1][j-1]):
                isPalin[i][j] = True

    # 2. Вычисление минимальных разрезов
    dp = [n] * n
    dp[0] = 0
    for i in range(1, n):
        if isPalin[0][i]:
            dp[i] = 0
        else:
            for j in range(i, 0, -1):
                if isPalin[j][i]:
                    dp[i] = min(dp[i], 1 + dp[j-1])
    return dp[n-1]
```

---

## Сложность

- **Время:** O(n²) на построение таблицы палиндромов и O(n²) на вычисление разрезов → общая O(n²).
- **Память:** O(n²) для таблицы палиндромов + O(n) для массива `dp`.
