## Определение

В бинарном дереве поиска (BST) все элементы упорядочены: левое поддерево содержит меньшие ключи, правое — большие. **Inorder-обход** (левое поддерево → узел → правое поддерево) даёт отсортированную последовательность ключей. Таким образом, k-й наименьший элемент — это k-й элемент при inorder-обходе дерева.

Задача: по корню BST и целому числу `k` найти значение k-го наименьшего элемента. Если такого элемента нет (k больше размера дерева), вернуть `-1`.

---

## Постановка задачи

**Вход:**
- `root` — корень BST.
- `k` — порядковый номер (1-индексация).

**Выход:** значение k-го наименьшего узла или `-1`, если k превышает количество узлов.

**Пример:**
```
        20
       /  \
      8    22
     / \
    4   12
       /  \
      10   14
```
Для `k = 3` inorder-обход: [4, 8, 10, 12, 14, 20, 22] → 3-й элемент = 10.

---

## Идея решения (итеративный inorder-обход с использованием стека)

Для поиска k-го элемента нам не нужно строить весь список inorder. Достаточно выполнить обход и остановиться, когда будет посещён k-й узел. Используем **стек** для симуляции рекурсивного inorder-обхода:

1. Начинаем с корня и идём по левым ссылкам, помещая узлы в стек, пока не дойдём до `None`.
2. Извлекаем верхний узел из стека (это самый левый узел текущего поддерева), увеличиваем счётчик посещённых узлов.
3. Если счётчик достиг `k`, возвращаем значение этого узла.
4. Переходим в правое поддерево извлечённого узла и повторяем процесс.

Этот алгоритм не требует рекурсии и использует O(h) памяти (глубина стека), где h — высота дерева.

---

## Алгоритм (пошагово)

1. Инициализировать `count = 0`, пустой стек `st`, текущий узел `cur = root`.
2. Пока `st` не пуст или `cur` не `None`:
   - Пока `cur` не `None`:
     - Поместить `cur` в стек.
     - Перейти к левому потомку `cur = cur.left`.
   - Извлечь узел из стека: `cur = st.pop()`.
   - Увеличить `count += 1`.
   - Если `count == k`, вернуть `cur.data`.
   - Перейти к правому потомку: `cur = cur.right`.
3. Если цикл завершился (т.е. k больше числа узлов), вернуть `-1`.

---

## Реализация на Python

```python
class Solution:
    def kthSmallest(self, root, k):
        count = 0
        stack = []
        cur = root
        while stack or cur:
            while cur:
                stack.append(cur)
                cur = cur.left
            cur = stack.pop()
            count += 1
            if count == k:
                return cur.data
            cur = cur.right
        return -1
```

---

## Сложность

- **Время:** O(h + k), где h — высота дерева. В худшем случае (для k близкого к n) O(n). В среднем для сбалансированного BST O(log n + k).
- **Память:** O(h) для стека (в сбалансированном дереве O(log n), в вырожденном O(n)).

---

## Примечания

- Если `k` больше количества узлов, цикл завершится и вернётся `-1`.
- Альтернативно можно использовать рекурсивный inorder-обход с глобальным счётчиком, но итеративный подход позволяет избежать переполнения стека при глубоких деревьях.
- В BST все ключи уникальны, поэтому порядок однозначен.
- Для поиска k-го **наибольшего** элемента можно выполнить обход в обратном порядке (right → node → left) или использовать модификацию алгоритма.
