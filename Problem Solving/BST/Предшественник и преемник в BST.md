## Определение

Для заданного ключа `key` в бинарном дереве поиска (BST):

- **Предшественник (predecessor)** — это узел с **наибольшим** значением, которое **строго меньше** `key` (в терминах inorder-обхода — узел, идущий непосредственно перед `key`).
- **Преемник (successor)** — это узел с **наименьшим** значением, которое **строго больше** `key` (в inorder-обходе — узел, идущий сразу после `key`).

Если такого узла нет, соответствующее значение должно быть `None` (или `null`).

**Пример:**
```
        50
       /  \
      30   70
     / \   / \
    20 40 60 80
```
Для `key = 65`: предшественник = 60, преемник = 70.

---

## Идея решения

Используем свойства BST и выполняем один проход по дереву без рекурсии.

- Начинаем с корня.
- Если текущий узел меньше `key`, то он может быть предшественником (поскольку он меньше, но нужно найти максимальный из меньших). Переходим в правое поддерево (там могут быть большие значения, но всё ещё < key).
- Если текущий узел больше `key`, то он может быть преемником (минимальный из больших). Переходим в левое поддерево (там могут быть меньшие, но всё ещё > key).
- Если текущий узел равен `key`:
  - Преемник находится в **правом поддереве** — это самый левый (минимальный) узел в правом поддереве.
  - Предшественник находится в **левом поддереве** — это самый правый (максимальный) узел в левом поддереве.
- Если `key` не найден, то по окончании обхода мы будем иметь последние кандидаты на pre и suc (в соответствии с условиями выше). Однако в данной задаче гарантируется, что ключ присутствует? В условии не сказано, но обычно предполагается, что ключ есть. В любом случае алгоритм корректно обрабатывает случай отсутствия, но тогда pre и suc будут определены как ближайшие меньший и больший (если есть).

### Алгоритм (пошагово)

1. Инициализировать `pre = None`, `suc = None`.
2. Установить `cur = root`.
3. Пока `cur` не `None`:
   - Если `cur.data < key`:
     - `pre = cur` (текущий узел кандидат в предшественники)
     - Перейти `cur = cur.right` (ищем большее значение, но всё ещё < key)
   - Иначе если `cur.data > key`:
     - `suc = cur` (текущий узел кандидат в преемники)
     - Перейти `cur = cur.left` (ищем меньшее, но всё ещё > key)
   - Иначе (`cur.data == key`):
     - Найти преемника: спуститься в правое поддерево и идти по левым ссылкам до конца; последний узел будет преемником (если правое поддерево не пусто).
     - Найти предшественника: спуститься в левое поддерево и идти по правым ссылкам до конца; последний узел будет предшественником (если левое поддерево не пусто).
     - Выйти из цикла (так как ключ найден).
4. Вернуть `[pre, suc]`.

В коде, если ключ найден, после установки `cur = cur.right` и `cur = cur.left` он ищет соответственно самый левый и самый правый узлы.

## Реализация на Python

```python
class Solution:
    def findPreSuc(self, root, key):
        pre, suc = None, None
        cur = root
        while cur:
            if cur.data < key:
                pre = cur
                cur = cur.right
            elif cur.data > key:
                suc = cur
                cur = cur.left
            else:
                # Нашли ключ: ищем преемника в правом поддереве
                temp = cur
                cur = cur.right
                while cur:
                    suc = cur
                    cur = cur.left
                # Ищем предшественника в левом поддереве
                cur = temp.left
                while cur:
                    pre = cur
                    cur = cur.right
                break  # можно не break, но цикл завершится, так как cur станет None
        return [pre, suc]
```

## Сложность

- **Время:** O(h), где h — высота BST. В сбалансированном дереве O(log n), в вырожденном — O(n).
- **Память:** O(1) — используется только несколько указателей (без рекурсии).

---

## Примечания

- Если ключ отсутствует в дереве, алгоритм всё равно найдёт ближайшие меньший и больший (если они существуют) и вернёт их как предшественника и преемника соответственно.
- В коде, после нахождения ключа, поиск преемника и предшественника выполняется отдельно с помощью циклов.
- Данный метод является классическим и не требует дополнительной памяти.
- Для inorder-обхода предшественник и преемник определяются именно так: предшественник — максимальный среди меньших, преемник — минимальный среди больших.
