## Определение

**Floor(x)** в бинарном дереве поиска (BST) — это **наибольший** элемент в дереве, который **меньше или равен** заданному значению `x`. Если такого элемента нет (т.е. `x` меньше всех значений в дереве), возвращается `-1`.

Например, для дерева `[10, 5, 15, 12, 30]` при `x = 14` floor будет `12`, так как 12 — это наибольшее значение, не превышающее 14.

---

## Алгоритм (итеративный)

Используем свойства BST для поиска Floor без рекурсии:

1. Начинаем с корня, инициализируем переменную `floor_value = -1`.
2. Пока текущий узел не равен `None`:
   - Если значение текущего узла **равно** `x`, то это и есть Floor — возвращаем его.
   - Если значение текущего узла **больше** `x`, переходим в **левое** поддерево (ищем меньшее значение).
   - Если значение текущего узла **меньше** `x`, то этот узел является потенциальным Floor, запоминаем его (`floor_value = node.data`) и переходим в **правое** поддерево (пытаемся найти значение побольше, но всё ещё ≤ `x`).
3. Если дошли до `None`, возвращаем последний запомненный `floor_value` (или `-1`, если его не было).

Этот подход не использует рекурсию, следовательно, требует O(1) дополнительной памяти.

---

## Реализация

```python
class Node:
    def __init__(self, data):
        self.data = data
        self.left = None
        self.right = None

def floor(root, x):
    floor_value = -1
    curr = root
    while curr:
        if curr.data == x:
            return x
        elif curr.data > x:
            curr = curr.left          # идём влево, чтобы уменьшить значение
        else:
            floor_value = curr.data   # текущий узел подходит
            curr = curr.right         # ищем большее, но всё ещё ≤ x
    return floor_value
```

---

## Сложность

- **Время:** O(h), где h — высота BST. В сбалансированном дереве O(log n), в вырожденном — O(n).
- **Память:** O(1) — используется только несколько переменных.

---

## Примечания

- Алгоритм работает как для уникальных, так и для неуникальных ключей (но в стандартном BST ключи обычно уникальны).
- Если дерево пустое, возвращается `-1`.
- Аналогично можно реализовать **Ceil** (наименьший элемент ≥ x), изменив логику: при значении узла < x переходить вправо, а при значении > x запоминать узел и идти влево.
- Итеративная версия предпочтительнее рекурсивной в языках с ограниченной глубиной стека или при работе с большими деревьями.
