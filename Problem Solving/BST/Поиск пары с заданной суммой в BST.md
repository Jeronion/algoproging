## Определение

В бинарном дереве поиска (BST) необходимо определить, существует ли пара узлов (два различных узла), значения которых в сумме дают заданное число `target`. Элементы в BST упорядочены, что позволяет эффективно искать пару, аналогично методу двух указателей в отсортированном массиве.

**Пример:**
```
        7
       / \
      3   8
     / \   \
    2   4   9
```
Для `target = 12` пара (4, 8) даёт сумму 12 → true.

---

## Постановка задачи

Дан корень BST и целое число `target`. Вернуть `True`, если существуют два различных узла, сумма значений которых равна `target`, иначе `False`.

Ограничения: количество узлов до 10⁵, значения и target до 10⁶.

---

## Идея решения (два итератора с использованием стеков)

Вместо того чтобы выполнять inorder-обход и сохранять все значения в массив (O(n) памяти), можно использовать **два указателя**, которые перемещаются по BST с помощью стеков:

- **Левый итератор (next)** — выдаёт следующий наименьший элемент (inorder-обход слева направо).
- **Правый итератор (prev)** — выдаёт следующий наибольший элемент (обратный inorder-обход справа налево).

Это позволяет получить два указателя: `l_val` и `r_val`, которые начинаются с минимального и максимального элементов соответственно. Далее, как в классическом алгоритме для отсортированного массива:

- Если `l_val + r_val == target` → пара найдена.
- Если `l_val + r_val > target` → сдвигаем правый указатель вниз (уменьшаем сумму).
- Если `l_val + r_val < target` → сдвигаем левый указатель вверх (увеличиваем сумму).

Итераторы реализованы с помощью стеков, что позволяет получать следующий элемент за амортизированное O(1) времени и использовать O(h) памяти (h — высота дерева).

---

## Алгоритм (пошагово)

1. Инициализировать два стека: `stL` для inorder-обхода (слева направо) и `stR` для обратного inorder-обхода (справа налево).
2. Установить `left = root`, `right = root`.
3. Реализовать функции:
   - `nextLeft()`: перемещает указатель `left` к следующему наименьшему элементу.
   - `nextRight()`: перемещает указатель `right` к следующему наибольшему элементу.
4. Получить первое значение: `l_val = nextLeft()`, `r_val = nextRight()`.
5. Пока оба указателя не достигли краёв (т.е. стек не пуст или текущий узел не None):
   - Если `l_val + r_val == target` → вернуть `True`.
   - Если `l_val + r_val > target` → `r_val = nextRight()`.
   - Иначе → `l_val = nextLeft()`.
6. Если цикл завершился — пары нет → вернуть `False`.

## Реализация на Python

```python
class Solution:
    def findTarget(self, root, target):
        # Итератор для inorder (возрастание)
        stL = []
        left = root
        def next_left():
            nonlocal left
            while left:
                stL.append(left)
                left = left.left
            node = stL.pop()
            val = node.data
            left = node.right
            return val
        
        # Итератор для reverse inorder (убывание)
        stR = []
        right = root
        def next_right():
            nonlocal right
            while right:
                stR.append(right)
                right = right.right
            node = stR.pop()
            val = node.data
            right = node.left
            return val
        
        l_val = next_left()
        r_val = next_right()
        
        while (stL or left) and (stR or right):
            if l_val + r_val == target:
                return True
            elif l_val + r_val > target:
                r_val = next_right()
            else:
                l_val = next_left()
        return False
```

## Сложность

- **Время:** O(n) в худшем случае, так как каждый узел посещается не более одного раза (в каждом итераторе). В среднем O(n) для проверки всех элементов.
- **Память:** O(h) для стеков, где h — высота BST. В сбалансированном дереве O(log n), в вырожденном O(n).

---

## Примечания

- Метод использует два указателя без явного построения списка значений, что экономит память.
- Оба указателя движутся от крайних значений к центру, что гарантирует, что если пара существует, она будет найдена.
- Если в дереве есть дублирующиеся значения, алгоритм всё равно работает (узлы считаются различными, но значения могут совпадать — проверка суммы корректна).
- Альтернативные подходы: сохранить все значения в множество и для каждого элемента проверять `target - val`. Это требует O(n) времени и O(n) памяти, но проще в реализации. Данный подход более эффективен по памяти.
