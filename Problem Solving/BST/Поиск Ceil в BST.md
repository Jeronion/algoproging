## Определение

**Ceil(x)** в бинарном дереве поиска (BST) — это наименьший элемент в дереве, который **больше или равен** заданному значению `x`. Если такого элемента нет, возвращается `-1`.

Например, для дерева `[10, 5, 11, 4, 7]` при `x = 6` ceil будет `7`, так как это ближайшее значение, не меньшее `6`.

---

## Алгоритм

Ищем Ceil, используя свойства BST:

1. Начинаем обход с корня.
2. Если значение текущего узла **равно** `x` — это и есть Ceil, возвращаем его.
3. Если значение текущего узла **больше** `x` — этот узел является потенциальным Ceil, запоминаем его и переходим в **левое** поддерево (чтобы найти меньшее значение, но всё ещё ≥ `x`).
4. Если значение текущего узла **меньше** `x` — Ceil не может быть в этом узле, переходим в **правое** поддерево (ищем большее значение).
5. Если дошли до `None` — возвращаем последний запомненный кандидат (или `-1`, если его не было).

### Итеративная реализация (предпочтительная)

```python
class Solution:
    def findCeil(self, root, x):
        ceil = -1
        curr = root
        while curr:
            if curr.data == x:
                return x
            elif curr.data > x:
                ceil = curr.data       # потенциальный ответ
                curr = curr.left       # ищем меньшее, но всё ещё >= x
            else:
                curr = curr.right      # ищем большее
        return ceil
```

### Рекурсивная реализация (для сравнения)

```python
class Solution:
    def findCeil(self, root, x):
        def dfs(node, x):
            if node is None:
                return -1
            if node.data == x:
                return x
            if node.data > x:
                # проверяем левое поддерево, но не забываем о текущем узле
                left_ceil = dfs(node.left, x)
                return left_ceil if left_ceil != -1 else node.data
            else:
                return dfs(node.right, x)
        return dfs(root, x)
```

---

## Сложность

- **Время:** O(h), где h — высота BST. В сбалансированном дереве O(log n), в худшем (вырожденном) O(n).
- **Память:** O(1) для итеративной версии, O(h) для рекурсивной (стек вызовов).

---

## Примечания

- Если дерево пустое, возвращается `-1`.
- Значения узлов уникальны, но это не критично для алгоритма.
- Аналогично можно реализовать **Floor** (наибольший элемент, не превосходящий `x`), изменив направление обхода.
