## Определение

Задача **Gas Station** (или «Заправка») — классическая задача на жадный алгоритм. Даны два массива: `gas[i]` — количество бензина, которое можно заправить на станции `i`, и `cost[i]` — расход бензина для переезда от станции `i` к станции `i+1` (по кругу). Автомобиль начинает путь с пустым баком. Требуется найти **индекс стартовой станции**, с которой можно совершить полный круг, не останавливаясь из‑за нехватки бензина. Если такой станции нет, вернуть `-1`. Гарантируется, что решение, если оно есть, единственно.

**Пример:**
```
gas  = [4, 5, 7, 4]
cost = [6, 6, 3, 5]
Ответ: 2 (начинаем со станции 2)
```

---

## Постановка задачи

**Вход:**
- `gas[]` — массив целых чисел, где `gas[i]` — объём топлива на станции `i`.
- `cost[]` — массив целых чисел, где `cost[i]` — расход топлива для переезда от `i` к `i+1` (последний переход от `n-1` к `0`).

**Выход:** индекс стартовой станции (0-индексация), или `-1`, если обход невозможен.

**Ограничения:** 1 ≤ n ≤ 10⁶, 1 ≤ gas[i], cost[i] ≤ 10³.

---

## Идея решения (жадный алгоритм)

Задача решается за один проход по массиву без использования вложенных циклов. Используются две переменные:

- `total` — сумма всех `(gas[i] - cost[i])`. Если `total < 0`, то обход невозможен (общий запас топлива меньше общего расхода).
- `cur` — текущий остаток топлива при движении от предполагаемого старта `start`.
- `start` — индекс кандидата на стартовую станцию.

Алгоритм основан на ключевом наблюдении: если на некотором отрезке накопленный остаток `cur` становится отрицательным, то ни одна из станций на этом отрезке (включая текущий кандидат) не может быть стартовой. Поэтому мы сбрасываем `cur = 0` и переносим `start` на следующую станцию `i+1`. В конце, если `total >= 0`, то `start` будет правильным ответом.

### Почему это работает?

Если `cur < 0` на станции `i`, то сумма `(gas - cost)` от `start` до `i` отрицательна. Это означает, что невозможно доехать от `start` до `i+1`. Но поскольку общий баланс `total` неотрицателен, то существует более поздняя станция, с которой можно начать. Новый кандидат `start = i+1` отбрасывает весь предыдущий отрезок, и алгоритм продолжает проверку. В конце, если `total >= 0`, найденный `start` гарантированно подходит.

---

## Алгоритм (пошагово)

1. Инициализировать:
   - `total = 0` — суммарный баланс топлива.
   - `cur = 0` — баланс от текущего кандидата `start`.
   - `start = 0` — предполагаемый старт.
2. Для каждой станции `i` от `0` до `n-1`:
   - `total += gas[i] - cost[i]`.
   - `cur += gas[i] - cost[i]`.
   - Если `cur < 0`, то:
     - Обнулить `cur = 0`.
     - Установить `start = i + 1` (следующая станция как новый кандидат).
3. После цикла:
   - Если `total < 0`, вернуть `-1`.
   - Иначе вернуть `start`.

## Реализация на Python

```python
class Solution:
    def startStation(self, gas, cost):
        n = len(gas)
        total = 0
        cur = 0
        start = 0

        for i in range(n):
            diff = gas[i] - cost[i]
            total += diff
            cur += diff

            # Если накопленный баланс отрицателен,
            # то старт не может быть на текущем отрезке.
            if cur < 0:
                cur = 0
                start = i + 1

        # Если общий баланс отрицателен, обход невозможен.
        if total < 0:
            return -1
        return start
```

## Сложность

- **Время:** O(n) — один проход по массиву.
- **Память:** O(1) — только несколько переменных.

---

## Примечания

- Алгоритм работает для **кругового маршрута**, поэтому важен переход от последней станции к первой — это учтено циклически через общий баланс.
- Если `total == 0`, решение существует и единственно (по условию).
- `start` может стать равным `n` (если последняя станция не подходит), но тогда `total` был бы отрицательным, и мы бы вернули `-1`.
- Альтернативное решение: моделирование с двумя указателями, но данный жадный подход является классическим и оптимальным.
