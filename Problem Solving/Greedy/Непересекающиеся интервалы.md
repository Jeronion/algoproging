## Определение

Дан массив интервалов `intervals[i] = [start_i, end_i]`. Требуется найти **минимальное количество интервалов**, которые необходимо удалить, чтобы все оставшиеся интервалы не пересекались. Два интервала считаются непересекающимися, если `end_i ≤ start_j` или `end_j ≤ start_i`.

**Пример:**
```
intervals = [[1,2], [2,3], [3,4], [1,3]]
Удалить достаточно [1,3] → остальные не пересекаются → ответ 1.
```

---

## Постановка задачи

**Вход:** список интервалов `intervals` (длина до 10⁵, значения до 5·10⁴).
**Выход:** минимальное число удалений.

---

## Идея решения (жадный алгоритм)

Сортируем интервалы по **началу** (по возрастанию start). Затем проходим по отсортированному списку и отслеживаем конец последнего выбранного (или сохранённого) интервала. При встрече интервала, который перекрывается с текущим сохранённым (т.е. его начало меньше конца сохранённого), мы должны удалить один из двух. Чтобы минимизировать удаления, **оставляем интервал с меньшим концом** (удаляем тот, который заканчивается позже). Если интервалы не перекрываются, просто обновляем сохранённый конец.

Этот жадный подход корректен, так как он всегда сохраняет интервал, который заканчивается раньше, оставляя больше места для последующих интервалов.

### Почему это работает?

При перекрытии двух интервалов удаление одного из них неизбежно. Выгодно удалить тот, который заканчивается позже, так как он потенциально может пересечься с большим количеством будущих интервалов. Это локально оптимальный выбор, который приводит к глобальному минимуму.

---

## Алгоритм

1. Отсортировать `intervals` по возрастанию `start` (а при равенстве — по `end`, но это не критично).
2. Инициализировать `cnt = 0` (количество удалений) и `end = intervals[0][1]` (конец первого интервала, который мы пока сохраняем).
3. Для каждого интервала `i` от `1` до `n-1`:
   - Если `intervals[i][0] < end` (перекрытие), то:
     - Увеличить `cnt` на 1 (один из двух удаляем).
     - Обновить `end = min(end, intervals[i][1])` (оставляем интервал с меньшим концом).
   - Иначе (перекрытия нет):
     - Обновить `end = intervals[i][1]` (теперь сохраняем текущий интервал).
4. Вернуть `cnt`.

---

## Реализация

```python
class Solution:
    def minRemoval(self, intervals):
        intervals.sort()  # сортировка по началу
        cnt = 0
        end = intervals[0][1]
        for i in range(1, len(intervals)):
            if intervals[i][0] < end:   # перекрытие
                cnt += 1
                end = min(end, intervals[i][1])  # оставляем тот, что заканчивается раньше
            else:
                end = intervals[i][1]
        return cnt
```

---

## Сложность

- **Время:** O(n log n) из-за сортировки, где n — количество интервалов.
- **Память:** O(1) дополнительно (не считая памяти под сортировку).

---

## Пример

```
intervals = [[1,2], [2,3], [3,4], [1,3]]
Сортируем: [[1,2], [1,3], [2,3], [3,4]]
end = 2
i=1: [1,3]: 1 < 2 → cnt=1, end=min(2,3)=2
i=2: [2,3]: 2 < 2? нет → end=3
i=3: [3,4]: 3 < 3? нет → end=4
Ответ: 1
```

---

## Примечания

- Условие перекрытия — строгое `start < end`. Если `start == end`, интервалы считаются непересекающимися (согласно условию задачи).
- Альтернативное решение: сортировка по концу и подсчёт максимального числа непересекающихся интервалов. Тогда ответ = n - max_непересекающихся. Оба подхода дают одинаковый результат.
- Алгоритм не требует хранения выбранных интервалов, только подсчёт удалений.
