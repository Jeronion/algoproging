## Определение

В магазине есть N видов конфет с ценами `prices[i]`. Действует акция: при покупке одной конфеты можно получить **до k других конфет бесплатно**. Нужно определить **минимальную** и **максимальную** сумму денег, необходимую для приобретения всех конфет, при условии, что в каждой покупке берётся максимально возможное количество бесплатных конфет.

**Пример:**
```
prices = [3, 2, 1, 4], k = 2
Минимум: купить конфету за 1, бесплатно взять 3 и 4, затем купить 2 → итого 1+2=3.
Максимум: купить конфету за 4, бесплатно взять 1 и 2, затем купить 3 → итого 4+3=7.
```

---

## Постановка задачи

**Вход:**
- `prices` — массив цен всех конфет.
- `k` — максимальное количество бесплатных конфет за одну покупку.

**Выход:** массив из двух чисел: `[min_cost, max_cost]`.

**Ограничения:** 1 ≤ n ≤ 10⁵, 0 ≤ k ≤ n, 1 ≤ prices[i] ≤ 10⁴.

---

## Идея решения (жадный алгоритм)

Чтобы минимизировать затраты, нужно **покупать самые дешёвые** конфеты и забирать бесплатно **самые дорогие**. Чтобы максимизировать затраты, нужно **покупать самые дорогие** конфеты и забирать бесплатно **самые дешёвые**.

Это классическая жадная стратегия: сортируем цены, затем для минимума используем два указателя, выбирая с левого конца (дешёвые) и пропуская k самых дорогих с правого конца; для максимума — наоборот.

### Почему это работает?

- При минимизации: если мы покупаем дорогую конфету, мы теряем возможность взять её бесплатно, а вместо этого должны заплатить. Поэтому выгодно покупать дешёвые и получать дорогие бесплатно.
- При максимизации: наоборот, выгодно покупать дорогие и получать дешёвые бесплатно, чтобы максимизировать сумму покупок.

Алгоритм гарантирует, что мы всегда берём максимальное количество бесплатных конфет (k штук) за каждую покупку.

---

## Алгоритм

1. Отсортировать массив `prices` по возрастанию.
2. **Для минимальной суммы:**
   - Установить указатели `l = 0`, `r = n-1`.
   - Пока `l <= r`:
     - Добавить к сумме `prices[l]` (покупаем самую дешёвую доступную).
     - Увеличить `l` на 1 (следующая дешёвая).
     - Уменьшить `r` на `k` (пропускаем k самых дорогих как бесплатные).
3. **Для максимальной суммы:**
   - Сбросить указатели `l = 0`, `r = n-1`.
   - Пока `l <= r`:
     - Добавить к сумме `prices[r]` (покупаем самую дорогую доступную).
     - Уменьшить `r` на 1.
     - Увеличить `l` на `k` (пропускаем k самых дешёвых как бесплатные).
4. Вернуть `[min_sum, max_sum]`.

## Реализация на Python

```python
class Solution:
    def minMaxCandy(self, prices, k):
        prices.sort()
        n = len(prices)
        min_cost = 0
        max_cost = 0
        
        # Минимальная сумма
        l, r = 0, n - 1
        while l <= r:
            min_cost += prices[l]   # покупаем самую дешёвую
            l += 1
            r -= k                  # k самых дорогих — бесплатно
        
        # Максимальная сумма
        l, r = 0, n - 1
        while l <= r:
            max_cost += prices[r]   # покупаем самую дорогую
            r -= 1
            l += k                  # k самых дешёвых — бесплатно
        
        return [min_cost, max_cost]
```

---

## Сложность

- **Время:** O(n log n) из-за сортировки.
- **Память:** O(1) дополнительно (не считая памяти под сортировку).

## Пример работы

```
prices = [3, 2, 1, 4], k = 2
Сортируем: [1, 2, 3, 4]
Минимум:
  l=0, r=3: +1, l=1, r=1
  l=1, r=1: +2, l=2, r=-1 → останов → сумма = 3.
Максимум:
  l=0, r=3: +4, r=2, l=2
  l=2, r=2: +3, r=1, l=4 → останов → сумма = 7.
Ответ: [3, 7]
```

---

## Примечания

- Если `k = 0`, то бесплатных конфет нет, и минимальная и максимальная суммы равны сумме всех цен.
- Если `k >= n`, то можно купить одну самую дешёвую (для минимума) или одну самую дорогую (для максимума) и получить все остальные бесплатно.
- Важно, что в каждой покупке мы берём **максимальное** количество бесплатных конфет, поэтому при подсчёте указатели сдвигаются на `k` за раз.
