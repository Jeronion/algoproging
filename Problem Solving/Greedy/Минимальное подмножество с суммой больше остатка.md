## Определение

Дан массив неотрицательных целых чисел. Требуется выбрать **минимальное количество элементов** так, чтобы сумма выбранных элементов была **строго больше** суммы всех остальных элементов массива.

**Пример:**
```
arr = [2, 17, 7, 3] → достаточно взять 17 (сумма остальных 12) → ответ 1.
arr = [20, 12, 18, 4] → нужно взять 12 + 18 = 30 > 24 (20+4) → ответ 2.
```

---

## Постановка задачи

**Вход:** массив `arr` размера n (1 ≤ n ≤ 10⁵, 1 ≤ arr[i] ≤ 10⁴).
**Выход:** минимальное количество элементов, которое нужно выбрать, чтобы их сумма превышала сумму оставшихся.

---

## Идея решения (жадный алгоритм)

Чтобы минимизировать количество выбранных элементов, нужно выбирать **наибольшие** элементы, так как они дают максимальный вклад в сумму за один элемент. Жадная стратегия:

1. Отсортировать массив по убыванию.
2. Подсчитать общую сумму всех элементов `total`.
3. Идти по отсортированному массиву, накапливая сумму выбранных `selected_sum` и уменьшая `total` (сумма остатка).
4. Как только `selected_sum > total`, возвращаем количество выбранных элементов.

Это работает, потому что выбор самых больших элементов гарантирует, что мы достигнем нужного порога за минимальное число шагов.

---

## Алгоритм

1. `arr.sort(reverse=True)`.
2. `total = sum(arr)`.
3. `selected = 0`, `count = 0`.
4. Для каждого `x` в `arr`:
   - `selected += x`
   - `total -= x`
   - `count += 1`
   - Если `selected > total`, вернуть `count`.
5. (По условию задачи гарантируется, что ответ всегда существует, так как можно взять все элементы.)

---

## Реализация на Python

```python
class Solution:
    def minSubset(self, arr):
        arr.sort(reverse=True)
        total = sum(arr)
        selected = 0
        for i, x in enumerate(arr):
            selected += x
            total -= x
            if selected > total:
                return i + 1
        return len(arr)  # на случай, если условие не выполнится (но по условию задачи всегда выполнится)
```

---

## Сложность

- **Время:** O(n log n) из-за сортировки.
- **Память:** O(1) дополнительно (если не считать памяти, используемой сортировкой).

---

## Пример работы

```
arr = [2, 17, 7, 3]
sort → [17, 7, 3, 2]
total = 29
i=0: selected=17, total=12, 17 > 12 → возвращаем 1.

arr = [20, 12, 18, 4]
sort → [20, 18, 12, 4]
total = 54
i=0: selected=20, total=34, 20 > 34? нет
i=1: selected=38, total=16, 38 > 16 → возвращаем 2.
```

---

## Примечания

- Задача решается жадным способом, так как выбор больших элементов всегда оптимален.
- Если элементы могут быть равны нулю, алгоритм всё равно работает (но по условию arr[i] ≥ 1).
- Если массив уже отсортирован по убыванию, можно обойтись без сортировки, но в общем случае сортировка необходима.
