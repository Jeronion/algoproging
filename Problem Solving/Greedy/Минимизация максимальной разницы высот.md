## Определение

Дан массив `arr[]`, представляющий высоты башен, и целое число `k`. Для каждой башни мы обязаны ровно один раз либо увеличить её высоту на `k`, либо уменьшить на `k`. После всех изменений необходимо найти **минимально возможную разницу** между максимальной и минимальной высотой в новом массиве. Результирующие высоты не должны быть отрицательными.

**Пример:**
```
k = 2, arr = [1, 5, 8, 10]
После изменений: [3, 3, 6, 8] → разница = 8 - 3 = 5.
```

---

## Идея решения

Мы можем рассматривать отсортированный массив и выбирать точку разделения: все элементы слева от неё (включая её) увеличиваем на `k`, все элементы справа — уменьшаем на `k`. Это гарантирует, что меньшие элементы станут больше, а большие — меньше, что минимизирует разброс.

Почему это работает: новые минимум и максимум будут определяться только крайними значениями после преобразований. Перебирая все возможные точки разделения, мы покрываем все способы распределения операций и находим минимальную разницу.

### Алгоритм

1. Отсортировать массив `arr`.
2. Инициализировать ответ `ans = arr[n-1] - arr[0]` (без изменений).
3. Вычислить `smallest = arr[0] + k` (минимальная возможная высота, если самый маленький элемент увеличить).
4. Вычислить `largest = arr[n-1] - k` (максимальная возможная высота, если самый большой элемент уменьшить).
5. Для каждого `i` от 1 до n-1:
   - Если `arr[i] - k < 0`, пропустить (нельзя получить отрицательную высоту).
   - `minH = min(smallest, arr[i] - k)` — минимальная высота среди всех преобразованных.
   - `maxH = max(largest, arr[i-1] + k)` — максимальная высота среди всех преобразованных.
   - Обновить `ans = min(ans, maxH - minH)`.
6. Вернуть `ans`.

---

## Реализация

```python
def getMinDiff(arr, k):
    n = len(arr)
    arr.sort()
    ans = arr[-1] - arr[0]

    smallest = arr[0] + k
    largest = arr[-1] - k

    for i in range(1, n):
        if arr[i] - k < 0:
            continue
        min_h = min(smallest, arr[i] - k)
        max_h = max(largest, arr[i-1] + k)
        ans = min(ans, max_h - min_h)

    return ans
```

---

## Сложность

- **Время:** O(n log n) — из-за сортировки.
- **Память:** O(1) дополнительно.

---

## Примечания

- Операция **обязательна** для каждого элемента (нельзя оставить без изменения).
- Отрицательные высоты запрещены, поэтому проверка `arr[i] - k >= 0` обязательна.
- Алгоритм работает для любого количества элементов, включая случай `n == 1` (ответ будет 0, так как только один элемент).
