## Определение

Дана строка `s`, состоящая из строчных латинских букв. Требуется определить, можно ли переставить её символы так, чтобы никакие два соседних символа не были одинаковыми. Если возможно, вернуть `true`, иначе `false`.

**Пример:**
```
s = "aaabc" → можно переставить в "abaca" → true
s = "aaaabc" → невозможно, так как один символ встречается слишком часто → false
```

---

## Постановка задачи

**Вход:** строка `s` длины n (1 ≤ n ≤ 10⁵).
**Выход:** `true`, если существует перестановка без одинаковых соседей, иначе `false`.

---

## Идея решения

Критерий возможности: строка может быть переставлена так, чтобы соседние символы различались, **тогда и только тогда**, когда максимальная частота любого символа не превышает `(n + 1) // 2`. Это необходимое и достаточное условие:

- Если какой-то символ встречается чаще чем половина (с округлением вверх), его неизбежно придётся ставить рядом с самим собой.
- Если максимальная частота не превосходит этого предела, всегда можно построить нужную перестановку (например, сначала ставить самые частые символы через один).

Таким образом, задача сводится к нахождению максимальной частоты. Её можно найти простым подсчётом через массив из 26 элементов или через `Counter`. В представленном решении используется **алгоритм Бойера-Мура** для нахождения кандидата на наиболее частый символ (без полного подсчёта), а затем подсчитывается его точная частота. Это эффективно и не требует хранения полной частотной таблицы, если мы не знаем алфавит, но в данном случае алфавит ограничен 26 буквами, поэтому проще использовать массив.

Однако алгоритм Бойера-Мура даёт корректного кандидата, если такой символ действительно имеет частоту > n/2. В нашем случае нам нужно точное значение частоты самого частого символа, поэтому после нахождения кандидата мы подсчитываем его частоту и сравниваем с `(n+1)//2`.

---

## Алгоритм (по коду)

1. Используем алгоритм голосования Бойера-Мура для поиска кандидата:
   - Инициализируем `candidate = -1`, `count = 0`.
   - Для каждого символа `ch` в `s`:
     - Если `count == 0`, установить `candidate = ch`, `count = 1`.
     - Иначе если `ch == candidate`, увеличить `count`.
     - Иначе уменьшить `count`.
2. Подсчитать точное количество вхождений `candidate` в строке.
3. Если `count_candidate <= (n + 1) // 2`, вернуть `true`, иначе `false`.

**Обоснование:**  
Кандидат от Бойера-Мура будет наиболее частым символом, если он встречается более чем в половине случаев. В противном случае кандидат может быть любым, но если какой-то символ имеет частоту > n/2, то алгоритм обязательно выдаст именно его. Поэтому проверка условия на этом кандидате достаточна.

## Реализация

```python
class Solution:
    def canRearrange(self, s):
        # Поиск кандидата (алгоритм Бойера-Мура)
        candidate = -1
        count = 0
        for ch in s:
            if count == 0:
                candidate = ch
                count = 1
            elif ch == candidate:
                count += 1
            else:
                count -= 1
        
        # Подсчёт реальной частоты кандидата
        freq = 0
        for ch in s:
            if ch == candidate:
                freq += 1
        
        # Проверка условия
        return freq <= (len(s) + 1) // 2
```

---

## Сложность

- **Время:** O(n) — два прохода по строке.
- **Память:** O(1) — используются только несколько переменных.

---

## Примечания

- Альтернативно можно использовать `collections.Counter` и взять максимальную частоту, но это требует O(n) времени и O(алфавит) памяти, что тоже допустимо. Предложенный метод более экономен по памяти и не зависит от размера алфавита.
- Условие `freq <= (n+1)//2` эквивалентно тому, что максимальная частота не превышает половину с округлением вверх. Для чётного n это `n/2`, для нечётного — `(n+1)/2`.
- Пример: `"aaabb"` (n=5, freq('a')=3, (5+1)//2=3 → true). `"aaaabc"` (n=6, freq('a')=4, (6+1)//2=3 → false).
