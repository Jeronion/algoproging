## Определение

**Задача о выборе активностей** — это классическая задача на жадный алгоритм. Даны наборы начал и концов активностей. Требуется выбрать максимальное количество активностей, которые может выполнить один человек, при условии, что он не может выполнять две активности одновременно (временные интервалы не должны перекрываться). Активности заданы массивами `start[]` и `finish[]`; каждая активность имеет время начала `start[i]` и время окончания `finish[i]`. Человек может начать следующую активность только после окончания предыдущей, причём нельзя начинать в тот же момент, когда закончилась предыдущая (т.е. `finish[i] < start[j]`).

**Пример:**
```
start = [1, 3, 0, 5, 8, 5]
finish = [2, 4, 6, 7, 9, 9]
Максимальное количество активностей: 4 (например, активности 0, 1, 3, 4)
```

---

## Постановка задачи

**Вход:**
- `start[]` — массив времени начала активностей.
- `finish[]` — массив времени окончания активностей (соответствует тому же индексу).
- Оба массива имеют одинаковую длину.

**Выход:** максимальное количество активностей, которые можно выполнить без перекрытия (с учётом условия `finish[i] < start[j]`).

**Ограничения:** 1 ≤ n ≤ 2·10⁵, 0 ≤ start[i] ≤ finish[i] ≤ 10⁹.

---

## Идея решения (жадный алгоритм)

Оптимальная стратегия: **всегда выбирать активность с наименьшим временем окончания** среди ещё не выбранных, которая не перекрывается с уже выбранными. Это стандартный жадный подход, который гарантирует максимальное количество активностей.

### Обоснование

Если мы выберем активность, которая заканчивается раньше, то у нас останется больше времени для остальных активностей. Таким образом, локально оптимальный выбор (самая ранняя окончание) приводит к глобально оптимальному решению.

### Алгоритм

1. Объединить пары `(start[i], finish[i])` и отсортировать их по времени окончания.
2. Выбрать первую активность (она заканчивается раньше всех) и запомнить её индекс.
3. Для каждой последующей активности в отсортированном списке:
   - Если её начало **строго больше** времени окончания последней выбранной активности, то выбрать её (увеличить счётчик и обновить индекс последней выбранной).
4. Вернуть количество выбранных активностей.

---

## Реализация на Python

```python
class Solution:
    def activitySelection(self, start, finish):
        # Объединяем start и finish в пары и сортируем по finish
        activities = list(zip(start, finish))
        activities.sort(key=lambda x: x[1])
        
        count = 1
        last = 0  # индекс последней выбранной активности
        for i in range(1, len(activities)):
            # Если начало текущей активности > окончание последней выбранной
            if activities[i][0] > activities[last][1]:
                count += 1
                last = i
        return count
```

---

## Сложность

- **Время:** O(n log n) из-за сортировки. Проход по отсортированному списку занимает O(n).
- **Память:** O(n) для хранения списка пар (можно также сортировать по finish, используя индексы, но в данном случае память O(n)).

## Пример работы

```
start = [1, 3, 0, 5, 8, 5]
finish = [2, 4, 6, 7, 9, 9]
Пары: (1,2), (3,4), (0,6), (5,7), (8,9), (5,9)
Сортируем по finish: (1,2), (3,4), (0,6), (5,7), (8,9), (5,9)
Выбираем (1,2): last=0, count=1
i=1: (3,4): 3 > 2 → выбираем, count=2, last=1
i=2: (0,6): 0 > 4? нет
i=3: (5,7): 5 > 4 → выбираем, count=3, last=3
i=4: (8,9): 8 > 7 → выбираем, count=4, last=4
i=5: (5,9): 5 > 9? нет
Ответ: 4
```

## Примечания

- В условии сказано, что нельзя начинать новую активность в момент окончания предыдущей, поэтому используется строгое сравнение `>`.
- Если бы разрешалось начинать в тот же момент (т.е. `finish[i] <= start[j]`), то использовалось бы `>=`.
- Задача решается за линейное время после сортировки, что оптимально.
- Если активности не отсортированы, сортировка является основным шагом.
