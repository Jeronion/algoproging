## Определение

**Задача о дробном рюкзаке** — это классическая задача оптимизации, где имеется набор предметов с заданными стоимостью и весом, а также рюкзак ограниченной вместимости. В отличие от дискретного (0/1) рюкзака, здесь разрешается брать **дробную часть** предмета (например, взять 1/2 предмета, если его вес превышает оставшееся место). Цель — максимизировать суммарную стоимость предметов, помещённых в рюкзак, при условии, что общий вес не превышает вместимости.

**Пример:**
```
val = [60, 100, 120], wt = [10, 20, 30], capacity = 50
Оптимальный выбор: взять целиком предметы весом 10 (60) и 20 (100), и 2/3 предмета весом 30 (120) → сумма = 240.
```

---

## Постановка задачи

Даны два массива `val[]` и `wt[]` одинаковой длины, где каждый элемент соответствует стоимости и весу предмета. Задана вместимость рюкзака `capacity`. Необходимо найти максимальную стоимость, которую можно получить, беря предметы целиком или дробно. Ответ должен быть с плавающей точкой (округлён до 6 знаков).

**Ограничения:** 1 ≤ n ≤ 10⁵, 1 ≤ capacity ≤ 10⁹, 1 ≤ val[i], wt[i] ≤ 10⁴.

---

## Идея решения (жадный алгоритм)

В дробном рюкзаке оптимальная стратегия — **всегда брать предметы с наибольшей удельной стоимостью** (стоимость за единицу веса). Это стандартный жадный подход, который гарантирует оптимальность благодаря возможности делить предметы.

### Обоснование

Если мы можем взять часть предмета, то любой предмет с более высокой удельной стоимостью даёт больше выгоды на каждый килограмм веса, чем предмет с меньшей удельной стоимостью. Поэтому, пока есть место, следует брать предметы в порядке убывания отношения `val[i] / wt[i]`.

### Алгоритм

1. Для каждого предмета вычислить отношение `value / weight`.
2. Отсортировать предметы по убыванию этого отношения.
3. Инициализировать `total_value = 0` и оставшуюся вместимость `remaining = capacity`.
4. Для каждого предмета в отсортированном порядке:
   - Если вес предмета `w` ≤ `remaining`, то взять его целиком: `total_value += val`, `remaining -= w`.
   - Иначе взять только часть предмета: долю `fraction = remaining / w`, `total_value += val * fraction`, и завершить алгоритм (рюкзак заполнен).
5. Вернуть `total_value`.

## Реализация на Python

```python
class Solution:
    def fractionalKnapsack(self, val, wt, capacity):
        # Сортируем предметы по убыванию отношения val/wt
        items = sorted(zip(val, wt), key=lambda x: x[0] / x[1], reverse=True)
        res = 0.0
        for v, w in items:
            if capacity <= 0:
                break
            if w <= capacity:
                res += v
                capacity -= w
            else:
                res += v * (capacity / w)
                capacity = 0
        return res
```

**Пояснение:** в коде используется `min(1, capacity / w)` для вычисления доли, но более явное условие с `if` проще для понимания.

---

## Сложность

- **Время:** O(n log n) из-за сортировки.
- **Память:** O(n) для хранения отсортированных предметов (можно сортировать на месте, но для простоты оставляем).

---

## Пример работы

```
val = [60, 100, 120], wt = [10, 20, 30], capacity = 50
Отношения: 60/10=6, 100/20=5, 120/30=4
Сортируем: (60,10), (100,20), (120,30)
- Берём 10: остаток 40, сумма 60
- Берём 20: остаток 20, сумма 160
- Берём 20/30 от 120: 120*(20/30)=80, сумма 240
Ответ: 240.000000
```

---

## Примечания

- В отличие от 0/1 рюкзака, здесь жадный подход даёт оптимальное решение.
- Если все предметы не помещаются, берётся дробь от последнего предмета.
- Результат должен быть возвращён как `float` с 6 знаками после запятой.
- Задача решается за линейное время после сортировки; для больших `n` сортировка является основным фактором.
