## Определение

**Задача о распределении печенья** — это классическая задача на жадный алгоритм. Даны два массива: `greed[]` (минимальный размер печенья, который удовлетворит каждого ребёнка) и `cookie[]` (размеры доступных печений). Каждый ребёнок может получить не более одного печенья. Ребёнок `i` будет удовлетворён, если получит печенье `j` размером `cookie[j] >= greed[i]`. Требуется найти максимальное число детей, которых можно удовлетворить.

**Пример:**
```
greed = [1, 10, 3], cookie = [1, 2, 3] → можно удовлетворить 2 детей (1 и 3).
greed = [10, 100], cookie = [1, 2] → 0.
```

---

## Постановка задачи

**Вход:**
- `greed` — массив целых чисел (минимальные требования детей).
- `cookie` — массив целых чисел (размеры печений).

**Выход:** максимальное количество детей, которых можно удовлетворить.

**Ограничения:** размеры массивов до 10⁵, значения до 10⁹.

---

## Идея решения (жадный алгоритм)

Оптимальная стратегия — **сортировать оба массива по возрастанию**, затем использовать два указателя:

- Один указатель `i` на текущего ребёнка (начиная с самого жадного? На самом деле, мы начинаем с наименьшего требования, чтобы использовать маленькие печенья для детей с малыми потребностями, оставляя большие печенья для более требовательных).
- Второй указатель `j` на текущее печенье.

Если текущее печенье `cookie[j]` может удовлетворить текущего ребёнка (`cookie[j] >= greed[i]`), то мы даём это печенье ребёнку, увеличиваем счётчик и переходим к следующему ребёнку. Иначе это печенье слишком маленькое даже для самого нетребовательного из оставшихся детей, значит, оно никому не подойдёт, и мы переходим к следующему печенью.

Такой подход гарантирует максимальное количество удовлетворённых детей.

### Почему это работает?

Если мы можем удовлетворить ребёнка с наименьшим требованием, мы должны это сделать, используя наименьшее подходящее печенье. Это оставляет большие печенья для детей с большими требованиями, что максимизирует общее количество.

---

## Алгоритм

1. Отсортировать `greed` по возрастанию.
2. Отсортировать `cookie` по возрастанию.
3. Инициализировать `i = 0` (индекс ребёнка), `count = 0`.
4. Для каждого печенья `cookie[j]` (перебирая `j` от 0 до конца):
   - Если `i == len(greed)`, то все дети уже удовлетворены → прервать цикл.
   - Если `cookie[j] >= greed[i]`, то дать это печенье ребёнку: `count++`, `i++`.
5. Вернуть `count`.

---

## Реализация на Python

```python
class Solution:
    def maxChildren(self, greed, cookie):
        greed.sort()
        cookie.sort()
        i = 0
        count = 0
        for c in cookie:
            if i == len(greed):
                break
            if c >= greed[i]:
                count += 1
                i += 1
        return count
```

---

## Сложность

- **Время:** O(n log n + m log m), где n и m — длины массивов `greed` и `cookie` (сортировка). После сортировки проход за O(n+m).
- **Память:** O(1) дополнительно (если не считать память для сортировки, которая обычно O(1) или O(log n) в зависимости от реализации).

## Пример работы

```
greed = [1, 10, 3] → сортируем: [1, 3, 10]
cookie = [1, 2, 3] → сортируем: [1, 2, 3]
i=0, c=1: 1 >= 1 → даём, count=1, i=1
c=2: 2 >= 3? нет → пропускаем
c=3: 3 >= 3 → даём, count=2, i=2
Все печенья просмотрены → ответ 2.
```

---

## Примечания

- Задача является классическим примером использования жадного алгоритма.
- Если печенье слишком маленькое даже для самого нетребовательного ребёнка, оно не может быть использовано ни для кого, поэтому его следует пропустить.
- Важно сортировать оба массива, чтобы применять жадную стратегию.
