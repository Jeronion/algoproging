## Определение

**Задача о минимизации денежного потока** заключается в том, чтобы среди группы людей, у которых есть задолженности друг перед другом, найти способ погашения всех долгов с **минимальным количеством транзакций**. Идея состоит в том, чтобы вместо того, чтобы каждый платил каждому, перераспределить платежи так, чтобы каждый человек совершил минимальное число переводов, а общая сумма переведённых денег осталась неизменной.

**Пример:**
```
Три человека:
0 должен 1: 1000, 0 должен 2: 2000, 1 должен 2: 5000
Итоговый оптимизированный поток:
0 платит 2: 3000, 1 платит 2: 4000
```

---

## Постановка задачи

Дана матрица `transaction`, где `transaction[i][j]` — сумма, которую человек `i` должен человеку `j`. Требуется построить новую матрицу `result`, представляющую минимальное количество переводов, которые нужно совершить, чтобы все долги были погашены. Суммы переводов должны быть неотрицательными, и итоговый баланс каждого человека должен остаться таким же, как в исходной матрице.

**Вход:** квадратная матрица `transaction` размера `n × n` (неотрицательные целые числа, где `transaction[i][i] = 0`).
**Выход:** матрица `result` тех же размеров, где `result[i][j]` — сумма, которую `i` переводит `j` в оптимизированной схеме.

---

## Идея решения (жадный алгоритм с приоритетной очередью)

1. **Вычислить чистый баланс** для каждого человека:
   - `net[i] = сумма полученных денег (входящие платежи) - сумма отданных денег (исходящие платежи)`.
   - Если `net[i] > 0`, человек является **кредитором** (ему должны).
   - Если `net[i] < 0`, человек является **должником** (он должен другим).
   - Если `net[i] == 0`, он не участвует в расчётах.

2. **Разделить на две группы**: должников (отрицательный баланс) и кредиторов (положительный баланс), поместив их в **max-heap** (приоритетные очереди) по абсолютной величине долга/требования. Используем отрицательные значения для имитации max-heap в Python.

3. **Пока есть должники**:
   - Извлечь наибольшего должника (с наибольшей суммой долга) и наибольшего кредитора (с наибольшей суммой к получению).
   - Перевести сумму `min(долг, требование)` от должника к кредитору.
   - Записать эту транзакцию в результирующую матрицу.
   - Уменьшить остатки должника и кредитора на эту сумму.
   - Если остаток должника > 0, вернуть его обратно в кучу; если остаток кредитора > 0, вернуть его обратно.

Этот жадный подход минимизирует количество транзакций, так как на каждом шаге мы закрываем максимально возможную сумму между двумя участниками, что уменьшает общее число переводов.

---

## Алгоритм (пошагово)

1. Инициализировать массив `net` размера `n` нулями.
2. Для каждой пары `(i, j)`:
   - `net[i] -= transaction[i][j]` (отдаёт деньги)
   - `net[j] += transaction[i][j]` (получает деньги)
3. Создать две пустые приоритетные очереди (min-heap с отрицательными значениями):
   - `debtors` для `net[i] < 0` (храним `(-abs(net[i]), i)`)
   - `creditors` для `net[i] > 0` (храним `(-net[i], i)`)
4. Инициализировать `result` нулевой матрицей `n × n`.
5. Пока `debtors` не пуста:
   - Извлечь `debtAmount, debtorId` из `debtors`; `debtAmount = -debtAmount`.
   - Извлечь `creditAmount, creditorId` из `creditors`; `creditAmount = -creditAmount`.
   - `transfer = min(debtAmount, creditAmount)`.
   - Записать `result[debtorId][creditorId] = transfer`.
   - Обновить остатки: `debtAmount -= transfer`, `creditAmount -= transfer`.
   - Если `debtAmount > 0`, вернуть `( -debtAmount, debtorId)` в `debtors`.
   - Если `creditAmount > 0`, вернуть `(-creditAmount, creditorId)` в `creditors`.
6. Вернуть `result`.

## Реализация на Python

```python
import heapq

def minCashFlow(transaction):
    n = len(transaction)
    net = [0] * n

    # Вычисление чистого баланса
    for i in range(n):
        for j in range(n):
            net[i] -= transaction[i][j]   # i отдаёт j
            net[j] += transaction[i][j]   # j получает от i

    debtors = []   # ( -долг, индекс )
    creditors = [] # ( -требование, индекс )

    for i in range(n):
        if net[i] < 0:
            heapq.heappush(debtors, (-abs(net[i]), i))
        elif net[i] > 0:
            heapq.heappush(creditors, (-net[i], i))

    result = [[0] * n for _ in range(n)]

    while debtors:
        debtAmount, debtorId = heapq.heappop(debtors)
        creditAmount, creditorId = heapq.heappop(creditors)
        debtAmount = -debtAmount
        creditAmount = -creditAmount

        transfer = min(debtAmount, creditAmount)
        result[debtorId][creditorId] = transfer

        debtAmount -= transfer
        creditAmount -= transfer

        if debtAmount > 0:
            heapq.heappush(debtors, (-debtAmount, debtorId))
        if creditAmount > 0:
            heapq.heappush(creditors, (-creditAmount, creditorId))

    return result
```

## Сложность

- **Время:** O(n² + k log n), где n — число людей, k — количество транзакций в оптимизированном решении. В худшем случае k = O(n). Основная часть — вычисление чистого баланса O(n²), затем каждая операция с кучей O(log n).
- **Память:** O(n²) для хранения результирующей матрицы, O(n) для куч и массива `net`.

---

## Примечания

- Алгоритм гарантирует, что сумма всех переводов останется неизменной и каждый человек получит/отдаст столько же, сколько в исходной схеме.
- Количество транзакций минимизируется, так как на каждом шаге мы максимально уменьшаем остаток одного участника.
- Если несколько должников или кредиторов имеют одинаковые суммы, порядок извлечения не влияет на итоговое количество транзакций (но может влиять на конкретные суммы переводов).
