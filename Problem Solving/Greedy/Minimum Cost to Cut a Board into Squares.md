## Определение

Дана доска размером `n × m` (количество строк и столбцов), которую нужно разрезать на `n*m` единичных квадратов. Разрезы можно делать вдоль вертикальных и горизонтальных линий. Массивы `x[]` и `y[]` содержат стоимости разрезания по каждой вертикальной и горизонтальной линии соответственно (в произвольном порядке, но каждую линию можно разрезать только один раз). При выполнении разреза его стоимость умножается на текущее количество сегментов в противоположном направлении (поскольку разрез пересекает все уже образовавшиеся части). Требуется найти минимальную общую стоимость, чтобы разрезать доску на квадраты.

**Пример:**
```
n = 4, m = 6, x = [2,1,3,1,4], y = [4,1,2]
Вывод: 42
```

---

## Идея решения

Это классическая **жадная задача**. Дорогие разрезы следует выполнять как можно раньше, потому что они будут умножены на меньшее количество сегментов в противоположном направлении.

- Каждый разрез по вертикали увеличивает количество вертикальных сегментов (`v`), а каждый разрез по горизонтали увеличивает количество горизонтальных сегментов (`h`). Изначально `h = 1`, `v = 1` (вся доска — один сегмент в каждом направлении).
- Стоимость вертикального разреза с ценой `cost` равна `cost * h` (пересекает все текущие горизонтальные сегменты).
- Стоимость горизонтального разреза с ценой `cost` равна `cost * v` (пересекает все текущие вертикальные сегменты).

Чтобы минимизировать общую стоимость, нужно на каждом шаге выбирать разрез с **наибольшей стоимостью** из оставшихся (как вертикальных, так и горизонтальных), так как это уменьшит коэффициент умножения для всех последующих разрезов (или, по крайней мере, для более дорогих). Это стандартная стратегия: отсортировать оба массива по убыванию и затем выполнять разрезы в порядке убывания стоимости, выбирая больший из двух текущих максимумов.

### Алгоритм

1. Отсортировать `x` и `y` по убыванию.
2. Инициализировать `h = 1`, `v = 1`, `total = 0`, указатели `i = 0` (по x), `j = 0` (по y).
3. Пока есть неиспользованные разрезы в обоих массивах:
   - Если `x[i] > y[j]`, то выполняем вертикальный разрез: `total += x[i] * h`, увеличиваем `v`, сдвигаем `i`.
   - Иначе выполняем горизонтальный разрез: `total += y[j] * v`, увеличиваем `h`, сдвигаем `j`.
4. Если остались разрезы только в одном массиве, добавляем их стоимость с умножением на текущее количество сегментов противоположного направления.
5. Возвращаем `total`.

## Реализация

```python
class Solution:
    def minCost(self, n, m, x, y):
        # Сортируем по убыванию
        x.sort(reverse=True)
        y.sort(reverse=True)

        h, v = 1, 1          # текущее количество сегментов
        i, j = 0, 0
        total = 0

        # Пока есть разрезы в обоих направлениях
        while i < m - 1 and j < n - 1:
            if x[i] > y[j]:
                total += x[i] * h
                v += 1
                i += 1
            else:
                total += y[j] * v
                h += 1
                j += 1

        # Остались вертикальные разрезы
        while i < m - 1:
            total += x[i] * h
            i += 1

        # Остались горизонтальные разрезы
        while j < n - 1:
            total += y[j] * v
            j += 1

        return total
```

---

## Сложность

- **Время:** O((n+m) log(n+m)) — из-за сортировки массивов.
- **Память:** O(1) дополнительно (не считая входных массивов).

## Пример работы

Для `n = 4`, `m = 6`:
- `x = [2,1,3,1,4]` → сортировка: `[4,3,2,1,1]`
- `y = [4,1,2]` → сортировка: `[4,2,1]`
- Итерации:
  - 4 (x) > 4 (y)? равно, берём y: `total += 4*1 = 4`, `h=2`, `v=1`
  - 4 (x) > 2 (y)? берём x: `total += 4*2 = 8`, `v=2`, `h=2`
  - 3 (x) > 2 (y)? берём x: `total += 3*2 = 6`, `v=3`
  - 2 (x) > 2 (y)? берём x (можно y, но равенство): `total += 2*2 = 4`, `v=4`
  - 1 (x) > 2 (y)? берём y: `total += 2*4 = 8`, `h=3`
  - 1 (x) > 1 (y)? берём x: `total += 1*3 = 3`, `v=5`
  - 1 (x) > 1 (y)? берём x: `total += 1*3 = 3`, `v=6`
  - остался y=1: `total += 1*6 = 6`
- Итого: 4+8+6+4+8+3+3+6 = 42.
