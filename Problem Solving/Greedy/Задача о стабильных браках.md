## Определение

**Задача о стабильных браках** — это задача нахождения стабильного паросочетания между двумя множествами равного размера (например, мужчинами и женщинами), где каждый участник имеет полный порядок предпочтений по отношению к представителям противоположного множества. Паросочетание называется **стабильным**, если не существует пары (мужчина, женщина), которые оба предпочли бы друг друга своим текущим партнёрам. Если такая пара существует, она называется «нестабильной» и может разрушить текущее паросочетание.

**Пример:**
```
Мужчины: m0, m1, m2
Женщины: w0, w1, w2
Предпочтения мужчин:
m0: [w1, w0, w2]
m1: [w2, w1, w0]
m2: [w0, w2, w1]
Предпочтения женщин:
w0: [m1, m0, m2]
w1: [m2, m1, m0]
w2: [m0, m2, m1]
Одно из стабильных паросочетаний: [1, 2, 0] (m0→w1, m1→w2, m2→w0)
```

---

## Постановка задачи

**Вход:**
- `n` — количество мужчин и женщин (нумерация от 0 до n-1).
- `men` — список предпочтений каждого мужчины (список индексов женщин в порядке убывания предпочтений).
- `women` — список предпочтений каждой женщины (аналогично).

**Выход:** массив `result` длины `n`, где `result[i]` — индекс женщины, назначенной мужчине `i`. Если возможно несколько стабильных паросочетаний, возвращается то, которое оптимально для мужчин (man-optimal).

**Примечание:** Гарантируется, что решение существует.

---

## Идея решения (алгоритм Гейла-Шепли)

Алгоритм Гейла-Шепли (Gale-Shapley) — это классический жадный алгоритм, который всегда находит стабильное паросочетание. Он работает следующим образом:

1. Все мужчины и женщины изначально свободны.
2. Пока есть свободный мужчина, который ещё не делал предложения всем женщинам:
   - Он делает предложение женщине, которую он предпочитает больше всех среди тех, кому ещё не предлагал.
   - Если женщина свободна, она принимает предложение, и они становятся парой.
   - Если женщина уже занята, она сравнивает своего текущего партнёра с новым предложившим. Если она предпочитает нового, она принимает его предложение, а текущий партнёр становится свободным; иначе предложение отклоняется.
3. Процесс продолжается, пока все мужчины не будут заняты.

Алгоритм гарантирует стабильность и завершается за O(n²) шагов. При этом получаемое паросочетание является оптимальным для мужчин (каждый мужчина получает наилучшую возможную среди всех стабильных паросочетаний).

---

## Алгоритм (пошагово)

1. Инициализировать:
   - `w_partner[w] = -1` (текущий партнёр каждой женщины, -1 означает свободна).
   - `m_partner[m] = -1` (текущий партнёр каждого мужчины).
   - `next_proposal[m] = 0` (индекс следующей женщины в списке предпочтений мужчины).
   - `free_man[m] = True` (свободен ли мужчина).
   - `free_count = n` (количество свободных мужчин).
2. Пока `free_count > 0`:
   - Найти любого свободного мужчину `m`.
   - `w = men[m][next_proposal[m]]` (следующая женщина по предпочтениям).
   - Увеличить `next_proposal[m]`.
   - Если `w_partner[w] == -1` (женщина свободна):
     - Обручить `m` и `w`: `w_partner[w] = m`, `m_partner[m] = w`.
     - `free_man[m] = False`, `free_count -= 1`.
   - Иначе (женщина занята мужчиной `m1`):
     - Если женщина предпочитает `m` своему текущему партнёру `m1`:
       - `w_partner[w] = m`, `m_partner[m] = w`.
       - `free_man[m] = False`, `free_man[m1] = True` (предыдущий партнёр становится свободным).
3. Вернуть `m_partner`.

## Реализация на Python

```python
from typing import List

def prefers(women: List[List[int]], w: int, m: int, m1: int) -> bool:
    """Возвращает True, если женщина w предпочитает мужчину m мужчине m1."""
    for candidate in women[w]:
        if candidate == m:
            return True
        if candidate == m1:
            return False
    return False  # никогда не достигается

def stableMarriage(men: List[List[int]], women: List[List[int]]) -> List[int]:
    n = len(men)
    w_partner = [-1] * n       # партнёр женщины
    m_partner = [-1] * n       # партнёр мужчины
    next_proposal = [0] * n    # следующее предложение для каждого мужчины
    free_man = [True] * n      # свободен ли мужчина
    free_count = n

    while free_count > 0:
        # Находим свободного мужчину
        m = next(i for i in range(n) if free_man[i])
        w = men[m][next_proposal[m]]
        next_proposal[m] += 1

        if w_partner[w] == -1:   # женщина свободна
            w_partner[w] = m
            m_partner[m] = w
            free_man[m] = False
            free_count -= 1
        else:
            m1 = w_partner[w]
            if prefers(women, w, m, m1):
                # женщина предпочитает нового мужчину
                w_partner[w] = m
                m_partner[m] = w
                free_man[m] = False
                free_man[m1] = True   # предыдущий партнёр становится свободным

    return m_partner

# Пример использования
if __name__ == "__main__":
    n = 3
    men = [
        [1, 0, 2],
        [2, 1, 0],
        [0, 2, 1]
    ]
    women = [
        [1, 0, 2],
        [2, 1, 0],
        [0, 2, 1]
    ]
    result = stableMarriage(men, women)
    print(result)  # [1, 2, 0]
```

## Сложность

- **Время:** O(n²) — каждый мужчина делает предложение каждой женщине не более одного раза, а сравнение предпочтений женщины (при проверке) также O(n) в худшем случае, но в среднем O(1). Общая сложность O(n²).
- **Память:** O(n²) для хранения предпочтений (или O(n) для дополнительных массивов, если предпочтения уже даны).

---

## Примечания

- Алгоритм Гейла-Шепли гарантирует стабильность и всегда находит решение.
- Результат зависит от того, кто делает предложения: если мужчины, то получается man-optimal (наилучшее для мужчин), если женщины — woman-optimal.
- В случае нескольких стабильных решений алгоритм выдаёт одно из них (конкретное зависит от порядка обработки свободных мужчин).
- Задача имеет множество применений: распределение студентов по университетам, назначение задач исполнителям, формирование пар в соревнованиях и т.д.
