## Определение

Дан односвязный список. Требуется определить, содержит ли он **цикл** (loop), т.е. существует ли узел, который можно достичь снова, следуя по указателям `next`, не доходя до `None`.

**Пример:**
```
1 → 3 → 4 → 3 (ссылка обратно на 3) → true
1 → 8 → 3 → 4 → None → false
```

---

## Идея решения (алгоритм Флойда)

Используются два указателя: `slow` (черепаха) перемещается на один шаг за раз, `fast` (заяц) — на два шага. Если в списке есть цикл, то `fast` и `slow` обязательно встретятся внутри цикла (догонят друг друга). Если цикла нет, `fast` достигнет `None`.

### Алгоритм

1. Инициализировать `slow = head`, `fast = head`.
2. Пока `fast` и `fast.next` не равны `None`:
   - `slow = slow.next`
   - `fast = fast.next.next`
   - Если `slow == fast`, то цикл найден → вернуть `True`.
3. Если цикл не обнаружен, вернуть `False`.

---

## Реализация на Python

```python
class Node:
    def __init__(self, x):
        self.data = x
        self.next = None

def detect_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            return True
    return False
```

---

## Сложность

- **Время:** O(n) — каждый узел посещается `slow` и `fast` ограниченное число раз.
- **Память:** O(1) — только два указателя.

---

## Примечания

- Алгоритм Флойда не требует дополнительной памяти и работает за линейное время.
- Если нужно не только обнаружить цикл, но и найти его начало или удалить, используются модификации того же метода.
- Наивный подход с хеш-таблицей (множеством) также возможен, но требует O(n) памяти и менее эффективен.
