## Определение

Дан односвязный список, в котором каждый узел содержит два указателя: `next` (на следующий узел) и `random` (на любой произвольный узел в том же списке, может быть `None`). Требуется создать **глубокую копию** (клон) этого списка – новый список, состоящий из новых узлов с теми же значениями и идентичными связями `next` и `random`. Исходный список не должен быть изменён.

**Пример:**
```
Оригинал: 1 -> 2 -> 3 -> 4 -> 5
random: 1→3, 2→1, 3→5, 4→3, 5→2
Клон должен иметь точно такие же связи.
```

---

## 1. Решение с использованием хеш-таблицы (O(n) времени, O(n) памяти)

### Идея

- Создать словарь (hash map), который для каждого узла исходного списка будет хранить соответствующий новый узел.
- Первый проход: создать все новые узлы и сохранить пару `(оригинал -> клон)`.
- Второй проход: для каждого исходного узла установить `next` и `random` клона, используя значения из словаря.
- Вернуть голову клона.

### Реализация

```python
class Node:
    def __init__(self, x):
        self.data = x
        self.next = None
        self.random = None

def clone_list_hash(head):
    if not head:
        return None
    # Словарь: оригинальный узел -> новый узел
    node_map = {}
    curr = head
    while curr:
        node_map[curr] = Node(curr.data)
        curr = curr.next
    curr = head
    while curr:
        clone = node_map[curr]
        clone.next = node_map.get(curr.next)
        clone.random = node_map.get(curr.random)
        curr = curr.next
    return node_map[head]
```

### Сложность

- **Время:** O(n) – два прохода по списку.
- **Память:** O(n) – словарь из n записей.

## 2. Оптимальное решение без дополнительной памяти (in-place) – O(n) времени, O(1) памяти

### Идея

1. **Вставка клонов:** Для каждого узла `X` создать его копию `X'` и вставить сразу после `X` (так, что `X -> X' -> следующий`).
2. **Установка random:** Для каждого клона `X'` установить `random` на `X.random.next` (если `X.random` существует), потому что `random` исходного узла указывает на какой-то узел `Y`, а после вставки его клон находится сразу за `Y`.
3. **Разделение списков:** Пройти по списку и восстановить исходные связи `next`, одновременно извлекая клоны в отдельный список.

### Реализация

```python
def clone_list_in_place(head):
    if not head:
        return None
    
    # Шаг 1: вставляем клоны между исходными узлами
    curr = head
    while curr:
        nxt = curr.next
        clone = Node(curr.data)
        curr.next = clone
        clone.next = nxt
        curr = nxt
    
    # Шаг 2: устанавливаем random для клонов
    curr = head
    while curr:
        if curr.random:
            curr.next.random = curr.random.next
        curr = curr.next.next
    
    # Шаг 3: разделяем списки
    curr = head
    clone_head = head.next
    while curr:
        clone = curr.next
        curr.next = clone.next
        clone.next = clone.next.next if clone.next else None
        curr = curr.next
    return clone_head
```

### Сложность

- **Время:** O(n) – три линейных прохода.
- **Память:** O(1) – дополнительные указатели, не считая памяти под новые узлы (которая всё равно нужна для результата).

---

## Примечания

- Оба подхода создают новый список, не изменяя исходный.
- Метод с хеш-таблицей проще для понимания и реализации, но требует дополнительной памяти.
- In-place метод более экономичен по памяти и часто ожидается на собеседованиях.
