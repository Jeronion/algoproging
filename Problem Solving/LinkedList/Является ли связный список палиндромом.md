## Определение

Дан односвязный список из целых чисел. Требуется определить, является ли он **палиндромом**, т.е. читается ли он одинаково слева направо и справа налево.

**Пример:**
```
1 → 2 → 1 → 1 → 2 → 1 → None  → true
10 → 20 → 30 → 40 → 50 → None → false
```

---

## Идея решения (O(n) времени, O(1) памяти)

Алгоритм состоит из трёх шагов:

1. **Найти середину** списка с помощью быстрого и медленного указателей (tortoise and hare). Для чётной длины — первый узел второй половины.
2. **Развернуть вторую половину** списка (начиная с узла после середины).
3. **Сравнить** элементы первой половины (от головы) с элементами развёрнутой второй половины. Если все пары совпали — палиндром.

### Детали

- Поиск середины: `slow` и `fast` стартуют с `head`. Пока `fast.next` и `fast.next.next` существуют, сдвигаем. В конце `slow` указывает на узел, после которого начинается вторая половина (для нечётной длины средний узел остаётся в первой половине).
- Разворот второй половины: стандартный алгоритм.
- Сравнение: проходим по двум спискам одновременно, пока не закончится вторая половина (она короче или равна первой). Если хотя бы одно значение не совпало — `False`, иначе `True`.

**Важно:** После разворота первая половина остаётся неизменной, вторая развёрнута. Сравниваем их узлы.

---

## Реализация

```python
class Solution:
    def isPalindrome(self, head):
        # Функция разворота списка
        def reverse(head):
            prev = None
            cur = head
            while cur:
                nxt = cur.next
                cur.next = prev
                prev = cur
                cur = nxt
            return prev
        
        # 1. Находим середину
        slow = fast = head
        while fast.next and fast.next.next:
            slow = slow.next
            fast = fast.next.next
        
        # 2. Разворачиваем вторую половину
        second_half = reverse(slow.next)
        slow.next = None   # разрываем связь (необязательно, но помогает)
        
        # 3. Сравниваем
        left = head
        right = second_half
        while right:
            if left.data != right.data:
                return False
            left = left.next
            right = right.next
        return True
```

---

## Сложность

- **Время:** O(n) — один проход для поиска середины, один для разворота, один для сравнения.
- **Память:** O(1) — константное количество указателей.

---

## Примечания

- Алгоритм изменяет вторую половину списка (разворачивает её на месте). Если исходный список нельзя менять, нужно предварительно скопировать значения в массив, но это потребует O(n) памяти.
- Для нечётной длины средний элемент не участвует в сравнении (он остаётся в первой половине).
- Обработка пустого списка или списка из одного узла: алгоритм корректно отработает (второй половины нет, цикл сравнения не выполнится, вернётся `True`).
