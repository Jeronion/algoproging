## Определение

Дан односвязный список и целое число `k`. Требуется выполнить **левый поворот** (вращение) списка на `k` позиций, то есть переместить первые `k` узлов в конец в том же порядке. Если `k` больше длины списка, берётся `k % length`. Вернуть голову изменённого списка.

**Пример:**
```
Вход: 10 → 20 → 30 → 40 → 50 , k = 4
Выход: 50 → 10 → 20 → 30 → 40
Пояснение:
После 1 поворота: 20→30→40→50→10
После 2: 30→40→50→10→20
После 3: 40→50→10→20→30
После 4: 50→10→20→30→40
```

---

## Идея решения

1. Найти длину списка `n` и последний узел `end`.
2. Нормализовать `k = k % n`. Если `k == 0`, поворот не требуется – вернуть исходную голову.
3. Найти узел, который станет **новым хвостом** – это узел на позиции `(n - k - 1)` при 0-индексации (то есть пройти от головы `n - k - 1` шагов). Обозначим его `cur`.
4. Новый голова – это `ans = cur.next`.
5. Разорвать связь: `cur.next = None`.
6. Присоединить старую голову в конец: `end.next = head`.
7. Вернуть `ans`.

---

## Реализация

```python
class Solution:
    def rotate(self, head, k):
        if head is None:
            return head
        
        # Находим длину и последний узел
        n = 1
        end = head
        while end.next:
            n += 1
            end = end.next
        
        k %= n
        if k == 0:
            return head
        
        # Ищем новый хвост (n - k - 1 шагов)
        cur = head
        for _ in range(n - k - 1):
            cur = cur.next
        
        ans = cur.next
        cur.next = None
        end.next = head
        return ans
```

---

## Сложность

- **Время:** O(n) – один проход для вычисления длины, один для поиска нового хвоста.
- **Память:** O(1) – только несколько указателей.

---

## Примечания

- Обрабатывается случай `k = 0` или `k = n` – список не меняется.
- Если `k` очень большое (до 10⁹), нормализация через `% n` обязательна.
- Алгоритм работает для списков длиной 1 и более.
