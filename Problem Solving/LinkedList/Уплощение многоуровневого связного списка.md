## Определение

Дан многоуровневый связный список, в котором каждый узел имеет два указателя: `next` (следующий узел на том же уровне) и `down` (указатель на дочерний список). Требуется **уплостить** список так, чтобы все узлы оказались в односвязном списке, причём сначала идут узлы **первого уровня**, затем узлы **второго уровня**, затем третьего и т.д. (обход в глубину). Исходная структура не сохраняется.

**Пример:**
```
5 → 10 → 19 → 28
↓         ↓
7         22
↓         ↓
8         50
↓
30

Уплощённый список: 5 → 7 → 8 → 30 → 10 → 19 → 22 → 50 → 28
```

---

## Идея решения

Необходимо выполнить **обход в глубину (DFS)** с посещением сначала `down`, затем `next` (или наоборот, в зависимости от порядка). В данной задаче требуется, чтобы элементы более глубоких уровней появлялись **сразу после своего родителя** (depth‑wise). Это значит, что при обходе узла мы должны сначала целиком обработать его дочерний список (рекурсивно), а затем переходить к следующему узлу.

Два основных способа:

1. **Рекурсивный** – естественно выражает DFS.
2. **С использованием стека** – итеративная версия, позволяет избежать переполнения стека для очень глубоких списков.

---

## 1. Стековый подход (итеративный)

Используем явный стек, чтобы имитировать DFS. Добавляем узлы в порядке, обратном желаемому обходу (чтобы правильно сохранить порядок `down` перед `next`).

Алгоритм:

1. Создать стек, положить в него `head`.
2. Инициализировать `prev = None`.
3. Пока стек не пуст:
   - Извлечь `curr` из стека.
   - Сначала положить в стек `curr.next` (если есть), затем `curr.down` (если есть) — порядок важен: чтобы `down` обработался раньше, его нужно класть **последним** (так как стек LIFO).
   - Связать `prev` с `curr` через `next`.
   - Обновить `prev = curr`.
4. В конце `prev` указывает на последний узел.

### Реализация

```python
def flatten_list(head):
    if head is None:
        return
    stack = [head]
    prev = None
    while stack:
        curr = stack.pop()
        # Сначала кладём next, потом down (чтобы down был сверху)
        if curr.next:
            stack.append(curr.next)
        if curr.down:
            stack.append(curr.down)
        if prev:
            prev.next = curr
        prev = curr
```

### Сложность

- **Время:** O(N) – каждый узел обрабатывается один раз.
- **Память:** O(K) для стека, где K – максимальное количество узлов, одновременно находящихся в стеке (может быть до N в худшем случае).

---

## Сравнение подходов

| Подход      | Память         | Риск переполнения | Код                |
|-------------|----------------|-------------------|--------------------|
| Рекурсивный | O(D) (глубина) | возможен          | прост и элегантен  |
| Стековый    | O(N) в худшем  | отсутствует       | чуть сложнее       |

## Примечания

- Оба метода модифицируют исходную структуру (поле `next`) и по окончании `down` указатели не обнуляются. Если требуется сохранить исходное состояние, нужно создать копии.
- При использовании стека порядок добавления `next` и `down` важен: так как мы хотим сначала обработать `down`, то кладём его в стек **после** `next`, чтобы он оказался на вершине.
- В приведённом рекурсивном коде используется список `prev` для передачи по ссылке, так как в Python нельзя изменить внешнюю переменную внутри функции без `nonlocal` (а здесь функция определена не внутри другой функции).
