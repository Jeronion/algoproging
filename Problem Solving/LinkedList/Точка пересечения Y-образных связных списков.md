## Определение

Даны два односвязных списка, которые в некоторой точке (включая эту точку) сливаются в один общий список (образуют Y-образную форму). Требуется найти узел, начиная с которого списки пересекаются (общий узел). Гарантируется, что пересечение существует.

**Пример:**
```
head1: 10 → 15 → 30 → None
head2: 3 → 6 → 9 → 15 → 30 → None
Точка пересечения: узел со значением 15.
```

---

## Идея решения (сведение к поиску цикла)

Алгоритм временно соединяет два списка в **кольцо**, затем находит начало этого кольца (точку пересечения) с помощью алгоритма Флойда.

### Шаги

1. **Соединить списки** – пройти до конца первого списка и присоединить его `next` к голове второго списка. Теперь структура стала замкнутой (цикл), причём начало цикла находится в точке пересечения исходных списков.
2. **Найти цикл** – применить алгоритм Флойда (быстрый и медленный указатели) к `head1`. Если цикл не обнаружен (нет пересечения), вернуть `None`. По условию пересечение есть, но на всякий случай проверяем.
3. **Найти начало цикла** – когда `slow` и `fast` встретились, один из указателей сбросить на `head1` и двигать оба по одному шагу до их новой встречи. Эта точка и будет началом цикла, т.е. узлом пересечения.
4. (Восстановление исходной структуры не требуется, так как задача не обязывает; в коде оно не выполняется.)

---

## Реализация

```python
class Solution:
    def intersect_point(self, head1, head2):
        if not head1 or not head2:
            return None
        
        # 1. Соединяем конец первого списка с началом второго
        temp = head1
        while temp.next:
            temp = temp.next
        temp.next = head2
        
        # 2. Поиск цикла методом Флойда
        slow = fast = head1
        has_cycle = False
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            if slow == fast:
                has_cycle = True
                break
        
        if not has_cycle:
            return None
        
        # 3. Поиск начала цикла (точки пересечения)
        slow = head1
        while slow != fast:
            slow = slow.next
            fast = fast.next
        return slow
```

---

## Сложность

- **Время:** O(n + m), где n и m – длины списков. Проход до конца первого списка, затем алгоритм Флойда O(n+m).
- **Память:** O(1) – только указатели.

---

## Примечания

- Алгоритм **модифицирует** исходные списки (временно создаёт цикл). Если требуется сохранить их структуру, перед возвратом нужно восстановить последний узел первого списка, установив его `next = None`.
- По условию задачи гарантируется наличие пересечения, поэтому случай `has_cycle == False` обрабатывается, но не должен возникать.
- Альтернативные решения: метод разницы длин (выравнивание указателей) не требует модификации и также работает за O(n+m) с O(1) памяти.
