## Определение

Подстрока — это непрерывная последовательность символов внутри строки. Для строки длины `n` существует `n * (n + 1) / 2` непустых подстрок. Требуется сгенерировать и вывести (или сохранить) все возможные подстроки.

**Пример:** для строки `"abc"` подстроки: `"a"`, `"ab"`, `"abc"`, `"b"`, `"bc"`, `"c"`.

---

## Итеративный подход (вложенные циклы)

Самый простой способ: внешний цикл фиксирует начальный индекс, внутренний — конечный.

```python
def all_substrings_iterative(s: str):
    n = len(s)
    res = []
    for i in range(n):
        for j in range(i, n):
            res.append(s[i:j+1])
    return res
```

**Как это работает:**
- `i` — начало подстроки (от 0 до n-1)
- `j` — конец подстроки (от i до n-1)
- Срез `s[i:j+1]` берёт подстроку.

**Сложность:**
- Время: O(n³) из-за того, что создание среза в Python занимает O(длина подстроки). На самом деле в большинстве реализаций строк срезы работают за O(k), где k — длина среза, поэтому суммарно O(n³). Можно улучшить до O(n²), если не копировать подстроки, а только выводить индексы.
- Память: O(n³) для хранения всех подстрок (если сохранять). Если только выводить, память O(1).

## Рекурсивный подход

Рекурсивно строим подстроки, добавляя по одному символу.

```python
def all_substrings_recursive(s: str):
    res = []
    def backtrack(start, current):
        if start == len(s):
            return
        # Добавляем следующий символ
        current += s[start]
        res.append(current)
        # Продолжаем с тем же началом, увеличивая длину
        backtrack(start + 1, current)
        # Начинаем новую подстроку со следующего индекса
        # (это происходит в основном вызове)
    # Запуск для каждой стартовой позиции
    for i in range(len(s)):
        backtrack(i, "")
    return res
```

Более лаконичная рекурсия (без внешнего цикла) возможна, но менее наглядна.

**Сложность:** аналогична итеративному методу — O(n³) времени и O(n³) памяти.

## Оптимизация (генерация без копирования)

Если требуется только вывести подстроки, можно обойтись без создания новых строк:

```python
def print_substrings(s):
    n = len(s)
    for i in range(n):
        for j in range(i, n):
            # печатаем символы от i до j
            for k in range(i, j+1):
                print(s[k], end='')
            print()
```

Этот вариант использует O(1) дополнительной памяти, но время остаётся O(n³).

## Использование генераторов (Python)

```python
def substrings_gen(s):
    n = len(s)
    for i in range(n):
        for j in range(i, n):
            yield s[i:j+1]

# Использование
for sub in substrings_gen("abc"):
    print(sub)
```

## Сложность

- **Время:** O(n³) в худшем случае из-за копирования символов при создании подстрок.  
  В языках с неизменяемыми строками (Python, Java) создание новой строки требует O(L) операций, где L — длина подстроки. Суммарно все подстроки дают O(n³).  
  Можно достичь O(n²) если использовать специальные структуры (например, суффиксный массив или дерево), но для простых задач это избыточно.

- **Память:** O(n³) если сохранять все подстроки. Если только выводить, то O(1) дополнительной памяти.

---

## Примечания

- Количество подстрок растёт квадратично: для n = 1000 это около 500 000 подстрок, что может быть проблемой по памяти.
- Задача часто встречается в учебных целях для понимания вложенных циклов и рекурсии.
- В реальных задачах обычно не нужно генерировать все подстроки, а достаточно эффективно обрабатывать их (например, с помощью скользящего окна или префиксных хешей).
