## Определение

Даны массив `job[]`, где `job[i]` — время выполнения i-й задачи, и целое число `k` — количество идентичных исполнителей. Каждый исполнитель тратит `t` единиц времени на выполнение одной единицы работы. Задачи должны быть распределены между исполнителями **непрерывными блоками** (подмассивами) без разбиения отдельных задач. Требуется найти **минимальное общее время**, за которое все задачи могут быть выполнены, если исполнители работают параллельно. Время выполнения для исполнителя равно сумме времён его задач, умноженной на `t` (так как каждый исполнитель работает со скоростью 1 единица работы за `t` единиц времени).

**Пример:**
```
job = [10, 7, 8, 12, 6, 8], k = 4, t = 5
Оптимальное распределение:
[10] → 10*5 = 50
[7,8] → 15*5 = 75
[12] → 12*5 = 60
[6,8] → 14*5 = 70
Максимальное время = 75 → ответ 75.
```

---

## Постановка задачи

**Вход:**
- `job[]` — массив положительных целых чисел (время каждой задачи).
- `k` — количество исполнителей.
- `t` — время выполнения одной единицы работы каждым исполнителем.

**Выход:** минимальное возможное максимальное время (с учётом множителя `t`), которое может потребоваться для выполнения всех задач при оптимальном разбиении на `k` непрерывных групп.

**Ограничения:** задачи нельзя переставлять, можно только разбивать массив на `k` подмассивов.

---

## Идея решения (бинарный поиск по ответу)

Задача эквивалентна поиску минимального значения `X` (времени без учёта множителя `t`), такого что задачи можно разбить на `k` групп, каждая из которых имеет сумму ≤ `X`. Тогда итоговое время = `X * t`.

- **Нижняя граница (`low`)** — максимальная задача (так как хотя бы одна задача должна быть выполнена, и её время не может быть меньше).
- **Верхняя граница (`high`)** — сумма всех задач (можно отдать всё одному исполнителю).

Используем бинарный поиск по `X`. Для каждого `mid` проверяем, можно ли разбить массив на не более чем `k` групп с суммой ≤ `mid`. Проверка выполняется жадным способом: идём по массиву, накапливая сумму в текущей группе. Если добавление следующей задачи превышает `mid`, начинаем новую группу и увеличиваем счётчик. Если количество групп ≤ `k`, то `mid` подходит.

После нахождения минимального `X`, умножаем на `t` и возвращаем результат.

### Почему это работает?

- Бинарный поиск по монотонной функции (возможность разбиения на `k` групп монотонна относительно `X`).
- Жадная проверка оптимальна, так как мы не можем переставлять задачи, и для заданного ограничения `X` наилучшая стратегия — набирать максимально возможную сумму в каждой группе, не превышая `X`.

## Алгоритм (пошагово)

1. Вычислить `total = sum(job)` и `max_job = max(job)`.
2. Установить `low = max_job`, `high = total`.
3. Инициализировать `ans = total` (верхняя граница).
4. Пока `low <= high`:
   - `mid = (low + high) // 2`.
   - Вызвать функцию `can_assign(mid)`:
     - `groups = 1`, `current_sum = 0`.
     - Пройти по `job`:
       - Если `current_sum + job[i] > mid`, то начать новую группу: `groups++`, `current_sum = job[i]` (если `job[i] > mid`, то `mid` слишком мал, можно сразу вернуть `False`).
       - Иначе `current_sum += job[i]`.
     - Вернуть `groups <= k`.
   - Если `can_assign(mid)` истинно, обновить `ans = min(ans, mid)` и `high = mid - 1`.
   - Иначе `low = mid + 1`.
5. Вернуть `ans * t`.

---

## Реализация на Python

```python
def can_assign(jobs, limit, k):
    groups = 1
    current_sum = 0
    for job in jobs:
        if job > limit:        # задача сама по себе больше лимита
            return False
        if current_sum + job > limit:
            groups += 1
            current_sum = job
        else:
            current_sum += job
    return groups <= k

def min_time_to_finish(jobs, k, t):
    low = max(jobs)
    high = sum(jobs)
    ans = high
    while low <= high:
        mid = (low + high) // 2
        if can_assign(jobs, mid, k):
            ans = min(ans, mid)
            high = mid - 1
        else:
            low = mid + 1
    return ans * t

# Пример
jobs = [10, 7, 8, 12, 6, 8]
k = 4
t = 5
print(min_time_to_finish(jobs, k, t))  # 75
```

---

## Сложность

- **Время:** O(n log(total)), где `total = sum(job)`. Бинарный поиск выполняется O(log total) итераций, каждая проверка O(n).
- **Память:** O(1) — используются только переменные.

---

## Примечания

- Задача является классическим вариантом "Allocate Minimum Pages" и "Painter's Partition Problem".
- Если `k >= n`, то можно отдать каждую задачу отдельному исполнителю, и ответ будет `max(job) * t`.
- Если `k = 1`, ответ будет `sum(job) * t`.
- Проверка `job > limit` в `can_assign` необходима, так как если одна задача превышает `limit`, то при любом разбиении этот исполнитель превысит лимит.
