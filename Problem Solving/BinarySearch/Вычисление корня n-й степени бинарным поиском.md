## Определение

Требуется вычислить **корень n-й степени** из неотрицательного числа $a$:

$$
\sqrt[n]{a}
$$

где $n$ — натуральное число ($n \ge 1$), а $a \ge 0$. Результат должен быть найден с заданной точностью $\varepsilon$ (в примере $\varepsilon = 10^{-11}$).

---

## Постановка задачи

Даны два числа:
- $a$ — вещественное неотрицательное;
- $n$ — целое положительное.

Найти значение $x$, такое что $x^n$ максимально близко к $a$ с точностью $\varepsilon$. Чаще всего требуется вычислить $x = a^{1/n}$.

---

## Идея решения

Поскольку функция $f(x) = x^n$ монотонно возрастает на $[0, +\infty)$ (при $n \ge 1$), можно применить **бинарный поиск на вещественных числах**.

1. Определяем границы поиска:
   - Если $a \ge 1$, то корень находится в $[0, a]$, так как $a^n \ge a$ при $a \ge 1$.
   - Если $0 \le a < 1$, то корень лежит в $[0, 1]$, потому что $x^n \le x \le 1$ при $x \le 1$.
   - Общая формула: $right = \max(1, a)$.

2. Пока разница между левой и правой границей больше $\varepsilon$:
   - Находим середину $mid = \frac{left + right}{2}$.
   - Вычисляем $mid^n$ и сравниваем с $a$.
   - Если $mid^n \le a$, значит корень находится в правой половине — сдвигаем $left$ в $mid$ (или чуть больше).
   - Иначе — сдвигаем $right$ в $mid$.

3. После выхода из цикла $left$ и $right$ приблизительно равны, можно вернуть любое из них.

---

## Выбор границ

| Диапазон $a$ | Нижняя граница | Верхняя граница |
|----------------|----------------|-----------------|
| $a \ge 1$    | 0              | $a$           |
| $0 \le a < 1$| 0              | 1               |

Итоговая верхняя граница: $right = \max(1, a)$.

---

## Реализация (корректная версия)

```python
def nth_root(a: float, n: int, eps: float = 1e-11) -> float:
    if a == 0:
        return 0
    left, right = 0.0, max(1.0, a)
    while abs(right - left) > eps:
        mid = (left + right) / 2
        if mid ** n <= a:
            left = mid
        else:
            right = mid
    return (left + right) / 2   # возвращаем среднее для большей точности
```

### Комментарии
- В отличие от исходного кода, здесь не используется прибавление/вычитание $\varepsilon$, что делает алгоритм более устойчивым.
- Цикл продолжается, пока интервал не станет меньше заданной точности.
- После выхода из цикла можно вернуть любую из границ или их среднее.

---

## Примеры

```python
print(nth_root(16, 4))       # ≈ 2.0
print(nth_root(2, 2))        # ≈ 1.41421356237
print(nth_root(0.25, 2))     # ≈ 0.5
print(nth_root(0.001, 3))    # ≈ 0.1
```

---

## Сложность

- **Количество итераций:** $O(\log_2(\frac{\text{right} - \text{left}}{\varepsilon}))$.  
  Для $a \le 10^9$ и $\varepsilon = 10^{-11}$ потребуется около 50–70 итераций.
- **Время:** каждая итерация выполняет возведение в степень $O(\log n)$ (в Python ** реализовано быстро). Для больших $n$ можно использовать быстрое возведение в степень.
- **Память:** $O(1)$.

---

## Примечания

- В исходном коде использовалось `left = mid + eps` и `right = mid - eps`. Это может привести к некорректному сужению интервала, особенно при малых значениях $eps$. Более безопасно просто сдвигать границы к `mid`.
- При $a = 0$ корень равен 0, и бинарный поиск не нужен.
- Если требуется высокая точность, стоит учесть, что операция `**` с плавающей точкой имеет погрешность; для особо требовательных вычислений можно использовать метод Ньютона (итерации сходятся быстрее).
- Вместо явного возведения в степень каждый раз можно предвычислить, но для учебных целей это допустимо.
