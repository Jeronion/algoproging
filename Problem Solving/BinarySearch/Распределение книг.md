## Определение

Дано `n` книг, каждая с `arr[i]` страницами. Нужно распределить книги между `k` учениками так, чтобы:

- Каждый ученик получил хотя бы одну книгу.
- Книги распределяются последовательно (нельзя перемешивать порядок).
- Максимальное количество страниц, которое читает один ученик, должно быть **минимально возможным**.

Требуется найти это минимальное значение.

Задача известна как **Allocate Minimum Number of Pages** (GeeksforGeeks) или аналогичная на LeetCode.

---

## Постановка задачи

**Вход:**
- `arr` — массив количества страниц в каждой книге.
- `k` — количество учеников.

**Выход:** минимальное значение максимальной суммы страниц у одного ученика. Если книг меньше, чем учеников (`n < k`), вернуть `-1`.

**Пример:**
```
arr = [12, 34, 67, 90], k = 2
Результат: 113
Пояснение: Оптимальное распределение: [12, 34, 67] (сумма 113) и [90] (сумма 90). Максимум = 113.
```

---

## Идея решения

Задача решается **бинарным поиском по ответу**.

1. **Нижняя граница (`left`)** — минимальная страница в одной книге (или 1). На самом деле, левая граница — это минимальное значение `max(arr[i])`, потому что если одна книга имеет 100 страниц, то ученик, который её получит, прочитает не менее 100.
2. **Верхняя граница (`right`)** — сумма всех страниц (все книги отдаём одному ученику).
3. **Функция проверки `can(m)`** — можно ли распределить книги так, чтобы каждый ученик получил не более `m` страниц, и при этом уложиться в `k` учеников?
   - Жадно идём по книгам, набираем текущему ученику сумму, пока добавление следующей книги не превысит `m`. Тогда начинаем нового ученика.
   - Если в какой-то момент `arr[i] > m` — невозможно, возвращаем `False`.
   - Если количество использованных учеников ≤ `k`, то `m` подходит.

4. Бинарный поиск ищет минимальное `m`, для которого `can(m)` истинно.

---

## Реализация

```python
class Solution:
    def findPages(self, arr, k):
        n = len(arr)
        if n < k:
            return -1
        
        def can(m):
            students = 1
            current_sum = 0
            for pages in arr:
                if pages > m:
                    return False
                if current_sum + pages > m:
                    students += 1
                    current_sum = pages
                else:
                    current_sum += pages
            return students <= k
        
        left, right = max(arr), sum(arr)
        while left <= right:
            mid = (left + right) // 2
            if can(mid):
                right = mid - 1
            else:
                left = mid + 1
        return left
```

---

## Сложность

- **Время:** O(n log S), где S — сумма всех страниц. Бинарный поиск даёт log S итераций, каждая проверка O(n).
- **Память:** O(1) дополнительно.

---

## Примечания

- Важно, что книги распределяются **последовательно**, поэтому жадный подход работает.
- Левая граница должна быть `max(arr)`, а не 1, иначе могут быть ложные срабатывания (когда одна книга больше предполагаемой границы).
- Если `k == n`, то ответ — `max(arr)` (каждый берёт одну книгу).
- Задача часто встречается в контексте "минимизации максимума" и решается бинарным поиском.
