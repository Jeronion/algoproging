## Условие

Дано `m` оленей с параметрами `a[i]` (строптивость) и `n` эльфов с параметрами `b[j]` (темперамент). Нужно выбрать максимальное количество оленей, каждому из которых сопоставить двух разных эльфов так, чтобы для оленя с параметром `a` выполнялось условие: один эльф имеет темперамент `< a`, а другой `> a` (или наоборот). Каждый эльф может быть использован не более одного раза. Требуется вывести максимальное количество `k` и один из возможных вариантов сопоставления (номер оленя и номера двух эльфов).

**Ограничения:** `m, n ≤ 100 000`, значения параметров до 10⁹.

**Пример:**
```
Вход:
4 6
2 3 4 5
1 3 2 2 5 2
Выход:
2
1 1 2
2 4 5
```

---

## Идея решения

Задача сводится к поиску максимального паросочетания между оленями и парами эльфов, где один эльф должен быть строго меньше параметра оленя, другой — строго больше. Очевидно, что каждый олень требует двух эльфов, поэтому максимальное возможное `k` не может превышать `min(m, n//2)`.

Если отсортировать эльфов по темпераменту, то для каждого оленя подходящая пара будет состоять из одного эльфа из левой части (меньше `a`) и одного из правой (больше `a`). При этом эльфы не могут повторяться, поэтому нужно выбрать `k` левых и `k` правых эльфов так, чтобы существовало `k` оленей, каждый из которых можно «вставить» между своей парой.

### Ключевое наблюдение

Отсортируем оленей по возрастанию `a`, а эльфов — по возрастанию `b`. Рассмотрим некоторое количество `k`. Если мы возьмём **k наименьших эльфов** (левая группа) и **k наибольших эльфов** (правая группа), то жадное сопоставление сработает следующим образом: будем перебирать оленей в порядке возрастания и пытаться найти для очередной пары `(left_elf, right_elf)` (где `left_elf` — следующий по порядку из левой группы, `right_elf` — следующий из правой группы) такого оленя, у которого параметр лежит между этими двумя эльфами. Если удаётся сопоставить все `k` пар, то `k` достижимо.

Почему это работает? 
- Если существует решение для `k`, то можно переупорядочить эльфов так, чтобы левая группа состояла из `k` наименьших, а правая — из `k` наибольших (потому что любой эльф, используемый в качестве «меньшего», должен быть меньше параметра своего оленя, а значит, все используемые меньшие эльфы не больше, чем `k`-й по величине эльф в общем списке, и аналогично для больших). Более формально, используется теорема Холла для интервальной структуры.
- Жадный перебор оленей в порядке возрастания и попытка сопоставить их с упорядоченными парами эльфов даёт оптимальный результат, так как если существует решение, то можно перенумеровать пары эльфов и оленей так, чтобы для каждого `l` выполнялось `left_elf[l] < a[олень] < right_elf[l]`.

## Алгоритм

1. Прочитать `m`, `n`, массивы `A` (олени) и `B` (эльфы). Сохранить исходные номера.
2. Отсортировать `A` по значению строптивости, `B` по темпераменту.
3. Определить границы бинарного поиска: `left = 0`, `right = min(m, n//2)`.
4. Для каждого `mid` (проверяемое количество оленей) выполнить `check(mid)`, которая:
   - Берёт первые `mid` эльфов из отсортированного списка (левая группа) и последние `mid` эльфов (правая группа).
   - Проходит по отсортированным оленям (индекс `i`) и пытается сопоставить каждой паре `(левая_группа[l], правая_группа[l])` первого подходящего оленя, который больше левого и меньше правого.
   - Если удаётся сопоставить все `mid` пар, возвращает `True`.
5. Бинарный поиск находит максимальное `k`, для которого `check(k)` истинно.
6. Ещё раз вызываем `check(k, build=True)`, чтобы вывести сами пары.

## Реализация на Python

```python
def solve():
    M, N = map(int, input().split())
    A = sorted((a, i) for i, a in enumerate(map(int, input().split()), start=1))
    B = sorted((b, i) for i, b in enumerate(map(int, input().split()), start=1))

    def check(k, build=False):
        if build:
            print(k)
        c = 0
        l = 0      # индекс в левой/правой группе
        i = 0      # индекс в списке оленей
        while l < k and i < M:
            # Проверяем, подходит ли текущий олень под пару (B[l], B[N - k + l])
            if B[l][0] < A[i][0] < B[N - k + l][0]:
                if build:
                    print(A[i][1], B[l][1], B[N - k + l][1])
                c += 1
                l += 1
            i += 1
        return c == k

    left, right = 0, min(M, N // 2)
    while left <= right:
        mid = (left + right) // 2
        if check(mid):
            left = mid + 1
        else:
            right = mid - 1

    check(right, build=True)

if __name__ == "__main__":
    solve()
```

## Сложность

- **Сортировка:** O(m log m + n log n)
- **Бинарный поиск:** O(log(min(m, n//2))) итераций.
- **Каждая проверка:** O(m + k) = O(m), так как проход по оленям и по группам эльфов линейный.
- **Итоговая сложность:** O((m + n) log (m + n)), что приемлемо для 10⁵.

## Пример работы

Входные данные из примера:
```
4 6
2 3 4 5
1 3 2 2 5 2
```

Отсортированные олени: (2,1), (3,2), (4,3), (5,4)  
Отсортированные эльфы: (1,1), (2,3), (2,4), (2,6), (3,2), (5,5)

Проверка k=2:
- Левая группа: (1,1), (2,3)  
- Правая группа: (3,2), (5,5)  
- Олени: 2,3,4,5  
  l=0, i=0: олень 2 — не подходит (2 не между 1 и 3? 1<2<3 — да) → берём, выводим (олень1, эльф1, эльф2)  
  l=1, i=1: олень 3 — проверяем между (2,5): 2<3<5 — да → берём (олень2, эльф3, эльф5)  
  Все 2 пары найдены. Максимальное k=2.

## Примечания

- В решении используется жадный подход, основанный на сортировке и проверке возможности «вложить» оленей между парами эльфов. Это классический метод для задач, где требуется выбрать максимальное количество объектов с двумя ограничивающими элементами.
- Если бы требовалось вывести все возможные решения, подошло бы любое; данный алгоритм выдаёт одно из них.
- Важно, что эльфы в левой и правой группах выбираются именно наименьшие и наибольшие — это следует из оптимальной структуры решения.
