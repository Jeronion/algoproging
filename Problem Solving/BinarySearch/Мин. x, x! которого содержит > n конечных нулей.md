## Условие

Дано целое число $n$ ($1 \le n \le 10^4$). Требуется найти наименьшее положительное целое $x$ такое, что $x!$ (факториал числа $x$) содержит **не менее $n$ конечных нулей** (т.е. делится на $10^n$).

**Примеры:**
- $n = 6$ → $x = 25$ (25! оканчивается на 6 нулей)
- $n = 1$ → $x = 5$ (5! = 120, один ноль)

---

## Идея решения

Количество конечных нулей в $x!$ равно показателю степени $10$ в разложении $x!$ на простые множители. Поскольку $10 = 2 \cdot 5$, а двоек в факториале всегда больше, чем пятёрок, количество нулей определяется **числом пятёрок** в разложении $x!$:

$$
\text{zeros}(x) = \left\lfloor \frac{x}{5} \right\rfloor + \left\lfloor \frac{x}{25} \right\rfloor + \left\lfloor \frac{x}{125} \right\rfloor + \dots
$$

Функция $\text{zeros}(x)$ **монотонно возрастает** с ростом $x$. Поэтому для заданного $n$ можно применить **бинарный поиск** по $x$, чтобы найти минимальное $x$, для которого $\text{zeros}(x) \ge n$.

---

## Алгоритм

1. **Функция `count_zeros(k)`** вычисляет количество конечных нулей в $k!$:
   ```
   count = 0
   factor = 5
   while factor <= k:
       count += k // factor
       factor *= 5
   return count
   ```

2. **Бинарный поиск**:
   - Левая граница: $l = 5$ (наименьшее число, дающее хотя бы один ноль).
   - Правая граница: $r = 5 \cdot n$ (эмпирическая верхняя оценка, т.к. каждый пятый даёт один ноль, а нужно $n$ нулей).
   - Пока $l \le r$:
     - $m = (l + r) // 2$
     - если `count_zeros(m) < n`, то $l = m + 1$ (нужно больше)
     - иначе $r = m - 1$ (нужно меньше)
   - Ответ: $l$.

---

## Реализация

```python
class Solution:
    def count_zeros(self, k: int) -> int:
        count = 0
        factor = 5
        while factor <= k:
            count += k // factor
            factor *= 5
        return count
    
    def findNum(self, n: int) -> int:
        left, right = 5, 5 * n   # верхняя граница достаточно большая
        while left <= right:
            mid = (left + right) // 2
            if self.count_zeros(mid) < n:
                left = mid + 1
            else:
                right = mid - 1
        return left
```

---

## Сложность

- **Время:** $O(\log n \cdot \log_5 n)$ — бинарный поиск по $x$ выполняется за $O(\log (5n)) = O(\log n)$ итераций, а внутри каждого вызова `count_zeros` работает за $O(\log_5 x) = O(\log n)$. Итоговая сложность: $O((\log n)^2)$.
- **Память:** $O(1)$.

---

## Примечания

- Верхняя граница $5n$ является грубой, но достаточной, так как каждый пятый вносит как минимум одну пятёрку, а также дополнительные пятёрки от степеней 25, 125 и т.д. Таким образом, при $x = 5n$ количество нулей заведомо превышает $n$.
- Для очень больших $n$ можно использовать более точную оценку, но для $n \le 10^4$ подходит.
- Задача эквивалентна поиску наименьшего $x$ такого, что $\sum_{i=1}^{\infty} \lfloor x / 5^i \rfloor \ge n$.
