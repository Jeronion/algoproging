## Определение

Отсортированный массив был сдвинут (повёрнут) в неизвестной точке. Требуется найти в нём элемент (или минимум). Алгоритм модифицирует бинарный поиск, учитывая, что одна из половин всегда отсортирована.

---

## 1. Поиск элемента в сдвинутом массиве (LeetCode 33)

```python
def search(nums: List[int], target: int) -> int:
    left, right = 0, len(nums) - 1
    while left <= right:
        mid = (left + right) // 2
        if nums[mid] == target:
            return mid
        # определяем, какая половина отсортирована
        if nums[mid] >= nums[left]:
            # левая половина отсортирована
            if nums[left] <= target < nums[mid]:
                right = mid - 1
            else:
                left = mid + 1
        else:
            # правая половина отсортирована
            if nums[mid] < target <= nums[right]:
                left = mid + 1
            else:
                right = mid - 1
    return -1
```

## 2. Поиск минимального элемента (LeetCode 153)

```python
def findMin(nums: List[int]) -> int:
    left, right = 0, len(nums) - 1
    while left < right:
        mid = (left + right) // 2
        if nums[mid] <= nums[right]:
            right = mid
        else:
            left = mid + 1
    return nums[left]
```

## Сложность

- Время: O(log n)
- Память: O(1)
