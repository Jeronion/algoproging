## Определение

Коко хочет съесть все бананы из `n` куч за `k` часов. В каждый час она может съесть некоторое количество бананов (скорость `m`), причём если в куче осталось меньше, чем `m`, она доедает эту кучу и не переходит к следующей до следующего часа. Нужно найти **минимальную целочисленную скорость** `m`, при которой Коко успеет съесть все бананы ровно за `k` часов или быстрее.

Задача известна как [LeetCode 875. Koko Eating Bananas](https://leetcode.com/problems/koko-eating-bananas/).

---

## Постановка задачи

**Вход:**
- `arr` — массив целых чисел, где `arr[i]` — количество бананов в i-й куче.
- `k` — количество часов, за которое нужно съесть все бананы.

**Выход:** минимальная скорость `m` (бананов в час), достаточная для того, чтобы съесть все бананы за `≤ k` часов.

**Пример:**
```
arr = [3,6,7,11], k = 8
Вывод: 4
Пояснение:
- Скорость 4: 3/4=1 час, 6/4=2 часа, 7/4=2 часа, 11/4=3 часа. Итого 1+2+2+3 = 8 часов.
- Скорость 3: 3/3=1, 6/3=2, 7/3=3, 11/3=4 → 1+2+3+4=10 >8.
- Скорость 4 — минимальная.
```

---

## Идея решения

Функция времени от скорости монотонно убывает: чем больше скорость, тем меньше часов требуется. Поэтому можно применить **бинарный поиск по скорости**.

1. **Нижняя граница (`l`)** — минимальная скорость 1 (можно есть 1 банан в час).
2. **Верхняя граница (`r`)** — максимальное количество бананов в одной куче (скорость, большая этого значения, бессмысленна, так как Коко не может съесть больше, чем в куче, за час).
3. **Функция `can(m)`** — вычисляем общее количество часов, необходимое при скорости `m`:
   - Для каждой кучи `arr[i]` требуется `ceil(arr[i] / m)` часов, что можно вычислить как `(arr[i] + m - 1) // m` или `arr[i] // m + (1 if arr[i] % m else 0)`.
   - Суммируем по всем кучам. Если сумма ≤ `k`, то скорость подходит (можно успеть).
4. Бинарный поиск ищет минимальную скорость, при которой `total_hours ≤ k`.

---

## Реализация

```python
class Solution:
    def kokoEat(self, arr, k):
        left, right = 1, max(arr)
        while left <= right:
            mid = (left + right) // 2
            total_hours = 0
            for bananas in arr:
                # ceil(bananas / mid)
                total_hours += (bananas + mid - 1) // mid
            if total_hours > k:
                left = mid + 1
            else:
                right = mid - 1
        return left
```

---

## Сложность

- **Время:** O(n log M), где n — количество куч, M — максимальное количество бананов в куче (правая граница). Бинарный поиск даёт log M итераций, каждая итерация O(n).
- **Память:** O(1) дополнительно.

## Примечания

- Код в запросе использует `arr[i] // m + bool(arr[i] % m)` — это эквивалентно `(arr[i] + m - 1) // m`.
- Важно, что Коко не может переходить к следующей куче в середине часа, поэтому время на кучу округляется вверх.
- Если `k` очень большое (например, больше суммы всех бананов), то минимальная скорость может быть 1.
- Если `k` меньше количества куч, то решение может не существовать (но по условию обычно гарантируется, что `k ≥ len(arr)`? В задаче LeetCode `k ≥ len(arr)` не обязательно, но скорость всегда можно увеличить; если скорость максимальна, то часов = количество куч. Если `k` меньше количества куч, то даже максимальная скорость не уложится, но тогда ответа нет? На практике LeetCode гарантирует, что решение существует, т.е. `k ≥ len(arr)`. В противном случае можно вернуть -1, но обычно не требуется.
