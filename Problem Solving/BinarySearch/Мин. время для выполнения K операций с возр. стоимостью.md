## Определение

Дано `k` операций, которые нужно выполнить на нескольких устройствах (процессорах). Каждое устройство `i` имеет базовую скорость `arr[i]` (чем меньше число, тем быстрее). Выполнение операций на каждом устройстве подчиняется закону: первая операция занимает `arr[i]` единиц времени, вторая — `2*arr[i]`, третья — `3*arr[i]` и так далее (время выполнения j-й операции на устройстве i равно `j * arr[i]`). Устройства могут работать параллельно. Нужно найти **минимальное время**, за которое можно выполнить все `k` операций, распределяя их между устройствами произвольным образом.

Задача известна как *«Minimum time to complete k tasks with increasing time per task»* (вариация).

---

## Постановка задачи

**Вход:**
- `arr` — массив целых чисел, `arr[i]` — время выполнения первой операции на i-м устройстве.
- `k` — количество операций.

**Выход:** минимальное время `T`, за которое можно выполнить `k` операций.

**Пример:**
```
arr = [1, 2], k = 3
Вывод: 2
Пояснение:
- Устройство 0 (скорость 1): за время 2 может выполнить операции: 1-я за 1, 2-я за 2 → 2 операции.
- Устройство 1 (скорость 2): за время 2 может выполнить только 1-ю операцию (за 2). Всего 3 операции.
```

---

## Идея решения

За время `t` каждое устройство `i` может выполнить `x_i` операций, где `x_i` — наибольшее целое, удовлетворяющее:

$$
\sum_{j=1}^{x_i} j \cdot arr[i] \le t.
$$

Сумма арифметической прогрессии: `arr[i] * x_i (x_i + 1) / 2 ≤ t`. Решая квадратное неравенство относительно `x_i`, получаем:

$$
x_i \le \frac{\sqrt{1 + \frac{8t}{arr[i]}} - 1}{2}.
$$

Таким образом, максимальное количество операций на устройстве `i`:

$$
\text{ops}_i(t) = \left\lfloor \frac{\sqrt{1 + \frac{8t}{arr[i]}} - 1}{2} \right\rfloor.
$$

Общее количество операций, которое можно выполнить за время `t`, равно `sum(ops_i(t))`. Нам нужно найти минимальное `t`, при котором эта сумма ≥ `k`.

Функция `sum(ops_i(t))` монотонно возрастает с `t`, поэтому можно применить **бинарный поиск** по `t`.

---

## Алгоритм

1. Отсортировать массив `arr` (необязательно, но для верхней границы удобно взять наименьший элемент).
2. Определить границы бинарного поиска:
   - `left = 0` (минимальное время).
   - `right = arr[0] * k * (k + 1) // 2` (время, если все операции выполняются на самом быстром устройстве).
3. Для каждого `mid` вычислить общее количество операций:
   ```
   total = 0
   for r in arr:
       total += floor((sqrt(1 + 8 * mid / r) - 1) / 2)
   ```
   Если `total ≥ k`, то `mid` подходит, ищем меньшее (`right = mid - 1`), иначе увеличиваем (`left = mid + 1`).
4. Ответ — `left` (первое значение, при котором условие выполнено).

## Реализация

```python
import math

class Solution:
    def minTime(self, arr, k):
        arr.sort()
        n = len(arr)
        
        def can(t):
            total = 0
            for r in arr:
                # количество операций на устройстве со скоростью r за время t
                ops = int((math.sqrt(1 + 8 * t / r) - 1) // 2)
                total += ops
                if total >= k:
                    return True
            return total >= k
        
        left, right = 0, arr[0] * k * (k + 1) // 2
        while left <= right:
            mid = (left + right) // 2
            if can(mid):
                right = mid - 1
            else:
                left = mid + 1
        return left
```

---

## Сложность

- **Время:** O(n log M), где M — правая граница. Бинарный поиск даёт O(log M) итераций, каждая проверка O(n).
- **Память:** O(1) дополнительно.

---

## Примечания

- В коде используется `can(t)`, возвращающая `True`, если за время `t` можно выполнить **не менее** `k` операций. В бинарном поиске мы ищем первое `t`, при котором `can(t)` истинно.
- Формула для `ops` может быть записана как `int((math.sqrt(1 + 8 * t / r) - 1) / 2)` без `//`, но важно целочисленное деление вниз.
- Верхняя граница `right` может быть увеличена для надёжности, например, `right = max(arr) * k * (k + 1) // 2` или даже `10**18`.
- Задача встречается на платформах вроде Codeforces, GeeksforGeeks (например, «Minimum time to complete K tasks»).
