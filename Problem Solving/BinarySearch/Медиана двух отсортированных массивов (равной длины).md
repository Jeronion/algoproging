## Определение

Даны два отсортированных массива `a` и `b` одинаковой длины `n`. Требуется найти **медиану** объединённого отсортированного массива.

Медиана — это средний элемент (если общее количество элементов нечётно) или среднее арифметическое двух средних элементов (если чётно). Поскольку `2n` всегда чётно, медиана вычисляется как среднее арифметическое `n`-го и `(n+1)`-го элементов объединённого массива (в 1-индексации).

Задача известна как [GeeksforGeeks: Median of two sorted arrays of same size](https://www.geeksforgeeks.org/median-of-two-sorted-arrays/).

---

## Идея решения

Наивный подход — слить два массива за O(n) и найти медиану. Оптимальный алгоритм использует **бинарный поиск** за O(log n).

Основная идея: разделить оба массива на две части так, чтобы все элементы левой части были меньше всех элементов правой части, и в левой части было ровно `n` элементов (половина от `2n`). Тогда медиана будет равна среднему арифметическому максимального элемента левой части и минимального элемента правой части.

Обозначим:
- `n` — длина каждого массива.
- Выбираем разбиение первого массива на две части: `mid1` элементов слева, `n - mid1` справа. Тогда во втором массиве слева должно быть `n - mid1` элементов (чтобы общее количество слева было `n`). Таким образом, `mid2 = n - mid1`.

Условия корректного разбиения:
- `max(left_part) ≤ min(right_part)`
- Для этого достаточно проверить:
  - `a[mid1 - 1] ≤ b[mid2]` (если индексы существуют)
  - `b[mid2 - 1] ≤ a[mid1]`

Если эти условия выполнены, то медиана вычисляется как `(max(l1, l2) + min(r1, r2)) / 2`.

Если `a[mid1 - 1] > b[mid2]`, то нужно сдвинуть разбиение влево (уменьшить `mid1`), иначе — вправо.

---

## Алгоритм

1. Убедимся, что `n <= len(b)` (в нашем случае длины равны). При необходимости можно поменять массивы местами, но здесь это не требуется.
2. Инициализируем `left = 0`, `right = n`.
3. Пока `left <= right`:
   - `mid1 = (left + right) // 2`
   - `mid2 = n - mid1`
   - Вычисляем:
     - `l1 = a[mid1 - 1]` если `mid1 > 0`, иначе `-inf`
     - `r1 = a[mid1]` если `mid1 < n`, иначе `+inf`
     - `l2 = b[mid2 - 1]` если `mid2 > 0`, иначе `-inf`
     - `r2 = b[mid2]` если `mid2 < n`, иначе `+inf`
   - Если `l1 <= r2` и `l2 <= r1`, то нашли правильное разбиение. Возвращаем `(max(l1, l2) + min(r1, r2)) / 2.0`.
   - Если `l1 > r2`, то `right = mid1 - 1` (нужно уменьшить `mid1`).
   - Иначе `left = mid1 + 1` (увеличить `mid1`).
4. Если цикл завершился (не должно произойти при корректных входных данных), возвращаем 0.

---

## Реализация

```python
class Solution:
    def medianOf2(self, a, b):
        n = len(a)
        left, right = 0, n
        while left <= right:
            mid1 = (left + right) // 2
            mid2 = n - mid1
            
            l1 = float('-inf') if mid1 == 0 else a[mid1 - 1]
            r1 = float('inf') if mid1 == n else a[mid1]
            l2 = float('-inf') if mid2 == 0 else b[mid2 - 1]
            r2 = float('inf') if mid2 == n else b[mid2]
            
            if l1 <= r2 and l2 <= r1:
                return (max(l1, l2) + min(r1, r2)) / 2.0
            elif l1 > r2:
                right = mid1 - 1
            else:
                left = mid1 + 1
        return 0
```

## Сложность

- **Время:** O(log n) — бинарный поиск по разбиению.
- **Память:** O(1) — используется только несколько переменных.

---

## Примечания

- Алгоритм работает только для массивов **одинаковой длины**. Для разной длины требуется модификация (бинарный поиск по меньшему массиву, обработка границ).
- Важно использовать `float('inf')` и `float('-inf')` для корректного сравнения на границах.
- При чётном общем количестве элементов (`2n`) медиана вычисляется как среднее двух центральных чисел.
- В приведённом коде `mid2 = n - mid1` автоматически даёт правильное количество элементов слева во втором массиве.
