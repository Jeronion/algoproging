## Постановка задачи

Дано кубическое уравнение:

$$
a x^3 + b x^2 + c x + d = 0
$$

с вещественными коэффициентами $a, b, c, d$. Требуется найти все его действительные корни на интервале $[-1000, 1000]$ с точностью до $10^{-6}$. Гарантируется, что все корни лежат в этом диапазоне (или программа рассматривает только этот отрезок).

---

## Идея решения

Алгоритм сочетает два подхода:

1. **Грубое сканирование** — с фиксированным шагом проходим по интервалу и ищем участки, где функция меняет знак или близка к нулю.
2. **Бинарное уточнение** — на каждом найденном интервале, где функция меняет знак, применяем метод бисекции для нахождения корня с высокой точностью.

Этот метод гарантированно находит все простые корни (нечётной кратности). Корни чётной кратности (например, кратные) могут быть обнаружены, если функция в них близка к нулю, но не меняет знак.

---

## Алгоритм (пошагово)

### 1. Выбор параметров

- **Диапазон поиска:** от -1000 до 1000 (достаточно для большинства задач).
- **Шаг сканирования:** `step = 0.5`. Такой шаг позволяет надёжно обнаружить смену знака, так как при типичных коэффициентах функция не успевает «проскочить» ноль между двумя точками.
- **Точность:** `eps = 1e-8` используется для проверки близости к нулю и в бинарном поиске.

### 2. Сканирование

Перебираем точки `x` от -1000 до 1000 с шагом `step`. Для каждого интервала `[x, x+step]`:

- Вычисляем `y1 = f(x)`, `y2 = f(x+step)`.
- Если `abs(y1) < eps` — считаем, что `x` является корнем (с учётом погрешности).
- Если `y1 * y2 < 0` — на интервале есть смена знака, значит, существует корень (нечётной кратности).

### 3. Уточнение корня

Для каждого интервала со сменой знака выполняем бинарный поиск (метод бисекции):

```
left, right = x, x+step
while right - left > eps:
    mid = (left + right) / 2
    if f(left) * f(mid) <= 0:
        right = mid
    else:
        left = mid
root = (left + right) / 2
```

Это стандартная реализация бисекции, гарантирующая сходимость к корню с заданной точностью.

### 4. Удаление дубликатов

При обнаружении корня проверяем, не совпадает ли он с последним найденным (с точностью `1e-6`). Это исключает повторное добавление одного и того же корня, который мог быть найден как «почти ноль» и позже как корень на интервале.

### 5. Сортировка и вывод

Собранные корни сортируются и выводятся с шестью знаками после запятой.

## Реализация на Python

```python
def f(a, b, c, d, x):
    return a * x**3 + b * x**2 + c * x + d

def find_roots(a, b, c, d):
    eps = 1e-8
    step = 0.5
    roots = []
    x = -1000.0
    while x <= 1000.0:
        x2 = x + step
        y1 = f(a, b, c, d, x)
        y2 = f(a, b, c, d, x2)

        # Если значение близко к нулю
        if abs(y1) < eps:
            root = x
        # Если есть смена знака
        elif y1 * y2 < 0:
            left, right = x, x2
            while right - left > eps:
                mid = (left + right) / 2
                if f(a, b, c, d, left) * f(a, b, c, d, mid) <= 0:
                    right = mid
                else:
                    left = mid
            root = (left + right) / 2
        else:
            x = x2
            continue

        # Проверка дубликатов
        if not roots or abs(roots[-1] - root) > 1e-6:
            roots.append(root)

        x = x2

    roots.sort()
    return roots

if __name__ == "__main__":
    a, b, c, d = map(float, input().split())
    for r in find_roots(a, b, c, d):
        print(f"{r:.6f}")
```

## Сложность

- **Время:**  
  Количество интервалов сканирования: `(2000 / step) ≈ 4000`.  
  На каждом интервале, где найден корень, выполняется бинарный поиск с ~30 итерациями (log₂(1/eps) ≈ 27).  
  Общее число итераций — несколько сотен тысяч, что для Python выполняется быстро.

- **Память:** O(1) дополнительно плюс хранение найденных корней (не более трёх для кубического уравнения).

---

## Примечания

- Метод не гарантирует нахождение **кратных корней чётной кратности** (например, двойного корня, где функция касается оси, но не пересекает её). В таких местах нет смены знака, и алгоритм их пропустит. Однако если значение в точке близко к нулю (согласно `eps`), корень будет добавлен. Для повышения надёжности можно проверять значения функции на всей сетке, а не только в точках.
- Выбор шага `0.5` — эмпирический. При очень «крутых» функциях (большие коэффициенты) может потребоваться уменьшить шаг.
- Точность `1e-8` в бинарном поиске даёт итоговую точность корня ~1e-8, а вывод форматируется до 6 знаков.
- Для кубического уравнения существует аналитическое решение (формула Кардано), но численный метод универсален и прост в реализации.
