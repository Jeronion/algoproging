## Определение

Дана очередь `q`. Требуется развернуть её так, чтобы первый элемент стал последним, а последний — первым, сохранив порядок остальных элементов в обратном порядке.

**Пример:**
```
Вход:  [5, 10, 15, 20, 25]
Выход: [25, 20, 15, 10, 5]
```

---

## Подход 1: Использование стека (O(n) времени, O(n) памяти)

Идея: стек работает по принципу LIFO (последним пришёл — первым вышел). Если извлечь все элементы из очереди и поместить в стек, а затем извлечь их из стека и снова поместить в очередь, порядок элементов обратится.

### Алгоритм

1. Создать пустой стек.
2. Пока очередь не пуста:
   - Извлечь элемент из начала очереди (`pop`/`popleft`).
   - Поместить его в стек.
3. Пока стек не пуст:
   - Извлечь элемент из стека (`pop`).
   - Поместить его в конец очереди (`append`).

### Реализация на Python

```python
from collections import deque

def reverse_queue(q):
    stack = []
    # Переносим все элементы из очереди в стек
    while q:
        stack.append(q.popleft())
    # Переносим элементы обратно из стека в очередь
    while stack:
        q.append(stack.pop())
    return q

# Пример
q = deque([5, 10, 15, 20, 25])
reverse_queue(q)
print(list(q))  # [25, 20, 15, 10, 5]
```

### Сложность

- **Время:** O(n) — каждый элемент перекладывается дважды.
- **Память:** O(n) — для хранения стека.

## Подход 2: Использование рекурсии (O(n) времени, O(n) памяти на стек вызовов)

Можно развернуть очередь рекурсивно, не используя явный стек. Идея: извлечь первый элемент, рекурсивно развернуть оставшуюся очередь, затем добавить извлечённый элемент в конец.

### Алгоритм

1. Если очередь пуста, вернуть.
2. Извлечь первый элемент `front` из очереди.
3. Рекурсивно вызвать функцию для оставшейся части.
4. Добавить `front` в конец очереди.

### Реализация

```python
from collections import deque

def reverse_queue_recursive(q):
    if not q:
        return
    front = q.popleft()
    reverse_queue_recursive(q)
    q.append(front)

# Пример
q = deque([5, 10, 15, 20, 25])
reverse_queue_recursive(q)
print(list(q))  # [25, 20, 15, 10, 5]
```

### Сложность

- **Время:** O(n) — каждый элемент обрабатывается один раз.
- **Память:** O(n) из-за глубины рекурсии (стек вызовов).

---

## Примечания

- Для Python рекомендуется использовать `collections.deque`, так как операции `popleft()` и `append()` выполняются за O(1).
- Оба метода изменяют исходную очередь. Если нужно сохранить оригинал, перед вызовом следует создать копию.
- Рекурсивный метод более элегантен, но может вызвать переполнение стека для очень больших очередей.
