## Определение

Требуется реализовать **очередь (Queue)**, используя только **стеки (Stacks)** (их стандартные операции push/pop). Очередь должна поддерживать операции: `enqueue(x)` (добавить в конец), `dequeue()` (удалить из начала), `front()` (вернуть первый элемент), `size()` (текущий размер). Решение должно использовать только стековые структуры, без дополнительных массивов или связных списков.

---

## Идея

Очередь работает по принципу FIFO (первый вошёл — первый вышел), стек — LIFO (последний вошёл — первый вышел). Чтобы имитировать очередь, нужно использовать два стека. Есть два основных подхода:

1. **Сделать операцию enqueue затратной** (перенос элементов при каждой вставке) — тогда `dequeue` будет O(1).
2. **Сделать операцию dequeue затратной** (перенос при удалении) — тогда `enqueue` будет O(1).

Оба подхода корректны, выбор зависит от сценария использования.

---

## 1. Подход: дорогой enqueue (O(n)), дешёвый dequeue (O(1))

### Принцип

- Два стека: `s1` и `s2`.
- При `enqueue(x)`:
  - Перемещаем все элементы из `s1` в `s2` (чтобы освободить порядок LIFO).
  - Кладём `x` в `s1`.
  - Перемещаем все элементы обратно из `s2` в `s1`. Теперь `s1` содержит элементы в порядке очереди (первый добавленный — наверху).
- При `dequeue()` просто извлекаем верхний элемент из `s1`.
- `front()` — верхний элемент `s1`.
- `size()` — размер `s1`.

### Реализация на Python

```python
class MyQueue:
    def __init__(self):
        self.s1 = []
        self.s2 = []

    def enqueue(self, x):
        # Перемещаем всё из s1 в s2
        while self.s1:
            self.s2.append(self.s1.pop())
        # Добавляем новый элемент
        self.s1.append(x)
        # Возвращаем элементы обратно
        while self.s2:
            self.s1.append(self.s2.pop())

    def dequeue(self):
        if not self.s1:
            return
        self.s1.pop()

    def front(self):
        return -1 if not self.s1 else self.s1[-1]

    def size(self):
        return len(self.s1)
```

### Сложность

- **enqueue:** O(n) (два переноса всех элементов)
- **dequeue:** O(1)
- **front:** O(1)
- **size:** O(1)
- **Память:** O(n)

## 2. Подход: дорогой dequeue (O(n)), дешёвый enqueue (O(1))

### Принцип

- При `enqueue(x)` просто кладём `x` в `s1`.
- При `dequeue()`:
  - Если `s2` пуст, перемещаем все элементы из `s1` в `s2` (теперь `s2` содержит элементы в порядке очереди).
  - Извлекаем верхний элемент из `s2`.
- `front()` — если `s2` не пуст, возвращаем верхний элемент `s2`, иначе переносим из `s1` в `s2` и возвращаем верхний `s2`.
- `size()` = `len(s1) + len(s2)`.

Этот подход часто предпочтительнее, так как амортизированное время `dequeue` и `front` составляет O(1) (каждый элемент переносится только один раз).

### Реализация

```python
class MyQueue:
    def __init__(self):
        self.s1 = []
        self.s2 = []

    def enqueue(self, x):
        self.s1.append(x)

    def dequeue(self):
        if not self.s2:
            while self.s1:
                self.s2.append(self.s1.pop())
        if self.s2:
            self.s2.pop()

    def front(self):
        if not self.s2:
            while self.s1:
                self.s2.append(self.s1.pop())
        return -1 if not self.s2 else self.s2[-1]

    def size(self):
        return len(self.s1) + len(self.s2)
```

### Сложность

- **enqueue:** O(1)
- **dequeue:** амортизированное O(1) (худший случай O(n) при редких переносах)
- **front:** амортизированное O(1)
- **size:** O(1)
- **Память:** O(n)

## 3. Реализация с одним стеком и рекурсией (теоретическая)

Можно реализовать очередь, используя только один стек и рекурсию. Идея: при `dequeue` рекурсивно извлекаем элементы, пока не дойдём до последнего (самого нижнего), который и является первым в очереди, затем возвращаем его, а остальные возвращаем на место. Однако такой подход имеет глубину рекурсии O(n) и неэффективен.

---

## Заключение

Для практической реализации очереди через стеки чаще используется второй подход (дорогой dequeue) из-за амортизированной эффективности. Оба метода корректны и показывают, как комбинация стеков может эмулировать поведение очереди.
