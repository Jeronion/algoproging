## Определение

Дана очередь `q` и целое число `k`. Требуется развернуть порядок первых `k` элементов очереди, оставив остальные элементы в том же порядке. Если `k` больше размера очереди, вернуть исходную очередь.

**Пример:**
```
q = [1, 2, 3, 4, 5], k = 3 → [3, 2, 1, 4, 5]
q = [4, 3, 2, 1], k = 4 → [1, 2, 3, 4]
```

---

## Идея решения

Используем вспомогательный **стек** для временного хранения первых `k` элементов. Поскольку стек работает по принципу LIFO, извлечение элементов из стека даст их в обратном порядке.

### Алгоритм

1. Если очередь пуста или `k <= 0`, вернуть исходную очередь.
2. Извлечь из очереди первые `k` элементов и поместить их в стек.
3. Извлечь элементы из стека (в обратном порядке) и добавить их обратно в очередь.
4. Теперь первые `k` элементов развёрнуты, но они находятся в конце очереди. Чтобы вернуть их в начало, нужно переместить оставшиеся `n - k` элементов (которые идут после развёрнутых) в конец очереди.
5. Для этого извлечь из начала `n - k` элементов и добавить их в конец.

---

## Реализация на Python

```python
from collections import deque

class Solution:
    def reverseFirstK(self, q, k):
        if not q or k <= 0 or k > len(q):
            return q
        stack = []
        # 1. Извлекаем первые k элементов в стек
        for _ in range(k):
            stack.append(q.popleft())
        # 2. Возвращаем их из стека в очередь (обратный порядок)
        while stack:
            q.append(stack.pop())
        # 3. Перемещаем оставшиеся элементы в конец
        for _ in range(len(q) - k):
            q.append(q.popleft())
        return q
```

---

## Сложность

- **Время:** O(n) — каждый элемент обрабатывается не более двух раз.
- **Память:** O(k) — для хранения стека.

---

## Примечания

- В Python для эффективной работы с очередью используется `collections.deque`, так как `popleft()` работает за O(1).
- Если `k` равно размеру очереди, то операция аналогична полному развороту очереди.
- Условие `k > len(q)` не обязательно проверять, так как по условию задачи `k` может быть больше, тогда возвращаем исходную очередь (код выше обрабатывает этот случай).
