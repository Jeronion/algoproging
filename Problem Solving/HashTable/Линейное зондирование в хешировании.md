## Определение

**Линейное зондирование** — метод разрешения коллизий в хеш-таблицах с открытой адресацией. При возникновении коллизии (ячейка уже занята) алгоритм последовательно проверяет следующие ячейки (с шагом 1) до нахождения свободной позиции. Формула для поиска индекса на шаге `i`:

```
index = (hash(key) + i) % m,   i = 0, 1, 2, ...
```

где `m` — размер таблицы.

---

## Постановка задачи

Дан массив целых чисел `arr[]` и размер хеш-таблицы `m`. Требуется вставить элементы массива в хеш-таблицу, используя линейное зондирование для разрешения коллизий. Правила:

1. Хеш-функция: `hash(value) = value % m`.
2. Если ячейка свободна (значение `-1`), вставляем элемент.
3. Если ячейка занята, но значение в ней **равно** вставляемому, то ничего не делаем (дубликат просто игнорируется, остаётся на том же месте).
4. Если ячейка занята другим элементом, начинаем линейное зондирование: `index = (index + 1) % m` до нахождения свободной ячейки или совпадения с дубликатом.
5. Если таблица полностью заполнена (`size == m`), оставшиеся элементы отбрасываются.
6. В результате получаем массив размера `m`, где `-1` означает пустую ячейку.

**Пример:**
```
arr = [4, 14, 24, 44], m = 10
Выход: [-1, -1, -1, -1, 4, 14, 24, 44, -1, -1]
```

---

## Алгоритм

1. Инициализируем `res` массивом длины `m`, заполненным `-1`.
2. Устанавливаем `size = 0` (количество заполненных ячеек).
3. Для каждого значения `v` из `arr`:
   - Если `size == m`, прекращаем вставку (`break`).
   - Вычисляем `idx = v % m`.
   - Пока `res[idx] != -1`:
     - Если `res[idx] == v`, выходим из цикла (дубликат уже есть).
     - Иначе `idx = (idx + 1) % m` (переход к следующей ячейке).
   - Если после цикла `res[idx] == -1`:
     - Устанавливаем `res[idx] = v`.
     - `size += 1`.
4. Возвращаем `res`.

---

## Реализация на Python

```python
class Solution:
    def linearProbing(self, arr, m):
        res = [-1] * m
        size = 0
        for v in arr:
            if size == m:
                break                     # таблица полна, остальные элементы игнорируем
            idx = v % m
            # ищем подходящее место (свободное или уже с таким же ключом)
            while res[idx] != -1:
                if res[idx] == v:
                    break                 # дубликат, не вставляем, но и не ищем дальше
                idx = (idx + 1) % m
            if res[idx] == -1:            # нашли свободное место
                res[idx] = v
                size += 1
        return res
```

---

## Сложность

- **Время:** O(len(arr) * m) в худшем случае (если много коллизий, каждый элемент может пройти почти всю таблицу). На практике, при хорошей хеш-функции и умеренном коэффициенте заполнения, ожидаемое время близко к O(len(arr)).
- **Память:** O(m) для выходного массива.

---

## Примечания

- Метод обрабатывает дубликаты: если элемент уже присутствует, он не вставляется повторно, но и не сдвигается.
- Если таблица заполняется, оставшиеся элементы отбрасываются, что соответствует условию задачи.
- Вместо `-1` можно использовать любой другой маркер пустой ячейки, если в массиве не встречаются отрицательные числа.
