## Определение

Дан массив целых неотрицательных чисел. Требуется найти длину **самой длинной подпоследовательности**, состоящей из **последовательных чисел** (конечные элементы подпоследовательности могут идти в любом порядке, но сами числа должны составлять непрерывный ряд). Подпоследовательность не обязана быть непрерывной в исходном массиве.

**Пример:**
```
arr = [2, 6, 1, 9, 4, 5, 3] → ответ 6 (последовательность 1,2,3,4,5,6)
arr = [1, 9, 3, 10, 4, 20, 2] → ответ 4 (1,2,3,4)
```

---

## Идея решения

Используется **хеш-множество** (set) для хранения всех элементов массива. Затем для каждого элемента проверяем, является ли он **началом** последовательности: то есть существует ли `v-1` в множестве. Если нет, то начинаем отсчёт длины последовательности, увеличивая `v` на 1, пока следующий элемент присутствует в множестве. Так мы находим все возможные последовательности, и берём максимальную длину.

Алгоритм:

1. Поместить все элементы массива в `set`.
2. Инициализировать `max_len = 0`.
3. Для каждого элемента `v` в массиве:
   - Если `v-1` отсутствует в множестве (это значит, что `v` — начало последовательности):
     - `current_len = 1`
     - Пока `v + current_len` в множестве, увеличивать `current_len`.
     - Обновить `max_len = max(max_len, current_len)`.
4. Вернуть `max_len`.

---

## Реализация на Python

```python
class Solution:
    def longestConsecutive(self, arr):
        num_set = set(arr)
        max_len = 0
        for v in arr:
            # если v - начало последовательности
            if v - 1 not in num_set:
                length = 1
                while v + length in num_set:
                    length += 1
                max_len = max(max_len, length)
        return max_len
```

---

## Сложность

- **Время:** O(n) — каждый элемент посещается не более двух раз: один раз при проверке на начало последовательности, и один раз при построении последовательности (внутренний цикл суммарно проходит по всем элементам, составляющим последовательности).
- **Память:** O(n) — для хранения множества.

---

## Примечания

- Метод использует тот факт, что последовательность может начинаться только с элемента, у которого нет `v-1` в множестве. Это позволяет не проверять одни и те же последовательности многократно.
- Алгоритм не требует сортировки и работает за линейное время.
- Подходит для неотрицательных чисел, но распространяется на любые целые.
- Если в массиве есть дубликаты, множество автоматически их удаляет, что не влияет на результат (дубликаты не меняют длины последовательности).
