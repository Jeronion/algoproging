## Определение

**Квадратичное зондирование** — метод разрешения коллизий в хеш-таблицах с открытой адресацией. При возникновении коллизии (ячейка уже занята) алгоритм ищет следующую свободную позицию с помощью квадратичной функции:

```
index = (hash(key) + i²) % m,   i = 0, 1, 2, ...
```

где `m` — размер таблицы. Это позволяет снизить первичную кластеризацию, характерную для линейного зондирования, за счёт «разбегания» шагов.

## Постановка задачи

Дан массив целых чисел `arr[]` и размер хеш-таблицы `m`. Требуется вставить элементы массива в хеш-таблицу, используя квадратичное зондирование для разрешения коллизий. Пустые ячейки обозначаются `-1`.

Правила:

1. Хеш-функция: `hash(value) = value % m`.
2. При коллизии последовательно проверяются позиции:
   ```
   newIndex = (hash(value) + i²) % m,   i = 1, 2, ...
   ```
3. Если элемент уже присутствует в таблице (дубликат), он не вставляется повторно (но занимает ту же позицию).
4. Размер таблицы гарантированно не меньше длины массива, но на всякий случай, если заполнена вся таблица, оставшиеся элементы игнорируются (хотя при `arr.size() ≤ m` этого не произойдёт).
5. В результат записывается итоговое состояние хеш-таблицы.

**Пример:**
```
arr = [21, 10, 33, 43], m = 5
Выход: [10, 21, -1, 33, 43]
```

---

## Алгоритм

1. Инициализировать `res` массивом длины `m`, заполненным `-1`.
2. Установить `size = 0` (количество заполненных ячеек).
3. Для каждого значения `v` из `arr`:
   - Если `size == m`, прекратить вставку (`break`).
   - Установить `i = 0`.
   - Пока `res[hash(v, i, m)]` не равно `-1` и не равно `v`:
     - Увеличить `i`.
   - Вычислить `idx = (v + i²) % m`.
   - Если `res[idx] == -1` (свободное место), то:
     - `res[idx] = v`
     - `size += 1`
4. Вернуть `res`.

## Реализация на Python

```python
class Solution:
    def _hash(self, v, i, m):
        return (v + i ** 2) % m
    
    def quadraticProbing(self, arr, m):
        res = [-1] * m
        size = 0
        for v in arr:
            if size == m:
                break                     # таблица полна
            i = 0
            # ищем свободное место или повтор ключа
            while res[self._hash(v, i, m)] != -1 and res[self._hash(v, i, m)] != v:
                i += 1
            idx = self._hash(v, i, m)
            if res[idx] == -1:            # нашли свободное место
                res[idx] = v
                size += 1
        return res
```

## Сложность

- **Время в среднем:** O(1) при хорошем коэффициенте заполнения.
- **Худший случай (много коллизий, таблица почти полна):** O(m) для одной вставки, общая O(m²).
- **Память:** O(m) для выходного массива.

## Преимущества и недостатки

**Преимущества:**
- Устраняет первичную кластеризацию (в отличие от линейного зондирования).
- Проще в реализации, чем двойное хеширование.

**Недостатки:**
- Вторичная кластеризация (свойства `i²` приводят к тому, что некоторые последовательности проб повторяются).
- Не всегда может найти свободную ячейку, даже если она есть (при неудачном выборе `m`).
- Требует, чтобы `m` было простым или степенью двойки для полного обхода таблицы.

---

## Примечания

- В приведённой реализации используется `while` для поиска подходящего индекса. Условие `res[...] != -1 and res[...] != v` гарантирует, что мы не остановимся на удалённом маркере (в данной задаче удаления нет, поэтому маркеры не нужны).
- Если в таблице уже есть **такой же ключ**, мы не вставляем его повторно и не увеличиваем счётчик `size`. Это соответствует обработке дубликатов.
- При `size == m` дальнейшие элементы игнорируются (по условию задачи).
