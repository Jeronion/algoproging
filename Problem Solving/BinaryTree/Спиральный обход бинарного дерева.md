## Определение

**Спиральный (зигзагообразный) обход** бинарного дерева — это обход уровней, при котором направление обхода на каждом уровне чередуется:

- Корень (уровень 0) обходится **справа налево** (или слева направо — в зависимости от реализации, но в условии обычно начинают с правого направления).
- Следующий уровень (уровень 1) обходится **слева направо**.
- Затем уровень 2 — справа налево, и так далее.

**Пример:**
```
        1
       / \
      2   3
     / \ / \
    7  6 5  4
```
Спиральный обход (начиная с правого направления на уровне 0): [1, 2, 3, 4, 5, 6, 7]

---

## Постановка задачи

Дан корень бинарного дерева. Требуется вернуть список значений узлов в порядке спирального обхода (уровень за уровнем, чередуя направление).

---

## Идея решения (два стека)

Используются два стека для хранения узлов текущего и следующего уровней. Один стек (s1) обрабатывает текущий уровень справа налево, другой (s2) — следующий уровень слева направо (или наоборот). На каждом шаге мы опустошаем один стек, добавляя детей в другой стек в нужном порядке. Это позволяет обходить дерево без явного запоминания уровней.

### Алгоритм

1. Если корень пуст, вернуть пустой список.
2. Создать два пустых стека: `s1` и `s2`.
3. Поместить корень в `s1`.
4. Пока `s1` или `s2` не пусты:
   - **Обработка `s1`** (текущий уровень, направление справа налево):
     - Извлекать узел из `s1`, добавлять его значение в результат.
     - Добавлять его **правого** потомка в `s2`, затем **левого** (чтобы при извлечении из `s2` они пошли в обратном порядке, т.е. слева направо).
   - **Обработка `s2`** (следующий уровень, направление слева направо):
     - Извлекать узел из `s2`, добавлять его значение в результат.
     - Добавлять его **левого** потомка в `s1`, затем **правого** (чтобы при извлечении из `s1` они пошли в обратном порядке, т.е. справа налево).
5. Вернуть результат.

### Почему это работает?

- `s1` обрабатывает узлы в порядке, обратном порядку их добавления (LIFO). При добавлении сначала правый, потом левый, извлекаем сначала левый → получаем порядок слева направо на следующем уровне.
- `s2` аналогично обеспечивает чередование.

## Реализация на Python

```python
class Node:
    def __init__(self, data):
        self.data = data
        self.left = None
        self.right = None

def spiral_order(root):
    if not root:
        return []
    res = []
    s1 = [root]   # текущий уровень (справа налево)
    s2 = []       # следующий уровень (слева направо)
    
    while s1 or s2:
        # Обрабатываем s1 (текущий уровень)
        while s1:
            node = s1.pop()
            res.append(node.data)
            # Добавляем детей в s2: сначала правый, потом левый
            if node.right:
                s2.append(node.right)
            if node.left:
                s2.append(node.left)
        
        # Обрабатываем s2 (следующий уровень)
        while s2:
            node = s2.pop()
            res.append(node.data)
            # Добавляем детей в s1: сначала левый, потом правый
            if node.left:
                s1.append(node.left)
            if node.right:
                s1.append(node.right)
    
    return res
```

## Сложность

- **Время:** O(n), где n — количество узлов. Каждый узел посещается ровно один раз.
- **Память:** O(n) в худшем случае (для хранения узлов в стеках).

---

## Примечания

- Альтернативные подходы:
  - **Рекурсивный:** вычисляется высота дерева, затем для каждого уровня рекурсивно обходится с указанием направления. Время O(n), память O(h) для стека рекурсии, но требует вычисления высоты заранее.
  - **Использование дека (deque):** можно обойтись одной двусторонней очередью, добавляя узлы с разных концов в зависимости от направления. Это также даёт O(n) времени и O(n) памяти.
- В реализации с двумя стеками не требуется знать высоту дерева или размер уровней.
- Начальное направление (справа налево для корня) можно изменить, поменяв порядок добавления детей в `s1` и `s2`.
