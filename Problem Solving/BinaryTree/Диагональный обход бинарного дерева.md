## Определение

**Диагональный обход** бинарного дерева — это обход, при котором узлы группируются по диагоналям с наклоном `-1`. Каждая диагональ соответствует прямой линии, проходящей от верхнего левого угла к нижнему правому (или наоборот, в зависимости от определения). В данной задаче диагонали определяются так: для каждого узла его левый потомок попадает на следующую диагональ (сдвиг вниз), а правый потомок остаётся на той же диагонали. Результат выводится в порядке: сначала все узлы первой диагонали (сверху вниз), затем второй, и так далее.

**Пример:**
```
        8
       / \
      3   10
     / \    \
    1   6    14
       / \   /
      4   7 13
```
Диагональный обход: `[8, 10, 14, 3, 6, 7, 13, 1, 4]`

## Постановка задачи

Дан корень бинарного дерева. Требуется вернуть список значений узлов при диагональном обходе, где все узлы на одной диагонали выводятся вместе, начиная с самой верхней диагонали (корень) и двигаясь вниз.

---

## Идея решения (DFS с хеш-таблицей)

Используется **предварительный обход (preorder)** с передачей номера диагонали (`dist`). Начальная диагональ корня равна `0`. При переходе к левому потомку номер диагонали увеличивается на `1` (левый потомок попадает на следующую диагональ). При переходе к правому потомку номер диагонали остаётся тем же (правый потомок находится на той же диагонали). Все узлы с одинаковым `dist` принадлежат одной диагонали.

Для хранения узлов каждой диагонали используется словарь `hm`, где ключ — номер диагонали, значение — список узлов в порядке их посещения (preorder). После обхода все диагонали выводятся по порядку возрастания номеров.

### Почему это работает?

- Обход в глубину (preorder) гарантирует, что узлы на каждой диагонали будут перечислены в порядке сверху вниз (так как корень посещается первым, затем левое поддерево, затем правое).
- Изменение `dist` при переходе налево и сохранение при переходе направо соответствует геометрической интерпретации диагоналей.

### Алгоритм

1. Создать словарь `hm = defaultdict(list)` для хранения узлов по диагоналям.
2. Запустить рекурсивную функцию `dfs(node, dist)`:
   - Если `node` равен `None`, завершить.
   - Добавить `node.data` в `hm[dist]`.
   - Рекурсивно вызвать `dfs(node.left, dist + 1)`.
   - Рекурсивно вызвать `dfs(node.right, dist)`.
3. После обхода собрать результаты: для `i` от `0` до максимального `dist` (которое можно отслеживать) добавить все элементы `hm[i]` в результирующий список.
4. Вернуть список.

---

## Реализация

```python
from collections import defaultdict

class Solution:
    def diagonal(self, root):
        hm = defaultdict(list)
        max_dist = 0
        
        def dfs(node, dist):
            nonlocal max_dist
            if node is None:
                return
            hm[dist].append(node.data)
            max_dist = max(max_dist, dist)
            dfs(node.left, dist + 1)
            dfs(node.right, dist)
        
        dfs(root, 0)
        res = []
        for d in range(max_dist + 1):
            res.extend(hm[d])
        return res
```

---

## Сложность

- **Время:** O(n), где n — количество узлов. Каждый узел посещается один раз.
- **Память:** O(n) для хранения словаря и стека рекурсии.

---

## Примечания

- Альтернативно можно использовать BFS с очередью, но DFS с хеш-таблицей проще и интуитивнее.
- Порядок узлов внутри одной диагонали соответствует предварительному обходу (корень, левое, правое), что гарантирует правильный порядок сверху вниз.
- В словаре `hm` ключи — номера диагоналей; после обхода они будут в порядке возрастания, так как мы перебираем от 0 до `max_dist`.
