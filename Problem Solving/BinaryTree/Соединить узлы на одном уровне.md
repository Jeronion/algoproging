## Определение

Дан бинарное дерево, в котором каждый узел имеет дополнительный указатель `nextRight` (изначально равен `None`). Требуется установить этот указатель для каждого узла на следующий узел на том же уровне справа. Если справа узла нет, оставить `None`. Это позволяет эффективно обходить дерево по уровням без использования очереди.

**Пример:**
```
        10
       /  \
      8    2
     /    /
    3    4
```
После установки `nextRight`:  
- `8.nextRight = 2`  
- `3.nextRight = 4`  
- для остальных `None`.

---

## Подход 1: Уровневый обход с очередью (O(n) времени, O(n) памяти)

Используем стандартный BFS с очередью. Для каждого уровня запоминаем количество узлов и, проходя по ним, устанавливаем `nextRight` на следующий узел в очереди (кроме последнего, для которого ставим `None`). Детей добавляем в очередь.

### Реализация

```python
from collections import deque

def connect(root):
    if not root:
        return root
    q = deque([root])
    while q:
        size = len(q)
        for i in range(size):
            node = q.popleft()
            if i == size - 1:
                node.nextRight = None
            else:
                node.nextRight = q[0]   # следующий узел в очереди
            if node.left:
                q.append(node.left)
            if node.right:
                q.append(node.right)
    return root
```

### Сложность

- **Время:** O(n) – каждый узел обрабатывается один раз.
- **Память:** O(n) – очередь может содержать до n/2 узлов (для полного дерева).

## Подход 2: Оптимальный (без дополнительной памяти) – O(n) времени, O(1) памяти

Используем уже установленные `nextRight` для обхода узлов текущего уровня. Для каждого узла обрабатываем его детей, устанавливая их `nextRight` на нужный узел на следующем уровне. Для поиска подходящего узла справа используется вспомогательная функция `get_next_right`, которая идёт по `nextRight` текущего узла, пока не найдёт узел с левым или правым ребёнком.

### Алгоритм

1. Установить `root.nextRight = None`.
2. Начать с первого узла уровня (`level_start = root`).
3. Пока `level_start` не `None`:
   - Проходим по всем узлам текущего уровня через `nextRight`.
   - Для каждого узла `cur`:
     - Если есть левый ребёнок:
       - Если есть правый ребёнок, то `left.nextRight = right`.
       - Иначе ищем подходящий узел через `get_next_right(cur)`.
     - Если есть правый ребёнок:
       - `right.nextRight = get_next_right(cur)`.
   - Переходим на следующий уровень: 
     - Если у `level_start` есть левый ребёнок, берём его как начало следующего уровня.
     - Иначе если есть правый – его.
     - Иначе ищем через `get_next_right(level_start)`.

### Вспомогательная функция

```python
def get_next_right(node):
    temp = node.nextRight
    while temp:
        if temp.left:
            return temp.left
        if temp.right:
            return temp.right
        temp = temp.nextRight
    return None
```

### Реализация

```python
def connect(root):
    if not root:
        return None
    root.nextRight = None
    level_start = root
    while level_start:
        cur = level_start
        while cur:
            if cur.left:
                if cur.right:
                    cur.left.nextRight = cur.right
                else:
                    cur.left.nextRight = get_next_right(cur)
            if cur.right:
                cur.right.nextRight = get_next_right(cur)
            cur = cur.nextRight
        # определяем начало следующего уровня
        if level_start.left:
            level_start = level_start.left
        elif level_start.right:
            level_start = level_start.right
        else:
            level_start = get_next_right(level_start)
    return root
```

### Сложность

- **Время:** O(n) – каждый узел обрабатывается константное число раз.
- **Память:** O(1) – не используется дополнительная структура (кроме стека вызовов, если бы использовалась рекурсия; здесь итеративный подход).

---

## Примечания

- Первый подход (с очередью) проще для понимания и быстрее пишется, но требует O(n) памяти.
- Второй подход более элегантный и часто ожидается на собеседованиях.
- Указатели `nextRight` для последнего узла на каждом уровне остаются `None` (по умолчанию).
- В примере `get_next_right` ищет первый доступный левый или правый ребёнок у узлов справа на том же уровне.
