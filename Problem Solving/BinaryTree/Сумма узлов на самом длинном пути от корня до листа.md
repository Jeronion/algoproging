## Определение

В бинарном дереве требуется найти сумму значений узлов на **самом длинном пути** от корня до любого листа. Если таких путей несколько (одинаковая длина), выбрать тот, у которого сумма узлов максимальна.

**Пример:**
```
        4
       / \
      2   5
     / \  / \
    7   1 2  3
         \
          6
```
Самый длинный путь: 4 → 2 → 1 → 6 (длина 4), сумма = 4 + 2 + 1 + 6 = 13.
```

---

## Постановка задачи

Дан корень бинарного дерева (узлы содержат целые числа). Необходимо найти сумму узлов на самом длинном пути от корня до листа. Если несколько путей имеют максимальную длину, вернуть максимальную сумму среди них.

---

## Идея решения (DFS с отслеживанием максимальной длины и суммы)

Используется **обход в глубину (DFS)**. При обходе мы передаём текущую глубину (`dist`) и накопленную сумму (`sm`). При достижении листового узла сравниваем текущую глубину с максимальной запомненной:

- Если глубина больше, обновляем максимальную глубину и соответствующую сумму.
- Если глубина равна, выбираем большую сумму.

Таким образом, мы обходим все пути и сохраняем только лучший.

---

## Реализация

```python
class Solution:
    def sumOfLongRootToLeafPath(self, root):
        # mx[0] — максимальная длина, mx[1] — сумма на этом пути
        mx = [0, 0]
        
        def dfs(node, depth, total):
            if node is None:
                return
            total += node.data
            # если достигли листа
            if node.left is None and node.right is None:
                if depth > mx[0]:
                    mx[0] = depth
                    mx[1] = total
                elif depth == mx[0]:
                    mx[1] = max(mx[1], total)
            else:
                dfs(node.left, depth + 1, total)
                dfs(node.right, depth + 1, total)
        
        dfs(root, 0, 0)
        return mx[1]
```

---

## Сложность

- **Время:** O(n) — каждый узел посещается ровно один раз.
- **Память:** O(h) для стека рекурсии (h — высота дерева). В худшем случае (вырожденное дерево) O(n).

---

## Примечания

- Для пустого дерева функция вернёт 0 (поскольку `mx[1]` изначально 0).
- Значения узлов неотрицательные, но алгоритм работает и с отрицательными числами.
- Альтернативно можно решить задачу с помощью BFS по уровням, но DFS естественнее для поиска пути.
