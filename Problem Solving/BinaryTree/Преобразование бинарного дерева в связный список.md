## Определение

Дано бинарное дерево. Требуется преобразовать его в «связный список» на месте, используя только указатели `right` и `left`. После преобразования:

- У каждого узла `left` должен указывать на `None`.
- `right` должен указывать на следующий узел в **прямом порядке (preorder)** обхода исходного дерева.

**Пример:**
```
Исходное дерево:
        1
       / \
      2   5
     / \   \
    3   4   6

Результат (список в preorder):
1 -> 2 -> 3 -> 4 -> 5 -> 6
```

---

## Идея решения (итеративный подход, O(1) дополнительной памяти)

Используется идея, похожая на **обход Морриса**. Алгоритм работает без дополнительной памяти (кроме нескольких переменных) и за O(n) времени.

### Алгоритм

1. Начать с корня `cur = root`.
2. Пока `cur` не равен `None`:
   - Если у `cur` есть левый ребёнок:
     - Найти **самый правый** узел в левом поддереве `cur` (обозначим `pre`).
     - Присоединить правое поддерево `cur` как правого потомка `pre`: `pre.right = cur.right`.
     - Перенести левое поддерево `cur` в правое: `cur.right = cur.left`.
     - Обнулить левый указатель: `cur.left = None`.
   - Перейти к правому потомку: `cur = cur.right`.

После завершения алгоритма дерево будет преобразовано в правый список, содержащий узлы в порядке preorder.

### Почему это работает?

- На каждом шаге мы берём левое поддерево текущего узла и вставляем его между текущим узлом и его правым поддеревом, сохраняя порядок preorder (корень, левое поддерево, правое поддерево).
- Обход Морриса гарантирует, что мы посещаем каждый узел ровно один раз и не используем дополнительную память.

## Реализация на Python

```python
class Node:
    def __init__(self, val):
        self.data = val
        self.left = None
        self.right = None

def flatten(root):
    cur = root
    while cur:
        if cur.left:
            # Находим самый правый узел в левом поддереве
            pre = cur.left
            while pre.right:
                pre = pre.right
            # Присоединяем правое поддерево cur к pre
            pre.right = cur.right
            # Переносим левое поддерево в правое
            cur.right = cur.left
            cur.left = None
        cur = cur.right
    return root
```

## Альтернативный рекурсивный подход (требует O(h) памяти на стек)

Хотя рекурсивный подход использует дополнительную память, он проще для понимания:

```python
def flatten_recursive(root):
    if not root or (not root.left and not root.right):
        return
    # Рекурсивно преобразуем левое поддерево
    flatten_recursive(root.left)
    # Сохраняем правое поддерево
    tmp = root.right
    # Переносим левое в правое
    root.right = root.left
    root.left = None
    # Находим конец нового правого списка
    tail = root.right
    while tail.right:
        tail = tail.right
    tail.right = tmp
    # Рекурсивно преобразуем правое поддерево (уже изменённое)
    flatten_recursive(root.right)
```

---

## Сложность

- **Время:** O(n) – каждый узел обрабатывается один раз.
- **Память:** O(1) для итеративного подхода (без учёта стека вызовов, так как рекурсия не используется). Рекурсивный подход требует O(h) дополнительной памяти.

---

## Примечания

- В результате список не является двусвязным, только односторонний `right`.
- Порядок узлов в списке соответствует **preorder** (корень, левое, правое).
- Алгоритм работает in‑place, не создавая новых узлов.
- Для пустого дерева возвращается `None`.
