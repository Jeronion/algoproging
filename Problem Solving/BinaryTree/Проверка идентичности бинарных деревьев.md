## Определение

Два бинарных дерева считаются **идентичными**, если они имеют одинаковую структуру (расположение узлов) и значения данных в соответствующих узлах совпадают.

Требуется по данным корневым узлам `r1` и `r2` определить, идентичны ли деревья.

---

## Идея решения (рекурсивный обход)

Используется **рекурсивный подход**: одновременно обходим оба дерева в одинаковом порядке (например, в глубину, начиная с корня и сначала левое поддерево, потом правое). На каждом шаге:

1. Если оба узла равны `None` → возвращаем `True` (достигли конца одновременно).
2. Если один из узлов `None`, а другой нет → возвращаем `False`.
3. Если значения узлов не совпадают → возвращаем `False`.
4. Иначе рекурсивно проверяем левые и правые поддеревья: `isIdentical(left1, left2) and isIdentical(right1, right2)`.

При обнаружении любого несовпадения рекурсия прекращается и возвращается `False`.

---

## Реализация

```python
class Solution:
    def isIdentical(self, r1, r2):
        def dfs(node1, node2):
            # базовые случаи
            if node1 is None and node2 is None:
                return True
            if node1 is None or node2 is None:
                return False
            if node1.data != node2.data:
                return False
            # рекурсивная проверка поддеревьев
            return dfs(node1.left, node2.left) and dfs(node1.right, node2.right)
        
        return dfs(r1, r2)
```

---

## Сложность

- **Время:** O(n), где n — минимальное количество узлов в двух деревьях (или общее число узлов, если деревья идентичны). Каждый узел посещается не более одного раза.
- **Память:** O(h), где h — высота дерева (глубина рекурсии). В худшем случае (вырожденное дерево) h = n.

---

## Примечания

- Алгоритм работает для деревьев с любыми значениями (поддерживается сравнение данных).
- Рекурсивный подход естественен для деревьев, но может вызвать переполнение стека для очень глубоких деревьев (можно использовать итеративный обход с явным стеком, но рекурсивный проще).
