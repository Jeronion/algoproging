## Определение

**Наименьший общий предок (LCA)** двух узлов `n1` и `n2` в бинарном дереве поиска (BST) — это самый глубокий узел, который является предком обоих узлов (узел может быть предком самого себя). В BST все ключи уникальны, и оба узла гарантированно присутствуют в дереве.

**Пример:**
```
        20
       /  \
      8   22
     / \
    4   12
       /  \
      10   14
```
- LCA(8, 14) = 8
- LCA(10, 14) = 12
- LCA(4, 22) = 20

---

## Постановка задачи

Даны корень BST и два узла `n1` и `n2` (значения уникальны, оба присутствуют). Найти узел, являющийся их наименьшим общим предком.

---

## Идея решения

В BST используется свойство упорядоченности: все значения в левом поддереве меньше корня, в правом — больше. Это позволяет найти LCA за один проход без рекурсии.

**Ключевое наблюдение:** LCA узлов `n1` и `n2` — это первый узел, значение которого лежит **между** значениями `n1` и `n2` (включая их самих). При обходе от корня:

- Если текущий узел меньше обоих искомых, то LCA находится в правом поддереве.
- Если текущий узел больше обоих, то LCA — в левом поддереве.
- В противном случае (текущий узел находится между значениями или равен одному из них) — это и есть LCA.

### Алгоритм (итеративный)

1. Упорядочить `n1` и `n2` так, чтобы `n1.data < n2.data` (для удобства сравнения).
2. Начать с корня.
3. Пока текущий узел не `None`:
   - Если `cur.data < n1.data`, то оба узла находятся справа → перейти `cur = cur.right`.
   - Если `cur.data > n2.data`, то оба узла слева → перейти `cur = cur.left`.
   - Иначе ( `n1.data <= cur.data <= n2.data` ) — текущий узел является LCA, вернуть его.

---

## Реализация

```python
class Solution:
    def LCA(self, root, n1, n2):
        # Гарантируем порядок n1 < n2
        if n1.data > n2.data:
            n1, n2 = n2, n1
        cur = root
        while cur:
            if cur.data < n1.data:
                cur = cur.right
            elif cur.data > n2.data:
                cur = cur.left
            else:
                return cur
```

---

## Сложность

- **Время:** O(h), где h — высота BST. В сбалансированном дереве O(log n), в вырожденном O(n).
- **Память:** O(1) — используется только один указатель, без рекурсии.

---

## Примечания

- Алгоритм работает только для BST с уникальными значениями.
- Если `n1` или `n2` не существуют в дереве, результат не определён (по условию они присутствуют).
- Код использует итеративный подход, что предпочтительнее рекурсивного из-за отсутствия риска переполнения стека.
- Можно также реализовать рекурсивно, но итеративная версия проще и эффективнее по памяти.
