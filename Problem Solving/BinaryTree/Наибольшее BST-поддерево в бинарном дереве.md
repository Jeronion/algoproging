## Определение

В бинарном дереве требуется найти **самое большое поддерево**, которое является корректным **бинарным деревом поиска (BST)**. Размер поддерева определяется количеством узлов в нём. Поддерево должно удовлетворять свойствам BST: для каждого узла все значения в левом поддереве меньше, а в правом — больше.

**Пример:**
```
        5
       / \
      2   4
     / \
    1   3
```
Наибольшее BST-поддерево — всё дерево? Нет, корень 5 имеет правого потомка 4, что нарушает BST (4 < 5). Самое большое BST: поддерево с корнем 2 (узлы 1,2,3) размера 3.

---

## Постановка задачи

Дан корень бинарного дерева. Найти размер (количество узлов) самого большого поддерева, которое является BST. Если таких несколько, вернуть размер любого. Гарантируется, что дерево содержит хотя бы один узел.

---

## Идея решения (постордерный обход)

Используем **рекурсивный обход снизу вверх** (postorder). Для каждого узла мы собираем информацию о его поддереве:

- Минимальное значение в поддереве.
- Максимальное значение в поддереве.
- Размер поддерева.

При обходе:

1. Если узел пуст → возвращаем (inf, -inf, 0) (минимальное = +∞, максимальное = -∞, размер = 0).
2. Рекурсивно обрабатываем левое и правое поддеревья.
3. Проверяем, является ли текущее поддерево BST:
   - Максимальное значение в левом поддереве **меньше** `node.data`.
   - Минимальное значение в правом поддереве **больше** `node.data`.
   - Левое и правое поддеревья должны быть BST (это гарантируется, если они не вернули «плохую» информацию).
4. Если условие выполняется, то текущее поддерево — BST. Возвращаем:
   - Минимум = `min(лево.мин, node.data)`
   - Максимум = `max(право.макс, node.data)`
   - Размер = `лево.размер + право.размер + 1`
5. Если условие не выполняется, то поддерево не является BST. Возвращаем «плохую» информацию:
   - Минимум = -∞ (или любое значение, которое не пройдёт проверку выше)
   - Максимум = +∞
   - Размер = `max(лево.размер, право.размер)` (берём максимальный размер из поддеревьев, так как где-то там может быть большое BST).

В конце возвращаем размер, накопленный для корня.

## Реализация на Python

```python
class Node:
    def __init__(self, val):
        self.data = val
        self.left = None
        self.right = None

class BSTInfo:
    def __init__(self, mn, mx, sz):
        self.mn = mn   # минимальное значение в поддереве
        self.mx = mx   # максимальное значение в поддереве
        self.sz = sz   # размер наибольшего BST в этом поддереве

class Solution:
    def largestBst(self, root):
        def check(node):
            if node is None:
                return BSTInfo(float('inf'), float('-inf'), 0)
            
            left = check(node.left)
            right = check(node.right)
            
            # Если поддеревья являются BST и текущий узел удовлетворяет условиям
            if left.mx < node.data and right.mn > node.data:
                return BSTInfo(
                    min(left.mn, node.data),
                    max(right.mx, node.data),
                    left.sz + right.sz + 1
                )
            
            # Иначе возвращаем "плохую" информацию с наибольшим размером из детей
            return BSTInfo(
                float('-inf'),
                float('inf'),
                max(left.sz, right.sz)
            )
        
        return check(root).sz
```

## Сложность

- **Время:** O(n), где n — количество узлов. Каждый узел посещается один раз.
- **Память:** O(h), где h — высота дерева (глубина рекурсии). В сбалансированном дереве O(log n), в вырожденном O(n).

---

## Примечания

- В объекте `BSTInfo` для не-BST поддеревьев устанавливаются `mn = -inf`, `mx = inf`, чтобы они не проходили проверку для родителя.
- Размер для не-BST поддерева — это максимальный размер из левого и правого поддеревьев, так как наибольшее BST может находиться в одном из них.
- Если всё дерево является BST, то размер будет равен общему количеству узлов.
- Альтернативный подход: можно было бы использовать массив для хранения информации, но объект делает код более читаемым.
