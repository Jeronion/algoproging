## Определение

**Левый обход** бинарного дерева — это множество узлов, видимых при взгляде на дерево с левой стороны. Для каждого уровня дерева (глубины) в левый обход попадает самый первый узел на этом уровне, если двигаться от корня вниз, приоритетно выбирая левое поддерево.

**Пример:**
```
        1
       / \
      2   3
     / \   \
    4   5   6
```
Слева видны узлы: 1 (корень), 2 (второй уровень), 4 (третий уровень) → [1, 2, 4].

---

## Постановка задачи

Дан корень бинарного дерева. Требуется вернуть список значений узлов, которые видны с левой стороны (левый обход). Если дерево пустое, вернуть пустой список.

---

## Идея решения (DFS)

Используем **поиск в глубину (DFS)** с передачей текущего уровня. Обход выполняется в порядке: корень → левое поддерево → правое поддерево. При этом мы запоминаем первый встреченный узел на каждом уровне (так как обход идёт слева направо, первым на уровне будет самый левый узел).

### Алгоритм

1. Инициализировать пустой список `ans` для хранения результата.
2. Рекурсивная функция `dfs(node, level)`:
   - Если `node` равен `None` → завершить.
   - Если `len(ans) == level` (то есть мы ещё не добавили узел для этого уровня), добавить `node.data` в `ans`.
   - Рекурсивно вызвать `dfs(node.left, level + 1)` (левое поддерево).
   - Рекурсивно вызвать `dfs(node.right, level + 1)` (правое поддерево).
3. Вызвать `dfs(root, 0)`.
4. Вернуть `ans`.

### Почему это работает?

При обходе DFS с приоритетом левого поддерева, на каждом уровне первым будет посещён самый левый узел. Проверка `len(ans) == level` гарантирует, что мы добавим только первый встреченный узел для каждого уровня, игнорируя остальные.

---

## Реализация

```python
class Solution:
    def leftView(self, root):
        ans = []
        def dfs(node, level):
            if node is None:
                return
            if len(ans) == level:
                ans.append(node.data)
            dfs(node.left, level + 1)
            dfs(node.right, level + 1)
        
        dfs(root, 0)
        return ans
```

---

## Сложность

- **Время:** O(n), где n — количество узлов. Каждый узел посещается ровно один раз.
- **Память:** O(h) для стека рекурсии (h — высота дерева) + O(n) для хранения результата (в худшем случае все узлы могут быть видны слева, но обычно меньше).

---

## Альтернативный подход (BFS)

Можно также использовать **поиск в ширину (BFS)** с очередью: проходить по уровням и добавлять в ответ первый узел на каждом уровне. Это даёт ту же сложность, но требует O(n) памяти для очереди.
