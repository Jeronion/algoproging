## Определение

**Диаметр бинарного дерева** — это длина (количество рёбер) **самого длинного пути** между любыми двумя узлами в дереве. Этот путь может проходить через корень, но не обязан.

**Пример:**
```
        1
       / \
      2   3
     / \
    4   5
```
Диаметр: путь 4 → 2 → 5 (2 ребра) или 4 → 2 → 1 → 3 (3 ребра) → максимум = 3.

---

## Постановка задачи

Дан корень бинарного дерева. Требуется найти диаметр дерева (количество рёбер на самом длинном пути между двумя любыми узлами). Если дерево состоит из одного узла, диаметр равен 0.

---

## Идея решения (постордерный обход)

Используется **обход в глубину (postorder)** для вычисления высоты каждого узла и одновременного обновления диаметра.

### Ключевое наблюдение

Для каждого узла `u` диаметр, проходящий через этот узел, равен сумме высот его левого и правого поддеревьев (количество рёбер от самого глубокого листа в левом поддереве до самого глубокого листа в правом поддереве через `u`). Глобальный диаметр — это максимальное значение этой суммы по всем узлам.

Высота узла определяется как максимальное расстояние (количество рёбер) от этого узла до любого листа в его поддереве.

### Алгоритм

1. Инициализировать глобальную переменную `mx = 0` для хранения максимального диаметра.
2. Рекурсивная функция `postorder(node)`:
   - Если `node` равен `None`, вернуть `0` (высота пустого поддерева).
   - Рекурсивно вычислить высоту левого поддерева: `l = postorder(node.left)`.
   - Рекурсивно вычислить высоту правого поддерева: `r = postorder(node.right)`.
   - Обновить диаметр через текущий узел: `mx = max(mx, l + r)`.
   - Вернуть высоту текущего узла: `max(l, r) + 1`.
3. После обхода всего дерева вернуть значение `mx`.

---

## Реализация

```python
class Solution:
    def diameter(self, root):
        mx = 0
        def postorder(node):
            nonlocal mx
            if node is None:
                return 0
            left = postorder(node.left)
            right = postorder(node.right)
            # диаметр через текущий узел
            mx = max(mx, left + right)
            # высота текущего узла
            return max(left, right) + 1
        
        postorder(root)
        return mx
```

---

## Сложность

- **Время:** O(n), где n — количество узлов. Каждый узел посещается ровно один раз.
- **Память:** O(h), где h — высота дерева (глубина стека рекурсии). В худшем случае (вырожденное дерево) O(n).

---

## Примечания

- В коде `l + r` даёт количество рёбер между самым глубоким листом левого поддерева и самым глубоким листом правого поддерева через текущий узел. Это корректно, так как высота измеряется в рёбрах (пустое поддерево имеет высоту 0).
- Если дерево пустое, функция вернёт 0 (так как `mx` останется 0).
- Альтернативный подход: можно использовать BFS или два DFS для нахождения диаметра, но постордерный метод является классическим и наиболее эффективным.
