## Определение

**Сбалансированное бинарное дерево** — это дерево, в котором для каждого узла высоты левого и правого поддеревьев отличаются **не более чем на 1**. Пустое дерево считается сбалансированным.

Задача: проверить, является ли данное бинарное дерево сбалансированным (по определению AVL-дерева).

---

## Идея решения (эффективный однопроходной алгоритм)

Наивный подход требует двух проходов: сначала рекурсивно вычислять высоту для каждого узла, затем проверять разницу. Это приводит к O(n²) в худшем случае.

Оптимальный подход использует **один рекурсивный обход**, совмещая вычисление высоты и проверку сбалансированности. Для каждого узла:

1. Рекурсивно проверяем левое и правое поддеревья.
2. Если хотя бы одно поддерево несбалансированно (возвращает `-1`), немедленно возвращаем `-1` для текущего узла.
3. Иначе вычисляем высоты левого и правого поддеревьев; если разница > 1, возвращаем `-1`.
4. Если все проверки пройдены, возвращаем высоту текущего узла (`max(высоты_поддеревьев) + 1`).

Таким образом, функция возвращает либо высоту (если поддерево сбалансировано), либо `-1` (если несбалансировано). Это позволяет избежать повторных вычислений высоты и работает за O(n) времени и O(h) памяти под стек рекурсии.

---

## Реализация на Python

```python
class Node:
    def __init__(self, data):
        self.data = data
        self.left = None
        self.right = None

def is_balanced_rec(root):
    if root is None:
        return 0   # высота пустого дерева = 0

    left_height = is_balanced_rec(root.left)
    right_height = is_balanced_rec(root.right)

    # Если какое-то поддерево несбалансированно (вернуло -1) или разница высот > 1
    if left_height == -1 or right_height == -1 or abs(left_height - right_height) > 1:
        return -1

    # Возвращаем высоту текущего узла
    return max(left_height, right_height) + 1

def is_balanced(root):
    return is_balanced_rec(root) > 0
```

---

## Сложность

- **Время:** O(n), где n — количество узлов. Каждый узел посещается ровно один раз.
- **Память:** O(h), где h — высота дерева (глубина рекурсии). В худшем случае для вырожденного дерева h = n.

---

## Примечания

- Функция `is_balanced_rec` возвращает `-1` для несбалансированного поддерева, что позволяет быстро прервать дальнейшие проверки.
- Высота пустого поддерева принимается равной 0.
- Проверка `abs(left_height - right_height) > 1` гарантирует выполнение условия сбалансированности.
- Этот метод также известен как **bottom-up подход**, так как проверка начинается с листьев и поднимается вверх.
