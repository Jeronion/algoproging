## Определение

**Sum Tree** — это бинарное дерево, в котором для каждого узла (кроме листьев) его значение равно сумме значений всех узлов в левом и правом поддеревьях. Пустое дерево считается Sum Tree, а листовой узел — тоже (так как сумма его поддеревьев равна 0, но по условию листья считаются корректными автоматически).

---

## Идея решения (рекурсивный подход)

Используем рекурсивную функцию, которая для каждого узла возвращает сумму всех узлов в его поддереве, если оно удовлетворяет свойству Sum Tree. Если свойство нарушено, функция возвращает `-1` (сигнал ошибки).

### Алгоритм

1. Если узел `None` → возвращаем `0` (сумма пустого поддерева).
2. Если узел — лист (нет левого и правого потомка) → возвращаем его значение (лист всегда корректен).
3. Рекурсивно вычисляем сумму левого поддерева `left_sum`.
   - Если `left_sum == -1`, поддерево не является Sum Tree — сразу возвращаем `-1`.
4. Рекурсивно вычисляем сумму правого поддерева `right_sum`.
   - Если `right_sum == -1`, аналогично возвращаем `-1`.
5. Проверяем условие: `left_sum + right_sum == node.data`.
   - Если не равно → возвращаем `-1`.
6. Если все проверки пройдены, возвращаем `left_sum + right_sum + node.data` (общая сумма поддерева).

Финальная проверка: если для корня функция не вернула `-1`, дерево является Sum Tree.

## Реализация на Python

```python
class Solution:
    def is_sum_tree(self, node):
        def dfs(root):
            if root is None:
                return 0
            # лист
            if root.left is None and root.right is None:
                return root.data
            
            left_sum = dfs(root.left)
            if left_sum == -1:
                return -1
            right_sum = dfs(root.right)
            if right_sum == -1:
                return -1
            
            if left_sum + right_sum != root.data:
                return -1
            
            return left_sum + right_sum + root.data
        
        return dfs(node) != -1
```

---

## Сложность

- **Время:** O(n) — каждый узел посещается один раз.
- **Память:** O(h) — глубина стека рекурсии (h — высота дерева). В худшем случае (вырожденное дерево) O(n).

---

## Примечания

- Функция `dfs` возвращает `-1` для некорректных поддеревьев, что позволяет немедленно прервать проверку и сэкономить время.
- Листовые узлы не проверяются на условие суммы, так как по определению они считаются корректными.
- Значения узлов могут быть большими (до 10⁵), сумма может превышать 32-битные границы, но в Python это не проблема.
