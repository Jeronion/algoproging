## Определение

Даны два массива: `inorder` и `preorder`, представляющие собой соответственно **центрированный** (inorder) и **прямой** (preorder) обходы некоторого бинарного дерева. Все значения узлов уникальны, и каждый элемент из `preorder` присутствует в `inorder`. Требуется восстановить исходное бинарное дерево и вернуть его корень.

**Пример:**
```
inorder = [3, 1, 4, 0, 5, 2]
preorder = [0, 1, 3, 4, 2, 5]
Восстановленное дерево:
        0
       / \
      1   2
     / \   \
    3   4   5
```

---

## Идея решения (рекурсивный подход с разделением по индексам)

### Ключевое наблюдение

В **preorder** обходе первым всегда идёт корень текущего поддерева. В **inorder** обходе все узлы левого поддерева находятся слева от корня, а все узлы правого — справа.

Таким образом, алгоритм:

1. Первый элемент `preorder` — корень дерева.
2. Находим позицию этого корня в `inorder`. Эта позиция делит `inorder` на две части: левое поддерево (слева от корня) и правое поддерево (справа от корня).
3. Зная размер левого поддерева (`left_size`), мы знаем, сколько элементов в `preorder` принадлежат левому поддереву (сразу после корня) и сколько — правому.
4. Рекурсивно строим левое и правое поддеревья, передавая соответствующие подмассивы `inorder` и `preorder`.

### Оптимизация

Для быстрого поиска позиции корня в `inorder` используется **хеш-таблица** (словарь), где ключ — значение узла, значение — его индекс в `inorder`.

---

## Алгоритм

1. Создать словарь `in_index`, отображающий значение узла на его индекс в массиве `inorder`.
2. Определить рекурсивную функцию `build(pre_left, pre_right, in_left, in_right)`:
   - Если `pre_left > pre_right` или `in_left > in_right`, вернуть `None`.
   - Корень — `preorder[pre_left]`.
   - Найти индекс корня в `inorder`: `root_in_idx = in_index[root_val]`.
   - Количество узлов в левом поддереве: `left_size = root_in_idx - in_left`.
   - Рекурсивно построить левое поддерево:
     - `pre_left + 1` до `pre_left + left_size`
     - `in_left` до `root_in_idx - 1`
   - Рекурсивно построить правое поддерево:
     - `pre_left + left_size + 1` до `pre_right`
     - `root_in_idx + 1` до `in_right`
   - Вернуть корень.
3. Вызвать `build(0, n-1, 0, n-1)`.

---

## Реализация

```python
class Solution:
    def buildTree(self, inorder, preorder):
        in_index = {val: i for i, val in enumerate(inorder)}
        n = len(preorder)

        def build(pre_left, pre_right, in_left, in_right):
            if pre_left > pre_right or in_left > in_right:
                return None

            root_val = preorder[pre_left]
            root = Node(root_val)
            root_in_idx = in_index[root_val]
            
            left_size = root_in_idx - in_left

            root.left = build(pre_left + 1,
                              pre_left + left_size,
                              in_left,
                              root_in_idx - 1)

            root.right = build(pre_left + left_size + 1,
                               pre_right,
                               root_in_idx + 1,
                               in_right)

            return root

        return build(0, n - 1, 0, n - 1)
```

---

## Сложность

- **Время:** O(n), где n — количество узлов. Построение словаря O(n), каждый узел создаётся ровно один раз.
- **Память:** O(n) для словаря и O(h) для стека рекурсии (h — высота дерева). В худшем случае (вырожденное дерево) O(n).

---

## Примечания

- Массивы содержат **уникальные** значения, что гарантирует однозначность восстановления.
- Аналогичный подход используется для построения по `inorder` и `postorder` (корень — последний элемент `postorder`).
- Данный метод является классическим и широко применяется в задачах на бинарные деревья.
