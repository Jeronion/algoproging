## Определение

**Нижний обзор** бинарного дерева — это список значений узлов, видимых, если смотреть на дерево снизу. Узлы упорядочиваются по горизонтальному расстоянию (HD) от корня:

- Корень имеет HD = 0.
- Левый потомок имеет HD = HD родителя - 1.
- Правый потомок имеет HD = HD родителя + 1.

Для каждого горизонтального расстояния берётся **самый нижний** узел (наибольшая глубина). Если несколько узлов имеют одинаковый HD и одинаковую глубину, выбирается тот, который появляется **позже** при обходе в ширину (level‑order).

Результат выводится в порядке возрастания HD (слева направо).

---

## Постановка задачи

Дан корень бинарного дерева. Требуется вернуть список значений узлов нижнего обзора.

**Пример:**
```
        20
       /  \
      8    22
     / \     \
    5   3     25
       / \    /
      10 14  28
```
Нижний обзор: [5, 10, 4, 28, 25]

---

## Идея решения (BFS)

Используется **обход в ширину (BFS)** с очередью. Для каждого узла храним его горизонтальное расстояние (HD). При посещении узла обновляем словарь `bottom_map[HD] = node.data`. Поскольку BFS обрабатывает узлы уровень за уровнем, последнее записанное значение для данного HD будет соответствовать самому нижнему узлу (или, при равной глубине, тому, что встретился позже в level‑order, что соответствует условию задачи).

### Алгоритм

1. Если дерево пустое, вернуть пустой список.
2. Создать `deque` и поместить в него корень с HD = 0.
3. Создать словарь `bottom_map` для хранения последнего значения для каждого HD.
4. Инициализировать `min_hd = 0` и `max_hd = 0` для границ.
5. Пока очередь не пуста:
   - Извлечь узел и его HD.
   - Обновить `bottom_map[HD] = node.data` (перезаписывая предыдущее значение для этого HD).
   - Обновить `min_hd` и `max_hd`.
   - Если есть левый потомок, добавить его с HD - 1.
   - Если есть правый потомок, добавить его с HD + 1.
6. После обхода, для каждого HD от `min_hd` до `max_hd` взять значение из `bottom_map` и добавить в результат.
7. Вернуть результат.

## Реализация

```python
from collections import deque

class Node:
    def __init__(self, val):
        self.data = val
        self.left = None
        self.right = None

def bottomView(root):
    if not root:
        return []
    
    bottom_map = {}
    min_hd = 0
    max_hd = 0
    q = deque([(root, 0)])
    
    while q:
        node, hd = q.popleft()
        bottom_map[hd] = node.data   # перезапись даёт нижний узел
        min_hd = min(min_hd, hd)
        max_hd = max(max_hd, hd)
        if node.left:
            q.append((node.left, hd - 1))
        if node.right:
            q.append((node.right, hd + 1))
    
    return [bottom_map[hd] for hd in range(min_hd, max_hd + 1)]
```

## Сложность

- **Время:** O(n), где n — количество узлов. Каждый узел обрабатывается один раз.
- **Память:** O(n) для словаря и очереди.

---

## Примечания

- BFS гарантирует, что при равной глубине последний посещённый узел для данного HD будет записан в словарь, так как мы перезаписываем значение каждый раз, когда встречаем узел с этим HD. Это соответствует условию задачи.
- Если несколько узлов имеют одинаковый HD, но разную глубину, более глубокий перезапишет более мелкий, так как BFS обрабатывает уровни последовательно.
- Альтернативный подход с DFS также возможен, но требует явного сравнения глубин; BFS-решение проще и соответствует условию.
