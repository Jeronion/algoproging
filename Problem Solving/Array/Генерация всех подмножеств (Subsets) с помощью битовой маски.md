## Определение

Дан массив **различных** целых чисел `arr` размера `n`. Требуется сгенерировать все **возможные подмножества** (включая пустое). Порядок подмножеств не важен (вывод может быть любым).

**Пример:**
```
arr = [1, 2, 3]
Вывод: [[], [1], [2], [3], [1,2], [1,3], [2,3], [1,2,3]]
```

---

## Идея решения (битовая маска)

В массиве из `n` элементов каждое подмножество можно закодировать **битовой маской** длины `n`:

- `1` на позиции `j` означает, что элемент `arr[j]` **включён** в подмножество.
- `0` — не включён.

Всего возможных масок ровно `2^n` (от `0` до `2^n - 1`). Маска `0` соответствует пустому подмножеству.

Для маски `i` (0 ≤ i < 2^n) формируем подмножество, проходя по всем `j` от `0` до `n-1` и проверяя, установлен ли `j`-й бит в `i`: `(i >> j) & 1`.

---

## Алгоритм

1. Инициализировать пустой список `result`.
2. Для каждого `i` от `0` до `2^n - 1`:
   - Создать временное подмножество `subset = []`.
   - Для каждого `j` от `0` до `n-1`:
     - Если `(i >> j) & 1` истинно, добавить `arr[j]` в `subset`.
   - Добавить `subset` в `result`.
3. Вернуть `result`.

---

## Реализация на Python (list comprehension)

```python
class Solution:
    def subsets(self, arr):
        n = len(arr)
        return [
            [arr[j] for j in range(n) if (i >> j) & 1]
            for i in range(1 << n)
        ]
```

## Пример работы

`arr = [1, 2, 3]`, `n = 3`, `2^3 = 8`.

| i (бинарно) | включённые индексы | подмножество |
|-------------|---------------------|--------------|
| 0 (000)     | —                   | []           |
| 1 (001)     | 0                   | [1]          |
| 2 (010)     | 1                   | [2]          |
| 3 (011)     | 0,1                 | [1,2]        |
| 4 (100)     | 2                   | [3]          |
| 5 (101)     | 0,2                 | [1,3]        |
| 6 (110)     | 1,2                 | [2,3]        |
| 7 (111)     | 0,1,2               | [1,2,3]      |

Результат: `[[],[1],[2],[1,2],[3],[1,3],[2,3],[1,2,3]]`.

---

## Сложность

- **Время:** O(2ⁿ · n) — перебор всех масок и для каждой маски проход по n битам.
- **Память:** O(2ⁿ · n) — для хранения всех подмножеств (обязательно, так как результат должен быть возвращён).

---

## Примечания

- Алгоритм работает только для **различных** элементов. Если есть повторяющиеся, нужно использовать другой подход или предварительно удалить дубликаты.
- Порядок подмножеств в выводе зависит от порядка битов. При необходимости их можно отсортировать, но по условию задачи это не требуется.
- Альтернативные методы: рекурсивный backtracking (DFS) — даёт ту же сложность, но часто лучше для пошагового построения.
- Битовый метод удобен при `n ≤ 20` (2²⁰ ≈ 1 млн), для больших `n` может быть слишком медленным.
