## Условие

Дан массив целых чисел `arr[]`, представляющий последовательность высот (или любых значений). Требуется вернуть **сжатую** версию массива, которая включает:

- Первый и последний элементы исходного массива.
- Каждый элемент, где **направление изменения** последовательности меняется: то есть переход от возрастания к убыванию или от убывания к возрастанию (локальные максимумы и минимумы).

**Примечание:** последовательные равные элементы (плоские участки) должны рассматриваться как один элемент и не влиять на определение смены направления.

### Примеры

**Пример 1:**
```
Вход: arr = [6, 4, 2, -2, 5, 3, 2, 2, -1, -1, 4]
Выход: [6, -2, 5, -1, 4]
```
Объяснение:  
- 6 — начало.  
- Идёт убывание до -2, -2 — локальный минимум (смена убывания на возрастание).  
- Затем возрастание до 5, 5 — локальный максимум (смена возрастания на убывание).  
- Затем убывание до -1, -1 — локальный минимум.  
- Затем возрастание до 4 — конец.

**Пример 2:**
```
Вход: arr = [6, 10, 11, 13]
Выход: [6, 13]
```
Объяснение: последовательность только возрастает, поэтому точек смены направления нет, остаются первый и последний элементы.

**Пример 3:**
```
Вход: arr = [1, 1]
Выход: [1]
```
Объяснение: после удаления последовательных дубликатов остаётся один элемент.

## Идея решения

1. **Сначала удаляем последовательные дубликаты.** Если массив пуст, возвращаем пустой массив.
2. Если после удаления дубликатов длина массива ≤ 2, возвращаем его (он уже является ответом).
3. Иначе проходим по полученному массиву, начиная со второго элемента и до предпоследнего. Для каждого элемента проверяем, является ли он **локальным экстремумом** относительно своих соседей:
   - Если `arr[i] > arr[i-1]` и `arr[i] > arr[i+1]` — локальный максимум.
   - Если `arr[i] < arr[i-1]` и `arr[i] < arr[i+1]` — локальный минимум.
   В обоих случаях добавляем элемент в результат.
4. Обязательно добавляем последний элемент (если он не был добавлен как часть экстремума или начало/конец).

### Почему это работает?

- Первый и последний элементы добавляются всегда, так как они не могут быть определены как локальные экстремумы по правилу (нет соседа с одной стороны).
- Локальные экстремумы — это точки, в которых направление движения меняется. Именно они нужны для сжатия.
- Удаление последовательных дубликатов гарантирует, что у нас нет плоских участков, которые могли бы создать ложные экстремумы (например, `[1,2,2,1]` после удаления дубликатов превращается в `[1,2,1]`, и `2` становится локальным максимумом).

## Реализация

```python
class Solution:
    def extractPoints(self, arr):
        # удаляем последовательные дубликаты
        if not arr:
            return []
        unique = [arr[0]]
        for i in range(1, len(arr)):
            if arr[i] != arr[i-1]:
                unique.append(arr[i])
        
        n = len(unique)
        if n <= 2:
            return unique
        
        res = [unique[0]]
        for i in range(1, n-1):
            # локальный максимум
            if unique[i] > unique[i-1] and unique[i] > unique[i+1]:
                res.append(unique[i])
            # локальный минимум
            elif unique[i] < unique[i-1] and unique[i] < unique[i+1]:
                res.append(unique[i])
        
        if res[-1] != unique[-1]:
            res.append(unique[-1])
        return res
```

---

## Сложность

- **Время:** O(n) — один проход для удаления дубликатов, один проход для поиска экстремумов.
- **Память:** O(n) в худшем случае (для хранения массива без дубликатов). Можно реализовать без дополнительной памяти, обрабатывая дубликаты на лету, но код становится сложнее. В текущей версии дополнительная память O(n).
