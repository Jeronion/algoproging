## Определение

Дана перестановка целых чисел (все элементы уникальны) в виде массива `arr`. Требуется найти **лексикографически следующую** перестановку (т.е. следующую по порядку в отсортированном списке всех перестановок). Если текущая перестановка является последней (наибольшей), нужно вернуть первую (наименьшую) перестановку, отсортированную по возрастанию.

Задача известна как [LeetCode 31. Next Permutation](https://leetcode.com/problems/next-permutation/).

**Примеры:**
- `[2,4,1,7,5,0]` → `[2,4,5,0,1,7]`
- `[3,2,1]` → `[1,2,3]` (последняя → первая)
- `[1,3,5,4,2]` → `[1,4,2,3,5]`

---

## Идея решения

Алгоритм работает за O(n) и использует только константную дополнительную память. Основная идея:

1. Найти самый правый индекс `pivot`, где нарушается убывающий порядок с конца, т.е. `arr[pivot] < arr[pivot+1]`. Если такого индекса нет, то массив уже максимален — переворачиваем его и завершаем.
2. Найти самый правый индекс `i` (справа от `pivot`), где `arr[i] > arr[pivot]` — это наименьший элемент, который больше `arr[pivot]` в суффиксе.
3. Поменять местами `arr[pivot]` и `arr[i]`.
4. Развернуть суффикс, начиная с `pivot+1` до конца (так как он был в убывающем порядке, после разворота станет возрастающим, что даёт минимальную возможную перестановку для этого суффикса).

---

## Пошаговый алгоритм

Пусть `n = len(arr)`.

1. Найти `pivot`:
   ```
   pivot = -1
   for i = n-2 downto 0:
       if arr[i] < arr[i+1]:
           pivot = i
           break
   ```
   Если `pivot == -1` → массив убывающий → `reverse(arr)` и выход.

2. Найти `successor` (самый правый элемент больше `arr[pivot]`):
   ```
   for i = n-1 downto pivot+1:
       if arr[i] > arr[pivot]:
           swap(arr[i], arr[pivot])
           break
   ```

3. Развернуть суффикс `arr[pivot+1 : n]` (можно двумя указателями).

## Реализация на Python

```python
def next_permutation(arr):
    n = len(arr)
    # 1. Находим pivot
    pivot = -1
    for i in range(n - 2, -1, -1):
        if arr[i] < arr[i + 1]:
            pivot = i
            break

    # Если pivot не найден, это последняя перестановка
    if pivot == -1:
        arr.reverse()
        return

    # 2. Находим элемент для замены (successor)
    for i in range(n - 1, pivot, -1):
        if arr[i] > arr[pivot]:
            arr[i], arr[pivot] = arr[pivot], arr[i]
            break

    # 3. Разворачиваем суффикс
    left, right = pivot + 1, n - 1
    while left < right:
        arr[left], arr[right] = arr[right], arr[left]
        left += 1
        right -= 1
```

## Пример работы

Возьмём `arr = [2,4,1,7,5,0]`.

1. Идём справа: 0 < 5? нет; 5 < 7? да → pivot = 3 (значение 7? нет, pivot — это индекс где arr[3]=7? проверяем: i=3, arr[3]=7, arr[4]=5, 7<5? нет, идём дальше. i=2: arr[2]=1, arr[3]=7, 1<7 → pivot=2. Значит pivot — это индекс 2 (значение 1).
2. Ищем справа элемент >1: идём с конца: 0>1? нет; 5>1? да → i=4 (значение 5). Меняем arr[2] и arr[4]: получаем `[2,4,5,7,1,0]`.
3. Разворачиваем суффикс с индекса 3: было `[7,1,0]`, после разворота `[0,1,7]`. Итог: `[2,4,5,0,1,7]`.

## Сложность

- **Время:** O(n) — один проход для поиска pivot, один для поиска successor, один для разворота.
- **Память:** O(1) — все операции выполняются на месте.

---

## Примечания

- Алгоритм работает для любых массивов с уникальными элементами (перестановок). Если есть дубликаты, он также даёт следующий лексикографический порядок (стандартное поведение).
- Встроенная функция `next_permutation` в C++ реализована именно так.
- В Python можно использовать `itertools.permutations`, но это потребует O(n!) времени, что неприемлемо для больших массивов.
