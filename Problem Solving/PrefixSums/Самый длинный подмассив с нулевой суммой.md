## Определение

Дан массив целых чисел (могут быть положительные, отрицательные и нули). Требуется найти длину **самого длинного непрерывного подмассива**, сумма элементов которого равна **0**.

**Пример:**
```
arr = [15, -2, 2, -8, 1, 7, 10] → ответ 5 (подмассив [-2, 2, -8, 1, 7])
arr = [1, 2, 3] → ответ 0 (нет подмассива с суммой 0)
arr = [1, 0, 3] → ответ 1 (подмассив [0])
```

---

## Идея решения (префиксные суммы + хеш-таблица)

Оптимальное решение O(n) использует **префиксные суммы** и **хеш-таблицу** для хранения первого вхождения каждого значения префиксной суммы.

**Ключевое наблюдение:** если для двух индексов `i` и `j` (`i < j`) префиксные суммы равны (`prefix[i] == prefix[j]`), то сумма элементов между `i+1` и `j` равна нулю (потому что `prefix[j] - prefix[i] = 0`). Длина такого подмассива = `j - i`.

Чтобы учесть подмассивы, начинающиеся с индекса 0, добавляем в хеш-таблицу `{0: -1}` (префиксная сумма 0 до начала массива).

### Алгоритм

1. Инициализировать `prefix_sum = 0`, `max_len = 0`, словарь `first_seen = {0: -1}`.
2. Пройти по массиву с индексом `i` от 0 до `n-1`:
   - `prefix_sum += arr[i]`.
   - Если `prefix_sum` уже есть в `first_seen`:
     - Длина текущего подмассива = `i - first_seen[prefix_sum]`.
     - Обновить `max_len = max(max_len, длина)`.
   - Иначе:
     - Запомнить `first_seen[prefix_sum] = i`.
3. Вернуть `max_len`.

---

## Реализация

```python
def longest_subarray_zero_sum(arr):
    prefix_sum = 0
    first_seen = {0: -1}   # сумма 0 до начала массива
    max_len = 0
    for i, val in enumerate(arr):
        prefix_sum += val
        if prefix_sum in first_seen:
            max_len = max(max_len, i - first_seen[prefix_sum])
        else:
            first_seen[prefix_sum] = i
    return max_len

# Пример
arr = [15, -2, 2, -8, 1, 7, 10]
print(longest_subarray_zero_sum(arr))  # 5
```

## Сложность

- **Время:** O(n) — один проход по массиву.
- **Память:** O(n) — в худшем случае хеш-таблица хранит все различные префиксные суммы.

---

## Примечания

- Наивный подход O(n²) перебирает все подмассивы — не подходит для больших массивов.
- Метод работает как для положительных, так и для отрицательных чисел.
- Если весь массив имеет нулевую сумму, ответом будет `n` (длина всего массива).
- Иногда требуется вернуть не длину, а сам подмассив — можно дополнительно хранить индексы.
