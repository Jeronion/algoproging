## Определение

Дан массив целых чисел `arr` и положительное целое число `k`. Требуется найти длину **самого длинного непрерывного подмассива**, сумма элементов которого **делится на `k`** (т.е. `sum % k == 0`).

**Пример:**
```
arr = [2, 7, 6, 1, 4, 5], k = 3 → ответ 4 (подмассив [7,6,1,4] сумма 18)
arr = [-2, 2, -5, 12, -11, -1, 7], k = 3 → ответ 5 (подмассив [2,-5,12,-11,-1] сумма -3)
```

---

## Идея решения (префиксные суммы по модулю + хеш-таблица)

Наивный подход O(n²) перебирает все подмассивы. Оптимальное решение использует **префиксные суммы** и их **остатки от деления на k**.

**Ключевое наблюдение:** если для двух индексов `i` и `j` (`i < j`) остатки от деления префиксных сумм на `k` равны, то сумма подмассива между `i+1` и `j` делится на `k` (поскольку `(prefix[j] - prefix[i]) % k == 0`). Также если сам `prefix[i] % k == 0`, то подмассив от начала до `i` уже подходит.

### Алгоритм

1. Инициализировать `prefix_mod = 0`, словарь `first_occurrence` (хранит первый индекс для каждого остатка), `max_len = 0`.
2. Пройти по массиву с индексом `i` от `0` до `n-1`:
   - `prefix_mod = (prefix_mod + arr[i]) % k`
   - **Важно:** в Python отрицательные числа при взятии модуля дают положительный остаток, но для корректной работы с отрицательными элементами нужно приводить остаток к диапазону `[0, k-1]` (например, `(prefix_mod + k) % k`). В стандартном `%` уже даёт неотрицательный результат, но если реализация в коде использует просто `%`, то для отрицательных чисел `(sum % k)` может быть отрицательным? В Python `%` всегда возвращает результат с тем же знаком, что и делитель (положительный). Поэтому `(sum + arr[i]) % k` уже корректен.
   - Если `prefix_mod == 0`, то `max_len = max(max_len, i+1)`.
   - Если `prefix_mod` уже встречался:
     - `max_len = max(max_len, i - first_occurrence[prefix_mod])`
   - Иначе:
     - `first_occurrence[prefix_mod] = i`
3. Вернуть `max_len`.

---

## Реализация

```python
def longest_subarray_divisible_by_k(arr, k):
    first_occ = {}
    prefix_mod = 0
    max_len = 0
    for i, val in enumerate(arr):
        prefix_mod = (prefix_mod + val) % k
        if prefix_mod == 0:
            max_len = max(max_len, i + 1)
        elif prefix_mod in first_occ:
            max_len = max(max_len, i - first_occ[prefix_mod])
        else:
            first_occ[prefix_mod] = i
    return max_len

# Пример
arr = [2, 7, 6, 1, 4, 5]
k = 3
print(longest_subarray_divisible_by_k(arr, k))  # 4
```

---

## Сложность

- **Время:** O(n) — один проход по массиву.
- **Память:** O(min(n, k)) — словарь хранит различные остатки (не более k).

---

## Примечания

- Важно, что остаток вычисляется на каждом шаге, и для отрицательных чисел Python даёт неотрицательный остаток, что корректно.
- Если в массиве есть элементы, кратные k, они также будут образовывать подмассивы длины 1.
- Алгоритм аналогичен поиску самого длинного подмассива с нулевой суммой, но вместо суммы используется сумма по модулю k.
