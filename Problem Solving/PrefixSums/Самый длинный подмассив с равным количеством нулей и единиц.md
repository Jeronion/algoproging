## Определение

Дан массив, состоящий только из 0 и 1. Требуется найти длину **самого длинного непрерывного подмассива**, в котором количество нулей равно количеству единиц.

**Пример:**
```
arr = [1, 0, 1, 1, 1, 0, 0] → ответ 6 (подмассив с индексов 1 по 6: [0,1,1,1,0,0] содержит три 0 и три 1)
arr = [0, 0, 1, 1, 0] → ответ 4 (например, [0,0,1,1] или [0,1,1,0])
arr = [0] → ответ 0
```

---

## Идея решения (хеш-таблица + префиксная сумма)

Заменяем каждый `0` на `-1`, а `1` оставляем как `1`. Тогда условие «равное количество 0 и 1» превращается в условие «сумма элементов подмассива равна 0». Задача сводится к **поиску самого длинного подмассива с нулевой суммой**, которая решается с помощью префиксных сумм и хеш-таблицы.

### Алгоритм

1. Инициализировать `prefix = 0`, словарь `first_seen` (хранит первый индекс каждого значения префиксной суммы), `max_len = 0`.
2. Добавить в словарь `{0: -1}` (чтобы обрабатывать подмассивы, начинающиеся с индекса 0).
3. Пройти по массиву с индексом `i`:
   - Если `arr[i] == 0`, добавить к `prefix` `-1`, иначе добавить `1`.
   - Если `prefix == 0`, то подмассив от начала до `i` уже имеет равное количество 0 и 1 → обновить `max_len = i+1`.
   - Если `prefix` уже есть в `first_seen`:
     - Длина подмассива = `i - first_seen[prefix]`
     - Обновить `max_len = max(max_len, длина)`
   - Иначе:
     - Запомнить `first_seen[prefix] = i`.
4. Вернуть `max_len`.

---

## Реализация

```python
def longest_subarray_equal_01(arr):
    first_seen = {}
    prefix = 0
    max_len = 0
    for i, val in enumerate(arr):
        prefix += 1 if val == 1 else -1
        if prefix == 0:
            max_len = i + 1
        elif prefix in first_seen:
            max_len = max(max_len, i - first_seen[prefix])
        else:
            first_seen[prefix] = i
    return max_len

# Пример
arr = [1, 0, 1, 1, 1, 0, 0]
print(longest_subarray_equal_01(arr))  # 6
```

## Сложность

- **Время:** O(n) — один проход по массиву.
- **Память:** O(n) — в худшем случае словарь хранит все различные префиксные суммы.

---

## Примечания

- Замена 0 на -1 — ключевой трюк, превращающий задачу в поиск нулевой суммы.
- Если весь массив имеет равное количество 0 и 1, ответом будет `n`.
- Метод работает для массивов любой длины (до 10⁵ и более).
