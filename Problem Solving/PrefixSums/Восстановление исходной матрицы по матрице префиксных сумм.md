## Определение

Дана матрица `after` размера `N × M`, которая получена из некоторой исходной матрицы `before` по правилу: каждый элемент `after[i][j]` равен сумме всех элементов `before[x][y]` для `0 ≤ x ≤ i` и `0 ≤ y ≤ j` (т.е. `after` — это двумерная префиксная сумма `before`). Требуется восстановить исходную матрицу `before`.

**Пример:**
```
after = [[1, 3, 6],
         [3, 7, 11]]
before = [[1, 2, 3],
          [2, 2, 1]]
```

---

## Идея решения

Восстановление `before` выполняется **обратным ходом**: из текущего значения `after[i][j]` вычитаем уже вычисленные суммы. Формула для `before[i][j]`:

```
before[i][j] = after[i][j]
              - (after[i-1][j] если i > 0)
              - (after[i][j-1] если j > 0)
              + (after[i-1][j-1] если i > 0 и j > 0)
```

Эта формула обратна к построению 2D префиксной суммы. Действительно, для `i,j` значение `after[i][j]` включает `before[i][j]` плюс всё, что лежит выше и левее. Вычитая уже восстановленные верхнюю и левую области, компенсируем двойное вычитание угла.

---

## Алгоритм

1. Создать матрицу `before` того же размера, заполненную нулями.
2. Пройти по всем `i` от `0` до `N-1`, `j` от `0` до `M-1`:
   - `total = after[i][j]`
   - Если `i > 0`: `total -= after[i-1][j]`
   - Если `j > 0`: `total -= after[i][j-1]`
   - Если `i > 0` и `j > 0`: `total += after[i-1][j-1]`
   - Присвоить `before[i][j] = total`
3. Вернуть `before`.

---

## Реализация на Python

```python
class Solution:
    def computeBeforeMatrix(self, N, M, after):
        before = [[0] * M for _ in range(N)]
        for i in range(N):
            for j in range(M):
                total = after[i][j]
                if i > 0:
                    total -= after[i-1][j]
                if j > 0:
                    total -= after[i][j-1]
                if i > 0 and j > 0:
                    total += after[i-1][j-1]
                before[i][j] = total
        return before
```

---

## Сложность

- **Время:** O(N·M) — один проход по матрице.
- **Память:** O(1) дополнительно (не считая входных данных), так как результат можно записывать в `before` или даже модифицировать `after` на месте, если разрешено.

---

## Примечания

- Матрица `after` должна быть согласованной (т.е. действительно являться префиксной суммой некоторой матрицы). Для любых целых входных данных восстановление выполняется однозначно.
- Эту операцию можно интерпретировать как вычисление **дискретной производной** (конечных разностей) двумерного массива.
