## Определение

Дан массив целых чисел `arr` и целое число `k`. Требуется найти длину **самого длинного непрерывного подмассива**, сумма элементов которого равна `k`. Если такого подмассива нет, вернуть `0`.

**Пример:**
```
arr = [10, 5, 2, 7, 1, -10], k = 15 → ответ 6 (подмассив [10,5,2,7,1,-10])
arr = [-5, 8, -14, 2, 4, 12], k = -5 → ответ 5 (подмассив [-5,8,-14,2,4])
arr = [10, -10, 20, 30], k = 5 → ответ 0
```

---

## Идея решения (хеш-таблица + префиксные суммы)

Наивный подход O(n²) перебирает все подмассивы. Оптимальное решение O(n) использует **префиксные суммы** и **хеш-таблицу**.

**Ключевое наблюдение:** если для индексов `i` и `j` (`i < j`) выполняется `prefix[j] - prefix[i] = k`, то сумма подмассива между `i+1` и `j` равна `k`. Длина такого подмассива = `j - i`.

Чтобы найти максимальную длину, нужно для каждого значения префиксной суммы хранить **самый ранний индекс**, где оно встретилось. При обходе массива:

- Текущая префиксная сумма `curr = prefix[i]`.
- Ищем `curr - k` в хеш-таблице: если есть индекс `prev`, то подмассив от `prev+1` до `i` имеет сумму `k`, его длина = `i - prev`.
- Если `curr` ещё не встречался, запоминаем `i` как первое вхождение.

Дополнительно обрабатываем случай, когда сам `curr == k` (подмассив от начала).

### Алгоритм

1. Инициализировать `prefix_sum = 0`, `max_len = 0`, словарь `first_seen = {}`.
2. Для `i` от `0` до `n-1`:
   - `prefix_sum += arr[i]`
   - Если `prefix_sum == k`:
     - `max_len = max(max_len, i + 1)`
   - Если `(prefix_sum - k)` в `first_seen`:
     - `max_len = max(max_len, i - first_seen[prefix_sum - k])`
   - Если `prefix_sum` нет в `first_seen`:
     - `first_seen[prefix_sum] = i`
3. Вернуть `max_len`.

---

## Реализация

```python
def longest_subarray_sum_k(arr, k):
    first_seen = {}
    prefix_sum = 0
    max_len = 0
    for i, val in enumerate(arr):
        prefix_sum += val
        if prefix_sum == k:
            max_len = max(max_len, i + 1)
        if (prefix_sum - k) in first_seen:
            max_len = max(max_len, i - first_seen[prefix_sum - k])
        if prefix_sum not in first_seen:
            first_seen[prefix_sum] = i
    return max_len

# Пример
arr = [10, 5, 2, 7, 1, -10]
k = 15
print(longest_subarray_sum_k(arr, k))  # 6
```

---

## Сложность

- **Время:** O(n) — один проход по массиву.
- **Память:** O(n) — в худшем случае словарь хранит все различные префиксные суммы.

---

## Примечания

- Алгоритм работает для любых целых чисел (включая отрицательные и нули).
- Если в массиве есть подмассив с суммой k, начинающийся не с первого элемента, он будет найден через разность префиксных сумм.
- Важно добавлять в словарь только **первое** вхождение каждой префиксной суммы, чтобы получить максимальную длину (более ранний индекс даёт большую длину).
