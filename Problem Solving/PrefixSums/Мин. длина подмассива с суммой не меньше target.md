## Условие

Задача известна как [LeetCode 209. Minimum Size Subarray Sum](https://leetcode.com/problems/minimum-size-subarray-sum/).

Дан массив целых положительных чисел `nums` и целевое число `target`. Требуется найти **минимальную длину** непрерывного подмассива, сумма элементов которого больше или равна `target`. Если такого подмассива нет, вернуть `0`.

**Вход:**  
- `target` — целое положительное число  
- `nums` — массив положительных целых чисел

**Выход:**  
Минимальная длина подмассива (непрерывной последовательности) с суммой ≥ `target`, либо `0`, если такого нет.

**Пример:**
```
target = 7
nums = [2,3,1,2,4,3]
Ответ: 2 (подмассив [4,3] или [3,4]? на самом деле [4,3] сумма 7, длина 2)
```

## Решение с префиксными суммами и двумя указателями

### Идея

Построим массив префиксных сумм `pref` длины `n+1`, где `pref[0] = 0`, а `pref[i]` — сумма первых `i` элементов (т.е. `nums[0..i-1]`). Тогда сумма подмассива от `l` до `r-1` (в 0-индексации) равна `pref[r] - pref[l]`.

Затем будем двигать два указателя `left` и `right` по префиксному массиву, поддерживая окно, в котором сумма `pref[right] - pref[left]` не меньше `target`. При выполнении условия пытаемся уменьшить окно, сдвигая `left`, иначе расширяем окно, сдвигая `right`. Это классический метод двух указателей на префиксных суммах.

### Реализация

```python
def minSubArrayLen(target: int, nums: list[int]) -> int:
    n = len(nums)
    pref = [0] * (n + 1)
    for i in range(1, n + 1):
        pref[i] = pref[i - 1] + nums[i - 1]
    
    res = float('inf')
    left = 0
    for right in range(1, n + 1):
        # сумма окна [left, right-1] = pref[right] - pref[left]
        while pref[right] - pref[left] >= target:
            res = min(res, right - left)
            left += 1
    return res if res != float('inf') else 0
```

В этом варианте `right` движется в цикле, а `left` поджимается изнутри. Это более читаемо, чем исходный код с while.

### Сложность

- **Время:** O(n) — каждый указатель проходит массив не более одного раза.
- **Память:** O(n) для префиксного массива (можно оптимизировать до O(1) без него).

## Классическое решение (скользящее окно без префиксного массива)

Можно обойтись без дополнительной памяти, используя два указателя на самом массиве и поддерживая текущую сумму.

```python
def minSubArrayLen(target: int, nums: list[int]) -> int:
    left = 0
    current_sum = 0
    res = float('inf')
    for right in range(len(nums)):
        current_sum += nums[right]
        while current_sum >= target:
            res = min(res, right - left + 1)
            current_sum -= nums[left]
            left += 1
    return res if res != float('inf') else 0
```

---

## Сложность

- **Время:** O(n) — каждый элемент добавляется и удаляется не более одного раза.
- **Память:** O(1) — только переменные.
