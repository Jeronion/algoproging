## Условие

Задача известна как [LeetCode 525. Contiguous Array](https://leetcode.com/problems/contiguous-array/).

Дан бинарный массив `nums`, состоящий только из 0 и 1. Требуется найти **максимальную длину** непрерывного подмассива, в котором количество 0 и 1 **равно**.

**Вход:** массив `nums` (длиной n), состоящий из 0 и 1.
**Выход:** максимальная длина такого подмассива. Если его нет, вернуть 0.

**Пример:**
```
nums = [0, 1, 0]
Вывод: 2 (подмассив [0,1] или [1,0] дают по одному 0 и одному 1)
```

---

## Примечания

- Подмассив должен быть непрерывным.
- Если весь массив содержит равное количество 0 и 1, ответом будет длина массива.
- Пустой подмассив не рассматривается, минимальная длина 2? Нет, может быть и длина 2, но если есть одиночный элемент, то он не подходит.

---

## Идея решения

Ключевое наблюдение: если заменить все 0 на -1, то условие равенства количества 0 и 1 эквивалентно тому, что сумма элементов подмассива равна 0. Таким образом, задача сводится к поиску самого длинного подмассива с нулевой суммой в преобразованном массиве.

Для этого используем **префиксные суммы**. Пусть `pref[i]` — сумма первых i элементов преобразованного массива (при i от 0 до n, `pref[0] = 0`). Тогда сумма подмассива от l до r-1 (в 0-индексации) равна `pref[r] - pref[l]`. Если эта сумма равна 0, то `pref[r] = pref[l]`. Следовательно, нам нужно найти два индекса i < j, для которых `pref[i] = pref[j]`, и максимизировать `j - i`.

Причём чем раньше встретится данная сумма, тем потенциально длиннее может быть подмассив, поэтому для каждого значения суммы сохраняем **самый ранний индекс**, где она встретилась.

### Алгоритм

1. Создаём массив `pref` длины `n+1`, где `pref[0] = 0`.
2. Для i от 1 до n:
   - Если `nums[i-1] == 1`, то `pref[i] = pref[i-1] + 1`
   - Если `nums[i-1] == 0`, то `pref[i] = pref[i-1] - 1` (или `pref[i-1] + (-1)`)
3. Создаём хеш-таблицу `hash_map`, где ключ — значение префиксной суммы, значение — первый индекс, где эта сумма встретилась.
4. Проходим по всем индексам i от 0 до n:
   - Если значение `pref[i]` ещё не в `hash_map`, сохраняем `hash_map[pref[i]] = i`.
   - Иначе (уже есть) вычисляем длину подмассива `i - hash_map[pref[i]]` и обновляем максимум.
5. Возвращаем максимум.

---

## Реализация

```python
def findMaxLength(nums: list[int]) -> int:
    max_len = 0
    n = len(nums)
    pref = [0] * (n + 1)
    for i in range(1, n + 1):
        # преобразуем 0 в -1, 1 в +1
        pref[i] = pref[i-1] + (1 if nums[i-1] else -1)

    hash_map = {}  # сумма -> первый индекс
    for i in range(n + 1):
        if pref[i] in hash_map:
            # нашли подмассив с нулевой суммой
            max_len = max(max_len, i - hash_map[pref[i]])
        else:
            hash_map[pref[i]] = i
    return max_len
```

## Сложность

- **Время:** O(n) — один проход для построения префиксов и один проход для проверки (можно совместить, но здесь разделено).
- **Память:** O(n) для хранения префиксного массива и хеш-таблицы. Можно уменьшить до O(n) только хеш-таблицы, если вычислять префиксную сумму на лету без хранения всего массива.

**Оптимизация памяти** (без хранения всего `pref`):

```python
def findMaxLength(nums):
    max_len = 0
    count = 0  # текущая префиксная сумма
    first_occurrence = {0: -1}  # сумма -> первый индекс, начинаем с -1 для удобства
    for i, num in enumerate(nums):
        count += 1 if num else -1
        if count in first_occurrence:
            max_len = max(max_len, i - first_occurrence[count])
        else:
            first_occurrence[count] = i
    return max_len
```
