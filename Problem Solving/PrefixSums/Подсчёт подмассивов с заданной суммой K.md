## Определение

Дан массив целых чисел `arr` (могут быть положительные, отрицательные и нули) и целое число `k`. Требуется подсчитать количество **непрерывных подмассивов**, сумма элементов которых равна `k`.

**Пример:**
```
arr = [10, 2, -2, -20, 10], k = -10 → ответ 3
(подмассивы: [10,2,-2,-20], [2,-2,-20,10], [-20,10])
```

---

## Идея решения (хеш-таблица + префиксные суммы)

Наивный подход O(n²) перебирает все подмассивы. Оптимальное решение O(n) использует **префиксные суммы** и **хеш-таблицу** для хранения частоты встречаемости каждой суммы.

**Ключевое наблюдение:** для каждого индекса `i` с префиксной суммой `curr`, количество подмассивов, заканчивающихся в `i` и имеющих сумму `k`, равно количеству предыдущих индексов `j`, для которых `prefix[j] = curr - k`. Это следует из равенства `curr - (curr - k) = k`.

### Алгоритм

1. Инициализировать `prefix_sum = 0`, `count = 0`, словарь `freq` (изначально пуст).
2. Для каждого элемента `val` в массиве:
   - `prefix_sum += val`
   - Если `prefix_sum == k`, то увеличить `count` на 1 (подмассив от начала до текущего индекса).
   - Если `(prefix_sum - k)` есть в `freq`, то добавить к `count` значение `freq[prefix_sum - k]` (количество подмассивов, оканчивающихся в текущей позиции).
   - Затем увеличить `freq[prefix_sum]` на 1.
3. Вернуть `count`.

---

## Реализация

```python
def cnt_subarrays_sum_k(arr, k):
    freq = {}
    prefix_sum = 0
    count = 0
    for val in arr:
        prefix_sum += val
        if prefix_sum == k:
            count += 1
        if (prefix_sum - k) in freq:
            count += freq[prefix_sum - k]
        freq[prefix_sum] = freq.get(prefix_sum, 0) + 1
    return count

# Пример
arr = [10, 2, -2, -20, 10]
k = -10
print(cnt_subarrays_sum_k(arr, k))  # 3
```

## Сложность

- **Время:** O(n) — один проход по массиву.
- **Память:** O(n) — словарь может хранить все различные префиксные суммы.

## Примечания

- Алгоритм является классическим расширением задачи «Subarray Sum Equals K» (LeetCode 560).
- Обязательно использовать `freq.get(prefix_sum, 0)` для корректного подсчёта, так как ключ может отсутствовать.
- Метод работает для любых целых чисел, включая отрицательные и нули.
- Если `k = 0`, то подмассивы с нулевой суммой также учитываются (включая пустые? нет, только непустые, так как мы начинаем с `prefix_sum = 0` и проверяем `prefix_sum == k` после добавления первого элемента).
