## Определение

Дана квадратная матрица `mat` размера `n × n`, где `1` означает свободную клетку, а `0` — стену (непроходимую). Крыса находится в левом верхнем углу `(0,0)` и должна добраться до правого нижнего `(n-1, n-1)`, двигаясь только вниз, вверх, влево, вправо (по четырём направлениям) и не заходя в стены. Требуется найти **все возможные пути**, выводя их в виде строк из букв: `'D'` (вниз), `'U'` (вверх), `'L'` (влево), `'R'` (вправо).

**Пример:**
```
mat = [[1, 0, 0, 0],
       [1, 1, 0, 1],
       [0, 1, 0, 0],
       [1, 1, 1, 1]]
Один из путей: "DRDDRR"
```

---

## Идея решения (backtracking)

Используем рекурсивный поиск с возвратом (backtracking). На каждом шаге:

1. Отмечаем текущую клетку как посещённую (временно зануляем).
2. Пытаемся перейти в одного из четырёх соседей (в порядке D, L, R, U — так получится лексикографический порядок, если он нужен).
3. Если соседняя клетка в пределах лабиринта и не является стеной (значение 1), то рекурсивно вызываем функцию, добавив в путь соответствующую букву.
4. После возврата из рекурсии откатываем изменения (восстанавливаем клетку).

Базовый случай: достигли правого нижнего угла — сохраняем текущий путь как строку.

---

## Алгоритм

1. Проверить, что стартовая клетка `(0,0)` не является стеной (иначе сразу вернуть пустой список).
2. Создать глобальный список `res` для хранения найденных путей.
3. Определить направления: `(1,0,'D')`, `(0,-1,'L')`, `(0,1,'R')`, `(-1,0,'U')`.
4. Запустить рекурсию `backtrack(path, i, j)`:
   - Если `(i,j)` — это `(n-1, n-1)`, добавить `path` в `res`.
   - Сохранить текущее значение клетки, временно установить её в 0 (чтобы не ходить по одному месту дважды).
   - Для каждого направления:
     - Вычислить новые координаты.
     - Если новые координаты в пределах матрицы и соответствующая клетка равна 1, вызвать `backtrack(path + [буква], ni, nj)`.
   - Восстановить исходное значение клетки.
5. Вернуть `res`.

## Реализация

```python
def rat_in_maze(mat):
    n = len(mat)
    m = len(mat[0])   # для квадратной матрицы n == m, но код общий
    res = []
    if mat[0][0] == 0:
        return res

    directions = [(1, 0, "D"), (0, -1, "L"), (0, 1, "R"), (-1, 0, "U")]

    def backtrack(path, i, j):
        if i == n - 1 and j == m - 1:
            res.append("".join(path))
            return
        old = mat[i][j]
        mat[i][j] = 0   # помечаем как посещённое
        for di, dj, letter in directions:
            ni, nj = i + di, j + dj
            if 0 <= ni < n and 0 <= nj < m and mat[ni][nj] == 1:
                path.append(letter)
                backtrack(path, ni, nj)
                path.pop()
        mat[i][j] = old

    backtrack([], 0, 0)
    return res
```

---

## Сложность

- **Время:** O(4^(n²)) в худшем случае (все клетки свободны), но на практике отсечения сокращают перебор.
- **Память:** O(n²) для стека рекурсии и временной модификации матрицы.

---

## Примечания

- Порядок направлений в `directions` влияет на лексикографический порядок путей (D → L → R → U даёт стандартный порядок, как в задачах GfG).
- Код модифицирует исходную матрицу. Если требуется сохранить оригинал, нужно передавать копию.
- Для квадратной матрицы `n == m`, но код работает и для прямоугольной.

## Мой код

```python
n, m = map(int, input().split())
mat = [list(map(int, input().split())) for _ in range(n)]
directions = [(1, 0, "D"), (0, -1, "L"), (0, 1, "R"), (-1, 0, "U")]
res = []

def backtrack(path, i, j):
    if i == n - 1 and j == m - 1:
        res.append("".join(path))
        return
    old = mat[i][j]
    mat[i][j] = 0  # mark as visited
    for di, dj, ch in directions:
        ni, nj = i + di, j + dj
        if 0 <= ni < n and 0 <= nj < m and mat[ni][nj] == 1:
            path.append(ch)
            backtrack(path, ni, nj)
            path.pop()
    mat[i][j] = old

if mat[0][0] == 1:
    backtrack([], 0, 0)

print(res)
```
