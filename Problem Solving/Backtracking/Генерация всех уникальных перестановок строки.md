## Определение

Дана строка `s`, которая может содержать **повторяющиеся символы**. Требуется сгенерировать все **уникальные** перестановки символов строки. Порядок вывода может быть любым.

**Пример:**
```
s = "ABC" → ["ABC", "ACB", "BAC", "BCA", "CAB", "CBA"]
s = "AAA" → ["AAA"]
```

---

## Идея решения (backtracking с пропуском дубликатов)

Классический метод генерации перестановок через рекурсивный перебор символов с использованием массива `used` для отметки уже взятых символов. Поскольку в строке могут быть дубликаты, необходимо **пропускать повторяющиеся символы на одном уровне рекурсии**, чтобы не генерировать одинаковые перестановки несколько раз.

### Алгоритм

1. **Отсортировать** строку `s` (чтобы одинаковые символы стояли рядом).
2. Создать массив `used` (boolean) длиной `n` для отметки использованных индексов.
3. Рекурсивная функция `backtrack(path)`:
   - Если длина `path` == `n`, добавить `path` в результат.
   - Иначе перебрать все индексы `i` от `0` до `n-1`:
     - Если `used[i]` уже `True`, пропустить.
     - Если `i > 0` и `s[i] == s[i-1]` и `not used[i-1]`, пропустить (это условие гарантирует, что на текущем уровне рекурсии из одинаковых символов будет выбран только первый, остальные пропускаются).
     - Отметить `used[i] = True`, добавить `s[i]` в `path`, рекурсивный вызов, затем откатить (`used[i] = False`, `path.pop()`).

---

## Реализация

```python
class Solution:
    def findPermutation(self, s: str):
        s = sorted(s)                     # сортируем для группировки одинаковых символов
        n = len(s)
        used = [False] * n
        res = []
        
        def backtrack(path):
            if len(path) == n:
                res.append("".join(path))
                return
            for i in range(n):
                if used[i]:
                    continue
                # пропускаем одинаковые символы, если предыдущий такой же не использован
                if i > 0 and s[i] == s[i-1] and not used[i-1]:
                    continue
                used[i] = True
                path.append(s[i])
                backtrack(path)
                path.pop()
                used[i] = False
        
        backtrack([])
        return res
```

---

## Сложность

- **Время:** O(n! · n) в худшем случае (для строки с уникальными символами). При наличии повторений количество перестановок меньше.
- **Память:** O(n) для стека рекурсии и массива `used`, плюс O(n! · n) для хранения результата (обязательно, так как нужно вернуть все перестановки).

---

## Примечания

- Условие `i > 0 and s[i] == s[i-1] and not used[i-1]` означает: если текущий символ такой же, как предыдущий, и предыдущий ещё не использован на этом уровне рекурсии, то мы уже обработали все варианты с первым из этих одинаковых символов, и повторный выбор приведёт к дубликатам. Поэтому пропускаем.
- Без этого условия для строки `"AA"` было бы сгенерировано две одинаковые перестановки `"AA"` (выбор сначала первого `A`, затем второго, и наоборот).
- Сортировка входной строки обязательна для корректной работы условия пропуска дубликатов.
