## Определение

Дана строка `s` и словарь `dictionary` (список слов). Требуется определить, можно ли разбить строку `s` на слова (возможно, с повторениями), используя только слова из словаря.

**Пример:**
```
s = "leetcode", dictionary = ["leet", "code"] → true
s = "applepenapple", dictionary = ["apple", "pen"] → true
s = "catsandog", dictionary = ["cats", "dog", "sand", "and", "cat"] → false
```

---

## Идея решения (рекурсия + мемоизация)

Используется рекурсивный поиск с возвратом (backtracking) и кэшированием результатов для каждого начального индекса. Для текущей позиции `i` в строке:

- Перебираем все слова из словаря.
- Если слово помещается (длина ≤ оставшейся части строки) и совпадает с префиксом `s[i:]`, то рекурсивно проверяем, можно ли разбить оставшуюся часть (`i + len(word)`).
- Если хотя бы одно слово привело к успеху, запоминаем `True` для `i` и возвращаем.
- Если ни одно слово не подошло, запоминаем `False`.

Благодаря мемоизации (кэшированию), каждый индекс обрабатывается не более одного раза, что даёт эффективное решение.

### Оптимизация

- Словарь преобразуется в множество для быстрой проверки членства (не требуется в данном подходе, так как мы итеративно сравниваем символы, но для другого подхода пригодилось бы).
- Условие `if len(p) > n - i: continue` отсекает слишком длинные слова.

---

## Алгоритм

1. Инициализировать `n = len(s)`, `memo = {}` (словарь для кэширования результатов по индексу).
2. Рекурсивная функция `backtrack(i)`:
   - Если `i == n`, вернуть `True`.
   - Если `i` есть в `memo`, вернуть `memo[i]`.
   - Для каждого слова `p` из `dictionary`:
     - Если длина `p` больше `n - i`, пропустить.
     - Сравнить символы `s[i : i+len(p)]` с `p`. Если все символы совпадают:
       - Вызвать `backtrack(i + len(p))`.
       - Если результат `True`, сохранить `memo[i] = True` и вернуть `True`.
   - Сохранить `memo[i] = False` и вернуть `False`.
3. Вернуть `backtrack(0)`.

---

## Реализация

```python
class Solution:
    def wordBreak(self, s, dictionary):
        word_set = set(dictionary)     # можно использовать для проверки, но здесь не нужно
        n = len(s)
        memo = {}
        
        def backtrack(i):
            if i == n:
                return True
            if i in memo:
                return memo[i]
            for w in dictionary:
                if len(w) > n - i:
                    continue
                # Проверяем совпадение
                if s[i:i+len(w)] == w:
                    if backtrack(i + len(w)):
                        memo[i] = True
                        return True
            memo[i] = False
            return False
        
        return backtrack()
```

## Сложность

- **Время:** O(n · m · L), где `n` — длина строки, `m` — количество слов в словаре, `L` — средняя длина слова. В худшем случае может быть экспоненциальным без мемоизации, но с мемоизацией каждый индекс обрабатывается один раз, и для каждого индекса проверяются все слова (сравнение строк за O(L)). Итоговая сложность O(n · m · L). На практике часто используют оптимизацию с хешированием префиксов, но для учебных целей и ограничений `n ≤ 300` решение подходит.
- **Память:** O(n) для кэша и стека рекурсии.

---

## Примечания

- В коде использована проверка через цикл по символам (вместо `s[i:i+len(w)] == w`) для возможной микрооптимизации, но для читаемости можно использовать срез.
- Множество `dictionary` не обязательно, так как мы перебираем исходный список (можно преобразовать в список для итерации).
- Задача также решается с помощью динамического программирования (DP снизу вверх), но рекурсия с мемоизацией здесь более наглядна.
