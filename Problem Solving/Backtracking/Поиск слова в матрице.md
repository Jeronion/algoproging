## Определение

Дана матрица `mat` размера `n × m`, содержащая буквы (строчные и прописные), и строка `word`. Требуется определить, можно ли составить `word`, перемещаясь по соседним клеткам (вертикально или горизонтально), без повторного использования одной клетки.

**Пример:**
```
mat = [['T','E','E'],
       ['S','G','K'],
       ['T','E','L']]
word = "GEEK" → true (путь: G→E→E→K)
```

---

## Идея решения (backtracking)

Используется поиск с возвратом (DFS) на матрице. Для каждой клетки, где начинается первая буква `word`, запускается рекурсивный обход.

### Алгоритм

1. Перебрать все клетки `(i, j)` матрицы.
2. Если `mat[i][j]` совпадает с `word[0]`, вызвать `backtrack(0, i, j)`.
3. В `backtrack(index, i, j)`:
   - Если `mat[i][j] != word[index]` → `False`.
   - Если `index == len(word)-1` → все буквы найдены → `True`.
   - Временно пометить текущую клетку как посещённую (например, заменить на `"$"`).
   - Для каждого из 4 соседей (вверх, вниз, влево, вправо):
     - Если сосед в пределах матрицы и `backtrack(index+1, ni, nj)` возвращает `True`, то вернуть `True`.
   - Восстановить исходное значение клетки.
   - Вернуть `False`.
4. Если ни одна стартовая клетка не привела к успеху, вернуть `False`.

## Реализация

```python
class Solution:
    def isWordExist(self, mat, word):
        n, m = len(mat), len(mat[0])
        directions = [(0,1), (1,0), (0,-1), (-1,0)]
        
        def backtrack(idx, i, j):
            if word[idx] != mat[i][j]:
                return False
            if idx == len(word) - 1:
                return True
            
            temp = mat[i][j]
            mat[i][j] = '#'   # маркер посещения
            for di, dj in directions:
                ni, nj = i + di, j + dj
                if 0 <= ni < n and 0 <= nj < m:
                    if backtrack(idx + 1, ni, nj):
                        return True
            mat[i][j] = temp
            return False
        
        for i in range(n):
            for j in range(m):
                if backtrack(0, i, j):
                    return True
        return False
```

## Сложность

- **Время:** O(n·m·4^L), где L = длина слова. В худшем случае (при L до 15 и малых n,m) перебор может быть значительным, но ограничения задачи (n,m ≤ 6) делают решение приемлемым.
- **Память:** O(L) для стека рекурсии.

## Примечания

- Клетка временно меняется на символ, не встречающийся в слове, чтобы избежать повторного посещения.
- Важно проверять соответствие буквы перед углублением рекурсии (раннее отсечение).
- Задача известна как **LeetCode 79. Word Search**.
