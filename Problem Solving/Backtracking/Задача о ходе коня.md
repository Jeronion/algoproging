## Определение

Дана шахматная доска размером `n × n`. Конь начинает в левом верхнем углу `(0, 0)`. Требуется найти **маршрут коня**, при котором он посетит **каждую клетку ровно один раз** (гамильтонов путь). Если решение существует, вернуть матрицу `n × n`, где в каждой клетке указан номер шага (начиная с `0`). Если решения нет, вернуть пустой список.

**Пример для `n = 5`:**
```
[[0, 11, 2, 17, 20],
 [3, 16, 19, 12, 7],
 [10, 1, 6, 21, 18],
 [15, 4, 23, 8, 13],
 [24, 9, 14, 5, 22]]
```

**Примечание:** для `n = 4` решения не существует (вернуть `[]`).

---

## Идея решения (backtracking)

Используется **поиск с возвратом (backtracking)**: последовательно делаем ходы, отмечая пройденные клетки. Если на каком-то шаге некуда идти, возвращаемся и пробуем другой вариант.

### Алгоритм

1. Создать матрицу `grid` размера `n × n`, заполненную `-1` (непосещённые клетки). Установить `grid[0][0] = 0`.
2. Определить 8 возможных ходов коня (массив `directions`).
3. Рекурсивная функция `backtrack(idx, i, j)`:
   - `idx` – номер текущего шага (сколько клеток уже посещено, начиная с 1, т.к. стартовая уже занята).
   - Если `idx == n*n`, то все клетки посещены → возвращаем `True`.
   - Для каждого хода `(di, dj)`:
     - Вычислить `ni, nj`.
     - Если координаты в пределах доски и `grid[ni][nj] == -1`:
       - Отметить `grid[ni][nj] = idx`.
       - Вызвать `backtrack(idx + 1, ni, nj)`. Если успешно – вернуть `True`.
       - Иначе откатить: `grid[ni][nj] = -1`.
   - Возвращаем `False` (ни один ход не привёл к решению).
4. Запустить `backtrack(1, 0, 0)`.
5. Если возвращено `True`, вернуть `grid`, иначе `[]`.

---

## Реализация

```python
class Solution:
    def knightTour(self, n):
        def backtrack(idx, i, j):
            if idx == n * n:
                return True
            for di, dj in directions:
                ni, nj = i + di, j + dj
                if 0 <= ni < n and 0 <= nj < n and grid[ni][nj] == -1:
                    grid[ni][nj] = idx
                    if backtrack(idx + 1, ni, nj):
                        return True
                    grid[ni][nj] = -1
            return False

        grid = [[-1] * n for _ in range(n)]
        grid[0][0] = 0
        directions = [
            [2, 1], [1, 2], [-1, 2], [-2, 1],
            [-2, -1], [-1, -2], [1, -2], [2, -1]
        ]
        if backtrack(1, 0, 0):
            return grid
        return []
```

## Сложность

- **Время:** экспоненциальное, в худшем случае O(8^(n²)). Для `n ≤ 6` перебор допустим (с отсечением).
- **Память:** O(n²) для хранения доски и стека рекурсии.

---

## Примечания

- Для `n = 4` решения нет, алгоритм вернёт пустой список.
- Для `n = 5` и `n = 6` решения существуют.
- Порядок ходов в массиве `directions` влияет на скорость поиска. Оптимально располагать ходы так, чтобы сначала шли ходы в менее загруженные области (эвристика Варнсдорфа), но в данной реализации используется фиксированный порядок, достаточный для `n ≤ 6`.
