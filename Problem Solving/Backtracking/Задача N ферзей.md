## Определение

Необходимо расставить `n` ферзей на шахматной доске `n × n` так, чтобы они не атаковали друг друга (не было двух на одной строке, столбце или диагонали). Требуется найти все возможные решения. Каждое решение представляется как список длины `n`, где `i`-й элемент (1-индексация) указывает номер столбца, в котором стоит ферзь в `i`-й строке.

**Пример:**
```
n = 4 → [[2,4,1,3], [3,1,4,2]]
n = 1 → [[1]]
n = 3 → []
```

---

## Идея решения (bitmask backtracking)

Классический рекурсивный перебор с расстановкой ферзей по строкам. Для ускорения и экономии памяти используются **битовые маски** для отслеживания занятых столбцов и диагоналей:

- `cols` – битовая маска занятых столбцов (бит j установлен, если столбец j уже занят).
- `dm` (diag1) – маска для диагоналей вида `(i + j)` (с наклоном вправо-вниз). Индекс диагонали: `i + j`. Чтобы сместить в диапазон `[0, 2n-1]`, часто используют `i + j`.
- `ds` (diag2) – маска для диагоналей вида `(j - i + n - 1)` или `(n + j - i)`. В коде используется `n + j - i`, что даёт значения от 1 до 2n-1.

На каждом шаге `i` (текущая строка) перебираем столбцы `j` от 0 до n-1. Если биты в `cols`, `dm`, `ds` для этой позиции не установлены, ставим ферзя, устанавливаем биты, рекурсивно переходим к следующей строке, затем откатываем изменения (XOR).

### Почему битовые маски?

- Ускорение: проверка и установка битов выполняются за O(1).
- Экономия памяти: не нужно хранить массивы `bool` размера n.
- Позволяет легко откатывать изменения (XOR вместо присваивания).

### Алгоритм

1. Инициализировать `cols = 0`, `dm = 0`, `ds = 0`, `rows = [0]*n`, `res = []`.
2. Рекурсивная функция `backtrack(i)`:
   - Если `i == n`, добавить копию `rows` в результат.
   - Иначе для `j` от 0 до n-1:
     - Если `(cols >> j) & 1` или `(dm >> (n + j - i)) & 1` или `(ds >> (i + j)) & 1`, пропустить.
     - Иначе: `rows[i] = j+1`, установить биты: `cols |= (1 << j)`, `dm |= (1 << (n + j - i))`, `ds |= (1 << (i + j))`.
     - Вызвать `backtrack(i+1)`.
     - Откатить: `cols ^= (1 << j)`, `dm ^= (1 << (n + j - i))`, `ds ^= (1 << (i + j))`.
3. Вернуть `res`.

## Реализация на Python

```python
class Solution:
    def nQueen(self, n):
        res = []
        rows = [0] * n
        cols = 0
        dm = 0   # диагонали (i + j)
        ds = 0   # диагонали (n + j - i)
        
        def backtrack(i):
            nonlocal cols, dm, ds
            if i == n:
                res.append(rows[:])
                return
            for j in range(n):
                if cols & (1 << j):
                    continue
                if dm & (1 << (i + j)):
                    continue
                if ds & (1 << (n + j - i)):
                    continue
                rows[i] = j + 1
                cols |= (1 << j)
                dm |= (1 << (i + j))
                ds |= (1 << (n + j - i))
                backtrack(i + 1)
                cols ^= (1 << j)
                dm ^= (1 << (i + j))
                ds ^= (1 << (n + j - i))
        
        backtrack(0)
        return res
```

## Сложность

- **Время:** O(n! * n) в худшем случае, но битовые маски ускоряют проверки. Для n ≤ 10 выполняется быстро.
- **Память:** O(n) для стека рекурсии и массивов, плюс O(количество решений * n) для результата.

---

## Примечания

- В стандартном решении N ферзей часто используют массивы `col`, `diag1`, `diag2` (булевые), но битовая версия компактнее и быстрее.
- Сдвиги `(1 << j)` для n=10 не выходят за пределы 32-битных целых.
- Нумерация диагоналей: `i + j` даёт значения от 0 до 2n-2, а `n + j - i` — от 1 до 2n-1 (индекс 0 не используется).
- Возвращаемые решения представлены в виде списков столбцов (1-индексация), что соответствует условию задачи.
