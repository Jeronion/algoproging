## Определение

Дана строка `s`. Требуется найти все возможные способы разбить её на подстроки так, чтобы **каждая подстрока была палиндромом**. Результат возвращается в виде списка списков строк.

**Пример:**
```
s = "geeks" → [["g","e","e","k","s"], ["g","ee","k","s"]]
s = "abcba" → [["a","b","c","b","a"], ["a","bcb","a"], ["abcba"]]
```

---

## Идея решения (DP + backtracking)

1. **Предварительное вычисление палиндромов** – строим матрицу `dp` размера `n×n`, где `dp[i][j]` (или `dp[j][i]` в зависимости от индексации) равно `1`, если подстрока `s[i..j]` (включительно) является палиндромом. При этом:
   - Одиночные символы – всегда палиндромы.
   - Два одинаковых символа подряд – палиндромы.
   - Для длин >2: `s[i] == s[j]` и подстрока между ними тоже палиндром (проверяется через `dp[i-1][j+1]`).

2. **Генерация разбиений с помощью backtracking** – рекурсивно строим разбиение, начиная с индекса `i`. На каждом шаге:
   - Для каждого `k` от `i` до `n-1`:
     - Если подстрока `s[i..k]` – палиндром (проверка по `dp`), то добавляем её в текущий путь и рекурсивно вызываем для `k+1`.
   - Когда дошли до конца (`i == n`), сохраняем текущий путь как одно из разбиений.

В коде используется вспомогательный массив `path` для хранения индексов концов подстрок (в 1-индексации), а затем уже по индексам извлекаются сами подстроки.

---

## Реализация (с комментариями)

```python
class Solution:
    def palinParts(self, s):
        n = len(s)
        # dp[i][j] будет 1, если s[i..j] (i <= j) — палиндром
        dp = [[0] * n for _ in range(n)]
        for i in range(n):
            dp[i][i] = 1                     # одиночный символ
        for i in range(n - 1):
            if s[i] == s[i + 1]:
                dp[i+1][i] = 1               # в коде используется dp[i+1][i] — это нестандартно, обычно dp[i][i+1]
        # Длинные палиндромы
        for i in range(1, n):
            for j in range(i):
                if s[i] == s[j] and dp[i-1][j+1]:
                    dp[i][j] = 1

        res = []
        path = [0]                           # хранит начальные индексы (0, затем концы+1)

        def backtrack(pos):
            if pos == n:
                # сформировать разбиение
                parts = [s[path[i]:path[i+1]] for i in range(len(path)-1)]
                res.append(parts)
                return
            for k in range(pos, n):
                if dp[k][pos] == 0:          # палиндром ли s[pos..k]
                    continue
                path.append(k + 1)
                backtrack(k + 1)
                path.pop()

        backtrack(0)
        return res
```

**Примечание:** в коде использована немного нестандартная индексация `dp[k][pos]`, где `k` — правый индекс, `pos` — левый. Главное, чтобы проверка была корректной.

## Сложность

- **Время:** O(n²) для построения `dp`, O(2ⁿ · n) для генерации всех разбиений в худшем случае (для строки из одинаковых букв, например, "aaaa..."). Верхняя граница: количество всех возможных разбиений строки длины `n` равно 2^(n-1), для каждого разбиения мы строим список подстрок.
- **Память:** O(n²) для `dp`, O(2ⁿ · n) для хранения всех разбиений (столько же, сколько и ответ).

---

## Примечания

- Построение `dp` выполняется за O(n²) и позволяет быстро отвечать на вопрос «является ли подстрока палиндромом?» за O(1).
- Backtracking перебирает все возможные варианты разбиения, отсекая заведомо непалиндромные подстроки.
- Ограничение `n ≤ 20` делает полный перебор допустимым (2^19 ≈ 524288 разбиений).
