## Определение

Дан бинарный массив `arr`, содержащий 0 и 1, и целое число `k`. Разрешено заменить **не более `k` нулей** на единицы. Требуется найти **максимальную длину** непрерывного подмассива, состоящего только из единиц, после выполнения операции (можно заменить нули не более `k` раз).

**Пример:**
```
arr = [1, 0, 1], k = 1 → ответ 3 (заменяем ноль на единицу → [1,1,1])
arr = [1, 0, 0, 1, 0, 1, 0, 1], k = 2 → ответ 5 (например, замена нулей на позициях 4 и 6 даёт подмассив с индексами 3..7: [1,1,1,1,1] — длина 5)
```

---

## Идея решения (скользящее окно)

Используем метод **двух указателей** (`l`, `r`). Окно `[l, r]` всегда должно содержать не более `k` нулей. При движении правого указателя `r` вправо:

- Если встретили ноль (`arr[r]==0`), увеличиваем счётчик нулей `m`.
- Если `m > k`, нужно сдвинуть левый указатель `l` вправо, чтобы уменьшить количество нулей в окне:
  - Пропускаем все единицы, пока не встретим ноль.
  - Сдвигаем `l` за этот ноль и уменьшаем `m` на 1.
- Текущая длина окна `r - l + 1` — потенциальный ответ.
- Обновляем глобальный максимум.

Этот алгоритм за O(n) находит максимальное окно, где число нулей ≤ k.

---

## Алгоритм

1. Инициализировать `l = 0`, `m = 0` (счётчик нулей в окне), `res = 0`.
2. Для `r` от `0` до `n-1`:
   - Если `arr[r] == 0`, то `m += 1`.
   - Если `m > k`:
     - Пока `arr[l] != 0`, увеличивать `l` (пропускать единицы).
     - `l += 1` (пропустить ноль).
     - `m -= 1`.
   - `res = max(res, r - l + 1)`.
3. Вернуть `res`.

---

## Реализация

```python
class Solution:
    def maxOnes(self, arr, k):
        n = len(arr)
        l = 0
        zeros = 0
        max_len = 0
        for r in range(n):
            if arr[r] == 0:
                zeros += 1
            while zeros > k:
                if arr[l] == 0:
                    zeros -= 1
                l += 1
            max_len = max(max_len, r - l + 1)
        return max_len
```

**Примечание:** В приведённом коде используется `while l < n and arr[l]` для пропуска единиц – это то же самое, что и условие выше, но записано иначе. Оба варианта корректны.

---

## Сложность

- **Время:** O(n) — каждый элемент входит в окно и выходит из него не более одного раза.
- **Память:** O(1) — только несколько переменных.

---

## Примечания

- Если `k` больше или равно количеству нулей во всём массиве, то можно получить длину всего массива.
- Алгоритм автоматически обрабатывает случай, когда `k=0` (просто поиск самой длинной непрерывной последовательности единиц).
- Эта задача также известна как "Longest Subarray with at most K zeros".
