## Определение

Дан массив целых чисел `arr` и целое число `k`. Требуется найти **максимальную сумму** любого непрерывного подмассива (окна) длины `k`.

**Пример:**
```
arr = [100, 200, 300, 400], k = 2 → 700 (300+400)
arr = [1, 4, 2, 10, 23, 3, 1, 0, 20], k = 4 → 39 (1+4+2+10 = 17? нет, 4+2+10+23 = 39)
```

---

## Идея решения (скользящее окно)

Наивный подход: перебрать все подмассивы длины `k` и вычислить сумму каждого. Это требует O(n·k) времени. Оптимальное решение использует **скользящее окно**:

1. Вычислить сумму первых `k` элементов: `window_sum = sum(arr[0:k])`.
2. Запомнить её как текущий максимум.
3. Сдвигать окно на один шаг вправо:
   - вычесть элемент, который выходит из окна (`arr[i]`),
   - добавить новый элемент, который входит в окно (`arr[i+k]`).
   - обновить максимум.
4. Повторять для `i` от `0` до `n-k-1`.

Это позволяет получить максимальную сумму за один проход O(n).

---

## Алгоритм

1. Инициализировать `window_sum = sum(arr[:k])`, `max_sum = window_sum`.
2. Для `i` от `0` до `n-k-1`:
   - `window_sum = window_sum - arr[i] + arr[i+k]`
   - `max_sum = max(max_sum, window_sum)`
3. Вернуть `max_sum`.

---

## Реализация на Python

```python
class Solution:
    def maxSubarraySum(self, arr, k):
        n = len(arr)
        window_sum = sum(arr[:k])
        max_sum = window_sum
        for i in range(n - k):
            window_sum = window_sum - arr[i] + arr[i + k]
            max_sum = max(max_sum, window_sum)
        return max_sum
```

---

## Сложность

- **Время:** O(n) — один проход по массиву.
- **Память:** O(1) — только несколько переменных.

---

## Примечания

- Алгоритм работает для любых целых чисел (в том числе отрицательных), но в условии задачи все `arr[i] ≥ 0`.
- Подходит для больших массивов (до 10⁶), так как не требует дополнительной памяти.
- Если `k > n` (по условию `k ≤ n`), задача не имеет смысла.
