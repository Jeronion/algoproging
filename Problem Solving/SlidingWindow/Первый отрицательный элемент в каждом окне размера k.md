## Определение

Дан массив целых чисел `arr` и целое число `k`. Требуется для каждого непрерывного подмассива (окна) длины `k` найти **первый отрицательный элемент** (если он есть). Если в окне нет отрицательных чисел, вернуть `0`.

**Пример:**
```
arr = [-8, 2, 3, -6, 10], k = 2
Окна:
[-8, 2] → -8
[2, 3] → 0
[3, -6] → -6
[-6, 10] → -6
Результат: [-8, 0, -6, -6]
```

---

## Идея решения (скользящее окно с указателем)

Наивный подход — для каждого окна сканировать его элементы, что даёт O(n·k). Оптимальное решение использует **два указателя**:

- Указатель `f` всегда указывает на первый отрицательный элемент в **текущем окне** (или на `n`, если такого нет).
- При сдвиге окна вправо мы:
  1. Увеличиваем `f`, если он указывает на элемент, который вышел из окна (его индекс ≤ i-k) или если элемент стал неотрицательным (не может быть, так как `f` всегда указывает на отрицательный; но в коде проверка на `arr[f] >= 0` нужна, если `f` каким-то образом мог сдвинуться на неотрицательный, например, при инициализации? На самом деле `f` всегда указывает на отрицательный, если такой есть, иначе на n; но условие `arr[f] >= 0` может быть полезно, если `f` по какой-то причине остановился на нуле или положительном, тогда его пропускаем).

В представленном коде используется именно такой подход: для каждого окна, начиная с индекса `i = k-1`, мы сначала корректируем `f` так, чтобы он указывал на первый отрицательный элемент внутри текущего окна. При этом если `f` находится левее начала окна (f ≤ i-k) или элемент по `f` неотрицательный, то увеличиваем `f`.

### Алгоритм

1. Инициализировать `f = 0` (указатель на первый отрицательный в окне).
2. Инициализировать пустой список `res`.
3. Для `i` от `k-1` до `n-1` (т.е. для каждого возможного правого конца окна):
   - Пока `f < i` и (`f <= i - k` или `arr[f] >= 0`), увеличивать `f`.
   - Если `f < n` и `arr[f] < 0`, добавить `arr[f]` в результат, иначе `0`.
4. Вернуть `res`.

## Реализация на Python

```python
class Solution:
    def firstNegInt(self, arr, k):
        f = 0
        res = []
        n = len(arr)
        for i in range(k - 1, n):
            # Перемещаем указатель f, пока он не будет указывать на отрицательный элемент внутри текущего окна
            while f < i and (f <= i - k or arr[f] >= 0):
                f += 1
            # Если нашли отрицательный — добавляем его, иначе 0
            if f < n and arr[f] < 0:
                res.append(arr[f])
            else:
                res.append(0)
        return res
```

**Пояснение:**
- `i` — правый индекс текущего окна.
- Левая граница окна: `i - k + 1`.
- Условие `f <= i - k` проверяет, что указатель `f` находится левее начала окна — тогда его нужно сдвинуть.
- Условие `arr[f] >= 0` проверяет, что элемент по `f` неотрицательный — тогда его тоже нужно пропустить.
- После цикла `f` либо указывает на первый отрицательный в окне (если есть), либо стал равен `i` или `n` (если отрицательных нет).

## Сложность

- **Время:** O(n) — каждый элемент массива обрабатывается не более двух раз (указатель `f` движется только вперёд).
- **Память:** O(1) дополнительно (не считая выходного массива).

---

## Примечания

- В коде используется `while f < i and (f <= i - k or arr[f] >= 0)`. Важно, что `f` никогда не может стать больше `i`, так как условие `f < i` ограничивает цикл.
- Если отрицательных нет во всей оставшейся части, `f` может достигнуть `n`, тогда `arr[f]` не существует, и условие `arr[f] < 0` проверять не нужно — в коде есть защита `f < n`.
- Алгоритм работает для любых целых чисел, не только для отрицательных/нулей/положительных.
