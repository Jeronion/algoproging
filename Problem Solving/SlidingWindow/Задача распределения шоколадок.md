## Определение

Дан массив `arr`, где каждый элемент представляет количество шоколадок в упаковке, и целое число `m` — количество учеников. Нужно раздать каждому ученику **ровно одну упаковку** так, чтобы разница между максимальным и минимальным количеством шоколадок среди выбранных упаковок была **минимальной**. Вернуть эту минимальную разницу.

**Пример:**
```
arr = [3, 4, 1, 9, 56, 7, 9, 12], m = 5
Отсортированный массив: [1, 3, 4, 7, 9, 9, 12, 56]
Выбираем 5 последовательных пакетов:
[1,3,4,7,9] → разница 9-1=8
[3,4,7,9,9] → 9-3=6
[4,7,9,9,12] → 12-4=8
[7,9,9,12,56] → 56-7=49
Минимальная разница = 6
```

---

## Идея решения

Если массив отсортировать, то **любые `m` последовательных элементов** в отсортированном массиве дадут наименьшую возможную разницу для этого окна (так как разница между максимальным и минимальным в любом подмножестве не может быть меньше, чем разница между крайними элементами в отсортированном порядке). Поэтому достаточно перебрать все окна длины `m` в отсортированном массиве и найти минимальную разницу `arr[i+m-1] - arr[i]`.

### Алгоритм

1. Сортировать массив.
2. Инициализировать `min_diff = ∞`.
3. Для `i` от `0` до `n-m`:
   - Вычислить `diff = arr[i+m-1] - arr[i]`.
   - Обновить `min_diff`.
4. Вернуть `min_diff`.

---

## Реализация

```python
class Solution:
    def findMinDiff(self, arr, m):
        n = len(arr)
        arr.sort()
        min_diff = float('inf')
        for i in range(n - m + 1):
            min_diff = min(min_diff, arr[i + m - 1] - arr[i])
        return min_diff
```

---

## Сложность

- **Время:** O(n log n) — из-за сортировки.
- **Память:** O(1) — дополнительная память не используется (сортировка in-place).

---

## Примечания

- Если `m == n`, то разница равна `max(arr) - min(arr)`.
- Задача гарантирует `1 ≤ m ≤ n`, поэтому окно всегда существует.
- Подход работает, так как мы выбираем ровно `m` пакетов и хотим минимизировать разброс. Сортировка позволяет рассматривать только последовательные окна.
