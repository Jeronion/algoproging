## Определение

Даны две строки `X` и `Y` одинаковой длины, состоящие из строчных латинских букв, и целое число `K`. Стоимость замены символа `X[i]` на `Y[i]` равна абсолютной разности их ASCII-кодов: `|ord(X[i]) - ord(Y[i])|`. Требуется найти максимальную длину **непрерывной подстроки**, которую можно изменить в `Y` (заменив соответствующие символы), чтобы суммарная стоимость замен не превышала `K`. Иными словами, найти самый длинный подмассив, для которого сумма `cost[i] ≤ K`, где `cost[i] = |ord(X[i]) - ord(Y[i])|`.

**Пример:**
```
X = "abcd", Y = "bcde", K = 3
cost: |a-b|=1, |b-c|=1, |c-d|=1, |d-e|=1
Можно взять первые 3 символа: сумма = 1+1+1 = 3 ≤ K → ответ 3.
```

---

## Постановка задачи

**Вход:** две строки `X` и `Y` длины `n`, целое `K`.
**Выход:** максимальная длина подстроки (непрерывной), для которой сумма стоимостей замен ≤ K.

---

## Идея решения (скользящее окно)

Используем метод **двух указателей** (скользящее окно). Поддерживаем окно `[left, right]` (включительно) и текущую сумму стоимостей в нём. Расширяем окно вправо, добавляя `cost[right]`. Если сумма превышает `K`, сдвигаем левую границу вправо, уменьшая сумму на `cost[left]`. На каждом шаге обновляем максимальную длину окна.

Этот подход работает за O(n) времени и O(1) дополнительной памяти.

---

## Алгоритм

1. Инициализировать `left = 0`, `current_cost = 0`, `max_len = 0`.
2. Для `right` от `0` до `n-1`:
   - `current_cost += abs(ord(X[right]) - ord(Y[right]))`.
   - Пока `current_cost > K`:
     - `current_cost -= abs(ord(X[left]) - ord(Y[left]))`.
     - `left += 1`.
   - `max_len = max(max_len, right - left + 1)`.
3. Вернуть `max_len`.

---

## Реализация на Python

```python
def max_equal_substring_length(X: str, Y: str, K: int) -> int:
    n = len(X)
    left = 0
    current_cost = 0
    max_len = 0
    for right in range(n):
        current_cost += abs(ord(X[right]) - ord(Y[right]))
        while current_cost > K:
            current_cost -= abs(ord(X[left]) - ord(Y[left]))
            left += 1
        max_len = max(max_len, right - left + 1)
    return max_len

# Пример
X = "abcd"
Y = "bcde"
K = 3
print(max_equal_substring_length(X, Y, K))  # 3
```

---

## Сложность

- **Время:** O(n) — каждый символ обрабатывается не более двух раз.
- **Память:** O(1) — только переменные.

---

## Примечания

- Задача эквивалентна поиску самого длинного подмассива в массиве `cost`, сумма элементов которого ≤ K (типичная задача скользящего окна).
- Альтернативное решение: построить массив префиксных сумм `prefix`, затем для каждого правого индекса бинарным поиском найти самый левый индекс, чтобы `prefix[r] - prefix[l-1] ≤ K`. Это требует O(n log n) времени и O(n) памяти.
