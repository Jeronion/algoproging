## Определение

Дан массив целых чисел `arr` и целое число `k`. Требуется подсчитать количество **непрерывных подмассивов**, которые содержат **ровно k различных элементов**.

**Пример:**
```
arr = [1, 2, 2, 3], k = 2 → подмассивы: [1,2], [1,2,2], [2,3] → итого 3? В примере вывода 4. Возможно, ответ 4 включает также [2,2]? Нет, [2,2] содержит только один различный элемент (2). Пересчитаем: [1,2] (2 разных), [1,2,2] (2), [2,3] (2), [2,2,3] (2 разных: 2 и 3) — это четвёртый. Да, [2,2,3] — подмассив от индекса 2 до 3? Индексы: 0:1,1:2,2:2,3:3. [2,2,3] – индексы 1..3 -> 3 элемента, различных 2 и 3 -> 2. Так что ответ 4.
arr = [3,1,2,2,3], k=3 → ответ 4.
```

---

## Идея решения (скользящее окно + разность)

Прямой перебор всех подмассивов требует O(n²) и не подходит для больших массивов. Используем формулу:

```
count_exactly_k = count_atmost_k - count_atmost_{k-1}
```

где `count_atmost_k` — количество подмассивов с *не более чем* k различными элементами. Эта функция вычисляется за O(n) с помощью скользящего окна.

### Подсчёт подмассивов с не более чем k различными элементами (atMostK)

Поддерживаем окно `[left, right]` и словарь частот. Расширяем `right`, добавляя элемент. Если количество различных элементов в окне становится > k, сдвигаем `left` вправо, удаляя элементы, пока не восстановится условие `distinct <= k`. На каждом шаге (для каждого `right`) все подмассивы, заканчивающиеся на `right` и начинающиеся в интервале `[left, right]`, имеют не более k различных элементов. Их количество = `right - left + 1`. Суммируем эти количества.

### Итоговый подсчёт

Вычисляем `atMostK(k)` и `atMostK(k-1)`. Их разность даёт количество подмассивов ровно с k различными элементами.

---

## Алгоритм (atMostK)

1. Инициализировать `left = 0`, `freq = defaultdict(int)`, `distinct = 0`, `result = 0`.
2. Для `right` от 0 до `n-1`:
   - Увеличить `freq[arr[right]]`.
   - Если `freq[arr[right]] == 1`, увеличить `distinct`.
   - Пока `distinct > k`:
     - Уменьшить `freq[arr[left]]`.
     - Если `freq[arr[left]] == 0`, уменьшить `distinct`.
     - `left += 1`.
   - `result += right - left + 1`.
3. Вернуть `result`.

---

## Реализация

```python
from collections import defaultdict

def atMostK(arr, k):
    if k < 0:
        return 0
    left = 0
    freq = defaultdict(int)
    distinct = 0
    count = 0
    for right in range(len(arr)):
        freq[arr[right]] += 1
        if freq[arr[right]] == 1:
            distinct += 1
        while distinct > k:
            freq[arr[left]] -= 1
            if freq[arr[left]] == 0:
                distinct -= 1
            left += 1
        count += right - left + 1
    return count

def exactlyK(arr, k):
    return atMostK(arr, k) - atMostK(arr, k - 1)

# Пример
arr = [1, 2, 2, 3]
k = 2
print(exactlyK(arr, k))  # 4
```

## Сложность

- **Время:** O(n) — каждый элемент добавляется и удаляется не более одного раза.
- **Память:** O(n) — для словаря частот (в худшем случае, если все элементы различны).

---

## Примечания

- Алгоритм работает для любых целых чисел (не только неотрицательных).
- Важно обработать случай `k = 0`: `atMostK(arr, -1)` возвращает 0, и формула даст корректный результат (0).
- Техника `atMostK(k) - atMostK(k-1)` является стандартной для задач подсчёта подмассивов с точным количеством уникальных элементов.
