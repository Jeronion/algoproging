## Определение

Дан бинарный массив `arr`, содержащий только 0 и 1. Требуется найти **минимальное количество обменов** (любых двух элементов), необходимое для того, чтобы все единицы оказались в одном непрерывном подмассиве. Если в массиве нет единиц, вернуть `-1`.

**Пример:**
```
arr = [1, 0, 1, 0, 1] → 1 (меняем 0 на позиции 1 с 1 на позиции 4 → [1,1,1,0,0])
arr = [1, 0, 1, 0, 1, 1] → 1 (можно обменять 0 на позиции 0 с 1 на позиции 3)
arr = [0,0,0] → -1
```

---

## Идея решения (скользящее окно)

Пусть общее количество единиц в массиве равно `k`. Если `k == 0`, ответ `-1`. Иначе, чтобы сгруппировать все единицы, нужно найти непрерывный подмассив длины `k`, в котором уже находится **максимальное количество единиц** (так как в этом случае потребуется меньше всего замен нулей на единицы извне). Тогда минимальное количество обменов = `k - max_ones_in_window` (т.е. количество нулей в этом окне, которые нужно заменить на единицы).

Таким образом, задача сводится к поиску максимального количества единиц в любом подмассиве длины `k` с помощью **скользящего окна**.

### Алгоритм

1. Вычислить `k = sum(arr)` (общее количество единиц).
2. Если `k == 0`, вернуть `-1`.
3. Вычислить количество единиц в первом окне длины `k`: `ones = sum(arr[:k])`.
4. Установить `max_ones = ones`.
5. Сдвигать окно:
   - При сдвиге вправо вычесть выходящий элемент, добавить входящий.
   - Обновить текущее количество единиц и `max_ones`.
6. Ответ = `k - max_ones`.

---

## Реализация

```python
class Solution:
    def minSwaps(self, arr):
        n = len(arr)
        k = sum(arr)                 # общее количество единиц
        if k == 0:
            return -1
        
        ones = sum(arr[:k])          # единицы в первом окне
        max_ones = ones
        for i in range(n - k):
            ones -= arr[i]           # убираем левый элемент
            ones += arr[i + k]       # добавляем правый элемент
            max_ones = max(max_ones, ones)
        return k - max_ones
```

---

## Сложность

- **Время:** O(n) — один проход по массиву.
- **Память:** O(1) — только константные переменные.

---

## Примечания

- Метод работает для любых бинарных массивов, в том числе больших (до 10⁶).
- Если `k` велико, окно может покрывать почти весь массив, но алгоритм всё равно линеен.
- Альтернативно можно было бы искать минимальное количество нулей в окне длины `k`, что эквивалентно.
