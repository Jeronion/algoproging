## Определение

Дан массив **неотрицательных** целых чисел `arr` и число `k` (тоже неотрицательное). Требуется подсчитать количество **непрерывных подмассивов**, сумма элементов которых **строго меньше** `k`.

**Пример:**
```
arr = [1, 11, 2, 3, 15], k = 10
Подмассивы с суммой < 10: [1], [2], [3], [2,3] → ответ 4.
```

---

## Идея решения (скользящее окно)

Используем два указателя `start` и `end` для поддержания окна, в котором сумма элементов `< k`. При расширении `end` вправо, мы добавляем `arr[end]` к текущей сумме `sum`. Если сумма становится `>= k`, то сдвигаем `start` вправо, вычитая `arr[start]` из суммы, пока сумма снова не станет `< k`. На каждом шаге все подмассивы, заканчивающиеся на `end` и начинающиеся в интервале `[start, end]`, имеют сумму `< k`. Их количество = `end - start + 1`. Суммируем эти количества.

Этот метод работает за O(n), так как каждый элемент добавляется и удаляется из окна не более одного раза.

---

## Алгоритм

1. Инициализировать `start = 0`, `sum = 0`, `count = 0`.
2. Для `end` от `0` до `n-1`:
   - `sum += arr[end]`.
   - Пока `sum >= k`:
     - `sum -= arr[start]`.
     - `start += 1`.
   - `count += end - start + 1`.
3. Вернуть `count`.

---

## Реализация на Python

```python
def count_subarrays_sum_less_than_k(arr, k):
    n = len(arr)
    start = 0
    current_sum = 0
    count = 0
    for end in range(n):
        current_sum += arr[end]
        while current_sum >= k:
            current_sum -= arr[start]
            start += 1
        count += end - start + 1
    return count

# Пример
arr = [1, 11, 2, 3, 15]
k = 10
print(count_subarrays_sum_less_than_k(arr, k))  # 4
```

---

## Сложность

- **Время:** O(n) — каждый элемент обрабатывается не более двух раз.
- **Память:** O(1) — несколько переменных.

---

## Примечания

- Алгоритм работает **только для неотрицательных чисел**. Если бы были отрицательные, сумма не была бы монотонной, и метод двух указателей не работал бы.
- Важно, что и `k`, и элементы массива неотрицательны (по условию).
- Подсчитываются все подмассивы (включая одиночные), сумма которых `< k`.
- Вместо проверки `>= k` используется условие строгого «меньше» (как в условии). Если бы требовалось `≤ k`, изменили бы условие на `> k` в цикле `while`.
