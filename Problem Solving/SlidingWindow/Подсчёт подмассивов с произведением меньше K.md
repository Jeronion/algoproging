## Определение

Дан массив **положительных** целых чисел и число `k`. Требуется подсчитать количество **непрерывных подмассивов**, произведение всех элементов которых **строго меньше** `k`.

**Пример:**
```
n = 4, k = 10, arr = [1, 2, 3, 4]
Все подмассивы: 
[1], [2], [3], [4], [1,2], [1,2,3], [2,3] → произведение < 10 → 7 подмассивов.
[1,2,3,4] (произведение 24), [2,3,4] (24), [3,4] (12) — не подходят.
```

---

## Идея решения (скользящее окно)

Используем два указателя `l` (левый) и `r` (правый), чтобы поддерживать окно, в котором произведение всех элементов `< k`. Переменная `p` хранит текущее произведение в окне.

1. Двигаем `r` от `0` до `n-1`, умножая текущий элемент на `p`.
2. Если `p >= k`, то сдвигаем левый указатель `l` вправо, деля произведение на `arr[l]`, пока `p` не станет `< k`. При этом, если `l == r` и `arr[l] >= k`, то окно становится пустым (`p = 1?` но по условию элементы положительные, и если один элемент уже >= k, то он не даст p < k, и мы просто пропустим этот индекс, сдвинув l и r? Обратите внимание: в коде используется условие `while l < r and p >= k`. Если `l == r` и `p >= k`, то цикл не выполнится, и мы не добавим `c` (так как `p >= k`). После этого на следующей итерации `r` увеличится, и `l` может остаться на том же месте, но произведение уже новое.
3. Если `p < k`, то все подмассивы, заканчивающиеся на `r` и начинающиеся в интервале `[l, r]`, имеют произведение `< k`. Их количество = `r - l + 1`. Добавляем к ответу.

Этот метод работает за O(n), так как каждый элемент добавляется и удаляется из окна не более одного раза.

---

## Алгоритм

1. Инициализировать `l = 0`, `count = 0`, `product = 1`.
2. Для `r` от `0` до `n-1`:
   - `product *= arr[r]`.
   - Пока `l < r` и `product >= k`:
     - `product //= arr[l]`
     - `l += 1`
   - Если `product < k`:
     - `count += r - l + 1`
3. Вернуть `count`.

---

## Реализация

```python
class Solution:
    def countSubArrayProductLessThanK(self, arr, n, k):
        l = 0
        count = 0
        product = 1
        for r in range(n):
            product *= arr[r]
            while l < r and product >= k:
                product //= arr[l]
                l += 1
            if product < k:
                count += r - l + 1
        return count
```

## Сложность

- **Время:** O(n) — каждый элемент обрабатывается не более двух раз.
- **Память:** O(1) — несколько переменных.

---

## Примечания

- Массив содержит только **положительные** числа (условие задачи). Если бы были нули или отрицательные, алгоритм не работал бы.
- При `product >= k` мы сдвигаем `l` только если `l < r`. Если `l == r` и `arr[l] >= k`, то условие `p < k` не выполнится, и текущий элемент не добавится (что правильно, так как подмассив из одного элемента уже не подходит).
- В конце цикла `count` содержит количество всех подмассивов с произведением `< k`.
