## Определение

Дан массив `A`, где `A[i]` — тип фрукта на i-м дереве. У вас есть **две корзины**, каждая может содержать фрукты **только одного типа** (но в любом количестве). Вы начинаете сбор с любого дерева и двигаетесь только **вправо**, последовательно добавляя фрукты в корзины. Если встречается третий тип фрукта, вы останавливаетесь. Требуется найти **максимальное количество** фруктов, которое можно собрать.

**Пример:**
```
A = [1, 2, 3, 2, 2] → ответ 4 (можно собрать [2,3,2,2], начиная с индекса 1)
A = [1, 2, 1] → ответ 3 (все дерево)
```

---

## Идея решения (скользящее окно)

Задача эквивалентна поиску **самого длинного подмассива, содержащего не более двух различных элементов**. Используем **два указателя** `start` и `end` для поддержания окна, в котором не более двух типов фруктов. Сдвигаем `end` вправо, добавляя фрукт. Если типов стало больше двух, сдвигаем `start` вправо, удаляя фрукты, пока типов не останется ≤ 2. На каждом шаге обновляем максимальную длину окна.

### Алгоритм

1. Инициализировать `basket = {}` (словарь: фрукт → количество в окне), `start = 0`, `max_len = 0`.
2. Для `end` от `0` до `n-1`:
   - Увеличить `basket[A[end]]` на 1.
   - Пока `len(basket) > 2`:
     - Уменьшить `basket[A[start]]` на 1.
     - Если `basket[A[start]] == 0`, удалить ключ.
     - `start += 1`.
   - Обновить `max_len = max(max_len, end - start + 1)`.
3. Вернуть `max_len`.

---

## Реализация

```python
def maxFruits(A):
    basket = {}          # fruit_type -> count
    start = 0
    max_len = 0
    for end in range(len(A)):
        basket[A[end]] = basket.get(A[end], 0) + 1
        while len(basket) > 2:
            basket[A[start]] -= 1
            if basket[A[start]] == 0:
                del basket[A[start]]
            start += 1
        max_len = max(max_len, end - start + 1)
    return max_len

# Пример
print(maxFruits([1, 2, 3, 2, 2]))  # 4
```

---

## Сложность

- **Время:** O(n) — каждый элемент добавляется и удаляется не более одного раза.
- **Память:** O(1) — словарь содержит не более 3 элементов.

---

## Примечания

- Задача также известна как "Longest Substring with At Most Two Distinct Characters".
- Если бы было разрешено более двух типов, обобщается заменой `2` на `k`.
- Наивный подход O(n²) перебирает все начальные позиции, что медленно для больших массивов.
