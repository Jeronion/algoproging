## Определение

Дан массив `arr` из **различных** целых чисел и число `K`. Для каждого непрерывного подмассива длины `K` вычислите **MEX** — наименьшее положительное целое число, отсутствующее в подмассиве. Найдите **максимальный** MEX среди всех таких подмассивов.

**Пример:**
```
arr = [3, 2, 1, 4], K = 2
Подмассивы длины 2:
[3,2] → MEX = 1
[2,1] → MEX = 3
[1,4] → MEX = 2
Максимальный MEX = 3
```

---

## Идея решения (скользящее окно + set)

Наивный подход O(K·N²) слишком медленный. Используем **set** для хранения чисел, которые **отсутствуют** в текущем окне. Поскольку числа в массиве различны, мы можем поддерживать set всех чисел от 1 до N+1 (это максимально возможный MEX для подмассива длины K). При сдвиге окна:

- Удаляем уходящий элемент (`arr[i-K]`) — добавляем его обратно в set (теперь он отсутствует в окне).
- Добавляем новый элемент (`arr[i]`) — удаляем его из set (теперь он присутствует).
- MEX текущего окна — это **наименьшее** число в set (так как set содержит отсутствующие числа, и они упорядочены).

Поскольку set в Python хранит элементы в отсортированном порядке, первый элемент set и есть MEX.

---

## Алгоритм

1. Инициализировать `S` — множество чисел от 1 до N+1 (N — длина массива). Все они изначально отсутствуют в пустом окне.
2. Для первого окна (индексы 0..K-1):
   - Удалить из `S` каждый элемент `arr[i]` (теперь эти числа присутствуют).
   - MEX первого окна = минимальный элемент в `S`.
3. Запомнить этот MEX как текущий максимум.
4. Для `i` от K до N-1:
   - Удалить из `S` `arr[i]` (новый элемент входит в окно).
   - Добавить в `S` `arr[i-K]` (старый элемент выходит из окна).
   - Текущий MEX = минимальный элемент `S`.
   - Обновить глобальный максимум.
5. Вывести максимальный MEX.

---

## Реализация на Python

```python
def maxMEX(arr, N, K):
    s = set(range(1, N + 2))          # 1..N+1 (максимальный возможный MEX)
    for i in range(K):
        s.remove(arr[i])              # элементы первого окна присутствуют
    mex = min(s)                      # первый элемент set
    for i in range(K, N):
        s.remove(arr[i])              # новый элемент в окне
        s.add(arr[i - K])             # старый элемент покинул окно
        mex = max(mex, min(s))
    return mex
```

---

## Сложность

- **Время:** O(N log N) — каждая операция добавления/удаления в set стоит O(log N), всего O(N) таких операций.
- **Память:** O(N) — для хранения множества.
