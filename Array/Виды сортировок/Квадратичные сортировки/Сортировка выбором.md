
Сортировка выбором — это алгоритм сортировки на основе сравнения. Он сортирует массив, последовательно выбирая наименьший (или наибольший) элемент из неотсортированной части и меняя его местами с первым элементом этой части.

**Принцип работы:**

1. Найти наименьший элемент и поменять его местами с первым элементом массива. Таким образом, наименьший элемент оказывается на своей правильной позиции.
    
2. Найти наименьший среди оставшихся элементов (или второй наименьший) и поменять его с вторым элементом.
    
3. Продолжать процесс до тех пор, пока все элементы не окажутся на своих местах.
    

---

## Python-программа для реализации сортировки выбором

```python
def selection_sort(arr):
    n = len(arr)
    for i in range(n - 1):
        # Предположим, что текущая позиция содержит минимальный элемент
        min_idx = i
        
        # находим минимум в неотсортированной части массива
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        
        # Перемещаем минимальный элемент на его правильную позицию
        arr[i], arr[min_idx] = arr[min_idx], arr[i]

def print_array(arr):
    for val in arr:
        print(val, end=" ")
    print()
```

## Анализ сложности сортировки выбором

- **Временная сложность:** `O(n²)`  
    Причина: два вложенных цикла:
    
    1. Один цикл выбирает элемент массива по одному → `O(n)`
        
    2. Второй цикл сравнивает этот элемент со всеми остальными → `O(n)`  
        Итоговая сложность: `O(n) * O(n) = O(n²)`
        
- **Дополнительная память:** `O(1)`  
    Используется только для временных переменных.
    

---

## Преимущества сортировки выбором

- Легко понять и реализовать — идеально для обучения базовым концепциям сортировки.
    
- Требует постоянной дополнительной памяти `O(1)`.
    
- Требует меньшее количество обменов по сравнению с другими стандартными алгоритмами сортировки. (Только циклическая сортировка (`Cycle Sort`) лучше по количеству записей в память.)
    

---

## Недостатки сортировки выбором

- Временная сложность `O(n²)` делает его медленнее, чем такие алгоритмы, как Quick Sort или Merge Sort.
    
- Не сохраняет относительный порядок равных элементов (не является стабильным алгоритмом).
    

---

## Применение сортировки выбором

- Отлично подходит для обучения фундаментальным механизмам сортировки и проектированию алгоритмов.
    
- Подходит для небольших списков, когда сложность более продвинутых алгоритмов не оправдана, а количество записей в память критично.
    
- Алгоритм **Heap Sort** основан на сортировке выбором.
