
Сортировка слиянием — это популярный алгоритм сортировки, известный своей эффективностью и стабильностью. Он следует подходу «Разделяй и властвуй». Алгоритм работает, рекурсивно разделяя входной массив на две половины, рекурсивно сортируя каждую из них и затем сливая отсортированные половины обратно, чтобы получить итоговый отсортированный массив.

Ниже приведено пошаговое объяснение работы сортировки слиянием:

1. Разделение (Divide):
Массив или список рекурсивно делится на две половины до тех пор, пока дальнейшее деление станет невозможным.

2. Завоевание / Сортировка (Conquer):
Каждый подмассив сортируется отдельно с использованием алгоритма сортировки слиянием.

3. Слияние (Merge):
Отсортированные подмассивы сливаются обратно в отсортированном порядке. Процесс продолжается до тех пор, пока все элементы из обоих подмассивов не будут объединены.

---

## Пример: сортировка массива `[38, 27, 43, 10]` с помощью Merge Sort

**Разделение:**

* `[38, 27, 43, 10]` делится на `[38, 27]` и `[43, 10]`
* `[38, 27]` делится на `[38]` и `[27]`
* `[43, 10]` делится на `[43]` и `[10]`

**Сортировка:**

* `[38]` — уже отсортирован
* `[27]` — уже отсортирован
* `[43]` — уже отсортирован
* `[10]` — уже отсортирован

**Слияние:**

* Слияние `[38]` и `[27]` даёт `[27, 38]`
* Слияние `[43]` и `[10]` даёт `[10, 43]`
* Слияние `[27, 38]` и `[10, 43]` даёт итоговый отсортированный массив `[10, 27, 38, 43]`

**Итог:**
Отсортированный список: `[10, 27, 38, 43]`

### Реализация на Python

```python
def merge(arr, left, mid, right):
	# Длины временных массивов
    n1 = mid - left + 1
    n2 = right - mid

    # Создание временных массивов
    L = [0] * n1
    R = [0] * n2

    # Копирование данных во временные массивы L[] и R[]
    for i in range(n1):
        L[i] = arr[left + i]
    for j in range(n2):
        R[j] = arr[mid + 1 + j]

    i = 0
    j = 0
    k = left

    # Слияние временных массивов обратно в arr[left..right]
    while i < n1 and j < n2:
        if L[i] <= R[j]:
            arr[k] = L[i]
            i += 1
        else:
            arr[k] = R[j]
            j += 1
        k += 1

    # Копирование оставшихся элементов L[], если они есть
    while i < n1:
        arr[k] = L[i]
        i += 1
        k += 1

    # Копирование оставшихся элементов R[], если они есть
    while j < n2:
        arr[k] = R[j]
        j += 1
        k += 1


def mergeSort(arr, left, right):
    if left < right:
        mid = (left + right) // 2
        mergeSort(arr, left, mid)
        mergeSort(arr, mid + 1, right)
        merge(arr, left, mid, right)
```

### Рекуррентное соотношение сортировки слиянием

Рекуррентное соотношение для Merge Sort:

* $T(n) = Θ(1)$, если $n = 1$
* $T(n) = 2T(n/2) + Θ(n)$, если $n > 1$

Где:

* $T(n)$ — общее время, затраченное алгоритмом на сортировку массива из n элементов
* $2T(n/2)$ — время на рекурсивную сортировку двух половин массива
* $Θ(n)$ — время, необходимое для слияния двух отсортированных половин

---

### Анализ сложности Merge Sort

**Временная сложность:**

* Лучший случай: $O(n \log n)$ — массив уже отсортирован или почти отсортирован
* Средний случай: $O(n \log n)$ — элементы расположены в случайном порядке
* Худший случай: $O(n \log n)$ — массив отсортирован в обратном порядке

**Дополнительная память:**

* $O(n)$ — требуется дополнительная память для временных массивов при слиянии

## Заключение

### Применения сортировки слиянием

* Сортировка больших наборов данных
* Внешняя сортировка (когда данные не помещаются в оперативной памяти)
* Решение задач: подсчёт инверсий, количество меньших элементов справа, Surpasser Count
* Используется в стандартных библиотечных методах языков программирования. Вариация Merge Sort — **TimSort** — применяется в Python, Java (Android) и Swift. Основная причина — стабильность, которой нет у QuickSort. В Java метод Arrays.sort использует QuickSort, а Collections.sort — Merge Sort
* Предпочтителен для сортировки связанных списков (Linked List)
* Легко распараллеливается, так как подмассивы можно сортировать независимо
* Функция слияния используется для эффективного решения задач объединения и пересечения двух отсортированных массивов

### Преимущества и недостатки Merge Sort

**Преимущества:**

* **Стабильность:** сохраняет относительный порядок равных элементов
* **Гарантированная производительность:** худшая временная сложность — $O(n \log n)$
* **Простота реализации:** понятный подход «разделяй и властвуй»
* **Естественная параллельность:** независимая обработка подмассивов

**Недостатки:**

* **Потребление памяти:** требует дополнительную память для временных массивов
* **Не является in-place алгоритмом:** сортировка происходит не на месте
* **Обычно медленнее QuickSort:** QuickSort более кэш-дружелюбен, так как работает in-place
