

**Radix Sort (поразрядная сортировка)** — это линейный алгоритм сортировки (при фиксированном количестве разрядов), который сортирует элементы, обрабатывая их **разряд за разрядом**. Он особенно эффективен для сортировки **целых чисел** или **строк с ключами фиксированной длины**.

Основные идеи алгоритма:

- Элементы многократно распределяются по «корзинам» (bucket) в зависимости от значения текущего разряда.
- В отличие от таких алгоритмов, как **Merge Sort** или **Quick Sort**, здесь элементы **не сравниваются напрямую**.
- Сортировка выполняется по значащим разрядам, начиная с **младшего** и заканчивая **старшим**, что в итоге даёт отсортированный массив.
- Для сортировки по отдельным разрядам используется **стабильный алгоритм**, например **Counting Sort**, чтобы сохранить корректность общего порядка.

## Реализация алгоритма (Python)

```python
def countingSort(arr, exp1):

    n = len(arr)

    # Выходной массив
    output = [0] * n

    # Массив для подсчёта цифр (0–9)
    count = [0] * 10

    # Подсчёт количества вхождений цифр
    for i in range(n):
        index = arr[i] // exp1
        count[index % 10] += 1

    # Изменяем count[i], чтобы он содержал
    # фактическую позицию цифры в output
    for i in range(1, 10):
        count[i] += count[i - 1]

    # Формируем выходной массив (справа налево — для стабильности)
    i = n - 1
    while i >= 0:
        index = arr[i] // exp1
        output[count[index % 10] - 1] = arr[i]
        count[index % 10] -= 1
        i -= 1

    # Копируем output обратно в arr
    for i in range(n):
        arr[i] = output[i]


# Основная функция Radix Sort
def radixSort(arr):

    # Находим максимальный элемент
    max1 = max(arr)

    # Сортируем по каждому разряду
    exp = 1
    while max1 // exp >= 1:
        countingSort(arr, exp)
        exp *= 10
```

## Анализ сложности Radix Sort

#### Временная сложность 
$$
O(d · (n + b))
$$
где:

- $d$ — количество разрядов,
- $n$ — количество элементов,
- $b$ — основание системы счисления (обычно 10).

#### Дополнительная память
$$
O(n + b)
$$

Она обусловлена:

- использованием дополнительных массивов (корзин) для каждого разряда;
- копированием элементов обратно в исходный массив после каждого прохода

## Заключение

Radix Sort — это эффективный алгоритм сортировки, который работает поразрядно и не использует прямые сравнения элементов. Благодаря этому при фиксированном количестве разрядов его временная сложность близка к линейной — $O(d \cdot (n + b))$, что делает его особенно выгодным для обработки больших массивов целых чисел или строк фиксированной длины.

Ключевым условием корректной работы алгоритма является использование стабильной сортировки на каждом разряде (например, Counting Sort), что обеспечивает сохранение относительного порядка элементов и формирование правильного итогового результата.

Однако Radix Sort требует дополнительной памяти и наиболее эффективен в случаях, когда количество разрядов ограничено и известно заранее. В таких условиях он способен существенно превосходить сравнительные алгоритмы сортировки, такие как Quick Sort или Merge Sort.

Таким образом, Radix Sort занимает важное место среди несравнительных алгоритмов сортировки и является мощным инструментом при работе с числовыми данными и ключами фиксированной длины.
