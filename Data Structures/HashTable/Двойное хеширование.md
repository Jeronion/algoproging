
## Определение

**Двойное хеширование** — метод открытой адресации для разрешения коллизий, использующий две хеш-функции. Первая определяет начальную позицию, вторая — шаг для последовательного просмотра. Это гарантирует равномерное распределение и отсутствие кластеризации.

Формула для вычисления индекса на шаге `i`:

```
index = (hash1(key) + i * hash2(key)) % size
```

где `i = 0, 1, 2, ...`, `hash2(key)` никогда не даёт 0 и взаимно просто с `size`.

## Реализация класса DoubleHash

```python
class DoubleHash:
    def __init__(self, size: int):
        """
        size: размер хеш-таблицы
        """
        self.size = size
        self.prime = self._largest_prime(size - 1)   # простое число для hash2
        self.table = [-1] * size                     # -1 = свободно
        self.count = 0                               # количество элементов

    def _largest_prime(self, n: int) -> int:
        """Возвращает наибольшее простое число <= n (для n >= 2)"""
        if n < 2:
            return 2
        is_prime = [True] * (n + 1)
        for i in range(2, int(n ** 0.5) + 1):
            if is_prime[i]:
                for j in range(i * i, n + 1, i):
                    is_prime[j] = False
        for i in range(n, 1, -1):
            if is_prime[i]:
                return i
        return 2

    def _hash1(self, key: int) -> int:
        return key % self.size

    def _hash2(self, key: int) -> int:
        return self.prime - (key % self.prime)

    def is_full(self) -> bool:
        return self.count == self.size

    def insert(self, key: int) -> None:
        """Вставляет ключ в таблицу."""
        if key in (-1, -2):
            raise ValueError("Key cannot be -1 or -2")
        if self.is_full():
            raise OverflowError("Hash table is full")

        h1 = self._hash1(key)
        h2 = self._hash2(key)
        i = 0
        while True:
            idx = (h1 + i * h2) % self.size
            if self.table[idx] in (-1, -2):   # свободно или удалено
                self.table[idx] = key
                self.count += 1
                return
            i += 1

    def search(self, key: int) -> bool:
        """Проверяет наличие ключа."""
        h1 = self._hash1(key)
        h2 = self._hash2(key)
        i = 0
        while True:
            idx = (h1 + i * h2) % self.size
            if self.table[idx] == -1:          # достигли пустой ячейки
                return False
            if self.table[idx] == key:
                return True
            i += 1
            if i > self.size:                 # защита от зацикливания
                return False

    def delete(self, key: int) -> bool:
        """Удаляет ключ, помечая ячейку как -2 (удалена)."""
        if not self.search(key):
            return False
        h1 = self._hash1(key)
        h2 = self._hash2(key)
        i = 0
        while True:
            idx = (h1 + i * h2) % self.size
            if self.table[idx] == key:
                self.table[idx] = -2           # маркер удаления
                self.count -= 1
                return True
            i += 1

    def display(self) -> None:
        """Выводит содержимое таблицы."""
        print(self.table)
```

## Пример использования

```python
if __name__ == "__main__":
    dh = DoubleHash(7)               # размер 7
    for val in [50, 700, 76, 85, 92, 73, 101]:
        dh.insert(val)
    dh.display()
    # Вывод: [76, 50, 73, -1, 92, 85, 700]

    print(dh.search(76))             # True
    dh.delete(76)
    dh.display()
    # Вывод: [-2, 50, 73, -1, 92, 85, 700]
```

## Сложность

- **Вставка, поиск, удаление в среднем:** O(1)
- **Худший случай (при сильных коллизиях):** O(n)
- **Память:** O(size)

---

## Примечания

- Значения `-1` и `-2` используются как маркеры свободных и удалённых ячеек. Обычные ключи не могут быть равны `-1` или `-2` (если допускаются, маски нужно изменить).
- Вторая хеш-функция `hash2(key) = prime - (key % prime)` гарантирует, что шаг всегда положителен и взаимно прост с размером (при условии, что `size` простое).
- При удалении ячейка помечается `-2`, чтобы не обрывать цепочку поиска.
