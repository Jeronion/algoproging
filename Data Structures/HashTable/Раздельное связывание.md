
## Определение

**Раздельное связывание (Separate Chaining)** — метод разрешения коллизий в хеш-таблицах, при котором каждая ячейка (бакет) содержит указатель на **связный список** (или другую структуру) элементов, имеющих одинаковое хеш-значение. При вставке новый элемент добавляется в соответствующий список. Этот подход прост, не ограничивает количество элементов и не требует маркеров удаления.

---

## Постановка задачи

Реализовать хеш-таблицу с цепочками, поддерживающую операции:

- **insert(key)** — вставка ключа.
- **remove(key)** — удаление ключа.
- **display()** — вывод содержимого.

Дополнительно реализовать вариант с **рехешированием** (rehashing), при котором размер таблицы удваивается при превышении коэффициентом заполнения `0.5`, чтобы поддерживать хорошую производительность.

---

## 1. Простая реализация (без рехеширования)

Хеш-таблица имеет фиксированное число бакетов `bucket_count`. Хеш-функция: `index = key % bucket_count`. Каждый бакет — список Python.

```python
class Hash:
    def __init__(self, buckets):
        self.bucket_count = buckets
        self.table = [[] for _ in range(self.bucket_count)]

    def get_hash_index(self, key):
        return key % self.bucket_count

    def insert(self, key):
        idx = self.get_hash_index(key)
        self.table[idx].append(key)

    def remove(self, key):
        idx = self.get_hash_index(key)
        if key in self.table[idx]:
            self.table[idx].remove(key)

    def display(self):
        for i in range(self.bucket_count):
            print(i, end="")
            for key in self.table[i]:
                print(f" --> {key}", end="")
            print()
```

### Пример использования

```python
keys = [7, 18, 12, 25]
hash_table = Hash(7)
for key in keys:
    hash_table.insert(key)

hash_table.remove(12)
hash_table.display()
```

**Вывод:**
```
0 --> 7
1
2
3
4 --> 18 --> 25
5
6
```

### Сложность (простая версия)

- **Поиск/вставка/удаление в среднем:** O(1 + α), где α = n/m — коэффициент заполнения.
- **Худший случай (все ключи в одном бакете):** O(n).
- **Память:** O(n + m).

## 2. Реализация с рехешированием (динамическое расширение)

При превышении коэффициента заполнения `0.5` размер таблицы удваивается, и все существующие ключи перехешируются в новую таблицу.

```python
class HashTable:
    def __init__(self, buckets):
        self.bucket_count = buckets
        self.num_of_elements = 0
        self.table = [[] for _ in range(buckets)]

    def get_hash_index(self, key):
        return key % self.bucket_count

    def get_load_factor(self):
        return self.num_of_elements / self.bucket_count

    def rehash(self):
        old_table = self.table
        self.bucket_count *= 2
        self.table = [[] for _ in range(self.bucket_count)]
        self.num_of_elements = 0
        for bucket in old_table:
            for key in bucket:
                self.insert(key)

    def insert(self, key):
        while self.get_load_factor() > 0.5:
            self.rehash()
        idx = self.get_hash_index(key)
        self.table[idx].append(key)
        self.num_of_elements += 1

    def remove(self, key):
        idx = self.get_hash_index(key)
        if key in self.table[idx]:
            self.table[idx].remove(key)
            self.num_of_elements -= 1

    def display(self):
        for i in range(self.bucket_count):
            print(i, end="")
            for key in self.table[i]:
                print(f" --> {key}", end="")
            print()
```

### Пример использования

```python
keys = [15, 11, 27]
hashTable = HashTable(5)
for key in keys:
    hashTable.insert(key)

hashTable.remove(11)
hashTable.display()

hashTable.insert(19)
print("\nПосле рехеширования:")
hashTable.display()
```

**Вывод (примерный):**
```
0 --> 15
1
2 --> 27
3
4

После рехеширования:
0 --> 15
1
2 --> 27
3
4 --> 19
```

### Сложность (с рехешированием)

- **Вставка в среднем:** O(1) амортизированно (редкое рехеширование O(n) распределяется на много вставок).
- **Удаление, поиск:** O(1 + α) в среднем.
- **Память:** O(n + m), где m динамически растёт.

## Преимущества и недостатки

| Преимущества | Недостатки |
|--------------|------------|
| Простая реализация | Дополнительная память на указатели |
| Не требует маркеров удаления | При плохой хеш-функции списки могут стать длинными |
| Количество элементов не ограничено | Ухудшение локальности кэша |
| Удаление — без перемещения элементов | — |

---

## Примечания

- В простой версии размер таблицы фиксирован; если коэффициент заполнения становится большим (>0.7), производительность падает.
- В версии с рехешированием таблица автоматически увеличивается, что сохраняет хорошее среднее время операций.
- Вместо списков Python можно использовать другие структуры (например, `set`), но для учебных целей списки наглядны.
