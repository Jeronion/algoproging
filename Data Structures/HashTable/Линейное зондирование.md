
## Определение

**Линейное зондирование** — метод разрешения коллизий в хеш-таблицах с открытой адресацией. При возникновении коллизии (ячейка уже занята) мы последовательно проверяем следующие ячейки таблицы (с шагом 1) до нахождения свободного места. Формула для поиска индекса на шаге `i`:

```
index = (hash(key) + i) % size,   i = 0, 1, 2, ...
```

Все элементы хранятся непосредственно в таблице, поэтому её размер должен быть больше количества ключей. Для поддержания хорошей производительности коэффициент заполнения (`load factor = количество ключей / размер`) не должен превышать `0.7`. При превышении рекомендуется увеличить таблицу и выполнить **рехеширование** (перевставку всех элементов).

---

## Операции

### Вставка (insert)

1. Вычислить начальный индекс: `hashIndex = key % capacity`.
2. Пока ячейка `arr[hashIndex]` не пуста и не помечена как удалённая, и ключ уже не существует, переходить к следующей ячейке: `hashIndex = (hashIndex + 1) % capacity`.
3. Если достигли пустой или удалённой ячейки, вставить новый элемент.

### Поиск (search / get)

1. Начать с `hashIndex = key % capacity`.
2. Идти по ячейкам последовательно, пока не встретим пустую ячейку (элемент не найден) или не найдём ключ.
3. Если нашли ключ, вернуть значение.

### Удаление (delete)

1. Найти позицию ключа тем же способом.
2. Если нашли, пометить ячейку как **удалённую** (dummy node), но не освобождать физически. Это позволяет не разрывать цепочку поиска для последующих элементов.
3. Уменьшить счётчик размера.

## Реализация на Python

```python
class HashNode:
    def __init__(self, key, value):
        self.key = key
        self.value = value

class HashMap:
    def __init__(self, capacity=20):
        self.capacity = capacity          # размер таблицы
        self.size = 0                     # количество реальных элементов
        self.table = [None] * capacity    # массив узлов или None
        self.dummy = HashNode(-1, -1)     # маркер удалённого узла

    def _hash(self, key):
        return key % self.capacity

    def insert(self, key, value):
        temp = HashNode(key, value)
        index = self._hash(key)

        # ищем свободное место (None или dummy)
        while self.table[index] is not None and self.table[index].key != -1:
            # если ключ уже существует, обновляем значение
            if self.table[index].key == key:
                self.table[index].value = value
                return
            index = (index + 1) % self.capacity

        # вставляем в найденную позицию
        self.table[index] = temp
        self.size += 1

    def get(self, key):
        index = self._hash(key)
        start = index
        while self.table[index] is not None:
            if self.table[index].key == key:
                return self.table[index].value
            index = (index + 1) % self.capacity
            if index == start:   # полный круг, элемент не найден
                break
        return -1

    def delete(self, key):
        index = self._hash(key)
        start = index
        while self.table[index] is not None:
            if self.table[index].key == key:
                # помечаем как удалённый
                self.table[index] = self.dummy
                self.size -= 1
                return True
            index = (index + 1) % self.capacity
            if index == start:
                break
        return False

    def display(self):
        for node in self.table:
            if node is not None and node.key != -1:
                print(f"key:{node.key} value:{node.value}")

    def is_empty(self):
        return self.size == 0

    def get_size(self):
        return self.size
```

## Пример использования

```python
if __name__ == "__main__":
    h = HashMap(7)
    h.insert(50, 50)
    h.insert(700, 700)
    h.insert(76, 76)
    h.insert(85, 85)
    h.insert(92, 92)
    h.insert(73, 73)
    h.insert(101, 101)
    h.display()
    print("Size:", h.get_size())
    print("Get 76:", h.get(76))
    h.delete(76)
    print("After deletion, size:", h.get_size())
    h.display()
```

```
key:700 value:700
```

## Сложность

| Операция | Средний случай | Худший случай |
|----------|----------------|----------------|
| Вставка  | O(1)           | O(n)           |
| Поиск    | O(1)           | O(n)           |
| Удаление | O(1)           | O(n)           |

- **Память:** O(capacity)

Худший случай происходит при сильной заполненности таблицы или плохой хеш-функции, когда все элементы попадают в одну «кучу» (кластеризацию).

## Преимущества и недостатки

**Преимущества:**
- Простота реализации.
- Отсутствие дополнительных структур (списков).
- Хорошая локальность кэша.

**Недостатки:**
- Первичная кластеризация — скопление занятых ячеек, что замедляет поиск.
- Необходимость маркеров удаления (dummy nodes), которые со временем накапливаются.
- Требуется рехеширование при высокой загрузке.
