
## Определение

**Бинарное дерево поиска (Binary Search Tree, BST)** — это бинарное дерево, обладающее следующим свойством: для каждого узла все значения в левом поддереве **меньше** значения узла, а все значения в правом поддереве — **больше** (при отсутствии дубликатов). Это свойство позволяет эффективно выполнять поиск, вставку и удаление элементов.

---

## Основные операции

| Операция   | Описание                              | Сложность (средняя) | Сложность (худшая) |
|------------|---------------------------------------|---------------------|--------------------|
| `insert(x)`| Вставка элемента                      | O(log n)            | O(n)               |
| `search(x)`| Поиск элемента                         | O(log n)            | O(n)               |
| `delete(x)`| Удаление элемента                      | O(log n)            | O(n)               |
| `min()`    | Поиск минимального элемента            | O(log n)            | O(n)               |
| `max()`    | Поиск максимального элемента           | O(log n)            | O(n)               |
| `inorder()`| Обход в порядке возрастания            | O(n)                | O(n)               |

*Худший случай возникает при вырождении дерева в линейный список (например, при вставке отсортированных элементов).*

---

## Свойства BST

- Все элементы левого поддерева узла **меньше** ключа узла.
- Все элементы правого поддерева узла **больше** ключа узла.
- Левое и правое поддеревья также являются BST.
- Обычно не содержит дубликатов (можно модифицировать для поддержки счётчиков).
- Inorder-обход даёт отсортированную последовательность.

---

## Структура узла

```python
class Node:
    def __init__(self, key):
        self.key = key
        self.left = None
        self.right = None
```

---
## Операции

### Вставка элемента (insert)

1. Если дерево пусто, создать новый узел как корень.
2. Иначе сравнить вставляемое значение с ключом текущего узла:
   - Если меньше — рекурсивно вставить в левое поддерево.
   - Если больше — вставить в правое поддерево.
   - (Если равно — обычно ничего не делаем или обрабатываем дубликаты).

```python
def insert(root, key):
    if root is None:
        return Node(key)
    if key < root.key:
        root.left = insert(root.left, key)
    elif key > root.key:
        root.right = insert(root.right, key)
    # Если ключ равен — игнорируем (можно добавить счётчик)
    return root
```

### Сложность
- **Время:** O(h), где h — высота дерева.

### Поиск элемента (search)

1. Если дерево пусто или ключ совпадает с корнем — вернуть узел.
2. Если ключ меньше — искать в левом поддереве.
3. Если больше — в правом.

```python
def search(root, key):
    if root is None or root.key == key:
        return root
    if key < root.key:
        return search(root.left, key)
    return search(root.right, key)
```

### Сложность
- **Время:** O(h).

### Удаление элемента (delete)

Возможны три случая:
1. **Узел — лист** (нет детей): просто удаляем его.
2. **Узел имеет одного ребёнка**: заменяем удаляемый узел его ребёнком.
3. **Узел имеет двух детей**: находим наименьший узел в правом поддереве (или наибольший в левом), копируем его значение в удаляемый узел, затем удаляем этот наименьший узел (он будет либо листом, либо иметь одного ребёнка).

```python
def min_value_node(node):
    current = node
    while current.left is not None:
        current = current.left
    return current

def delete(root, key):
    if root is None:
        return root

    # Ищем узел для удаления
    if key < root.key:
        root.left = delete(root.left, key)
    elif key > root.key:
        root.right = delete(root.right, key)
    else:
        # Нашли узел
        # Случай 1: нет детей или один ребёнок
        if root.left is None:
            return root.right
        elif root.right is None:
            return root.left

        # Случай 2: два ребёнка
        # Находим наименьший узел в правом поддереве
        temp = min_value_node(root.right)
        root.key = temp.key
        # Удаляем этот наименьший узел
        root.right = delete(root.right, temp.key)

    return root
```

### Сложность
- **Время:** O(h).

## Сбалансированные деревья

Обычный BST может вырождаться в линейный список, что ухудшает сложность до O(n). Для гарантии логарифмической высоты используются самобалансирующиеся деревья:

- [[AVL-дерево]]
- **Красно-чёрное дерево**
- **B-дерево**

Они поддерживают высоту O(log n) ценой дополнительных операций при вставке/удалении.
