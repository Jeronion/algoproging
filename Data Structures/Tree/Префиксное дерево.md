
## Определение

**Бор (Trie, префиксное дерево)** — это структура данных для эффективного хранения и поиска строк. В отличие от хеш-таблиц, бор не требует вычисления хеш-функций и разрешения коллизий, а также позволяет выполнять операции вставки, поиска и поиска по префиксу за O(L), где L — длина строки. Бор особенно полезен для задач автодополнения, проверки орфографии, поиска по словарю и подсчёта частот слов.

Каждый узел бора представляет собой один символ строки. Корень не содержит символа. Путь от корня до узла образует префикс строки. Узлы могут помечаться как "конец слова", что позволяет различать полноценные слова и префиксы.

---

## Структура узла

Узел бора обычно содержит массив (или словарь) дочерних узлов и флаг, указывающий на окончание слова. В классической реализации для строчных латинских букв используется массив из 26 указателей.

```python
class TrieNode:
    def __init__(self):
        self.children = [None] * 26
        self.is_end = False
```

---

## Операции

### 1. Вставка (insert)

- Начинаем с корня.
- Для каждого символа строки вычисляем индекс (например, `ord(c) - ord('a')`).
- Если дочерний узел для этого индекса отсутствует, создаём новый узел.
- Переходим к дочернему узлу.
- После обработки всех символов помечаем текущий узел как конец слова.

```python
def insert(root, word):
    curr = root
    for ch in word:
        idx = ord(ch) - ord('a')
        if curr.children[idx] is None:
            curr.children[idx] = TrieNode()
        curr = curr.children[idx]
    curr.is_end = True
```

### 2. Поиск (search)

- Аналогично вставке проходим по символам.
- Если на каком-то шаге дочерний узел отсутствует, слово отсутствует.
- После прохода всех символов возвращаем `curr.is_end`.

```python
def search(root, word):
    curr = root
    for ch in word:
        idx = ord(ch) - ord('a')
        if curr.children[idx] is None:
            return False
        curr = curr.children[idx]
    return curr.is_end
```

### 3. Поиск по префиксу (starts_with)

- Проходим по символам префикса.
- Если дочерний узел отсутствует, префикс отсутствует.
- Если все символы найдены, возвращаем `True` (независимо от `is_end`).

```python
def starts_with(root, prefix):
    curr = root
    for ch in prefix:
        idx = ord(ch) - ord('a')
        if curr.children[idx] is None:
            return False
        curr = curr.children[idx]
    return True
```

### 4. Удаление (delete) — опционально

Удаление требует рекурсивного обхода и удаления узлов, которые не являются концом других слов и не имеют детей. Это более сложная операция, но часто реализуется рекурсивно.

## Реализация на Python (класс)

```python
class TrieNode:
    def __init__(self):
        self.children = [None] * 26
        self.is_end = False

class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, word: str) -> None:
        curr = self.root
        for ch in word:
            idx = ord(ch) - ord('a')
            if curr.children[idx] is None:
                curr.children[idx] = TrieNode()
            curr = curr.children[idx]
        curr.is_end = True

    def search(self, word: str) -> bool:
        curr = self.root
        for ch in word:
            idx = ord(ch) - ord('a')
            if curr.children[idx] is None:
                return False
            curr = curr.children[idx]
        return curr.is_end

    def starts_with(self, prefix: str) -> bool:
        curr = self.root
        for ch in prefix:
            idx = ord(ch) - ord('a')
            if curr.children[idx] is None:
                return False
            curr = curr.children[idx]
        return True
```

## Сложность

| Операция | Время | Память |
|----------|-------|--------|
| Вставка  | O(L)  | O(L)   |
| Поиск    | O(L)  | O(1)   |
| Поиск по префиксу | O(L) | O(1) |
| Удаление | O(L)  | O(1)   |

где L — длина строки.

---

## Применение

- **Автодополнение и автокоррекция** (поиск по префиксу).
- **Словари и проверка орфографии.**
- **Поиск по паттерну с подстановочными символами** (с модификацией).
- **Подсчёт количества слов с данным префиксом** (хранение дополнительного счётчика в узле).
- **Сортировка строк** (обход бора даёт алфавитный порядок).

---

## Примечания

- Бор требует значительной памяти из-за большого количества узлов (каждый узел содержит массив из 26 указателей). Для экономии памяти можно использовать словарь дочерних узлов (`dict`), особенно при работе с большим алфавитом.
- Операции выполняются за время, пропорциональное длине строки, что делает бор предпочтительным для задач с большим количеством строк и короткой длиной.
- Бор может быть использован как основа для более сложных структур, таких как сжатый бор (Radix tree) или суффиксное дерево.
