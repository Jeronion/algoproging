
## Определение

**Правый (или циклический) поворот массива** — это операция, при которой каждый элемент смещается вправо на заданное количество позиций `d`, а элементы, выходящие за пределы, «заворачиваются» в начало. Например, массив `[1,2,3,4,5,6]` после правого поворота на 2 позиции превращается в `[5,6,1,2,3,4]`.

---

## Основная идея

Правый поворот на `d` позиций эквивалентен переносу последних `d` элементов в начало, а остальных — в конец. Если `d` больше длины массива, достаточно повернуть на `d % n`, где `n` — длина массива.

---

## Способы реализации

### 1. Поэлементный сдвиг (Rotate one by one)

На каждой итерации сдвигаем все элементы на одну позицию вправо, повторяем `d` раз.

```python
def rotate_one_by_one(arr, d):
    n = len(arr)
    d %= n
    for _ in range(d):
        last = arr[-1]
        for i in range(n - 1, 0, -1):
            arr[i] = arr[i - 1]
        arr[0] = last
```

**Сложность:** O(n·d) времени, O(1) памяти.

### 2. Временный массив

Создаём новый массив, куда сначала копируем последние `d` элементов, затем первые `n-d`.

```python
def rotate_with_temp(arr, d):
    n = len(arr)
    d %= n
    temp = [0] * n
    # копируем последние d элементов в начало temp
    for i in range(d):
        temp[i] = arr[n - d + i]
    # копируем первые n-d элементов
    for i in range(n - d):
        temp[i + d] = arr[i]
    # копируем обратно
    for i in range(n):
        arr[i] = temp[i]
```

**Сложность:** O(n) времени, O(n) памяти.

### 3. Алгоритм «Жонглирование» (Juggling Algorithm)

Разбивает массив на циклы, определяемые НОД(n, d). Элементы каждого цикла сдвигаются по цепочке.

```python
from math import gcd

def rotate_juggling(arr, d):
    n = len(arr)
    d %= n
    cycles = gcd(n, d)
    for i in range(cycles):
        curr_idx = i
        curr_val = arr[i]
        while True:
            next_idx = (curr_idx + d) % n
            next_val = arr[next_idx]
            arr[next_idx] = curr_val
            curr_val = next_val
            curr_idx = next_idx
            if curr_idx == i:
                break
```

**Сложность:** O(n) времени, O(1) памяти.

### 4. Алгоритм с тремя реверсами (Reversal Algorithm)

Наиболее элегантный и практичный способ:

1. Развернуть весь массив.
2. Развернуть первые `d` элементов.
3. Развернуть оставшиеся `n-d` элементов.

```python
def rotate_reversal(arr, d):
    n = len(arr)
    d %= n
    # разворот всего массива
    arr.reverse()
    # разворот первых d элементов
    arr[:d] = reversed(arr[:d])
    # разворот остальных
    arr[d:] = reversed(arr[d:])
```

**Сложность:** O(n) времени, O(1) памяти.

---

## Пример

```python
arr = [1, 2, 3, 4, 5, 6]
d = 2
rotate_reversal(arr, d)
print(arr)   # [5, 6, 1, 2, 3, 4]
```

---

## Сравнение методов

| Метод               | Время      | Память    | Примечание                         |
|---------------------|------------|-----------|------------------------------------|
| Поэлементный сдвиг  | O(n·d)     | O(1)      | Медленный при больших d            |
| Временный массив    | O(n)       | O(n)      | Простой, но требует доп. памяти    |
| Жонглирование       | O(n)       | O(1)      | Эффективен, сложнее в реализации   |
| Три реверса         | O(n)       | O(1)      | **Рекомендуется** (прост и быстр)  |
