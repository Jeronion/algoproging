## Определение

**Сортировка вставками** — алгоритм сортировки, который последовательно вставляет каждый элемент **неотсортированной части массива** на своё место в **уже отсортированной части**.

### Примечания

- **Стабильный** алгоритм: сохраняет относительный порядок равных элементов
- Работает **на месте**, не требует дополнительной памяти
- Эффективен для **небольших или почти отсортированных массивов**

---

## Принцип работы

1. Разделяем массив на **отсортированную** и **неотсортированную** части
2. Берём следующий элемент из неотсортированной части
3. Вставляем его на правильное место среди уже отсортированных элементов
4. Повторяем, пока весь массив не будет отсортирован

**Схема:**

```
[23]                ← первый элемент отсортирован
[1, 23]             ← вставили 1 перед 23
[1, 10, 23]         ← вставили 10 между 1 и 23
[1, 5, 10, 23]      ← вставили 5 между 1 и 10
[1, 2, 5, 10, 23]   ← вставили 2 между 1 и 5
```

---

## Реализация на Python

```python
def insertionSort(arr):
    for i in range(1, len(arr)):
        key = arr[i]
        j = i - 1
	    
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]
            j -= 1
			
        arr[j + 1] = key
```

## Сложность

|Показатель|Значение|
|---|---|
|Временная (лучший)|O(n)|
|Временная (средняя)|O(n²)|
|Временная (худшая)|O(n²)|
|Память|O(1)|

> Лучший случай достигается, если массив уже отсортирован, тогда не происходит сдвигов элементов.

---

## Заключение

- Простая, **стабильная** и адаптивная сортировка
- Эффективна для **небольших массивов** и почти отсортированных данных
- Работает **на месте**, минимальные требования к памяти
- Используется в **гибридных алгоритмах сортировки** (IntroSort, TimSort) на малых подмассивах
- Для больших массивов уступает Merge Sort и Quick Sort
