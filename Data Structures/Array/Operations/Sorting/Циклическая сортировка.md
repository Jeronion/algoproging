
## Определение

**Cycle Sort** — алгоритм сортировки, который минимизирует количество **записей** в память. Он основан на разбиении массива на циклы, где каждый элемент либо уже стоит на своём месте, либо перемещается по циклу на правильную позицию. Алгоритм **нестабильный** и работает **in‑place**.

Основная идея: для каждого элемента находим его правильную позицию в отсортированном массиве (подсчётом количества меньших элементов). Если элемент не на месте, запускаем цепочку обменов, пока не вернёмся в начало цикла.

---

## Принцип работы

1. Проходим по массиву слева направо. Для текущего элемента `item`:
   - Считаем, сколько элементов меньше `item` — это и будет его правильный индекс `pos` (если элементы уникальны).
   - Если `pos` совпадает с текущей позицией — элемент уже на месте, переходим к следующему.
   - Иначе помещаем `item` на место `pos`. При этом элемент, который был на позиции `pos`, вытесняется и становится новым `item`.
2. Повторяем процесс для вытесненного элемента, пока не замкнём цикл (не вернёмся к исходной позиции).
3. Продолжаем с исходной позиции + 1.

Для случая, когда массив содержит элементы от `1` до `N` (или от `0` до `N`), существует оптимизированная версия с линейным временем.

## Алгоритм (общий случай, O(n²))

```python
def cycle_sort(arr):
    n = len(arr)
    for cycle_start in range(n - 1):
        item = arr[cycle_start]
        # 1. Найти правильную позицию для item
        pos = cycle_start
        for i in range(cycle_start + 1, n):
            if arr[i] < item:
                pos += 1
        # 2. Если уже на месте, пропустить
        if pos == cycle_start:
            continue
        # 3. Пропустить дубликаты
        while item == arr[pos]:
            pos += 1
        # 4. Поместить item на правильное место, item = вытесненный элемент
        arr[pos], item = item, arr[pos]
        # 5. Продолжить цикл, пока не вернёмся к cycle_start
        while pos != cycle_start:
            pos = cycle_start
            for i in range(cycle_start + 1, n):
                if arr[i] < item:
                    pos += 1
            while item == arr[pos]:
                pos += 1
            arr[pos], item = item, arr[pos]
```

## Оптимизированная версия для диапазона [1…N] (O(n))

Если массив содержит числа от `1` до `N` (или от `0` до `N`), можно использовать прямую подстановку: правильный индекс элемента `value` — это `value - 1` (или `value`).

```python
def cycle_sort_range(arr):
    i = 0
    n = len(arr)
    while i < n:
        correct = arr[i] - 1   # для диапазона 1..N
        if arr[i] != arr[correct]:
            arr[i], arr[correct] = arr[correct], arr[i]
        else:
            i += 1
```

## Пример

**Вход:** `[2, 4, 5, 1, 3]`

- Цикл 1: `2` → его правильный индекс 1 (0‑индексация). Ставим `2` на позицию 1, вытесняя `4`. `4` → индекс 3, вытесняет `1`. `1` → индекс 0, вытесняет `2` (замыкание).
- Цикл 2: `5` → индекс 4, вытесняет `3`. `3` → индекс 2, вытесняет `5` (замыкание).
- Результат: `[1,2,3,4,5]`.

## Сложность

| Случай                     | Время      | Память |
|----------------------------|------------|--------|
| Общий случай               | O(n²)      | O(1)   |
| Оптимизированный (1..N)    | O(n)       | O(1)   |

- **Минимальное число записей:** каждый элемент записывается в свою позицию не более одного раза.
- **Неустойчивость:** порядок равных элементов не сохраняется.

---

## Преимущества и недостатки

### Преимущества
- Минимальное количество операций записи (важно для EEPROM, FLASH).
- Не требует дополнительной памяти.
- Оптимизированная версия работает за O(n) для перестановок из 1..N.

### Недостатки
- В общем случае медленный (O(n²)).
- Неустойчивый.
- Не подходит для больших массивов.

## Применение

- Сортировка данных в энергонезависимой памяти (где запись дороже чтения).
- Задачи на перестановки, например:
  - «Найти минимальное количество обменов для сортировки»,
  - «Найти пропущенное положительное число» (LeetCode 41, 268).
