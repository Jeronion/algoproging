
## Определение

**Radix Sort (поразрядная сортировка)** — алгоритм сортировки **без прямых сравнений**, который обрабатывает элементы **разряд за разрядом**.

### Примечания:

* Эффективен для **целых чисел** или **строк с ключами фиксированной длины**
* Сортировка выполняется **по значащим разрядам** (обычно от младшего к старшему)
* На каждом разряде используется **стабильный алгоритм**, например **Counting Sort**

---

## Принцип работы

1. Определить максимальный элемент массива, чтобы узнать количество разрядов.
2. Для каждого разряда (единицы, десятки, сотни…) распределить элементы по «корзинам» в зависимости от значения текущего разряда.
3. Копировать элементы обратно в массив, сохраняя относительный порядок (стабильность).
4. Повторять процесс для всех разрядов до старшего.
5. В результате массив оказывается полностью отсортированным.

> Важно: стабильность сортировки на каждом разряде обеспечивает корректный итоговый порядок элементов.

## Реализация на Python

```python id="al7jra"
def countingSort(arr, exp1):
    n = len(arr)
    output = [0] * n
    count = [0] * 10

    # Подсчёт цифр
    for i in range(n):
        index = arr[i] // exp1
        count[index % 10] += 1

    # Префиксные суммы
    for i in range(1, 10):
        count[i] += count[i - 1]

    # Формирование выходного массива справа налево
    i = n - 1
    while i >= 0:
        index = arr[i] // exp1
        output[count[index % 10] - 1] = arr[i]
        count[index % 10] -= 1
        i -= 1

    # Копирование обратно
    for i in range(n):
        arr[i] = output[i]

def radixSort(arr):
    max1 = max(arr)
    exp = 1
    while max1 // exp >= 1:
        countingSort(arr, exp)
        exp *= 10
```

## Сложность

| Показатель | Значение       |
| ---------- | -------------- |
| Временная  | O(d · (n + b)) |
| Память     | O(n + b)       |

* d — количество разрядов
* n — количество элементов
* b — основание системы счисления (обычно 10)

---

## Применения и ограничения

* Эффективен при **фиксированном или малом количестве разрядов**
* На каждом разряде используется **стабильная сортировка**
* Требует **дополнительной памяти** для массивов «корзин»
* Менее эффективен, если количество разрядов велико или данные не числовые

---

## Заключение

Radix Sort — мощный несравнительный алгоритм, который позволяет сортировать массивы почти линейно при ограниченном количестве разрядов.

**Плюсы:**

* Близкая к линейной временная сложность при фиксированном d
* Стабильная сортировка
* Идеален для чисел и ключей фиксированной длины

**Минусы:**

* Дополнительная память O(n + b)
* Требует фиксированного количества разрядов

> Используется в задачах, где важна высокая скорость сортировки целых чисел или строк с фиксированной длиной ключей, особенно в сочетании с Counting Sort для стабильности.
