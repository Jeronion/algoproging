
## Определение

**Алгоритм Хоара** — это схема разбиения массива относительно опорного элемента (pivot), предложенная Тони Хоаром — создателем быстрой сортировки. В отличие от схемы Ломуто, Хоар использует два указателя, движущихся навстречу друг другу, что обычно приводит к меньшему числу обменов и большей эффективности.

Задача: переставить элементы массива так, чтобы:
- Все элементы **меньшие или равные** pivot оказались в левой части (не обязательно строго до pivot).
- Все элементы **большие или равные** pivot — в правой части.
- Позиция, в которой указатели пересекаются, является точкой раздела.

Обычно в качестве pivot выбирается **первый элемент** массива.

---

## Алгоритм (пошагово)

Пусть `arr` — массив длины `n`, `pivot = arr[0]`.

1. Инициализируем два указателя:
   - `i = -1` (будет двигаться вправо)
   - `j = n` (будет двигаться влево)
2. В бесконечном цикле:
   - Двигаем `i` вправо (`i += 1`), пока `arr[i] < pivot`.
   - Двигаем `j` влево (`j -= 1`), пока `arr[j] > pivot`.
   - Если `i >= j`, выходим из цикла.
   - Меняем местами `arr[i]` и `arr[j]`.
3. После выхода из цикла возвращаем `j` — индекс, разделяющий левую и правую части.

В результате:
- Все элементы `arr[0..j]` ≤ pivot (но не обязательно все меньше pivot).
- Все элементы `arr[j+1..n-1]` ≥ pivot.
- Сам pivot может оказаться в любом месте левой части.

## Пример

```
Исходный массив: [5, 3, 8, 4, 2, 7, 1, 10]
pivot = 5

i = -1, j = 8
Цикл:
  i: 0 (arr[0]=5 ≥5? да, останавливаемся) → i=0
  j: 7 (arr[7]=10 >5? нет, останавливаемся) → j=7
  i<j → swap(0,7) → [10,3,8,4,2,7,1,5]
  i=0, j=7

  i: 0 (arr[0]=10 ≥5? да) → i=0? нет, i уже 0, но условие i+=1 даст i=1? На самом деле алгоритм в коде сначала увеличивает i, потом проверяет. В классике: do i++ while arr[i] < pivot. После swap i не сбрасывается, а продолжается. Лучше покажу по шагам с правильным кодом:

По стандартному алгоритму (do-while):
  i = -1, j = 8
  do i++ while arr[i] < pivot → i=0 (arr[0]=5 не <5) → i=0
  do j-- while arr[j] > pivot → j=7 (arr[7]=10 >5) → j=6 (arr[6]=1 >5? нет) → j=6
  i=0, j=6 → i<j → swap(0,6) → [1,3,8,4,2,7,5,10]
  do i++ while arr[i] < pivot → i=1 (3<5) → i=2 (8 не <5) → i=2
  do j-- while arr[j] > pivot → j=5 (7>5) → j=4 (2>5? нет) → j=4
  i=2, j=4 → i<j → swap(2,4) → [1,3,2,4,8,7,5,10]
  do i++ while arr[i] < pivot → i=3 (4<5) → i=4 (8 не <5) → i=4
  do j-- while arr[j] > pivot → j=3 (4>5? нет) → j=3
  i=4, j=3 → i>j → выход.
Возвращаем j = 3. Левая часть: [1,3,2,4] (все ≤5), правая: [8,7,5,10] (все ≥5). Pivot 5 теперь на позиции 6 (не на границе). Это нормально для Хоара.

Результат: [1, 3, 2, 4, 8, 7, 5, 10]
```

## Реализация на Python

```python
def hoare_partition(arr):
    n = len(arr)
    pivot = arr[0]
    i = -1
    j = n
    while True:
        i += 1
        while arr[i] < pivot:
            i += 1
        j -= 1
        while arr[j] > pivot:
            j -= 1
        if i >= j:
            return j   # индекс раздела
        arr[i], arr[j] = arr[j], arr[i]
```

**Примечание:** В классической реализации Хоара используются циклы `do-while`, которые в Python эмулируются через `while True` с ручным управлением.

## Сложность

- **Время:** O(n) — один проход по массиву.
- **Память:** O(1) — все операции на месте.

## Сравнение с разбиением Ломуто

| Характеристика       | Хоар                           | Ломуто                        |
|----------------------|--------------------------------|-------------------------------|
| Количество обменов   | Меньше (в среднем ~n/6)       | Больше (в среднем ~n/3)      |
| Скорость на практике | Быстрее                        | Медленнее                     |
| Простота реализации  | Немного сложнее                | Проще                         |
| Возвращаемое значение| Индекс разделения (j)          | Индекс pivot (i)              |
| Стабильность         | Нестабилен                     | Нестабилен                    |
| Типичное применение  | Производительные реализации    | Учебные цели                  |

## Примечания

- Разбиение Хоара не помещает pivot на окончательную позицию (в отличие от Ломуто). После разбиения pivot может находиться где угодно в левой части.
- При рекурсивной быстрой сортировке после разбиения Хоара нужно вызывать сортировку для `[left .. j]` и `[j+1 .. right]` (где `j` — индекс, возвращённый функцией).
- Алгоритм работает и при выборе другого pivot (например, случайного), достаточно поменять местами выбранный элемент с первым.
- Из-за того, что условие включает равенства (`arr[i] >= pivot`, `arr[j] <= pivot`), массив с повторяющимися значениями разбивается более равномерно, что предотвращает деградацию быстрой сортировки до O(n²) на массивах с одинаковыми элементами.
