
## Определение

**Сортировка подсчётом (Counting Sort)** — алгоритм сортировки **без сравнений**, который распределяет элементы массива по количеству их вхождений.

### Примечания:

- Эффективен, когда **диапазон значений невелик** по сравнению с размером массива
- **Стабильный** алгоритм: сохраняет относительный порядок равных элементов
- Требует дополнительной памяти O(N + M), где N — размер массива, M — диапазон значений
- Не подходит для вещественных чисел без предварительной обработки

---

## Принцип работы

1. Создать [[Массив подсчёта]] `cntArr[]` размером `max(arr[]) + 1` и инициализировать нулями
2. Подсчитать количество вхождений каждого элемента: `cntArr[arr[i]]++`
3. Вычислить [[Префиксные суммы]] для каждого индекса `cntArr[]` для обеспечения **стабильности**
4. Создать выходной массив `ans[]` размера N
5. Пройти по массиву `arr[]` **справа налево**, чтобы разместить элементы на правильных позициях:  
    - `ans[cntArr[arr[i]] - 1] = arr[i]`, затем `cntArr[arr[i]] -= 1`

---

## Реализация на Python

```python
def counting_sort(arr):
	N = len(arr)
	MAX = max(arr)
	MIN = min(arr)
	
	cntArr = [0] * (MAX - MIN + 1)
	
	for v in arr:
		cntArr[v - MIN] += 1
	
	for i in range(1, MAX - MIN + 1):
		cntArr[i] += cntArr[i - 1]
	
	ans = [0] * N
	
	for i in range(N - 1, -1, -1):
		v = arr[i]
		ans[cntArr[v - MIN] - 1] = v
		cntArr[v - MIN] -= 1
	
	return ans
```

## Сложность

|Показатель|Значение|
|---|---|
|Временная (лучший/средняя/худшая)|O(N + M)|
|Память|O(N + M)|

- N — количество элементов массива
- M — диапазон возможных значений элементов

---

## Применения и ограничения

- Отлично работает при **малом диапазоне значений**
- Используется в качестве вспомогательного метода для **Radix Sort** и **Bucket Sort**
- Требует **дополнительной памяти**, становится неэффективной при большом диапазоне
- Не подходит для **вещественных чисел** без масштабирования

>Используется как самостоятельный метод или компонент более сложных алгоритмов сортировки.
