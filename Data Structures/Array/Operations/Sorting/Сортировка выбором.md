
## Определение

**Сортировка выбором** — алгоритм сортировки на основе сравнения, который **последовательно выбирает наименьший (или наибольший) элемент из неотсортированной части массива** и меняет его местами с первым элементом этой части.

### Примечания:

- **Не стабильный** алгоритм (при обмене элементов одинакового значения их порядок может меняться)
- Работает **на месте**, не требует дополнительной памяти

---

## Принцип работы

1. Найти **наименьший элемент** в неотсортированной части массива
2. Поменять его местами с первым элементом этой части
3. Повторять процесс для оставшейся части массива, пока все элементы не окажутся на своих местах

**Схема:**

```
[7, 5, 2, 4]  → выбрали min=2 → [2, 5, 7, 4]
[2, 5, 7, 4]  → выбрали min=4 → [2, 4, 7, 5]
[2, 4, 7, 5]  → выбрали min=5 → [2, 4, 5, 7]
```

---

## Реализация на Python

```python
def selection_sort(arr):
    n = len(arr)
    for i in range(n - 1):
        min_idx = i
        # ищем минимальный элемент в оставшейся части
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        # меняем местами с текущей позицией
        arr[i], arr[min_idx] = arr[min_idx], arr[i]

def print_array(arr):
    print(" ".join(map(str, arr)))
```

## Сложность

|Показатель|Значение|
|---|---|
|Временная (лучший)|O(n²)|
|Временная (средняя)|O(n²)|
|Временная (худшая)|O(n²)|
|Память|O(1)|

> Причина: два вложенных цикла — внешний выбирает позицию, внутренний ищет минимум.

---

## Заключение

- Простая для понимания и реализации сортировка
- Работает **на месте**, не требует дополнительной памяти
- **Неэффективна для больших массивов** $O(n²)$
- Часто используется **для малых массивов** или как учебный пример
