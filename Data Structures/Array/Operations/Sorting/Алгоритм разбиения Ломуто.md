
## Определение

**Алгоритм Ломуто** — это схема разбиения массива относительно опорного элемента (pivot), используемая в быстрой сортировке. В отличие от схемы Хоара, Ломуто проще в реализации, но обычно выполняет больше обменов.

Задача: переставить элементы массива так, чтобы:
- Все элементы **меньше** pivot оказались **до** него.
- Все элементы **больше или равны** pivot оказались **после** него.
- Сам pivot в итоге занимает свою окончательную позицию.

Обычно в качестве pivot выбирается последний элемент массива.

---

## Алгоритм (пошагово)

Пусть `arr` — массив длины `n`, `pivot = arr[n-1]` (последний элемент).

1. Инициализируем указатель `i = 0` — это граница, левее которой находятся элементы, строго меньшие pivot.
2. Проходим указателем `j` от `0` до `n-2` (все элементы, кроме pivot):
   - Если `arr[j] < pivot`, то меняем местами `arr[i]` и `arr[j]`, затем увеличиваем `i` на 1.
3. После цикла меняем местами `arr[i]` и `arr[n-1]` (ставим pivot на своё место).
4. Возвращаем индекс `i` — позицию pivot.

В результате:
- `arr[0..i-1]` < pivot
- `arr[i]` = pivot
- `arr[i+1..n-1]` ≥ pivot

---

## Пример

```
Исходный массив: [5, 3, 8, 4, 2, 7, 1, 10]
pivot = 10 (последний элемент)

i = 0
j = 0: arr[0]=5 < 10 → swap(arr[0],arr[0]) → [5,3,8,4,2,7,1,10], i=1
j = 1: 3 < 10 → swap(arr[1],arr[1]) → i=2
j = 2: 8 < 10 → swap(arr[2],arr[2]) → i=3
j = 3: 4 < 10 → swap(arr[3],arr[3]) → i=4
j = 4: 2 < 10 → swap(arr[4],arr[4]) → i=5
j = 5: 7 < 10 → swap(arr[5],arr[5]) → i=6
j = 6: 1 < 10 → swap(arr[6],arr[6]) → i=7
После цикла: [5,3,8,4,2,7,1,10], i=7
Меняем arr[7] и arr[7] (ничего не меняется)
Результат: [5,3,8,4,2,7,1,10] (все элементы меньше 10, pivot на месте)
```

Более интересный пример, когда pivot не максимальный:

```
Массив: [5, 3, 8, 4, 2, 7, 1, 6] (pivot = 6)
i = 0
j=0: 5<6 → swap(0,0) → i=1
j=1: 3<6 → swap(1,1) → i=2
j=2: 8<6? нет → пропуск
j=3: 4<6 → swap(2,3) → [5,3,4,8,2,7,1,6], i=3
j=4: 2<6 → swap(3,4) → [5,3,4,2,8,7,1,6], i=4
j=5: 7<6? нет
j=6: 1<6 → swap(4,6) → [5,3,4,2,1,7,8,6], i=5
После цикла: ставим pivot на место i=5: swap(5,7) → [5,3,4,2,1,6,8,7]
Результат: элементы <6: [5,3,4,2,1] → pivot = 6 → элементы ≥6: [8,7].
```

## Реализация на Python

```python
def lomuto_partition(arr):
    n = len(arr)
    pivot = arr[-1]          # последний элемент
    i = 0
    for j in range(n - 1):
        if arr[j] < pivot:
            arr[i], arr[j] = arr[j], arr[i]
            i += 1
    # ставим pivot на место
    arr[i], arr[-1] = arr[-1], arr[i]
    return i   # индекс pivot
```

---

## Сложность

- **Время:** O(n) — один проход по массиву.
- **Память:** O(1) — все операции на месте.

---

## Сравнение с разбиением Хоара

| Характеристика       | Ломуто                         | Хоар                         |
|----------------------|--------------------------------|------------------------------|
| Количество обменов   | Больше (в среднем ~n/3)       | Меньше                       |
| Простота реализации  | Проще                          | Немного сложнее              |
| Стабильность         | Нестабилен                     | Нестабилен                   |
| Выбор pivot          | Обычно последний элемент       | Обычно первый или средний    |
| Типичное применение  | Учебные цели, простота кода    | Производительные реализации  |

---

## Примечания

- Разбиение Ломуто часто используется в учебных примерах быстрой сортировки из-за своей простоты.
- Неэффективно при большом количестве равных элементов (все элементы, равные pivot, попадают в правую часть, что может привести к O(n²) в быстрой сортировке).
- Для повышения эффективности pivot можно выбирать случайно (тогда маловероятно попадание на максимум/минимум).
