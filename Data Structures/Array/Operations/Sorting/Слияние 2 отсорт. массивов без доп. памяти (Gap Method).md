
## Определение

Даны два отсортированных массива `a` и `b` (размеры `n` и `m`). Требуется объединить их в один отсортированный массив, используя **только константную дополнительную память**. Результат должен быть расположен так, чтобы первые `n` элементов (минимальные) оказались в массиве `a`, а оставшиеся `m` элементов (максимальные) — в массиве `b`. Иными словами, после выполнения алгоритма массивы `a` и `b` вместе образуют единую отсортированную последовательность.

Алгоритм, основанный на **методе Gap** (или методе Шелла), выполняет сравнение и обмен элементов на расстоянии `gap`, постепенно уменьшая его. Этот подход позволяет обойтись без дополнительной памяти.

---

## Идея решения

Алгоритм имитирует процесс [[Сортировка с убыв. смещ.]] для объединённого массива длиной `n + m`. Однако физического слияния не происходит — элементы сравниваются и обмениваются в зависимости от того, находятся ли они в первом массиве, во втором или пересекают границу.

**Основные шаги:**

1. Вычисляем начальный `gap = ceil((n + m) / 2)`.
2. Пока `gap > 0`:
   - Проходим по объединённому массиву двумя указателями: `left = 0`, `right = gap`.
   - Для каждой пары `(left, right)`:
     - Определяем, где находятся указатели (в `a`, в `b` или оба в `b`).
     - Если значение в левом указателе больше значения в правом, меняем их местами.
     - Увеличиваем оба указателя на 1.
   - После прохода обновляем `gap = ceil(gap / 2)` (для следующей итерации).
3. Завершаем, когда `gap == 0`.

Этот алгоритм гарантирует, что после завершения все элементы массива `a` будут меньше или равны элементам массива `b`, и каждый массив останется отсортированным по отдельности, а вместе они образуют единую отсортированную последовательность.

---

## Алгоритм (пошагово)

1. **Инициализация:**  
   `gap = (n + m + 1) // 2` (это `ceil((n+m)/2)`).

2. **Внешний цикл:** пока `gap > 0`:
   - Установить `i = 0`, `j = gap`.
   - **Внутренний цикл:** пока `j < n + m`:
     - Если оба указателя в `a` (`j < n`):  
       сравнить `a[i]` и `a[j]`, при необходимости обменять.
     - Если `i` в `a`, а `j` в `b` (`i < n` и `j >= n`):  
       сравнить `a[i]` и `b[j - n]`, при необходимости обменять.
     - Если оба в `b` (`i >= n`):  
       сравнить `b[i - n]` и `b[j - n]`, при необходимости обменять.
     - Увеличить `i` и `j` на 1.
   - Если `gap == 1`, выйти (оптимизация, иначе можно продолжать до `gap == 0`).
   - Обновить `gap = (gap + 1) // 2` (ceil(gap/2)).

3. После завершения `a` содержит наименьшие `n` элементов, `b` — наибольшие `m` элементов, оба массива отсортированы.

---

## Реализация на Python

```python
def merge_arrays(a, b):
    n, m = len(a), len(b)
    gap = (n + m + 1) // 2

    while gap > 0:
        i = 0
        j = gap
        while j < n + m:
            # оба в a
            if j < n and a[i] > a[j]:
                a[i], a[j] = a[j], a[i]
            # i в a, j в b
            elif i < n <= j and a[i] > b[j - n]:
                a[i], b[j - n] = b[j - n], a[i]
            # оба в b
            elif i >= n and b[i - n] > b[j - n]:
                b[i - n], b[j - n] = b[j - n], b[i - n]
            i += 1
            j += 1
        # Если gap == 1, дальнейшие проходы не нужны
        if gap == 1:
            break
        gap = (gap + 1) // 2
```

## Пример

```
a = [1, 5, 9, 10, 15, 20]
b = [2, 3, 8, 13]
```

После выполнения:

```
a = [1, 2, 3, 5, 8, 9]
b = [10, 13, 15, 20]
```

Объединённая отсортированная последовательность:  
`1, 2, 3, 5, 8, 9, 10, 13, 15, 20`.

---

## Сложность

- **Время:** O((n+m) log (n+m)) — внешний цикл выполняется O(log(n+m)) раз, внутренний O(n+m) раз.
- **Память:** O(1) — все операции выполняются на месте.

---

## Примечания

- Алгоритм не требует выделения дополнительного массива, что важно при работе с большими данными.
- Метод Gap является адаптацией сортировки Шелла для двух массивов.
- Важно правильно вычислять индексы при переходе из `a` в `b` (вычитание `n`).
- После завершения алгоритма `a` и `b` по отдельности отсортированы, а все элементы `a` ≤ всех элементов `b`.
