
## Определение

**Merge Sort** — алгоритм сортировки по принципу **«Разделяй и властвуй»**.

### Примечания:

- **Стабильный алгоритм:** сохраняет относительный порядок равных элементов
- Требует дополнительной памяти: `O(n)`
- Временная сложность: `O(n log n)` во всех случаях

---

## Принцип работы

1. **Разделение (Divide):**  
    Массив делится на две половины до единичного элемента.

2. **Сортировка подмассивов (Conquer):**  
    Каждый подмассив сортируется рекурсивно.

3. **Слияние (Merge):**  
    Отсортированные подмассивы сливаются обратно в один отсортированный массив.
	- Есть слияние без затраты доп. памяти (но оно медленнее) - [[Слияние 2 отсорт. массивов без доп. памяти (Gap Method)]]

## Реализация на Python

```python
def merge(arr, left, mid, right):
    n1 = mid - left + 1
    n2 = right - mid

    L = [0] * n1
    R = [0] * n2

    for i in range(n1):
        L[i] = arr[left + i]
    for j in range(n2):
        R[j] = arr[mid + 1 + j]

    i = j = 0
    k = left

    while i < n1 and j < n2:
        if L[i] <= R[j]:
            arr[k] = L[i]
            i += 1
        else:
            arr[k] = R[j]
            j += 1
        k += 1

    while i < n1:
        arr[k] = L[i]
        i += 1
        k += 1

    while j < n2:
        arr[k] = R[j]
        j += 1
        k += 1


def mergeSort(arr, left, right):
    if left < right:
        mid = (left + right) // 2
        mergeSort(arr, left, mid)
        mergeSort(arr, mid + 1, right)
        merge(arr, left, mid, right)
```

|Временная (средняя)|O(n log n)|
|Временная (худшая)|O(n log n)|
|Память|O(n)|
|Стабильность|Да|
|In-place|Нет|

---

## Преимущества

- **Стабильность** — порядок равных элементов сохраняется
- **Гарантированная производительность** — худший случай `O(n log n)`
- **Простая реализация** — легко понять и применять
- **Параллелизм** — подмассивы можно сортировать независимо

---

## Недостатки

- **Дополнительная память** — требуется `O(n)`
- **Не in-place** — массив копируется при слиянии
- **Более медленный, чем QuickSort** на больших массивах из-за кэш-неэффективности
