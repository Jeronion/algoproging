
## Определение

**Бинарный поиск** — это алгоритм поиска элемента в **упорядоченном массиве** или числовом диапазоне, который **последовательно делит область поиска пополам**, сокращая её на каждом шаге.
### Примечания:

- Работает только для **отсортированных данных** или **монотонных функций**.
- Возвращает индекс элемента или значение-ответ, если элемент найден, иначе $-1$.
- Используется для ускоренного поиска по сравнению с линейным обходом (O(log N) вместо O(N)).

## 1. Бинарный поиск в упорядоченных массивах

**Идея:**

1. Задаём область поиска `[left, right]`
2. Выбираем середину `mid`
3. Сравниваем `array[mid]` с искомым `x`:
    
    - `x < array[mid]` → ищем в левой половине
    - `x > array[mid]` → ищем в правой половине
    - `x == array[mid]` → нашли

**Реализация на Python (итеративная):**

```python
def binary_search(arr, x):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] < x:
            left = mid + 1
        elif arr[mid] > x:
            right = mid - 1
        else:
            return mid
    return -1
```

**Сложность:** 
- Время: O(log N)
- Память: O(1)

**Реализация на Python (рекурсивная):**

```python
def exponential_search(arr, x):
    n = len(arr)
    if n == 0:
        return -1

    i = 1
    while i < n and arr[i] < x:
        i *= 2

    left = i // 2
    right = min(i, n - 1)

    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == x:
            return mid
        elif arr[mid] < x:
            left = mid + 1
        else:
            right = mid - 1
    return -1
```

## 2. Бинарный поиск для монотонных функций

**Определение:** функция $f(x)$ монотонна, если $x1 > x2 ⇒ f(x1) > f(x2)$.

**Пример:** поиск кубического корня числа $x ≥ 1$ с точностью `eps = 10**-11`

```python
eps = 1e-11
x = 20

left, right = 1, x
while right - left > eps:
    mid = (left + right) / 2
    if mid ** 3 < x:
        left = mid
    else:
        right = mid

print(left)
```

## 3. Бинарный поиск по ответу

Используется, когда нужно найти **некоторое число-ответ**, проверка возможна, прямое вычисление затруднено.

**Пример: задача «Дипломы»**

- Дано n прямоугольников $w × h$, нужно разместить на квадратной доске минимального размера `x`.
- Проверка: $(x // w) * (x // h) >= n$


```python
w, h, n = map(int, input().split())
left, right = max(w, h), max(w*n, h*n)

while left <= right:
    mid = (left + right) // 2
    if (mid // w) * (mid // h) >= n:
        right = mid - 1
    else:
        left = mid + 1

print(left)
```

---

## 4. Особенности

- Быстро сокращает область поиска: O(log N)
- Применим к массивам, монотонным функциям, задачам «поиск по ответу»
- Основан на принципе **деления области пополам**

---

## 5. Когда использовать

- Отсортированные массивы
- Монотонные функции
- Оптимизационные задачи с проверкой условия
