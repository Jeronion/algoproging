
## Определение

**Тернарный поиск** — алгоритм поиска экстремума (минимума или максимума) **унимодальной** функции на заданном интервале. В отличие от бинарного поиска, который делит отрезок на две части, тернарный поиск делит его на три равные части и отбрасывает ту треть, где экстремум заведомо не может находиться. Сложность алгоритма — $O(\log_3 n)$, но обычно на практике он чуть медленнее бинарного, так как требует двух сравнений на шаг.

> **Унимодальная функция** — функция, которая сначала строго убывает до некоторой точки (минимума), а затем строго возрастает (для поиска минимума). Для поиска максимума — сначала возрастает, потом убывает.

---

## Основная идея

Пусть у нас есть унимодальная функция $f(x)$ на отрезке $[l, r]$, и мы ищем точку минимума (или максимума). Вычислим две внутренние точки:

$$
m_1 = l + \frac{r - l}{3}, \quad m_2 = r - \frac{r - l}{3}.
$$

Если $f(m_1) < f(m_2)$, то минимум находится в левой части (включая $m_1$), поэтому правую треть $[m_2, r]$ можно отбросить. Если $f(m_1) > f(m_2)$ — минимум в правой части, отбрасываем левую треть $[l, m_1]$. Если $f(m_1) = f(m_2)$, то минимум лежит между ними, можно сузить диапазон до $[m_1, m_2]$.

Таким образом, на каждой итерации интервал уменьшается в $\frac{2}{3}$ раза (или до $\frac{1}{3}$ при равенстве). После $k$ итераций длина интервала становится примерно $(\frac{2}{3})^k$ от исходной.

## Алгоритм для поиска минимума в массиве (унимодальная последовательность)

```python
def ternary_search_min(arr):
    """
    Предполагается, что arr сначала строго убывает, затем строго возрастает.
    Возвращает индекс минимального элемента.
    """
    left, right = 0, len(arr) - 1
    while right - left > 2:  # пока интервал достаточно широк
        m1 = left + (right - left) // 3
        m2 = right - (right - left) // 3
        if arr[m1] < arr[m2]:
            right = m2 - 1
        elif arr[m1] > arr[m2]:
            left = m1 + 1
        else:
            left, right = m1, m2
            break
    # На маленьком интервале находим минимум линейным перебором
    min_idx = left
    for i in range(left, right + 1):
        if arr[i] < arr[min_idx]:
            min_idx = i
    return min_idx
```

### Пример

```python
arr = [9, 7, 1, 2, 3, 6, 10]   # убывает до 1, затем возрастает
print(ternary_search_min(arr))  # 2 (индекс элемента 1)
```

## Сложность

- **Время:** $O(\log_3 n)$ итераций, но на практике обычно используют бинарный поиск (логарифм по основанию 2) из-за меньшего числа сравнений.
- **Память:** $O(1)$.

---

## Применение

- Поиск экстремума унимодальной функции, заданной аналитически (например, $f(x) = -x^2 + 5x$).
- Задачи на массивах, где гарантированно есть одна точка поворота (горный массив, массив-долина).
- Оптимизация в численных методах.

---

## Сравнение с бинарным поиском

| Характеристика       | Тернарный поиск                | Бинарный поиск               |
|----------------------|--------------------------------|------------------------------|
| Деление отрезка      | На 3 части                     | На 2 части                   |
| Количество сравнений | 2 на итерацию                  | 1 на итерацию                |
| Скорость сходимости  | $O(\log_3 n)$                  | $O(\log_2 n)$                |
| Область применения   | Унимодальные функции           | Монотонные функции           |

Для поиска экстремума унимодальной функции тернарный поиск — естественный выбор. Для поиска корня монотонной функции предпочтительнее бинарный.

---

## Примечания

- Алгоритм требует, чтобы функция была **строго** унимодальной, иначе могут быть проблемы.
- В коде выше, чтобы избежать бесконечного цикла, при равенстве $f(m_1) = f(m_2)$ интервал сужается до $[m_1, m_2]$, а затем ищется линейно.
- Тернарный поиск часто используется на соревнованиях по программированию для задач вида "найти минимум функции" без производной.
