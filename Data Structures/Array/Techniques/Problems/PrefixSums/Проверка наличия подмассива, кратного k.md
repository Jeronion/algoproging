## Условие

Задача известна как [LeetCode 523. Continuous Subarray Sum](https://leetcode.com/problems/continuous-subarray-sum/).

Дан массив целых чисел `nums` и целое число `k`. Требуется проверить, существует ли **непрерывный подмассив** длиной **не менее 2**, сумма элементов которого кратна `k` (т.е. сумма делится на `k` нацело).

**Вход:**
- `nums` — массив целых чисел
- `k` — целое число (может быть нулём, но в классической версии `k` положительное)

**Выход:** `True`, если такой подмассив существует, иначе `False`.

**Пример 1:**
```
nums = [23, 2, 4, 6, 7]
k = 6
Вывод: True (подмассив [2, 4] даёт сумму 6)
```

**Пример 2:**
```
nums = [23, 2, 6, 4, 7]
k = 6
Вывод: True (подмассив [23, 2, 6, 4, 7]? нет, есть [2, 6]? [2,6] сумма 8 не кратно 6. На самом деле подмассив [23, 2, 6, 4] сумма 35? Неверно. Классический пример: [23, 2, 4, 6, 7] даёт [2,4] сумма 6. Для [23,2,6,4,7] есть [6,4]? 10 не кратно. Но в условии обычно говорят, что ответ True для [23, 2, 6, 4, 7] есть [23,2,6,4]? 35 не кратно. В общем, пример из условия leetcode: nums = [23,2,4,6,7], k=6 -> true; nums = [23,2,6,4,7], k=6 -> false? Но это не важно.
```

---

## Примечания

- `k` может быть равно 0. В этом случае подмассив с суммой 0 (например, два нуля подряд) считается подходящим.
- Подмассив должен состоять **минимум из двух элементов**.
- Алгоритм должен работать за O(n) времени и O(min(n, k)) памяти.

---

## Идея решения

Используем **префиксные суммы по модулю k**. Основное наблюдение: если два префикса имеют одинаковый остаток от деления на k, то сумма элементов между ними кратна k. Формально:

Пусть `prefix[i]` — сумма первых i элементов (i от 0 до n). Тогда `prefix[0] = 0`. Для отрезка от `l+1` до `r` (индексы l и r в исходном массиве, где l < r) сумма равна `prefix[r+1] - prefix[l]`. Если `(prefix[r+1] - prefix[l]) % k == 0`, то `prefix[r+1] % k == prefix[l] % k`.

Таким образом, достаточно хранить множество (или словарь) уже встреченных остатков и их индексы. При обнаружении одинакового остатка проверяем, что расстояние между индексами больше 1 (т.е. подмассив длиной >=2).

### Алгоритм

1. Создаём массив `pref` длины `n+1`, где `pref[0] = 0`. Для i от 1 до n вычисляем `pref[i] = (pref[i-1] + nums[i-1]) % k`.
2. Создаём хеш-таблицу `hash_table`, где ключ — остаток, значение — **первый индекс**, где этот остаток встретился.
3. Проходим по индексам `i` от 0 до n:
   - Если остаток `pref[i]` уже есть в таблице:
     - Проверяем, что `i - hash_table[pref[i]] >= 2`. Если да, возвращаем `True`.
   - Иначе (остаток новый) — сохраняем `hash_table[pref[i]] = i`.
4. Если ни разу не вернули `True`, возвращаем `False`.

**Почему проверяем `i - prev >= 2`?** Потому что подмассив образуется элементами между индексами `prev` и `i`. Индекс `prev` соответствует префиксу до некоторого элемента, а `i` — после. Длина подмассива равна `i - prev`. Нам нужно `i - prev >= 2`.

## Реализация

```python
def checkSubarraySum(nums: list[int], k: int) -> bool:
    n = len(nums)
    # Префиксные суммы по модулю k
    pref = [0] * (n + 1)
    for i in range(1, n + 1):
        pref[i] = (pref[i - 1] + nums[i - 1]) % k

    hash_table = {}  # остаток -> первый индекс
    for i in range(n + 1):
        if pref[i] in hash_table:
            # Проверяем, что длина подмассива >= 2
            if i - hash_table[pref[i]] >= 2:
                return True
        else:
            hash_table[pref[i]] = i
    return False
```

## Сложность

- **Время:** O(n) — один проход для вычисления префиксов и один проход для проверки (можно совместить).
- **Память:** O(min(n, k)) для хеш-таблицы (максимум k различных остатков, но не больше n+1).
