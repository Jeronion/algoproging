
## Определение

**Negative Marking** — это техника для решения задач на массивах **без использования дополнительной памяти** (in-place). Идея: использовать знак элемента (отрицательный/положительный) как флаг, отмечающий, что некоторое значение уже было встречено. Обычно применяется в задачах, где числа находятся в диапазоне от 1 до n (или 0 до n-1) и нужно найти дубликаты или пропущенные элементы.

---

## Принцип работы

Проходим по массиву. Для каждого элемента `x` (абсолютное значение) берём индекс `i = abs(x) - 1` (если числа от 1 до n). Если элемент `arr[i]` положителен, делаем его отрицательным — тем самым помечаем, что число `i+1` уже встречалось. Если элемент уже отрицательный, значит число-индекс встречалось второй раз — это дубликат.

---

## Пример: поиск дубликата в массиве [1..n]

Дан массив длины n+1, содержащий числа от 1 до n, одно число повторяется дважды. Найти дубликат без дополнительной памяти.

```python
def findDuplicate(nums):
    for num in nums:
        idx = abs(num) - 1
        if nums[idx] < 0:
            return abs(num)
        nums[idx] = -nums[idx]
    return -1
```

Пример: `nums = [1,3,4,2,2]`
- num=1 → idx=0, nums[0]=1 >0 → nums[0] = -1
- num=3 → idx=2, nums[2]=4 >0 → nums[2] = -4
- num=4 → idx=3, nums[3]=2 >0 → nums[3] = -2
- num=2 → idx=1, nums[1]=3 >0 → nums[1] = -3
- num=2 → idx=1, nums[1] = -3 <0 → дубликат 2.

---

## Сложность

- **Время:** O(n)
- **Память:** O(1) (модифицируем исходный массив, но это разрешено)

---

## Применение

- Поиск дубликата (LeetCode 287)
- Поиск пропущенного числа (LeetCode 448 — "Find All Numbers Disappeared in an Array")
- Поиск первого пропущенного положительного числа (LeetCode 41 — здесь используется похожая техника с перестановками)

**Важно:** метод изменяет исходный массив. Если нельзя менять массив, используют другие подходы (например, циклическую сортировку или черепаху и зайца).
