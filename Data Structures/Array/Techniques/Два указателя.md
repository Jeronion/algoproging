## Определение

**Два указателя** — метод, при котором используются два индекса (указателя), проходящие по массиву, списку или строке:

- навстречу друг другу
- или в одном направлении

Цель — решать задачи **эффективнее**, чем наивный перебор всех вариантов.

---

## Когда использовать

1. **Отсортированные данные**
    
    - Массив можно отсортировать заранее или он уже отсортирован
    - Пример: найти пару чисел с заданной суммой

2. **Пары или подмассивы**
    
    - Находим подмассивы/диапазоны с условиями
    - Примеры:
        
        - Самая длинная подстрока без повторов
        - Максимальное количество подряд идущих единиц
        - Проверка палиндрома

3. **Скользящее окно (Sliding Window)**
    
    - Поддерживаем окно элементов, расширяем или сужаем по условию
    - Примеры:
        
        - Наименьший подмассив с суммой ≥ K
        - Перенос нулей в конец массива, сохраняя порядок

4. **Связанные списки (медленный–быстрый указатели)**
    
    - Проверка палиндрома
    - Обнаружение циклов (алгоритм Флойда)
    - Нахождение середины списка

## Наивный метод (O(n²) по времени, O(1) по памяти)

```python
def two_sum(arr, target):
    n = len(arr)
    for i in range(n):
        for j in range(i + 1, n):
            if arr[i] + arr[j] == target:
                return True
    return False
```

- Генерируем все пары и проверяем сумму
- **Неэффективно** для больших массивов

---

## Два указателя (O(n) по времени, O(1) по памяти)

### Алгоритм

1. Инициализация:
    
    - `left = 0`
    - `right = n - 1`

2. Пока `left < right`:
    
    - `sum = arr[left] + arr[right]`
    - Если `sum == target` → пара найдена
    - Если `sum < target` → `left += 1`
    - Если `sum > target` → `right -= 1`

---

### Реализация

```python
def two_sum(arr, target):
    left, right = 0, len(arr) - 1
    while left < right:
        s = arr[left] + arr[right]
        if s == target:
            return True
        elif s < target:
            left += 1
        else:
            right -= 1
    return False
```

---

## Логика действий

1. **Увеличиваем `left`**
    
    - Сумма меньше `target`
    - Элемент `arr[left]` не может участвовать в решении с текущим `right` или любым левым элементом

2. **Уменьшаем `right`**
    
    - Сумма больше `target`
    - Элемент `arr[right]` не может участвовать в решении с текущим `left` или любым правым элементом

## Заключение

- Универсальный метод для задач с **парами, подмассивами, диапазонами**
- Позволяет сократить асимптотику с O(n²) до O(n)
- Основное преимущество: **простота + эффективность**
- Не всегда применим — подходит только, когда можно использовать свойства входных данных (упорядоченность, непрерывность, линейный проход)
