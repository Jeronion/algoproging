## Определение

Разностный массив `D` для массива `A` хранит **разницы соседних элементов**:

```id="diff_def"
D[1] = A[1]
D[i] = A[i] - A[i - 1]  # для i > 1
```

- Восстановление исходного массива через префиксную сумму:

```id="diff_recover"
A[i] = D[1] + D[2] + ... + D[i]
```

- **Идея:** обычный массив = префиксная сумма разностного массива

Он использует принцип [[Префиксные суммы]].

---

## Магия обновлений диапазона

Чтобы увеличить все элементы `[l, r]` на `k`:

```id="diff_update"
D[l] += k
D[r + 1] -= k
```

- Восстановление через префиксную сумму даст корректные значения
- Меняются **только две ячейки**, не весь диапазон

---

## Почему это работает

- Накопление изменений через префиксную сумму
- После `r` изменение «отменяется»
- Аналог включателя/выключателя на отрезке

---

## Классический пример: задача «Носки»

- Дано `N` отрезков `[l, r]`
- Нужно узнать, сколько отрезков покрывают каждую точку

```python
L, N, M = map(int, input().split())

diff = [0] * (L + 2)

for _ in range(N):
    l, r = map(int, input().split())
    diff[l] += 1
    diff[r + 1] -= 1

cover = [0] * (L + 1)
cur = 0
for i in range(1, L + 1):
    cur += diff[i]
    cover[i] = cur

for _ in range(M):
    x = int(input())
    print(cover[x])
```

---

## Сложность

|Этап|Время|
|---|---|
|Обработка всех отрезков|O(N)|
|Восстановление массива|O(L)|
|Ответы на запросы|O(M)|

---

## Когда использовать

- Много операций изменения на отрезках
- Подсчёт покрытия точек отрезками
- События, расписания, интервалы

---

## Ключевые моменты

- Не забывать про `r + 1` при уменьшении
- Размер массива должен быть `L + 2` или больше для безопасного доступа
- Разностный массив = «обратная» структура к префиксной сумме
