## Условие

Задача известна как [LeetCode 215. Kth Largest Element in an Array](https://leetcode.com/problems/kth-largest-element-in-an-array/).

Дан неотсортированный массив целых чисел `nums` и целое число `k`. Требуется найти **k-й по величине** элемент (имеется в виду k-й наибольший, т.е. при сортировке по убыванию элемент на позиции `k`). Необходимо вернуть его значение.

**Пример:**
```
nums = [3,2,1,5,6,4], k = 2
Вывод: 5 (после сортировки: [6,5,4,3,2,1], второй элемент = 5)
```

---

## Идея решения

Один из способов решения — построить **max-кучу** из всех элементов, а затем извлечь из неё `k-1` максимальных элементов. Тогда корень оставшейся кучи будет k-м по величине.

Этот подход гарантирует, что после `k-1` извлечений в корне кучи останется искомый элемент.

### Алгоритм

1. Преобразовать массив `nums` в max-кучу (например, с помощью `heapify_max`, если доступна).
2. Повторить `k-1` раз: извлечь максимальный элемент из кучи (обычно `heappop_max`).
3. Вернуть значение, оставшееся в корне кучи (т.е. `nums[0]`).

---

## Реализация

```python
def findKthLargest(self, nums: List[int], k: int) -> int:
    heapify_max(nums)          # превращаем nums в max-кучу
    for _ in range(k - 1):
        heappop_max(nums)      # удаляем наибольший элемент k-1 раз
    return nums[0]              # теперь в корне k-й по величине
```

### Альтернативная реализация с использованием стандартного heapq (min-куча)

Чтобы обойтись без специальных функций, можно использовать min-кучу размером `k`:

```python
import heapq

def findKthLargest(nums, k):
    heap = nums[:k]
    heapq.heapify(heap)          # min-куча из первых k элементов
    for num in nums[k:]:
        if num > heap[0]:
            heapq.heapreplace(heap, num)  # заменяем минимальный, если текущий больше
    return heap[0]                # в корне остаётся k-й по величине
```

Этот подход работает за O(n log k) и использует O(k) памяти.

---

## Сложность

- **Время:** O(n + k log n) для построения кучи (O(n)) и k-1 извлечений (каждое O(log n)). В худшем случае k близко к n, тогда O(n log n). Альтернативный метод с min-кучей размера k даёт O(n log k), что предпочтительнее при k << n.
- **Память:** O(1) дополнительно, если модифицировать исходный массив (in-place). В альтернативном методе требуется O(k) памяти.
