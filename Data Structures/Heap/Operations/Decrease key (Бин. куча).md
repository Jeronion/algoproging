## Определение

**decrease_key** — операция, которая уменьшает значение существующего элемента в куче (min-heap) до нового, меньшего значения.

---

## Примечания

- **Предусловие:** новое значение должно быть **меньше** старого. Если оно больше, операция может нарушить свойство кучи в обратную сторону, и потребуется просеивание вниз, что уже не является `decrease_key`.
- Эта операция часто используется вместе с **структурой данных, поддерживающей отображение "элемент → индекс"** (например, словарь), чтобы быстро находить индекс нужного элемента. В противном случае поиск элемента по значению занял бы O(n).

---

## Как работает decrease_key (код)

```python
def decrease_key(self, i, new_val):
    self.arr[i] = new_val               # 1. обновляем значение
    while i != 0 and self.arr[self.parent(i)] > self.arr[i]:
        p = self.parent(i)               # индекс родителя
        self.arr[i], self.arr[p] = self.arr[p], self.arr[i]  # обмен
        i = p                             # поднимаемся выше
```

### Пошаговое объяснение

1. **Обновление значения**  
   Элементу с индексом `i` присваивается новое, меньшее значение `new_val`. Предполагается, что `new_val` действительно меньше старого значения (иначе операция не имеет смысла, и нужно было бы использовать другой механизм, например, увеличение ключа с последующим просеиванием вниз).

2. **Проверка и всплытие**  
   После изменения текущий элемент может оказаться меньше своего родителя, что нарушает свойство min-кучи (родитель должен быть не больше потомка). Цикл `while` выполняется, пока:
   - `i != 0` (текущий узел не является корнем)
   - **и** значение родителя (`arr[parent(i)]`) **больше** значения текущего узла (`arr[i]`).

   В теле цикла:
   - Вычисляется индекс родителя `p`.
   - Элементы `arr[i]` и `arr[p]` меняются местами (меньший элемент поднимается вверх).
   - Индекс `i` обновляется на `p`, чтобы продолжить проверку для нового положения элемента.

3. **Завершение**  
   Цикл прекращается, когда либо достигается корень (`i == 0`), либо родитель оказывается не больше текущего узла (свойство кучи восстановлено).

---

## Сложность:
- **Время:** O(log n) в худшем случае — элемент может подняться от своей позиции до корня, высота кучи не превышает log n.
- **Память:** O(1) дополнительной памяти.

## Сравнение с операцией вставки (insert)

| Характеристика             | `decrease_key`                                  | `insert`                         |
| -------------------------- | ----------------------------------------------- | -------------------------------- |
| Исходное состояние         | Элемент уже есть в куче                         | Элемента нет, добавляется новый  |
| Действие                   | Изменяем значение существующего узла на меньшее | Добавляем новый узел в конец     |
| Восстановление             | Просеивание вверх от изменённого узла           | Просеивание вверх от нового узла |
| Необходимость знать индекс | Да (или иметь быстрый способ его найти)         | Нет                              |
