## Определение

**Вставка** — операция добавления нового элемента в кучу (min-heap) с сохранением свойства упорядоченности: значение каждого узла должно быть **не больше** значений его детей. Для вставки используется процедура **просеивания вверх (sift up / heapify up / bubble up)**.

---

## Реализация

```python
def insert(self, x):
    self.arr.append(x)                 # 1. добавляем элемент в конец массива
    i = len(self.arr) - 1               # индекс нового элемента

    while i > 0 and self.arr[self.parent(i)] > self.arr[i]:
        p = self.parent(i)               # индекс родителя
        self.arr[i], self.arr[p] = self.arr[p], self.arr[i]  # обмен
        i = p                             # поднимаемся выше
```

### Пошаговое объяснение

1. **Добавление в конец**  
   Новый элемент `x` добавляется в конец динамического массива `self.arr`. Так как куча хранится как полное двоичное дерево, новый узел становится самым правым листом на последнем уровне. После этого свойство кучи может быть нарушено (новый элемент может быть меньше своего родителя).

2. **Инициализация индекса**  
   Переменная `i` получает индекс только что добавленного элемента (последний индекс массива).

3. **Цикл просеивания вверх**  
   Цикл выполняется, пока:
   - `i > 0` (узел не является корнем)
   - **и** значение родительского узла `arr[parent(i)]` **больше** значения текущего узла `arr[i]` (нарушение min-heap: родитель не должен быть больше потомка).

   В теле цикла:
   - Вычисляется индекс родителя `p`.
   - Элементы `arr[i]` и `arr[p]` меняются местами (меньший элемент поднимается вверх, больший опускается).
   - Индекс `i` обновляется на `p`, чтобы продолжить проверку для нового положения элемента (теперь он на месте бывшего родителя).

4. **Завершение**  
   Цикл прекращается, когда либо достигается корень (`i == 0`), либо родитель оказывается не больше текущего узла (свойство кучи восстановлено).

---

## Сложность

- **Время:** O(log n) в худшем случае. Элемент может подняться от листа до корня, высота кучи — log n.
- **Память:** O(1) дополнительной памяти (без учёта роста массива). Рост массива при добавлении может требовать O(n) в редких случаях (переаллокация), но амортизированно это O(1) на операцию.

## Сравнение с heapify (просеивание вниз)

| Характеристика     | `insert` (sift up)                        | `heapify` (sift down)                        |
| ------------------ | ----------------------------------------- | -------------------------------------------- |
| Направление        | снизу вверх (от листа к корню)            | сверху вниз (от корня к листьям)             |
| Условие применения | После вставки нового элемента в конец     | После удаления корня или при построении кучи |
| Исходные данные    | Нарушение может быть только вверх по пути | Предполагается, что поддеревья уже корректны |
