
## Определение

**extract_min** — это операция удаления и возврата наименьшего элемента из min-кучи. В min-куче минимальный элемент всегда находится в корне (индекс 0). После его удаления свойство кучи нарушается, и его нужно восстановить с помощью процедуры **heapify** (просеивание вниз).

---

## Реализация

```python
def extract_min(self):
    if len(self.arr) == 0:
        return None

    if len(self.arr) == 1:
        return self.arr.pop()

    root = self.arr[0]                # 1. запоминаем минимальный элемент
    self.arr[0] = self.arr.pop()      # 2. заменяем корень последним элементом
    self.heapify(0)                   # 3. восстанавливаем свойство кучи
    return root                        # 4. возвращаем сохранённый минимум
```

### Пошаговое объяснение

1. **Проверка на пустоту**  
   Если куча пуста (`len(self.arr) == 0`), возвращаем `None` (или можно выбросить исключение).

2. **Случай с одним элементом**  
   Если в куче всего один элемент, то этот элемент и есть минимум. Удаляем его с помощью `pop()` и возвращаем. Никакого восстановления не требуется.

3. **Сохранение корня**  
   Минимальный элемент находится в корне (`self.arr[0]`). Сохраняем его в переменную `root`, чтобы потом вернуть.

4. **Замена корня последним элементом**  
   `self.arr.pop()` удаляет последний элемент массива и возвращает его. Мы присваиваем это значение корню. Теперь:
   - Куча стала на один элемент меньше (последний элемент удалён).
   - В корне теперь бывший последний элемент, который, скорее всего, нарушает свойство min-кучи (он может быть больше своих детей).

4. **Восстановление свойства кучи [[Heapify (Бин. куча)]]**  
   Вызываем `self.heapify(0)` — процедуру просеивания вниз для корня. Она сравнивает корень с детьми и при необходимости меняет местами с меньшим ребёнком, затем рекурсивно спускается дальше, пока свойство не восстановится. 

5. **Возврат результата**  
   После восстановления кучи возвращаем сохранённое значение `root`.

---

## Сложность:
- **Время:** O(log n) — вызов `heapify` занимает O(log n), остальные операции O(1).
- **Память:** O(1) дополнительной памяти (без учёта хранения самой кучи).
