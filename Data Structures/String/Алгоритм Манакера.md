
## Определение

**Алгоритм Манакера** — это алгоритм для нахождения всех палиндромных подстрок в строке за **линейное время** O(n). Он используется для решения задач, связанных с палиндромами, таких как поиск самой длинной палиндромной подстроки и ответы на запросы «является ли подстрока палиндромом?» за O(1) после предобработки.

В отличие от наивных методов (O(n²)) или хеширования с бинарным поиском (O(n log n)), алгоритм Манакера достигает оптимальной сложности за счёт использования симметрии палиндромов и избегания повторных сравнений.

---

## Постановка задачи

**Вход:** строка `s` длиной n.  
**Выход:** либо самая длинная палиндромная подстрока, либо массив радиусов палиндромов для каждого центра, позволяющий отвечать на запросы.

**Пример:**
```
s = "abba"
Самая длинная палиндромная подстрока: "abba" (длина 4)
```

---

## Почему другие методы неэффективны?

| Метод                         | Время      | Память   | Проблема                         |
| ----------------------------- | ---------- | -------- | -------------------------------- |
| Перебор всех подстрок         | O(n³)      | O(1)     | Слишком медленно                 |
| Динамическое программирование | O(n²)      | O(n²)    | Слишком много памяти             |
| Центрированное расширение     | O(n²)      | O(1)     | Квадратичное время               |
| Хеширование + бинарный поиск  | O(n log n) | O(n)     | Коллизии, логарифмический фактор |
| **Алгоритм Манакера**         | **O(n)**   | **O(n)** | **Оптимально**                   |

## Идея решения

### 1. Предобработка строки

Чтобы унифицировать обработку чётных и нечётных палиндромов, строка преобразуется:

- Добавляются разделители `#` между символами и по краям.
- Добавляются стражи (sentinel) — уникальные символы, не встречающиеся в строке (например, `@` в начале и `$` в конце).

**Пример:**
```
s = "abba"
Преобразованная строка: @#a#b#b#a#$
```

Теперь все палиндромы в преобразованной строке имеют **нечётную длину**, и каждый символ (включая `#`) может быть центром палиндрома.

### 2. Массив радиусов `p[i]`

Для каждой позиции `i` в преобразованной строке вычисляется `p[i]` — радиус наибольшего палиндрома с центром в `i` (количество символов в одну сторону). Например, для `@#a#b#b#a#$`:
- `p[4] = 2` (центр на `#`, палиндром `#b#b#` радиуса 2)

Итоговый палиндром в исходной строке имеет длину `p[i]` (без учёта разделителей).

### 3. Основной цикл с зеркальным отражением

Алгоритм поддерживает интервал `[l, r]` — границы самого правого из найденных палиндромов. При обработке позиции `i`:

- Если `i < r`, используется зеркальная позиция `mirror = l + r - i`. Тогда `p[i]` инициализируется как `min(r - i, p[mirror])` (безопасное копирование).
- Затем выполняется расширение влево и вправо, пока символы совпадают.
- Если новый палиндром выходит за `r`, интервал обновляется.

Это ключевая оптимизация: зеркальное отражение позволяет избежать повторных сравнений, обеспечивая линейное время.

## Реализация на Python

### Класс Manacher (предобработка)

```python
class Manacher:
    def __init__(self, s):
        # Преобразование строки с разделителями и стражами
        self.t = "@"
        for ch in s:
            self.t += "#" + ch
        self.t += "#$"
        
        self.n = len(self.t)
        self.p = [0] * self.n
        self._build()

    def _build(self):
        l, r = 0, 0
        for i in range(1, self.n - 1):
            # Зеркальное копирование
            if i < r:
                mirror = l + r - i
                self.p[i] = min(r - i, self.p[mirror])
            # Расширение
            while self.t[i + 1 + self.p[i]] == self.t[i - 1 - self.p[i]]:
                self.p[i] += 1
            # Обновление границ
            if i + self.p[i] > r:
                l, r = i - self.p[i], i + self.p[i]

    def get_longest(self, center, odd):
        """Возвращает длину палиндрома с центром в center (в исходной строке)."""
        # center — индекс в исходной строке
        pos = 2 * center + 2 + (0 if odd else 1)  # смещение в преобразованной строке
        return self.p[pos]

    def is_palindrome(self, l, r):
        """Проверка, является ли подстрока s[l:r+1] палиндромом за O(1)."""
        length = r - l + 1
        center = (l + r) // 2
        return length <= self.get_longest(center, length % 2)
```

### Поиск самой длинной палиндромной подстроки

```python
def longest_palindrome(s):
    if not s:
        return ""
    man = Manacher(s)
    best_len = 1
    best_start = 0
    
    for i in range(len(s)):
        # Нечётная длина
        odd_len = man.get_longest(i, 1)
        if odd_len > best_len:
            best_len = odd_len
            best_start = i - best_len // 2
        # Чётная длина
        even_len = man.get_longest(i, 0)
        if even_len > best_len:
            best_len = even_len
            best_start = i - best_len // 2 + 1
    
    return s[best_start:best_start + best_len]
```

### Ответы на запросы о палиндромности подстрок

```python
def palindrome_queries(s, queries):
    man = Manacher(s)
    return [man.is_palindrome(l, r) for l, r in queries]
```

## Сложность

- **Время:** O(n) — каждый символ обрабатывается константное число раз благодаря оптимизации с зеркальным отражением.
- **Память:** O(n) — массив `p` и преобразованная строка.

---

## Примечания

- Разделители `#` и стражи `@`, `$` должны быть уникальными и не встречаться в исходной строке.
- Массив `p[i]` хранит радиус **в символах преобразованной строки**. Длина палиндрома в исходной строке равна `p[i]` (без учёта `#`).
- Алгоритм часто используется для:
  - **Longest Palindromic Substring** (LeetCode 5)
  - **Palindrome Queries** (проверка подстрок за O(1))
  - **Count Palindromic Substrings** (LeetCode 647)
