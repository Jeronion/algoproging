
## Определение

**Z-функция** — это алгоритм линейного времени для поиска всех вхождений шаблона в текст, а также для решения других задач на строки, связанных с префиксами. Он строит **Z-массив** для строки: для каждой позиции `i` (начиная с 1) `z[i]` равно длине наибольшего общего префикса строки `s` и её суффикса, начинающегося с `i`. Алгоритм работает за O(n) времени и часто проще в реализации, чем KMP.

**Пример:**
```
s = "aabxaab"
Z-массив: [0, 1, 0, 0, 3, 1, 0]
Объяснение: z[4] = 3, так как подстрока "aab" с позиции 4 совпадает с префиксом "aab".
```

---

## Постановка задачи

**Вход:** текст `text` и шаблон `pattern` (строчные латинские буквы).  
**Выход:** массив 0-индексированных позиций начала всех вхождений `pattern` в `text`.

---

## Идея решения

1. Построить новую строку `s = pattern + '$' + text`, где `$` — разделитель, не встречающийся в тексте (например, `'$'` или `'#'`).
2. Вычислить Z-массив для `s`.
3. Для каждой позиции `i` в `s`, начиная с `len(pattern)+1` (то есть после разделителя), если `z[i] == len(pattern)`, то в тексте найдено вхождение на позиции `i - (len(pattern)+1)`.

### Как работает Z-массив?

Z-массив вычисляется слева направо, поддерживая интервал `[l, r]` — самый правый отрезок, который совпадает с префиксом строки. При обработке позиции `i`:

- Если `i ≤ r`, то можно использовать уже вычисленное `z[i-l]` для предварительной оценки, так как `s[i..]` соответствует `s[i-l..]` внутри отрезка.
- Затем, пока символы совпадают с началом строки, расширяем `z[i]`.
- Обновляем `[l, r]`, если найден более длинный отрезок.

Это позволяет вычислить все `z[i]` за O(n).

---

## Алгоритм (по шагам)

1. **Функция `z_function(s)`**:
   - Инициализировать `z = [0]*n`, `l = 0, r = 0`.
   - Для `i` от `1` до `n-1`:
     - Если `i ≤ r`, то `z[i] = min(r - i + 1, z[i - l])`.
     - Пока `i + z[i] < n` и `s[z[i]] == s[i + z[i]]`, увеличивать `z[i]`.
     - Если `i + z[i] - 1 > r`, обновить `l = i, r = i + z[i] - 1`.
   - Вернуть `z`.

2. **Поиск вхождений**:
   - Создать `combined = pattern + '$' + text`.
   - Вычислить `z = z_function(combined)`.
   - Для `i` от `len(pattern)+1` до `len(combined)-1`:
     - Если `z[i] == len(pattern)`, добавить `i - len(pattern) - 1` в результат.

---

## Реализация на Python

```python
def z_function(s):
    n = len(s)
    z = [0] * n
    l = r = 0
    for i in range(1, n):
        if i <= r:
            z[i] = min(r - i + 1, z[i - l])
        while i + z[i] < n and s[z[i]] == s[i + z[i]]:
            z[i] += 1
        if i + z[i] - 1 > r:
            l, r = i, i + z[i] - 1
    return z

def search(text, pattern):
    combined = pattern + '$' + text
    z = z_function(combined)
    m = len(pattern)
    res = []
    for i in range(m + 1, len(z)):
        if z[i] == m:
            res.append(i - m - 1)
    return res

# Пример
text = "aabxaabxaa"
pattern = "aab"
print(search(text, pattern))  # [0, 4]
```

---

## Сложность

- **Время:** O(n + m), где n — длина текста, m — длина шаблона. Строка `combined` имеет длину `n+m+1`, и каждый символ обрабатывается константное число раз.
- **Память:** O(n + m) для хранения объединённой строки и Z-массива.

---

## Преимущества и недостатки

**Преимущества:**
- Линейное время.
- Проще в реализации, чем KMP.
- Универсален: используется не только для поиска, но и для задач на префиксы (например, поиск наименьшего периода строки).

**Недостатки:**
- Требует дополнительной памяти O(n+m) (можно оптимизировать, но обычно приемлемо).
- Для очень длинных текстов может быть менее эффективен по памяти, чем KMP (который использует только массив префиксов).
