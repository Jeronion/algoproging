
## Определение

**Алгоритм Кнута-Морриса-Пратта (KMP)** — это эффективный алгоритм поиска всех вхождений [[Строка]]-образца (pattern) в строке-тексте (text). Он работает за линейное время O(n + m), где n — длина текста, m — длина образца, и не требует возврата по тексту (т.е. не перечитывает уже просмотренные символы).

В отличие от наивного алгоритма, который при несовпадении сдвигает образец на одну позицию и начинает сравнение заново, KMP использует информацию о **совпавшей части** образца, чтобы пропустить заведомо не подходящие позиции.

---

## Проблема наивного алгоритма

Наивный алгоритм (поиск перебором) работает так: для каждой позиции в тексте проверяем, совпадает ли образец. При несовпадении сдвигаемся на 1 и начинаем всё сначала. Это может быть очень медленно, особенно если образец и текст содержат много повторяющихся символов.

Пример: текст `"AAAAAAAAAAAAAAAAAB"`, образец `"AAAAB"`. Наивный алгоритм будет проверять каждую позицию, делая много лишних сравнений.

---

## Идея KMP

Когда происходит несовпадение, мы уже знаем, какая часть образца совпала с текстом. Например, мы сравнили `pat[0..j-1]` с `txt[i-j..i-1]` и обнаружили, что `pat[j]` не равен `txt[i]`. Вместо того чтобы начинать всё сначала с `j=0` и сдвигать образец на 1, мы можем сдвинуть образец так, чтобы **префикс образца совпал с суффиксом уже совпавшей части**. Это позволяет продолжить сравнение без повторной проверки уже совпавших символов.

Для этого нам нужна информация о самом образце: для каждой позиции `j` знать длину **наибольшего собственного префикса, который одновременно является суффиксом** для подстроки `pat[0..j]`. Эта информация хранится в массиве **LPS** (Longest Proper Prefix which is also Suffix).

---

## Что такое LPS (примеры)

LPS[i] = длина наибольшего **собственного** префикса (не всей строки), который также является суффиксом для `pat[0..i]`.

Собственный префикс — это префикс, не равный всей строке.

Примеры:
- Для `"a"` → нет собственного префикса → 0.
- Для `"aa"` → префикс `"a"` = суффикс `"a"` → 1.
- Для `"ab"` → нет → 0.
- Для `"aba"` → префикс `"a"` = суффикс `"a"` → 1.
- Для `"abab"` → префикс `"ab"` = суффикс `"ab"` → 2.
- Для `"aabaaac"` → LPS: [0,1,0,1,2,2,0] (как в статье).

---

## Построение LPS (префикс-функция)

Алгоритм построения LPS использует два указателя: `i` (идёт по строке) и `len_` (текущая длина совпавшего префикса-суффикса).

```python
def build_lps(pat):
    m = len(pat)
    lps = [0] * m
    len_ = 0
    i = 1
    while i < m:
        if pat[i] == pat[len_]:
            len_ += 1
            lps[i] = len_
            i += 1
        else:
            if len_ != 0:
                len_ = lps[len_ - 1]  # откатываемся
            else:
                lps[i] = 0
                i += 1
    return lps
```

**Как это работает?**
- Мы пытаемся увеличить `len_` (длину совпавшего префикса).
- Если символы совпадают — увеличиваем.
- Если нет — уменьшаем `len_`, используя уже вычисленные значения LPS, пока не найдём подходящий или не дойдём до 0.

## Алгоритм поиска

Имея LPS, проходим по тексту с двумя указателями: `i` по тексту, `j` по образцу.

```python
def kmp_search(txt, pat):
    n, m = len(txt), len(pat)
    lps = build_lps(pat)
    i = j =3 0
    res = []
    while i < n:
        if txt[i] == pat[j]:
            i += 1
            j += 1
            if j == m:
                res.append(i - j)   # нашли вхождение
                j = lps[j - 1]      # продолжаем поиск
        else:
            if j != 0:
                j = lps[j - 1]
            else:
                i += 1
    return res
```

**Пояснение:**
- Если символы совпадают — двигаем оба указателя.
- Если совпал весь образец — сохраняем индекс и откатываем `j` по LPS, чтобы найти перекрывающиеся вхождения.
- Если не совпадают и `j > 0` — откатываем `j` по LPS (не трогаем `i`!).
- Если `j = 0` — просто двигаем `i`.

---

## Пример работы

Текст: `"aabaacaadaabaaba"`, образец: `"aaba"`.

Построим LPS для `"aaba"`:
- i=1: `'a'` vs `'a'` (len=0) → совпали → len=1, lps[1]=1
- i=2: `'b'` vs `'a'` (len=1) → не совпали, len=lps[0]=0 → lps[2]=0
- i=3: `'a'` vs `'a'` (len=0) → совпали → len=1, lps[3]=1
LPS = [0,1,0,1]

Поиск:
- i=0..3: совпадает "aaba" → вхождение в 0.
- После совпадения j = lps[3] = 1 (т.е. следующий поиск начнётся с того, что уже есть "a").
- Продолжаем, находим вхождения на 9 и 12.

---

## Сложность

- **Время:** O(n + m) — каждый символ текста и образца обрабатывается ограниченное число раз.
- **Память:** O(m) — для хранения LPS.

---

## Зачем это нужно?

KMP используется везде, где нужен быстрый поиск подстроки:
- Команда `Ctrl+F` в текстовых редакторах.
- Поиск в базах данных, поисковых системах.
- Биоинформатика (поиск фрагментов ДНК).
- Проверка на плагиат.
- Сетевые анализаторы (поиск сигнатур).

## Реализация на Python (полная)

```python
def build_lps(pat):
    m = len(pat)
    lps = [0] * m
    length = 0
    i = 1
    while i < m:
        if pat[i] == pat[length]:
            length += 1
            lps[i] = length
            i += 1
        else:
            if length != 0:
                length = lps[length - 1]
            else:
                lps[i] = 0
                i += 1
    return lps

def kmp_search(txt, pat):
    n, m = len(txt), len(pat)
    if m == 0:
        return []
    lps = build_lps(pat)
    i = j = 0
    result = []
    while i < n:
        if txt[i] == pat[j]:
            i += 1
            j += 1
            if j == m:
                result.append(i - j)
                j = lps[j - 1]
        else:
            if j != 0:
                j = lps[j - 1]
            else:
                i += 1
    return result

# Пример
txt = "aabaacaadaabaaba"
pat = "aaba"
print(kmp_search(txt, pat))  # [0, 9, 12]
```
