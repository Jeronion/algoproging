
## Структура реализации дека через двусвязный список

Для реализации дека (двусторонней очереди) используется **двусвязный список**. Это позволяет эффективно добавлять и удалять элементы с обоих концов.

Узел (**Node**) содержит:

- `data` — значение элемента
- `prev` — ссылка на предыдущий узел
- `next` — ссылка на следующий узел


В самом деке (**Deque**) хранятся два указателя:

- `front` — первый элемент дека
- `rear` — последний элемент дека

Дополнительно хранится размер `_size` для быстрого получения количества элементов.

Начальное состояние:

```
front = None
rear = None
_size = 0
```

Это означает **пустой дек**.

---

# Структура Node

```python
class Node:
    def __init__(self, data):
        self.data = data
        self.prev = None
        self.next = None
```

---

# Основные операции дека

| Операция         | Описание                     |
| ---------------- | ---------------------------- |
| `push_front(x)`  | добавить элемент в начало    |
| `push_back(x)`   | добавить элемент в конец     |
| `pop_front()`    | удалить элемент из начала    |
| `pop_back()`     | удалить элемент из конца     |
| `peek_front()`   | получить первый элемент      |
| `peek_back()`    | получить последний элемент   |
| `is_empty()`     | проверка пустоты             |
| `size()`         | количество элементов         |

---

# PushFront (добавление в начало)

Добавляет элемент **в начало дека**.

### Шаги

1. Создать новый узел.
2. Если дек пуст:
   - `front = rear = new_node`
3. Иначе:
   - `new_node.next = front`
   - `front.prev = new_node`
   - `front = new_node`
4. Увеличить размер.

```python
def push_front(self, data):
    new_node = Node(data)
    if not self.front:
        self.front = self.rear = new_node
    else:
        new_node.next = self.front
        self.front.prev = new_node
        self.front = new_node
    self._size += 1
```

### Сложность
- **Время:** O(1)

---

# PushBack (добавление в конец)

Добавляет элемент **в конец дека**.

### Шаги

1. Создать новый узел.
2. Если дек пуст:
   - `front = rear = new_node`
3. Иначе:
   - `new_node.prev = rear`
   - `rear.next = new_node`
   - `rear = new_node`
4. Увеличить размер.

```python
def push_back(self, data):
    new_node = Node(data)
    if not self.rear:
        self.front = self.rear = new_node
    else:
        new_node.prev = self.rear
        self.rear.next = new_node
        self.rear = new_node
    self._size += 1
```

### Сложность
- **Время:** O(1)

---

# PopFront (удаление из начала)

Удаляет элемент **из начала дека** и возвращает его значение.

### Алгоритм

1. Если дек пуст → вернуть `None`.
2. Сохранить данные `front.data`.
3. Передвинуть `front` на `front.next`.
4. Если новый `front` существует:
   - обнулить его `prev` (`front.prev = None`)
   - иначе (дек стал пуст) → `rear = None`.
5. Уменьшить размер и вернуть сохранённые данные.

```python
def pop_front(self):
    if not self.front: 
        return None
    data = self.front.data
    self.front = self.front.next
    if self.front: 
        self.front.prev = None
    else: 
        self.rear = None
    self._size -= 1
    return data
```

### Сложность
- **Время:** O(1)

---

# PopBack (удаление из конца)

Удаляет элемент **из конца дека** и возвращает его значение.

### Алгоритм

1. Если дек пуст → вернуть `None`.
2. Сохранить данные `rear.data`.
3. Передвинуть `rear` на `rear.prev`.
4. Если новый `rear` существует:
   - обнулить его `next` (`rear.next = None`)
   - иначе (дек стал пуст) → `front = None`.
5. Уменьшить размер и вернуть сохранённые данные.

```python
def pop_back(self):
    if not self.rear: 
        return None
    data = self.rear.data
    self.rear = self.rear.prev
    if self.rear: 
        self.rear.next = None
    else: 
        self.front = None
    self._size -= 1
    return data
```

### Сложность
- **Время:** O(1)

---

# PeekFront / PeekBack (просмотр элементов)

Возвращают значение первого/последнего элемента без удаления.

```python
def peek_front(self): 
    return None if not self.front else self.front.data

def peek_back(self): 
    return None if not self.rear else self.rear.data
```

### Сложность
- **Время:** O(1)

---

# Проверка пустоты и размер

```python
def is_empty(self): 
    return self._size == 0

def size(self): 
    return self._size
```

# Полная реализация

```python
class Node:
    def __init__(self, data):
        self.data = data
        self.prev = None
        self.next = None

class Deque:
    def __init__(self):
        self.front = None
        self.rear = None
        self._size = 0

    def push_front(self, data):
        new_node = Node(data)
        if not self.front:
            self.front = self.rear = new_node
        else:
            new_node.next = self.front
            self.front.prev = new_node
            self.front = new_node
        self._size += 1

    def push_back(self, data):
        new_node = Node(data)
        if not self.rear:
            self.front = self.rear = new_node
        else:
            new_node.prev = self.rear
            self.rear.next = new_node
            self.rear = new_node
        self._size += 1

    def pop_front(self):
        if not self.front: 
            return None
        data = self.front.data
        self.front = self.front.next
        if self.front: 
            self.front.prev = None
        else: 
            self.rear = None
        self._size -= 1
        return data

    def pop_back(self):
        if not self.rear: 
            return None
        data = self.rear.data
        self.rear = self.rear.prev
        if self.rear: 
            self.rear.next = None
        else: 
            self.front = None
        self._size -= 1
        return data

    def peek_front(self): 
        return None if not self.front else self.front.data

    def peek_back(self): 
        return None if not self.rear else self.rear.data

    def is_empty(self): 
        return self._size == 0

    def size(self): 
        return self._size
```

---

# Пример использования

```python
d = Deque()

d.push_back(10)
d.push_back(20)
d.push_front(5)

print("Front:", d.peek_front())   # 5
print("Back:", d.peek_back())     # 20

print("Pop front:", d.pop_front()) # 5
print("Pop back:", d.pop_back())   # 20
print("Size:", d.size())           # 1
print("Is empty?", d.is_empty())   # False
```

### Вывод

```
Front: 5
Back: 20
Pop front: 5
Pop back: 20
Size: 1
Is empty? False
```

| size        | O(1)      |

# Дек на двусвязном списке vs Дек на массиве (кольцевом)

| Свойство                | Дек на двусвязном списке                  | Дек на кольцевом массиве                             |
| ----------------------- | ----------------------------------------- | ---------------------------------------------------- |
| Размер                  | динамический                              | фиксированный или расширяемый                        |
| Использование памяти    | под каждый элемент – узел с доп. ссылками | массив, возможны неиспользуемые ячейки               |
| Операции в начале/конце | O(1) всегда                               | O(1) амортизированно (при расширении)                |
| Простота реализации     | Требуется аккуратность с ссылками         | Проще для понимания, но требует управления индексами |
| Кэш-локальность         | Низкая (узлы разбросаны)                  | Высокая (элементы в смежной памяти)                  |

# Заключение

Реализация дека через двусвязный список обеспечивает константное время выполнения всех основных операций. Это делает её удобной для задач, где требуется частая вставка/удаление с обоих концов, а также когда размер заранее неизвестен. Однако за гибкость приходится платить дополнительной памятью под ссылки и возможным ухудшением кэш-производительности по сравнению с массивом.
