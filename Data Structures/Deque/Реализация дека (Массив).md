
## Реализация дека через циклический массив

**Deque** (Double Ended Queue) — это обобщение обычной очереди, позволяющее вставлять и удалять элементы с обоих концов (спереди и сзади).

## Основные операции дека

| Операция       | Описание                        |
|----------------|---------------------------------|
| `insertFront(x)` | добавить элемент в начало     |
| `insertRear(x)`  | добавить элемент в конец      |
| `deleteFront()`  | удалить элемент из начала     |
| `deleteRear()`   | удалить элемент из конца      |
| `frontEle()`     | получить первый элемент       |
| `rearEle()`      | получить последний элемент    |

**Примечание:** все операции выполняются за O(1), память требуется O(n) для хранения элементов.

---

# Структура реализации дека через циклический массив

Для реализации используется **массив фиксированного размера** и два индекса, которые перемещаются по кругу (циклически). Вместо хранения индекса `rear` отдельно, мы храним:
анно
- `arr` — массив для элементов (фиксировй ёмкости `capacity`)
- `front` — индекс первого элемента
- `size` — текущее количество элементов

Индекс последнего элемента (rear) вычисляется по формуле:
```
rear = (front + size - 1) % capacity
```

Начальное состояние:
```
arr = [None] * capacity
front = 0
size = 0
```
Это означает **пустой дек**.

---

# Вставка элемента в конец (insertRear)

Добавляет элемент **в конец дека**.

### Алгоритм

1. Проверить, не заполнен ли дек (`size == capacity`). Если да — выход.
2. Вычислить индекс для нового элемента:
   ```
   rear = (front + size) % capacity
   ```
3. Поместить элемент в `arr[rear]`.
4. Увеличить `size` на 1.

```python
def insertRear(self, key):
    if self.size == self.capacity:
        print("Deque is full")
        return
    rear = (self.front + self.size) % self.capacity
    self.arr[rear] = key
    self.size += 1
```

### Сложность
- **Время:** O(1)

---

# Вставка элемента в начало (insertFront)

Добавляет элемент **в начало дека**.

### Алгоритм

1. Проверить, не заполнен ли дек (`size == capacity`). Если да — выход.
2. Сдвинуть индекс `front` на одну позицию назад по кругу:
   ```
   front = (front - 1 + capacity) % capacity
   ```
3. Поместить элемент в `arr[front]`.
4. Увеличить `size` на 1.

```python
def insertFront(self, key):
    if self.size == self.capacity:
        print("Deque is full")
        return
    self.front = (self.front - 1 + self.capacity) % self.capacity
    self.arr[self.front] = key
    self.size += 1
```

### Сложность
- **Время:** O(1)

---

# Удаление элемента из конца (deleteRear)

Удаляет элемент **из конца дека** и возвращает его значение.

### Алгоритм

1. Проверить, не пуст ли дек (`size == 0`). Если да — вернуть `None` или ошибку.
2. Вычислить индекс последнего элемента:
   ```
   rear = (front + size - 1) % capacity
   ```
3. Сохранить значение `arr[rear]` как результат.
4. Уменьшить `size` на 1.
5. Вернуть сохранённое значение.

```python
def deleteRear(self):
    if self.size == 0:
        print("Deque is empty")
        return -1
    rear = (self.front + self.size - 1) % self.capacity
    res = self.arr[rear]
    self.size -= 1
    return res
```

### Сложность
- **Время:** O(1)

---

# Удаление элемента из начала (deleteFront)

Удаляет элемент **из начала дека** и возвращает его значение.

### Алгоритм

1. Проверить, не пуст ли дек (`size == 0`). Если да — вернуть `None` или ошибку.
2. Сохранить значение `arr[front]` как результат.
3. Сдвинуть индекс `front` на одну позицию вперёд по кругу:
   ```
   front = (front + 1) % capacity
   ```
4. Уменьшить `size` на 1.
5. Вернуть сохранённое значение.

```python
def deleteFront(self):
    if self.size == 0:
        print("Deque is empty")
        return -1
    res = self.arr[self.front]
    self.front = (self.front + 1) % self.capacity
    self.size -= 1
    return res
```

### Сложность
- **Время:** O(1)

# Получение первого и последнего элементов

Возвращают значение первого/последнего элемента без удаления.

```python
def frontEle(self):
    # предполагается, что дек не пуст
    return self.arr[self.front]

def rearEle(self):
    rear = (self.front + self.size - 1) % self.capacity
    return self.arr[rear]
```

### Сложность
- **Время:** O(1)

# Полная реализация класса Deque на циклическом массиве

```python
class myDeque:
    def __init__(self, c):
        self.l = [None] * c          # массив фиксированного размера
        self.cap = c                  # ёмкость
        self.size = 0                 # текущее количество элементов
        self.front = 0                 # индекс первого элемента

    def deleteFront(self):
        if self.size == 0:
            return None
        res = self.l[self.front]
        self.front = (self.front + 1) % self.cap
        self.size -= 1
        return res

    def insertFront(self, x):
        if self.size == self.cap:
            return
        self.front = (self.front - 1 + self.cap) % self.cap
        self.l[self.front] = x
        self.size += 1

    def insertRear(self, x):
        if self.size == self.cap:
            return
        new_rear = (self.front + self.size) % self.cap
        self.l[new_rear] = x
        self.size += 1

    def deleteRear(self):
        if self.size == 0:
            return None
        rear = (self.front + self.size - 1) % self.cap
        res = self.l[rear]
        self.size -= 1
        return res

    def frontEle(self):
        return self.l[self.front]

    def rearEle(self):
        rear = (self.front + self.size - 1) % self.cap
        return self.l[rear]
```

# Пример использования

```python
if __name__ == "__main__":
    dq = myDeque(4)

    dq.insertRear(10)
    dq.insertFront(20)
    dq.insertRear(30)

    print("Front:", dq.frontEle(), ", Rear:", dq.rearEle())
    # Front: 20, Rear: 30

    dq.deleteFront()
    print("Front:", dq.frontEle(), ", Rear:", dq.rearEle())
    # Front: 10, Rear: 30

    dq.deleteRear()
    print("Front:", dq.frontEle(), ", Rear:", dq.rearEle())
    # Front: 10, Rear: 10
```

### Вывод

```
Front: 20, Rear: 30
Front: 10, Rear: 30
Front: 10, Rear: 10
```

# Дек на циклическом массиве vs Дек на двусвязном списке

| Свойство            | Дек на циклическом массиве                      | Дек на двусвязном списке                   |
| ------------------- | ----------------------------------------------- | ------------------------------------------ |
| Размер              | Фиксированный (задаётся при создании)           | Динамический                               |
| Память              | Массив фиксированной длины                      | Под каждый элемент – узел с двумя ссылками |
| Переполнение        | Может возникнуть при заполнении                 | Только при нехватке памяти                 |
| Простота реализации | Требуется аккуратное управление индексами       | Требуется аккуратность с ссылками          |
| Кэш-локальность     | Высокая (элементы в неприрывной памяти)         | Низкая (узлы разбросаны)                   |
| Расширение          | Требуется создание нового массива и копирование | Лёгкое добавление новых узлов              |
