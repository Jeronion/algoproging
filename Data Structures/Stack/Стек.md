## Определение

**Stack (Стек)** — линейная структура данных, работающая по принципу [[LIFO]], где последний добавленный элемент извлекается первым.

---

## Основные термины

- **Top (Вершина):** Указатель на последний добавленный элемент. Все операции (push, pop, peek) всегда работают только с этим элементом.
- **Bottom (Дно):** Противоположный конец стека. Первый добавленный элемент.
- **Stack Overflow (Переполнение стека):** Ошибка, возникающая при попытке выполнить **Push** в стек, который уже полностью заполнен (достиг предела).
- **Stack Underflow (Опустошение стека):** Ошибка, возникающая при попытке выполнить **Pop** или **Peek** из пустого стека.

### Array Stack vs Linked List Stack

|Свойство|Array Stack|Linked List Stack|
|---|---|---|
|Размер|фиксированный|динамический|
|Overflow|возможен|почти невозможен|
|Память|компактная|больше overhead|
|Реализация|массив|связанный список|

## Операции:

Для того, чтобы делать манипуляции в стеке, нам предоставляются определенные операции.

- **push()** - Вставка.
- **pop()** - Удаление.
- **top()** - Возврат верх. элемента.
- **isEmpty()** - Проверка пустоты.
- **isFull()** - Проверка заполненности.
- **len()** - Возврат длины.

---

## Типы:

### Фиксированный стек размера

- Стек фиксированного размера имеет предопределенную емкость.
- Как только он становится полным, появляется **Stack Overflow**.
- При удалении элемента из пустого стека возникает **Stack Underflow**
- Обычно осуществляется с использованием статического массива.

### Динамический размер Stack

- Динамический стек размера может расти и уменьшаться по мере необходимости.
- Если стек заполнен, его емкость расширяется, чтобы позволить больше элементов.
- Когда элементы удаляются, использование памяти также может уменьшаться.
- Может быть реализовано с помощью: 
	
	- **Связанный список** → растет/уменьшается естественным образом.  
	- **Dynamic Array** (например, вектор в C++, ArrayList в Java или List в Python) → автоматически изменяет размеры.
