
## Структура реализации

Для реализации используются:

### Node

Узел содержит:

- `data` — значение элемента
- `next` — ссылка на следующий узел

### Указатели очереди

Для очереди хранятся два указателя:

- `front` — первый элемент очереди
- `rear` — последний элемент очереди

Начальное состояние:

```
front = None
rear = None
```

Это означает **пустую очередь**.

---

# Структура Node

```python
class Node:

    def __init__(self, new_data):
        self.data = new_data
        self.next = None
```

# Основные операции

|Операция|Описание|
|---|---|
|`enqueue(x)`|добавить элемент|
|`dequeue()`|удалить элемент|
|`getFront()`|получить первый элемент|
|`isEmpty()`|проверка пустоты|
|`size()`|количество элементов|

---

# Enqueue (добавление)

Добавляет элемент **в конец очереди**.

### Шаги

1. Создать новый узел
2. Если очередь пустая → `front = rear = new_node`
3. Иначе:
    
    - `rear.next = new_node`
    - `rear = new_node`

```python
def enqueue(self, new_data):

    new_node = Node(new_data)

    if self.isEmpty():
        self.front = self.rear = new_node
    else:
        self.rear.next = new_node
        self.rear = new_node
```

### Сложность
- **Время:** O(1)

---

# Dequeue (удаление)

Удаляет элемент **из начала очереди**.

### Алгоритм

1. Проверить пустоту
2. Сохранить значение `front.data`
3. Передвинуть `front`

```
front = front.next
```

4. Если `front == None` → очередь стала пустой → `rear = None`

```python
def dequeue(self):

    if self.isEmpty():
        print("Queue Underflow")
        return -1

    removedData = self.front.data

    self.front = self.front.next

    if self.front is None:
        self.rear = None

    return removedData
```

### Сложность
- **Время:** O(1)

---

# Проверка пустоты

```python
def isEmpty(self):
    return self.front is None
```

# Получить первый элемент

Возвращает элемент **в начале очереди**, не удаляя его.

```python
def getFront(self):

    if self.isEmpty():
        print("Queue is empty")
        return -1

    return self.front.data
```

# Полная реализация

```python
class Node:

    def __init__(self, new_data):
        self.data = new_data
        self.next = None


class QueueLinkedList:

    def __init__(self):

        self.front = None
        self.rear = None
        self.currSize = 0


    def isEmpty(self):
        return self.front is None


    def enqueue(self, new_data):

        new_node = Node(new_data)

        if self.isEmpty():
            self.front = self.rear = new_node
        else:
            self.rear.next = new_node
            self.rear = new_node

        self.currSize += 1


    def dequeue(self):

        if self.isEmpty():
            print("Queue Underflow")
            return -1

        removedData = self.front.data

        self.front = self.front.next

        if self.front is None:
            self.rear = None

        self.currSize -= 1

        return removedData


    def getFront(self):

        if self.isEmpty():
            print("Queue is empty")
            return -1

        return self.front.data


    def size(self):
        return self.currSize
```

---

# Пример использования

```python
q = QueueLinkedList()

q.enqueue(10)
q.enqueue(20)

print("Dequeue:", q.dequeue())

q.enqueue(30)

print("Front:", q.getFront())
print("Size:", q.size())
```

### Вывод

```
Dequeue: 10
Front: 20
Size: 2
```

# Array Queue vs Linked List Queue

|Свойство|Array Queue|Linked List Queue|
|---|---|---|
|Размер|фиксированный|динамический|
|Использование памяти|может быть неэффективным|эффективное|
|Overflow|возможен|только при нехватке памяти|
|Enqueue|O(1)|O(1)|
|Dequeue|O(1)|O(1)|
