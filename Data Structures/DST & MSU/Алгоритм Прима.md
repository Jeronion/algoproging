
## Определение

**Алгоритм Прима** — это жадный алгоритм для построения **минимального остовного дерева (Minimum Spanning Tree, MST)** взвешенного связного неориентированного графа. В отличие от алгоритма Крускала, который добавляет рёбра, Прим начинает с одной вершины и постепенно **наращивает дерево**, добавляя на каждом шаге ребро минимального веса, соединяющее уже построенную часть с ещё не включённой вершиной.

---

## Постановка задачи

Дан взвешенный связный неориентированный граф с `V` вершинами и `E` рёбрами. Требуется найти остовное дерево минимального веса (сумма весов рёбер минимальна).

**Вход:**
- `V` — количество вершин.
- `adj` — список смежности (для эффективной реализации) или матрица смежности (для простой).

**Выход:** общий вес MST (или список рёбер).

---

## Идея решения (жадный подход)

Алгоритм Прима основан на **свойстве разреза (cut property)**: если разделить вершины на два множества — уже включённые в MST и ещё нет, — то ребро минимального веса, пересекающее этот разрез, обязательно принадлежит некоторому MST.

Алгоритм работает так:

1. Начинаем с произвольной вершины (например, 0) и помещаем её в MST.
2. На каждом шаге находим ребро **минимального веса**, которое соединяет уже включённую вершину с ещё не включённой.
3. Добавляем это ребро и новую вершину в MST.
4. Повторяем, пока все вершины не будут включены.

Для эффективного поиска минимального ребра используется **приоритетная очередь (min-heap)**.

---

## Алгоритм (пошагово)

1. Инициализировать:
   - `visited = [False] * V` — посещённые вершины.
   - `pq = [(0, start)]` — куча (вес, вершина), начинаем с веса 0 для стартовой вершины.
   - `total_weight = 0`.
2. Пока куча не пуста:
   - Извлечь `(w, u)` — ребро минимального веса к вершине `u`.
   - Если `visited[u]`, пропустить.
   - Добавить `w` к `total_weight`, отметить `u` как посещённую.
   - Для каждого соседа `(v, weight)` из `adj[u]`:
     - Если `v` не посещена, добавить в кучу `(weight, v)`.
3. Вернуть `total_weight`.

---

## Реализация на Python (эффективная, с кучей)

```python
import heapq

def prim_mst(V, adj):
    visited = [False] * V
    pq = [(0, 0)]   # (вес, вершина)
    total_weight = 0

    while pq:
        w, u = heapq.heappop(pq)
        if visited[u]:
            continue
        visited[u] = True
        total_weight += w

        for v, weight in adj[u]:
            if not visited[v]:
                heapq.heappush(pq, (weight, v))

    return total_weight
```

### Пример использования

```python
V = 3
adj = [
    [(1, 5), (2, 1)],
    [(0, 5), (2, 3)],
    [(0, 1), (1, 3)]
]
print(prim_mst(V, adj))  # 4
```

## Альтернативная реализация (для матрицы смежности)

Простой вариант без кучи работает за O(V²), что подходит для плотных графов (когда V невелико). Он использует массив `key` для хранения минимальных весов рёбер до каждой вершины.

```python
import sys

def prim_matrix(V, graph):
    key = [sys.maxsize] * V
    parent = [-1] * V
    visited = [False] * V
    key[0] = 0

    for _ in range(V):
        # выбираем непосещённую вершину с минимальным key
        u = min((key[i], i) for i in range(V) if not visited[i])[1]
        visited[u] = True
        for v in range(V):
            if graph[u][v] and not visited[v] and graph[u][v] < key[v]:
                key[v] = graph[u][v]
                parent[v] = u

    return sum(key)   # сумма весов MST
```

---

## Сложность

- **С кучей (рекомендуется):**
  - Время: O((V + E) log V) — каждая вершина извлекается один раз, каждое ребро может быть добавлено в кучу.
  - Память: O(V + E) для хранения графа и кучи.

- **С матрицей смежности (O(V²)):**
  - Подходит для плотных графов, но медленнее для разреженных.

---

## Сравнение с алгоритмом Крускала

| Характеристика | Прим | Крускал |
|----------------|------|---------|
| Тип графа | Подходит для плотных и разреженных | Лучше для разреженных |
| Структура данных | Куча (priority queue) | DSU (система непересекающихся множеств) |
| Способ построения | Добавление вершин | Добавление рёбер |
| Сложность | O((V+E) log V) | O(E log E) |
| Начало | С произвольной вершины | Сортировка всех рёбер |
