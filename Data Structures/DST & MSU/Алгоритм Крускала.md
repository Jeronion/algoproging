
## Определение

**Минимальное остовное дерево (Minimum Spanning Tree, MST)** взвешенного связного неориентированного графа — это подмножество рёбер, которое соединяет все вершины графа, не содержит циклов и имеет минимально возможную суммарную стоимость (вес). **Алгоритм Крускала** — это жадный алгоритм для построения MST. Он сортирует все рёбра по весу и добавляет их в остов, если они не создают цикл, до тех пор, пока в дереве не окажется `V-1` рёбер (где `V` — число вершин). Для проверки циклов используется структура данных **[[Система непересекающихся множеств]] (DSU)**.

---

## Идея решения

Алгоритм Крускала основывается на принципе жадного выбора: на каждом шаге выбирается ребро с наименьшим весом, которое не нарушает ацикличность текущего остова. Это гарантирует, что итоговый набор рёбер будет минимальным по весу.

### Шаги алгоритма

1. **Сортировка рёбер** по возрастанию веса.
2. **Инициализация DSU** — каждая вершина является отдельным множеством.
3. **Последовательная обработка рёбер**:
   - Для каждого ребра `(u, v, w)` в отсортированном списке:
     - Если `u` и `v` принадлежат разным множествам (нет цикла), то добавляем ребро в остов, объединяем множества и добавляем вес `w` к общей стоимости.
     - Если `u` и `v` уже в одном множестве, ребро пропускается.
4. Процесс продолжается, пока не будет добавлено `V-1` рёбер (или не закончатся рёбра).
5. Если количество добавленных рёбер меньше `V-1`, граф был несвязным, и MST не существует.

## Реализация на Python

```python
class DSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, a, b):
        ra = self.find(a)
        rb = self.find(b)
        if ra == rb:
            return
        if self.rank[ra] < self.rank[rb]:
            self.parent[ra] = rb
        elif self.rank[ra] > self.rank[rb]:
            self.parent[rb] = ra
        else:
            self.parent[rb] = ra
            self.rank[ra] += 1

def kruskal_mst(V, edges):
    # edges: список кортежей (u, v, weight)
    edges.sort(key=lambda x: x[2])
    dsu = DSU(V)
    mst_weight = 0
    edges_used = 0
    for u, v, w in edges:
        if dsu.find(u) != dsu.find(v):
            dsu.union(u, v)
            mst_weight += w
            edges_used += 1
            if edges_used == V - 1:
                break
    return mst_weight if edges_used == V - 1 else None
```

## Пример

Пусть граф с вершинами `0,1,2,3` и рёбрами:
```
(0,1,10), (1,3,15), (2,3,4), (2,0,6), (0,3,5)
```
Отсортированные рёбра: `(2,3,4)`, `(0,3,5)`, `(2,0,6)`, `(0,1,10)`, `(1,3,15)`.
- Добавляем (2,3,4) → остов: {2-3}, вес=4.
- Добавляем (0,3,5) → {2-3, 0-3}, вес=9.
- Ребро (2,0,6) создало бы цикл (2-3-0-2) → пропускаем.
- Добавляем (0,1,10) → {2-3, 0-3, 0-1}, вес=19.
Итоговый MST весом 19.

## Сложность

- **Время:** O(E log E) из-за сортировки рёбер. Операции DSU выполняются почти за O(1) (с оптимизациями), так что общая сложность O(E log E).
- **Память:** O(V + E) для хранения рёбер и массивов DSU.

---

## Сравнение с алгоритмом Прима

| Критерий | Крускал | Прим |
|----------|---------|------|
| Тип графа | Подходит для разреженных и плотных | Эффективен для плотных графов |
| Структура данных | DSU | Очередь с приоритетом (min-heap) |
| Способ построения | Добавляет рёбра | Добавляет вершины |
| Сложность | O(E log E) | O(E log V) |

## Применение

- Построение минимальной стоимости соединения узлов (сети, дороги, проводка).
- Кластеризация (например, алгоритм Single-linkage).
- В задачах, где важна минимизация суммарной длины соединений.
