
## Определение

**Топологическая сортировка** — это упорядочение вершин **ориентированного ациклического графа (DAG)** в линейную последовательность, такую что для каждого направленного ребра `u → v` вершина `u` предшествует вершине `v` в этом порядке. Топологическая сортировка возможна **только** для DAG, поскольку циклы создают противоречивые зависимости (например, `u` должен быть до `v`, `v` до `w`, `w` до `u` — невозможно). В одном графе может существовать несколько топологических порядков.

---

## Описание работы

Топологическая сортировка представляет собой **упорядочение по зависимостям**: вершина, от которой зависят другие, должна идти раньше них. Существуют два основных подхода:

- **[[Обход графа (DFS)]] с записью в стек** — обход в глубину, где вершина добавляется в стек после обработки всех своих соседей. Это гарантирует, что все потомки окажутся выше в стеке, а родители — ниже, и при извлечении из стека родители будут идти раньше потомков.
- **Алгоритм Кана ([[Обход графа (BFS)]])** — использует подсчёт входящих рёбер (степени захода). Вершины с нулевой входящей степенью (без зависимостей) добавляются в очередь и последовательно удаляются, уменьшая входящие степени соседей. Этот процесс повторяется, пока все вершины не будут обработаны.

## Реализации

### Рекурсивная DFS

Использует стек вызовов и явный стек для накопления порядка.

```python
def dfs_topological(adj):
    n = len(adj)
    visited = [False] * n
    stack = []

    def dfs(v):
        visited[v] = True
        for u in adj[v]:
            if not visited[u]:
                dfs(u)
        stack.append(v)   # добавляем после всех соседей

    for v in range(n):
        if not visited[v]:
            dfs(v)

    return stack[::-1]   # разворачиваем стек

# Пример графа (список смежности)
graph = [[1], [2], [], [2, 4], []]
print(dfs_topological(graph))  # [0, 3, 1, 4, 2] или другой порядок
```

**Как это работает:**  
Обход начинается с каждой непосещённой вершины. После обработки всех соседей текущая вершина помещается в стек. В результате, когда мы извлекаем элементы из стека, вершины, не имеющие потомков, оказываются в начале, а корневые — в конце, что и даёт топологический порядок.

### Итеративная BFS (алгоритм Кана)

Использует очередь и подсчёт входящих рёбер.

```python
from collections import deque

def kahns_topological(adj):
    n = len(adj)
    indegree = [0] * n
    for u in range(n):
        for v in adj[u]:
            indegree[v] += 1

    queue = deque([v for v in range(n) if indegree[v] == 0])
    result = []

    while queue:
        u = queue.popleft()
        result.append(u)
        for v in adj[u]:
            indegree[v] -= 1
            if indegree[v] == 0:
                queue.append(v)

    return result if len(result) == n else []   # если есть цикл, вернём пустой список

# Пример графа
graph = [[1], [2], [], [2, 4], []]
print(kahns_topological(graph))  # [0, 3, 1, 4, 2]
```

**Как это работает:**  
Сначала вычисляем входящие степени всех вершин. Вершины с нулевой степенью (без предшественников) помещаем в очередь. Затем извлекаем вершину, добавляем в результат и уменьшаем входящие степени её соседей. Если степень соседа становится нулевой, добавляем его в очередь. Процесс продолжается, пока очередь не опустеет. Если в результате мы обработали не все вершины, значит граф содержит цикл.

---

## Сложность

- **Время:** O(V + E), где V — число вершин, E — число рёбер. Каждая вершина и каждое ребро обрабатываются один раз в обоих подходах.
- **Память:**
  - DFS: O(V) для стека рекурсии и явного стека, плюс массив `visited`.
  - BFS (Кана): O(V) для очереди и массива `indegree`.

---

## Порядок обхода и замечания

- Топологический порядок **не единственный** — в зависимости от порядка обхода соседей или выбора вершин с нулевой степенью можно получить разные корректные последовательности.
- Алгоритм **не работает для графов с циклами**. Если граф содержит цикл, DFS не сможет корректно определить порядок, а алгоритм Кана вернёт список, содержащий меньше, чем `V` вершин.
- Топологическая сортировка широко применяется в планировании задач, управлении зависимостями (например, системы сборки), построении учебных планов, обнаружении мёртвых блокировок в операционных системах.
