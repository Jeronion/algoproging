
## Определение

**Алгоритм Флойда-Уоршелла** — это алгоритм для нахождения кратчайших путей между **всеми парами вершин** во взвешенном ориентированном или неориентированном графе. Он работает с графами, содержащими отрицательные веса рёбер, но **не допускает отрицательных циклов** (циклов с суммой весов меньше нуля). Алгоритм основан на динамическом программировании и последовательно улучшает оценки расстояний, используя промежуточные вершины.

---

## Постановка задачи

Дана матрица `dist[][]` размера `n × n`, где `dist[i][j]` — вес ребра от вершины `i` к вершине `j`. Если ребра нет, `dist[i][j] = INF` (бесконечность). Диагональные элементы `dist[i][i] = 0`. Требуется найти кратчайшие расстояния между всеми парами вершин (возможно, с отрицательными весами, но без отрицательных циклов).

**Пример:**
```
Входная матрица:
[[0, 4, INF, 5, INF],
 [INF, 0, 1, INF, 6],
 [2, INF, 0, 3, INF],
 [INF, INF, 1, 0, 2],
 [1, INF, INF, 4, 0]]

Выходная матрица (кратчайшие расстояния):
[[0, 4, 5, 5, 7],
 [3, 0, 1, 4, 6],
 [2, 6, 0, 3, 5],
 [3, 7, 1, 0, 2],
 [1, 5, 5, 4, 0]]
```

---

## Идея решения

Алгоритм рассматривает каждую вершину `k` как возможный промежуточный узел на пути между `i` и `j`. Для каждой пары `(i, j)` он проверяет, не даст ли путь через `k` меньшую длину: `dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])`. Итерации выполняются для всех `k` от 0 до V-1, и на каждом шаге используются уже улучшенные расстояния. В конце матрица содержит кратчайшие пути между всеми парами.

### Оптимальная подструктура

Любой кратчайший путь от `i` до `j` либо не проходит через вершину `k`, либо является комбинацией кратчайших путей от `i` до `k` и от `k` до `j`. Это свойство позволяет применить [[Динамическое программирование]].

---

## Алгоритм (пошагово)

1. Инициализировать матрицу `dist` размера `n × n`:
   - `dist[i][j] = вес ребра (i, j)`, если ребро существует, иначе `INF`.
   - `dist[i][i] = 0`.
2. Для каждого `k` от 0 до `n-1`:
   - Для каждого `i` от 0 до `n-1`:
     - Для каждого `j` от 0 до `n-1`:
       - Если `dist[i][k]` и `dist[k][j]` конечны, то обновить `dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])`.
3. В результате `dist[i][j]` содержит кратчайшее расстояние от `i` до `j`.

---

## Реализация на Python

```python
def floyd_warshall(dist):
    n = len(dist)
    INF = 10**8
    # Копируем матрицу (если нужно сохранить исходную)
    # dist = [row[:] for row in dist]  # если не хотим менять исходную
    
    for k in range(n):
        for i in range(n):
            for j in range(n):
                if dist[i][k] != INF and dist[k][j] != INF:
                    dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
    return dist

# Пример использования
INF = 10**8
graph = [
    [0, 4, INF, 5, INF],
    [INF, 0, 1, INF, 6],
    [2, INF, 0, 3, INF],
    [INF, INF, 1, 0, 2],
    [1, INF, INF, 4, 0]
]
result = floyd_warshall(graph)
for row in result:
    print(' '.join(map(str, row)))
```

## Сложность

- **Время:** O(V³), где V — количество вершин. Три вложенных цикла по V.
- **Память:** O(V²) для хранения матрицы расстояний.

---

## Примечания

- Алгоритм не работает при наличии **отрицательных циклов** (сумма весов в цикле меньше нуля). В таком случае расстояния можно сделать сколь угодно малыми.
- Для разреженных графов (мало рёбер) более эффективны алгоритмы, основанные на многократном запуске Дейкстры (с двоичной кучей) или алгоритм Джонсона.
- Алгоритм прост в реализации и хорошо подходит для плотных графов (много рёбер), где число рёбер близко к V².
