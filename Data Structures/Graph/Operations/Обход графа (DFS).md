## Определение

**DFS (Depth-First Search, поиск в глубину)** — алгоритм обхода графа или дерева, который начинает с корневой (или произвольной) вершины и идет вглубь по каждому возможному пути до тех пор, пока не достигнет конца, после чего возвращается (backtrack) и исследует следующие пути. DFS использует стек (явно или через рекурсию) для запоминания вершин, которые нужно посетить.

---

## Описание работы

1. Начинаем с выбранной стартовой вершины, помечаем её как посещённую.
2. Исследуем первого соседа (если есть). Переходим к нему и рекурсивно (или через стек) повторяем процесс.
3. Когда все соседи текущей вершины исследованы, возвращаемся к предыдущей вершине и продолжаем с непосещёнными соседями.
4. Алгоритм завершается, когда все достижимые вершины посещены.

## Реализации

### Рекурсивная реализация

Использует стек вызовов Python.

```python
def dfs_recursive(graph, vertex, visited=None):
    if visited is None:
        visited = set()
    visited.add(vertex)
    print(vertex, end=' ')  # обработка вершины
    
    for neighbor in graph[vertex]:
        if neighbor not in visited:
            dfs_recursive(graph, neighbor, visited)
    return visited

# Пример графа (список смежности)
graph = {
    'A': ['B', 'C'],
    'B': ['A', 'D', 'E'],
    'C': ['A', 'F'],
    'D': ['B'],
    'E': ['B', 'F'],
    'F': ['C', 'E']
}

dfs_recursive(graph, 'A')  # A B D E F C (или другой порядок в зависимости от порядка соседей)
```

### Итеративная реализация

Использует явный стек для избежания глубины рекурсии.

```python
def dfs_iterative(graph, start):
    visited = set()
    stack = [start]
    
    while stack:
        vertex = stack.pop()
        if vertex not in visited:
            visited.add(vertex)
            print(vertex, end=' ')
            # Добавляем соседей в стек. Чтобы порядок был как в рекурсивном, 
            # можно добавить в обратном порядке.
            for neighbor in reversed(graph[vertex]):
                if neighbor not in visited:
                    stack.append(neighbor)
    return visited

dfs_iterative(graph, 'A')  # A B D E F C (при использовании reversed)
```

### Порядок обхода

Порядок посещения вершин зависит от порядка добавления соседей. В рекурсивной реализации обычно выбирается первый сосед в списке. В итеративной — от порядка в стеке. Чтобы получить тот же порядок, что и при рекурсивном вызове, можно добавлять соседей в обратном порядке.

---

## Сложность

- **Время:** O(V + E), где V — число вершин, E — число рёбер. Каждая вершина и каждое ребро посещаются один раз.
- **Память:** 
  - Рекурсивный: O(V) в худшем случае (глубина рекурсии может быть равна числу вершин).
  - Итеративный: O(V) для стека.
