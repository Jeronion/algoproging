
## Определение

**Алгоритм Беллмана-Форда** — это алгоритм для нахождения кратчайших расстояний от одной вершины (источника) до всех остальных вершин во взвешенном графе. В отличие от алгоритма Дейкстры, он **работает с рёбрами отрицательного веса** и может **обнаруживать отрицательные циклы** (циклы с суммой весов меньше нуля). Если отрицательный цикл достижим из источника, то кратчайшего пути не существует (можно бесконечно уменьшать расстояние), и алгоритм сообщает об этом.

---

## Постановка задачи

Даны:
- `V` — количество вершин.
- `edges` — список рёбер вида `[u, v, weight]` (ребро из `u` в `v` с весом `weight`).
- `src` — исходная вершина.

Требуется найти массив `dist[]` длины `V`, где `dist[i]` — кратчайшее расстояние от `src` до `i`. Если вершина недостижима, `dist[i] = 10^8` (условная бесконечность). Если существует отрицательный цикл, достижимый из `src`, вернуть `[-1]`.

**Примеры:**
```
V = 5, edges = [[0,1,5], [1,2,1], [1,3,2], [2,4,1], [4,3,-1]], src = 0
Вывод: [0, 5, 6, 6, 7]
```

---

## Идея решения

Алгоритм основан на **релаксации рёбер**: для каждого ребра `(u, v, w)` проверяем, можно ли улучшить расстояние до `v`, пройдя через `u`: если `dist[u] + w < dist[v]`, то обновляем `dist[v] = dist[u] + w`. Релаксация выполняется `V-1` раз (количество рёбер в кратчайшем пути без циклов не превышает `V-1`). После этого делается ещё один проход: если на нём удаётся улучшить какое-либо расстояние, значит, существует отрицательный цикл, и алгоритм возвращает `-1`.

**Ключевые моменты:**
- Работает с отрицательными весами.
- Обнаруживает отрицательные циклы.
- Медленнее Дейкстры: O(V·E) против O((V+E) log V).

---

## Алгоритм (пошагово)

1. Инициализировать `dist` массивом `INF` (например, 10^8), `dist[src] = 0`.
2. Повторить `V-1` раз:
   - Для каждого ребра `(u, v, w)`:
     - Если `dist[u] != INF` и `dist[u] + w < dist[v]`, обновить `dist[v]`.
3. Проверить наличие отрицательного цикла:
   - Ещё раз пройти по всем рёбрам. Если найдётся ребро `(u, v, w)`, такое что `dist[u] + w < dist[v]`, то вернуть `-1`.
4. Вернуть `dist`.

---

## Реализация на Python

```python
def bellman_ford(V, edges, src):
    INF = 10**8
    dist = [INF] * V
    dist[src] = 0

    # Релаксация V-1 раз
    for i in range(V - 1):
        for u, v, w in edges:
            if dist[u] != INF and dist[u] + w < dist[v]:
                dist[v] = dist[u] + w

    # Проверка на отрицательный цикл (ещё один проход)
    for u, v, w in edges:
        if dist[u] != INF and dist[u] + w < dist[v]:
            return [-1]   # отрицательный цикл

    return dist

# Пример
V = 5
edges = [[0, 1, 5], [1, 2, 1], [1, 3, 2], [2, 4, 1], [4, 3, -1]]
src = 0
print(bellman_ford(V, edges, src))  # [0, 5, 6, 6, 7]
```

---

## Сложность

- **Время:** O(V·E) — V-1 проходов по всем рёбрам.
- **Память:** O(V) — массив расстояний.

---

## Примечания

- Алгоритм **не работает** с отрицательными циклами (если они достижимы из источника). В таких случаях кратчайший путь не определён, и возвращается `-1`.
- По сравнению с Дейкстрой, Беллман-Форд медленнее, но универсальнее (поддерживает отрицательные веса).
- Применяется в задачах, где возможны отрицательные веса, а также для обнаружения отрицательных циклов (например, валютные арбитражные схемы).
