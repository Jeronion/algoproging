
## Определение

**Граф** — это нелинейная структура данных, представляющая собой совокупность объектов (**вершин** или узлов) и связей между ними (**рёбер**).

Формально граф обозначается как **G = (V, E)**, где:

- **V** — множество вершин (nodes)
- **E** — множество рёбер (edges), каждое из которых соединяет пару вершин

---

## Основные термины

- **Вершина (Vertex / Node)** — элементарная единица графа.
- **Ребро (Edge)** — связь между двумя вершинами.
- **Смежные вершины (Adjacent vertices)** — вершины, соединённые ребром.
- **Степень вершины (Degree)** — количество рёбер, инцидентных вершине (для ориентированных графов различают входящую и исходящую степень).
- **Путь (Path)** — последовательность вершин, соединённых рёбрами.
- **Цикл (Cycle)** — путь, у которого начальная и конечная вершины совпадают, и все рёбра различны.
- **Связный граф (Connected graph)** — между любой парой вершин существует путь.
- **Компонента связности (Connected component)** — максимальный связный подграф.
- **Вес ребра (Weight)** — числовая характеристика ребра (стоимость, длина и т.д.).

---

## Виды графов

### По направленности рёбер

| Тип | Описание | Пример |
|------|----------|--------|
| **Неориентированный** | Рёбра не имеют направления. Связь симметрична (A-B = B-A). | Дороги с двусторонним движением |
| **Ориентированный (орграф)** | Рёбра имеют направление (A→B не равно B→A). | Подписки в Instagram, ссылки на сайтах |

### По наличию весов

| Тип | Описание |
|------|----------|
| **Невзвешенный** | Все рёбра равнозначны |
| **Взвешенный** | Каждому ребру сопоставлен вес (расстояние, стоимость) |

### По структуре

| Тип | Описание |
|------|----------|
| **Связный** | Между любой парой вершин есть путь |
| **Несвязный** | Состоит из нескольких компонент связности |
| **Циклический** | Содержит хотя бы один цикл |
| **Ациклический** | Не содержит циклов (например, дерево) |
| **Полный** | Каждая вершина соединена со всеми остальными |
| **Двудольный** | Вершины можно разделить на два множества так, что рёбра соединяют только вершины из разных множеств |

## Способы представления графов

### 1. [[Матрица]] смежности (Adjacency Matrix)

Граф с **n** вершинами представляется двумерным массивом размера **n × n**. Элемент `matrix[i][j]` указывает наличие ребра между вершинами `i` и `j` (для взвешенного графа — вес ребра, иначе 0 или 1).

#### Реализация на Python

```python
# Неориентированный невзвешенный граф
n = 4
adj_matrix = [[0] * n for _ in range(n)]

# Добавление ребра между 0 и 1
adj_matrix[0][1] = 1
adj_matrix[1][0] = 1  # для неориентированного

# Добавление ребра между 0 и 2
adj_matrix[0][2] = 1
adj_matrix[2][0] = 1
```

**Плюсы:**

- Проверка наличия ребра за O(1)
- Простота реализации

**Минусы:**

- Память O(n²) — неэффективно для разреженных графов
- Добавление вершины требует пересоздания матрицы

### 2. [[Массив]] смежности (Adjacency List)

Для каждой вершины хранится список смежных с ней вершин (или список рёбер с весами). Обычно реализуется как словарь: ключ — вершина, значение — список соседей.

#### Реализация на Python

```python
# Неориентированный граф
adj_list = {
    0: [1, 2],
    1: [0, 3],
    2: [0],
    3: [1]
}

# Для взвешенного можно хранить кортежи (сосед, вес)
adj_list_weighted = {
    0: [(1, 5), (2, 3)],
    1: [(0, 5), (3, 7)],
    2: [(0, 3)],
    3: [(1, 7)]
}
```

**Плюсы:**

- Экономия памяти для разреженных графов: O(n + m), где m — число рёбер
- Удобно для обходов

**Минусы:**

- Проверка наличия конкретного ребра может занять O(deg(v))
- Чуть сложнее в реализации

---

## Основные операции

| Операция | Описание | Сложность (список смежности) | Сложность (матрица смежности) |
|----------|----------|-------------------------------|-------------------------------|
| Добавление вершины | Добавить новую изолированную вершину | O(1) (добавление в словарь) | O(n²) (пересоздание матрицы) |
| Удаление вершины | Удалить вершину и все связанные рёбра | O(deg(v)) + поиск | O(n²) |
| Добавление ребра | Соединить две вершины | O(1) (добавить в список) | O(1) |
| Удаление ребра | Удалить связь между вершинами | O(deg(v)) (поиск в списке) | O(1) |
| Проверка наличия ребра | Есть ли связь между u и v | O(deg(u)) | O(1) |
| Обход всех соседей | Получить список смежных вершин | O(deg(v)) | O(n) |

---

## Обходы графа

- **[[Обход графа (DFS)]]** — рекурсивный обход вглубь.
- **[[Обход графа (BFS)]]** — обход по уровням с использованием очереди.

Оба алгоритма имеют сложность **O(n + m)** при использовании списков смежности.

## Сравнение представлений

| Характеристика           | Матрица смежности    | Список смежности   |
| ------------------------ | -------------------- | ------------------ |
| Память                   | O(n²)                | O(n + m)           |

| Получение всех соседей u | O(n)                 | O(deg(u))          |

| Добавление ребра         | O(1)                 | O(1)               |
| Удаление ребра           | O(1)                 | O(deg(u))          |
| Лучше для                | Плотных графов       | Разреженных графов |
