## Определение

Задача «Поиск слова» (Word Search) — классическая задача на **поиск с возвратом (backtracking)**. Дана двумерная сетка (матрица) символов и слово. Требуется определить, можно ли составить это слово, перемещаясь по соседним ячейкам (горизонтально или вертикально), причём каждая ячейка может быть использована не более одного раза.

---

## Условие

Задача известна как [LeetCode 79. Word Search](https://leetcode.com/problems/word-search/).

**Входные данные:**
- `board` — матрица символов размером `m × n` (1 ≤ m, n ≤ 6)
- `word` — строка, длиной от 1 до 15 символов (состоит из букв латинского алфавита)

**Выходные данные:** `True`, если слово существует в сетке, иначе `False`.

**Пример 1:**

Вход:
```
board = [["A","B","C","E"],
         ["S","F","C","S"],
         ["A","D","E","E"]]
word = "ABCCED"
```
Выход: `True` (путь: A→B→C→C→E→D)

**Пример 2:**
```
word = "SEE"
```
Выход: `True` (S→E→E)

**Пример 3:**
```
word = "ABCB"
```
Выход: `False` (нет пути без повторного использования ячейки).

---

## Идея решения

Основной подход — **поиск в глубину (DFS) с возвратом (backtracking)**. Мы пытаемся построить слово, начиная с каждой клетки. Для текущей позиции проверяем:

- Если вышли за границы сетки — возвращаем `False`.
- Если символ в текущей ячейке не совпадает с ожидаемым символом слова — возвращаем `False`.
- Если все символы слова совпали (достигли длины) — возвращаем `True`.
- Иначе временно помечаем ячейку как посещённую (например, заменяем на пустую строку), рекурсивно пробуем все четыре соседних направления. Если хотя бы одно возвращает `True`, успех.
- После проверки возвращаем ячейке исходный символ (восстанавливаем для других путей).

### Важные детали

- Для отметки посещённых ячеек не нужно заводить отдельную матрицу `visited`, можно изменять саму `board`, а затем возвращать символ обратно (это экономит память и ускоряет проверку).
- Поскольку размеры сетки и длина слова невелики (m, n ≤ 6, длина ≤ 15), перебор всех возможных путей допустим.

## Реализация

```python
def exist(board: list[list[str]], word: str) -> bool:
    def backtrack(i: int, j: int, d: int) -> bool:
        # Проверка границ и соответствия символа
        if i < 0 or i >= len(board) or j < 0 or j >= len(board[0]):
            return False
        if board[i][j] != word[d]:
            return False

        # Если дошли до конца слова — успех
        if d == len(word) - 1:
            return True

        # Временно помечаем ячейку как посещённую
        temp = board[i][j]
        board[i][j] = ""

        # Рекурсивно идём во все четыре стороны
        if (backtrack(i + 1, j, d + 1) or
            backtrack(i - 1, j, d + 1) or
            backtrack(i, j + 1, d + 1) or
            backtrack(i, j - 1, d + 1)):
            return True

        # Восстанавливаем ячейку
        board[i][j] = temp
        return False

    # Пробуем начать с каждой клетки
    for i in range(len(board)):
        for j in range(len(board[0])):
            if backtrack(i, j, 0):
                return True
    return False
```

---

## Сложность

- **Время:** O(m·n·4^L), где L — длина слова, m·n — количество стартовых позиций. На каждом шаге до 4 вариантов (кроме первого). Из-за ограничений m,n ≤ 6, L ≤ 15 это приемлемо.
- **Память:** O(L) для стека рекурсии (глубина рекурсии равна длине слова).

## Примечания

- В условиях LeetCode размер сетки очень мал (до 6×6), поэтому даже простой перебор проходит. Однако при больших сетках можно применить оптимизацию: **обратный поиск** (начинать с конца слова), если частота первого символа значительно выше последнего.
- Метка посещённости через изменение ячейки на пустую строку (`""`) работает, потому что в слове могут быть только буквы, а пустая строка никогда не совпадёт с буквой.
- Вместо `board[i][j] = ""` можно использовать `board[i][j] = "#"` или временно хранить символ в переменной.
- В тестах LeetCode есть случай, когда множество ячеек заполнено одинаковыми буквами, что приводит к экспоненциальному перебору. Оптимизация через предварительную проверку, что количество букв в слове не превышает количество доступных букв в сетке, может помочь, но в рамках ограничений не обязательна.
