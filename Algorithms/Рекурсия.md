## Определение

**Рекурсия** — это приём программирования, используемый при решении задач, относящихся к принципу [[Разделяй и властвуй]], при котором функция **вызывает саму себя** для решения подзадач, постепенно сводя их к более простым случаям. Рекурсия особенно полезна для задач, имеющих самоподобную структуру (деревья, графы, комбинаторика).

---

## Обязательные компоненты

Любая корректная рекурсивная функция содержит:

1. **Базовый случай** — условие, при котором рекурсия завершается и возвращается конкретное значение без повторного вызова.
2. **Рекурсивный шаг** — вызов этой же функции с аргументами, приближающими к базовому случаю.

Без базового случая функция будет вызывать себя бесконечно, что приведёт к переполнению стека (stack overflow).

---

## Рекурсия vs итерация

| Рекурсия                                                | Итерация                                                               |
| ------------------------------------------------------- | ---------------------------------------------------------------------- |
| Код часто короче и ближе к математическому определению  | Требует явного управления состоянием (циклы, переменные)               |
| Естественно работает с рекурсивными структурами данных  | Может быть сложнее для самоподобных структур                           |
| Расходует стек вызовов (риск переполнения)              | Использует фиксированный объём памяти (если без динамических структур) |
| Может быть медленнее из-за накладных расходов на вызовы | Обычно быстрее                                                         |

Часто рекурсию можно преобразовать в итерацию с использованием явного стека, но ценой читаемости.
