## Условие

Дана последовательность, определённая рекуррентно:

- $a_0 = 1,\ a_1 = 1$
- $a_{2n} = a_n + a_{n-1}$  (для чётных индексов)
- $a_{2n+1} = a_n - a_{n-1}$ (для нечётных индексов)

Требуется вычислить $a_n$ для заданного $n$ ($1 \le n \le 1000$).

**Пример:**  
$n = 4$ → $a_4 = a_2 + a_1 = (a_1 + a_0) + a_1 = (1+1)+1 = 3$.

---

## Идея решения

Прямая рекурсия с вычислением по формулам приводит к экспоненциальному росту вызовов, так как каждый вызов порождает два новых, но индексы быстро уменьшаются. Однако можно вычислить последовательность **динамически снизу вверх**, заполняя массив от 0 до \(n\) по формулам, так как для любого \(i\) индексы \(i/2\) и \(i/2-1\) меньше \(i\).

### Алгоритм

1. Создать массив `a` длины \(n+1\).
2. `a[0] = a[1] = 1`.
3. Для `i` от 2 до `n`:
   - Если `i` чётное: `a[i] = a[i//2] + a[i//2 - 1]`
   - Если `i` нечётное: `a[i] = a[i//2] - a[i//2 - 1]`
4. Вывести `a[n]`.

---

## Реализация

```python
def strange_sequence():
    n = int(input())
    a = [0] * (n + 1)
    a[0] = a[1] = 1
    for i in range(2, n + 1):
        if i % 2 == 0:                # чётное
            a[i] = a[i // 2] + a[i // 2 - 1]
        else:                         # нечётное
            a[i] = a[i // 2] - a[i // 2 - 1]
    print(a[n])
```

---

## Сложность

- **Время:** O(n) — один проход по массиву.
- **Память:** O(n) для хранения массива (можно сократить до O(log n), вычисляя рекурсивно с мемоизацией, но для n ≤ 1000 это не критично).
