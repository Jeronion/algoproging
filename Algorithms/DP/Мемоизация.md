## Определение

**Мемоизация** — это техника оптимизации, применимая к [[Рекурсия]], при которой результаты выполнения функций сохраняются в кэше, чтобы при повторном вызове с теми же аргументами возвращать сохранённое значение, а не вычислять заново. В контексте динамического программирования мемоизация используется в **подходе сверху вниз (top-down)**.

---

## Основная идея

1. Функция рекурсивно решает задачу.
2. Перед вычислением проверяется, не было ли уже вычислено значение для данных аргументов (обычно с помощью словаря или массива).
3. Если было — возвращается сохранённый результат.
4. Если нет — значение вычисляется, сохраняется и возвращается.

Таким образом, каждое состояние вычисляется только один раз.

## Пример: числа Фибоначчи

Без мемоизации рекурсивное вычисление Фибоначчи имеет экспоненциальную сложность. С мемоизацией — линейная.

```python
def fib(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    memo[n] = fib(n-1, memo) + fib(n-2, memo)
    return memo[n]

print(fib(10))  # 55
```

Можно также использовать встроенный декоратор `functools.lru_cache`:

```python
from functools import lru_cache

@lru_cache(maxsize=None)
def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)
```

## Плюсы и минусы

**Плюсы:**
- Интуитивно понятна (прямая рекурсивная формулировка).
- Вычисляются только необходимые подзадачи.
- Легко реализовать поверх рекурсии.

**Минусы:**
- Риск переполнения стека при глубокой рекурсии.
- Накладные расходы на рекурсивные вызовы и хранение кэша.
- Не всегда эффективна по памяти по сравнению с итеративными методами.
