
## Определение

**Табуляция** — это итеративный подход в динамическом программировании, при котором заполняется таблица (обычно массив) значений для всех возможных состояний, начиная с базовых случаев и двигаясь к целевой задаче. Этот подход также называют **методом снизу вверх (bottom-up)**.

---

## Основная идея

1. Определяются базовые случаи и их значения.
2. Строится таблица (обычно одномерный или двумерный массив) размером, соответствующим количеству состояний.
3. Последовательно заполняются ячейки таблицы, используя уже вычисленные значения меньших подзадач.
4. Итоговый ответ находится в последней ячейке (или другой, в зависимости от задачи).

Табуляция полностью исключает рекурсию и использует циклы.

---

## Пример: числа Фибоначчи

```python
def fib_tabulation(n):
    if n <= 1:
        return n
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]

print(fib_tabulation(10))  # 55
```

**Оптимизация по памяти** — можно хранить только два последних значения:

```python
def fib_optimized(n):
    if n <= 1:
        return n
    prev2, prev1 = 0, 1
    for _ in range(2, n + 1):
        curr = prev1 + prev2
        prev2, prev1 = prev1, curr
    return prev1
```

---

## Плюсы и минусы

**Плюсы:**
- Нет рекурсии → нет риска переполнения стека.
- Часто быстрее из-за отсутствия накладных расходов на вызовы.
- Проще контролировать порядок вычислений.
- Легко оптимизировать память (скользящие массивы).

**Минусы:**
- Иногда сложнее определить порядок заполнения таблицы.
- Может вычислять состояния, которые не нужны для ответа (если не продумана структура).
- Для некоторых задач трудно представить в виде итеративного процесса.

## Сравнение с мемоизацией

| Характеристика          | Мемоизация (Top-Down)       | Табуляция (Bottom-Up)         |
|--------------------------|------------------------------|-------------------------------|
| Подход                   | Рекурсивный                  | Итеративный                   |
| Вычисление               | Только нужные подзадачи      | Часто все подзадачи           |
| Память                   | Зависит от глубины рекурсии  | Явная таблица                 |
| Риск переполнения стека  | Есть                         | Нет                           |
| Простота реализации      | Часто проще                  | Требуется определить порядок  |
