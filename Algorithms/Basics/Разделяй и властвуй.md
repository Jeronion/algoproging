
## Определение

**Разделяй и властвуй (Divide and Conquer)** — это парадигма проектирования алгоритмов, основанная на рекурсивном разбиении задачи на несколько подзадач того же типа, решении этих подзадач и последующем объединении результатов для получения ответа к исходной задаче.

---

## Основные шаги

1. **Разделение (Divide)** — исходная задача разбивается на меньшие непересекающиеся подзадачи (обычно одного размера).
2. **Преодоление (Conquer)** — каждая подзадача решается рекурсивно (если размер подзадачи достаточно мал, решение находится непосредственно).
3. **Объединение (Combine)** — решения подзадач комбинируются в решение исходной задачи.

Некоторые алгоритмы могут опускать явный шаг объединения (например, бинарный поиск), а разделение может быть как тривиальным (деление пополам), так и сложным (выбор опорного элемента в быстрой сортировке).

---

## Примеры алгоритмов

| Алгоритм               | Разделение                                 | Преодоление                             | Объединение                                    |
|------------------------|--------------------------------------------|-----------------------------------------|------------------------------------------------|
| **Бинарный поиск**     | Сравнить с серединой, отбросить половину  | Рекурсивный поиск в одной половине      | Не требуется (ответ сразу)                     |
| **Сортировка слиянием**| Разделить массив пополам                   | Рекурсивно отсортировать каждую половину| Слияние двух отсортированных половин            |
| **Быстрая сортировка** | Разделить относительно опорного элемента   | Рекурсивно сортировать две части        | Не требуется (массив уже разделён на месте)     |
| **Ближайшая пара точек**| Разделить по координате x на две половины | Рекурсивно найти минимум в каждой       | Проверить полосу вокруг границы и обновить ответ|

## Асимптотический анализ

Для большинства алгоритмов этого класса время работы описывается рекуррентным соотношением:

$$ T(n) = aT(n/b) + f(n) $$

где:
- $a$ — количество подзадач,
- $n/b$ — размер каждой подзадачи,
- $f(n)$ — затраты на разделение и объединение.

Для решения таких соотношений используется **[[Теорема Мастера]]**, которая даёт три основных случая:

- Если $f(n) = O(n^{\log_b a - \epsilon})$, то $T(n) = \Theta(n^{\log_b a})$.
- Если $f(n) = \Theta(n^{\log_b a})$, то $T(n) = \Theta(n^{\log_b a} \log n)$.
- Если $f(n) = \Omega(n^{\log_b a + \epsilon})$ и выполнено условие регулярности, то $T(n) = \Theta(f(n))$.

Примеры:
- Сортировка слиянием: $T(n) = 2T(n/2) + O(n) \Rightarrow \Theta(n \log n)$.
- Бинарный поиск: $T(n) = T(n/2) + O(1) \Rightarrow \Theta(\log n)$.
- Обход дерева (рекурсивный): $T(n) = 2T(n/2) + O(1)$ даёт $O(n)$? На самом деле для обхода всех узлов — $O(n)$.
